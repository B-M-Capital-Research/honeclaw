## Goal

Ship v0.14.1 with a fully visible desktop finance-calendar modal and a stable persistent WebKit data store for the standalone macOS user app so login cookies survive app restarts and upgrades.

## Scope

- Render the finance-calendar dialog at the page root and keep it inside the desktop viewport with internal scrolling.
- Give `hone-user-app` a deterministic macOS WebKit data-store identifier while preserving the default persistent fallback on macOS 12/13.
- Build, inspect, publish, download, and independently verify the Universal macOS DMG.
- Align all formal-release versions, publish bilingual notes, push `main`, and publish annotated tag `v0.14.1`.

## Validation

- Web typecheck, 229 Web tests, public Web production build, and `1000x620` desktop viewport verification passed.
- Workspace Rust check, 4 macOS user-app tests, app-specific check, and version-aware iOS remote-shell contract passed.
- Local Universal App/DMG inspection passed for version, `arm64/x86_64`, strict ad-hoc signature, DMG checksum, and minimal bundle contents.
- Two packaged-app launches reused `WebsiteDataStore/484f4e45-5c14-4a71-9b83-636f6d686f6e` and loaded production `/chat` successfully.
- GitHub Actions release run `29189572109` completed successfully. The downloaded Apple checksum and DMG verified, and the mounted remote App remained version `0.14.1`, Universal, strictly signed, and minimal.

## Documentation Sync

- Added `docs/releases/v0.14.1.md`, updated `docs/runbooks/public-user-macos-app.md`, and recorded the final handoff/archive entry.
- Architecture SVG was not updated because the remote-only Apple boundary, runners, channels, tools, database schema, and storage layout did not change.

## Risks / Open Questions

- macOS 14+ users may sign in once when moving from the previous default store to the fixed named store; future releases must preserve the identifier.
- The published DMG remains ad-hoc signed because Developer ID and notarization credentials are not configured.
