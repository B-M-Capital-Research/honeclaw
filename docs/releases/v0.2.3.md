# v0.2.3

发布日期：2026-04-22

English readers: please scroll down for the English version below.

## 用户应该知道的

- 投资组合现在区分“真实持仓”和“仅关注标的”。你可以把一个 ticker 加入关注列表而不填写股数 / 成本，后续仍会进入事件订阅和主动提醒。
- `portfolio` 工具新增 `watch` / `unwatch` 操作；对已关注标的执行 `add` 会自动升级为真实持仓，避免用 `shares=0` 伪造关注项。
- Web 管理端的投资组合页面会分开展示持仓和关注列表，汇总里的 `holdings_count` / `total_shares` 不再把仅关注标的算作真实持仓。
- Event Engine 新增社交源轮询能力：可配置 Telegram 公开频道和 Truth Social 公开账号，产出 `SocialPost` 事件并进入现有 digest / LLM 仲裁链路。
- News LLM classifier 的 OpenRouter 模型从 `google/gemini-3-flash-preview` 切到 `openai/gpt-oss-20b:nitro`，避开当前 Gemini 3 Flash 在 OpenRouter 上的 provider TOS 403，降低不确定来源新闻一直降级为 Low 的概率。
- 本次 release 也补充了几份排障文档，记录 event engine 高优先级新闻未即时推送、news classifier 403 fallback、scheduler heartbeat 等问题的当前结论。

## 谁应该尽快更新

- 需要跟踪“还没买入但想收到提醒”的股票 / ETF / 期权标的的用户。
- 依赖 Telegram 公开频道或 Truth Social 公开账号作为主动事件源的用户。
- 近期看到 `google/gemini-3-flash-preview` 通过 OpenRouter 返回 403，导致 news classifier 一直失败的部署。
- 依赖 Web 管理端查看多用户持仓和关注范围的管理员。

## 升级后会有什么变化

- 旧的 portfolio JSON 继续兼容；缺失 `tracking_only` 的记录会按真实持仓处理。
- 仅关注项会参与订阅匹配和事件推送，但不会计入总股数、真实持仓数量或成本汇总。
- `unwatch` 只删除关注项；如果同一标的是真实持仓，不会被误删。
- 社交源事件默认是 Low，并通过 `source_class=uncertain` 进入 LLM 仲裁；是否升 Medium 仍取决于用户 / actor 的重要性规则和全局路由策略。
- 新的 classifier 模型会消耗 reasoning tokens。当前默认 `llm.openrouter.max_tokens` 足够；如果部署方手动把 max tokens 调得很低，可能看到空回复并保守降级。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- Telegram 公开频道轮询走 `https://t.me/s/<handle>` web preview，不需要 bot token，但依赖 Telegram 页面可访问性。
- Truth Social 账号如果未配置 `account_id`，启动后会先通过公开搜索解析一次账号 ID；想减少启动时外部请求，可在配置里显式填写 `account_id`。
- 本次新增的社交源是主动事件输入，不代表所有社交帖子都会即时推送；Low / Medium 仍按 digest 机制合并发送。

## English

Release date: 2026-04-22

### What Users Should Know

- Portfolios now distinguish real holdings from watch-only symbols. You can track a ticker without entering shares or cost basis, and it will still participate in event subscriptions and proactive alerts.
- The `portfolio` tool now supports `watch` and `unwatch`. Running `add` on a watched ticker promotes it into a real holding instead of relying on fake `shares=0` records.
- The Web portfolio view separates holdings from the watchlist. Summary fields such as `holdings_count` and `total_shares` no longer count watch-only entries as real positions.
- Event Engine can now poll social sources: Telegram public channels and Truth Social public accounts can emit `SocialPost` events that flow into the existing digest and LLM arbitration path.
- The news LLM classifier now uses OpenRouter `openai/gpt-oss-20b:nitro` instead of `google/gemini-3-flash-preview`, avoiding the current provider TOS 403 path seen with Gemini 3 Flash on OpenRouter.
- This release also adds troubleshooting notes for event-engine high news delivery, news classifier 403 fallback, scheduler heartbeat issues, and related operational findings.

### Who Should Update Soon

- Users who want alerts for symbols they are watching but do not currently hold.
- Users relying on Telegram public channels or Truth Social accounts as proactive event sources.
- Deployments seeing OpenRouter 403 failures for `google/gemini-3-flash-preview` in the news classifier.
- Admins who use the Web console to inspect multiple users' holdings and watch scopes.

### What Will Change After Upgrading

- Existing portfolio JSON remains compatible. Records without `tracking_only` are treated as real holdings.
- Watch-only entries participate in subscription matching and event routing, but they do not count toward total shares, real holding counts, or cost aggregates.
- `unwatch` only removes watchlist entries. It will not delete a real holding for the same symbol.
- Social source events start as Low severity and use `source_class=uncertain` for LLM arbitration; promotion to Medium still depends on actor-specific importance rules and the router policy.
- The new classifier model uses reasoning tokens. The default `llm.openrouter.max_tokens` is sufficient; deployments that set very low max-token limits may see empty responses and conservative fallback behavior.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- Telegram public channel polling uses the `https://t.me/s/<handle>` web preview and does not require a bot token, but it depends on that page being reachable.
- Truth Social accounts without an explicit `account_id` perform a public search once on startup to resolve the account ID. Set `account_id` in config if you want to avoid that startup lookup.
- The new social sources are proactive event inputs, not guaranteed immediate pushes for every post. Low and Medium events still go through digest batching.
