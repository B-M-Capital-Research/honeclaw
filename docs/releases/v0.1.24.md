# v0.1.24

发布日期：2026-04-15

English readers: please scroll down for the English version below.

## 用户应该知道的

- Web research proxy 现在对 `research_api_base` 做了更严格的 URL 边界校验，不再接受带认证信息、query/fragment 或非 loopback 的明文 HTTP 地址
- session 与 company profile 的本地文件访问现在会拒绝包含非法路径组件的 id，降低路径逃逸风险
- Web console 不再把明文 `user_id` 打进后端日志
- 前端 PDF 导出依赖 `jspdf` 已升级到 `4.2.1`，并补跑了实际构建验证

## 谁应该尽快更新

- 公开暴露 Web console 或需要更稳妥 research proxy 边界的部署者
- 维护本地 session / company profile 存储安全性的开发者
- 依赖 GitHub security / quality 门禁的仓库维护者

## 升级后会有什么变化

- research proxy 只接受 HTTPS 远端 endpoint，或者 `localhost / 127.0.0.1 / ::1` 上的 HTTP endpoint
- profile / session 存储层会拒绝 `..`、多段路径等非法文件组件
- GitHub Actions 默认 token 权限被收紧为只读 `contents`
- Rust 侧一批高优 transitive dependency 已抬到修复版本：`aws-lc-sys`、`quinn-proto`、`rustls-webpki`、`salvo`、`tar`、`rand 0.9.x`

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- `glib` 相关告警仍在 desktop GTK/Tauri 依赖链上，未纳入本次 patch release 的默认验证范围
- `rand 0.10.0` 仍由 `feishu-sdk -> salvo_core` 链带入；它在 GitHub 当前告警中属于 low severity，本次未继续为其扩大上游升级面
- GitHub code scanning / dependabot 页面需要等 push 后重新扫描，告警清零不是同步即时完成

## English

Release date: 2026-04-15

### What Users Should Know

- The web research proxy now validates `research_api_base` more strictly and rejects embedded credentials, query/fragment suffixes, and non-loopback plain HTTP targets
- Session and company-profile storage now rejects invalid filesystem path components before reading or writing local files
- The web console backend no longer logs raw `user_id` values
- The frontend PDF export dependency `jspdf` has been upgraded to `4.2.1` and validated with a production build

### Who Should Update Soon

- Deployments that expose the web console or need a tighter research-proxy boundary
- Developers maintaining local session / company-profile storage safety
- Repository maintainers who rely on GitHub security / quality gates

### What Will Change After Upgrading

- The research proxy only accepts HTTPS remote endpoints or HTTP endpoints on `localhost / 127.0.0.1 / ::1`
- Session / profile storage rejects parent-dir and multi-segment path components
- GitHub Actions now declares read-only `contents` permissions by default
- A batch of high-priority transitive Rust dependencies has been moved to fixed versions: `aws-lc-sys`, `quinn-proto`, `rustls-webpki`, `salvo`, `tar`, and `rand 0.9.x`

### Install and Update

- `curl | bash` users: rerun the install script to move to the new version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- The remaining `glib` advisory is in the desktop GTK/Tauri chain and was not pulled into this patch release's default validation scope
- `rand 0.10.0` is still pulled in through the `feishu-sdk -> salvo_core` path; GitHub currently rates it as low severity, so this release stops short of a broader upstream dependency push
- GitHub code scanning and dependabot pages need a post-push rescan before the alert counts fully reflect these changes
