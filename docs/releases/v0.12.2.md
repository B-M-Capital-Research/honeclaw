# v0.12.2

发布日期：2026-05-14

English readers: please scroll down for the English version below.

## 用户应该知道的

- **修复了心跳预览的重复抑制 Bug**：飞书等渠道在 scheduler running 行未正确终结、跨任务去重出现“假阳性 skip”的情况下，可能漏发或重复抑制提醒。新版本补齐 `hone-channels` 调度器的 revision dedupe 逻辑，行为回归预期。
- **聊天分享卡片字体改造**：分享卡里的代码块从 JetBrains Mono 切到系统中文字体栈（PingFang SC / Microsoft YaHei），CJK 文本和英文行高对齐，不再出现“代码框里中文字符基线漂移”的视觉问题。
- **新增分享卡预览页（仅开发用）**：挂载 `/__share-preview`，提供示例消息和 PNG 导出按钮，方便在不走完整聊天流程的情况下验证分享卡渲染。
- **LLM 诊断脚本对齐新配置路径**：`scripts/diagnose_llm.sh` 跟随存储配置布局的最新调整，定位 LLM 配置时不会再 “找不到文件”。
- 内部还包含 event-engine digest helpers / global digest 分支的扁平化重构，对外行为无变化。

## 谁应该尽快更新

- 使用飞书或其它 channel scheduler 推送心跳/预览的部署
- 依赖公开聊天分享卡（特别是分享含中文代码块的会话）的用户
- 依赖 `scripts/diagnose_llm.sh` 排查 LLM 链路的运维

## 升级后会有什么变化

- 飞书 scheduler 不再遗留 running 状态行；跨任务去重不再误判
- 分享卡的代码块视觉一致性提升，CJK 字符不再错位
- LLM 诊断脚本能正确读取新位置下的配置

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 架构 SVG 不需要更新：本次发版未触达 pollers / EventKind / agent runners / tools / channels / storage layout

## English

Release date: 2026-05-14

### What Users Should Know

- **Heartbeat preview dedupe bugfix**: Feishu (and other channel scheduler) deployments could leave `running` rows un-finalized and trigger false-positive cross-job duplicate suppression, leading to missed or wrongly-skipped reminders. The `hone-channels` scheduler now carries proper revision dedupe so behavior matches expectations.
- **Share card font overhaul**: Code blocks in the public chat share card now use the system Chinese font stack (PingFang SC / Microsoft YaHei) instead of JetBrains Mono. CJK and ASCII characters share consistent baselines and line height — no more shifted Chinese glyphs inside fenced code.
- **New share-card preview route (dev-only)**: A `/__share-preview` page with sample messages and a PNG export button is mounted so you can verify share-card rendering without running the full public chat flow.
- **LLM diagnostic script aligned with new config layout**: `scripts/diagnose_llm.sh` follows the latest storage config paths and no longer fails to find LLM config files.
- Internal: flattened event-engine digest helpers and global digest branches; no external behavior change.

### Who Should Update Soon

- Deployments using Feishu or other channel schedulers to push heartbeat / preview reminders
- Users who rely on the public chat share card, especially for sessions with Chinese-text code blocks
- Operators using `scripts/diagnose_llm.sh` to triage LLM pipeline issues

### What Will Change After Upgrading

- Feishu scheduler stops leaking `running` rows; cross-job dedupe no longer false-skips
- Share-card code blocks render with consistent visual weight across CJK and ASCII
- LLM diagnostic script reads config from the updated locations

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- Architecture SVG was not updated: this release does not touch pollers / EventKind / agent runners / tools / channels / storage layout
