# v0.5.0

发布日期：2026-04-29

English readers: please scroll down for the English version below.

> v0.4.2 之后的破坏性收敛 release。主线是 **digest 调度统一 + 死配置/死类型清理**：原来"两个特殊命名 slot (pre_market / post_market)"+"全局 schedules 列表"+"prefs.digest_windows"三套并行的 digest 触发机制塌成一份 `Vec<DigestSlot>` N 元数组；`EventKind` 里三个从未被 poller emit 过的死变体（PressRelease / VolumeSpike / Buyback）整体下线；NL tool action `set_digest_windows` 改名为 `set_digest_slots`。同时吸收了 v0.4.2 之后合入 main 的多轮 scheduler / heartbeat / multi-agent / desktop / empty-answer 修复，以及 public 端 contact email 持久化能力。

## 用户应该知道的

### 1. Digest 配置 schema 重构（**breaking**）

`event_engine.digest.{pre_market, post_market}` 两个硬编码字段已被替换为 `default_slots: Vec<DefaultDigestSlot>`，每条 slot 可独立配 `time`（本地 `HH:MM`）+ 可选 `label`（渲染 header 用）。同一节里的 `event_engine.global_digest.schedules` 字段（v0.4.x 已标 deprecated 不读）整体删除。

```yaml
# v0.4.x（仍能读，但 v0.5.0 启动会反序列化失败）
event_engine:
  digest:
    pre_market: "08:30"
    post_market: "09:00"
  global_digest:
    schedules: ["07:30", "21:00"]

# v0.5.0
event_engine:
  digest:
    default_slots:
      - {time: "08:30", label: "盘前摘要"}
      - {time: "09:00", label: "晨间摘要"}
  # global_digest.schedules 字段已删除 —— 触发由 per-actor prefs.digest_slots 驱动
```

scheduler 在每个 default slot 时刻 + actor 可在自己的 `prefs.digest_slots` 里覆盖。`SourceSchedule::CronAligned` 顺便从 `{pre_prefetch, post_prefetch}` 改为 `prefetch_at: Vec<String>`，cron-aligned poller 可在任意数量默认 slot 之前 prefetch。

### 2. `prefs.digest_windows` 硬删（**breaking**，影响仅限非 null 老数据）

per-actor `NotificationPrefs.digest_windows: Option<Vec<String>>` 已从结构体里删除。serde 不会因老 JSON 残留 `digest_windows` 字段报错（unknown_fields 默认忽略），但里面写的值会**直接丢弃**。本 release 出货前请检查所有 `data/notif_prefs/*.json`：

```bash
jq -r 'select(.digest_windows != null and .digest_windows != []) | .digest_windows' data/notif_prefs/*.json
```

非空命中需手工迁移成 `digest_slots`：`["07:00", "19:00"]` → `[{"id":"s0","time":"07:00"}, {"id":"s1","time":"19:00"}]`。

### 3. `EventKind::PressRelease` / `VolumeSpike` / `Buyback` 删除（**breaking**，零运行影响）

这三个变体 enum 里挂着，但从未有任何 poller 产出。本次直接从 `EventKind` 删除：

- `events.sqlite3` 已确认零历史行使用这三个 kind（部署前可自行 `SELECT kind FROM events WHERE kind IN (...)` 验证）
- 配套 match 分支、`ALL_KIND_TAGS`、`kind_label` / `kind_short` / `event_category` / `KindBucket` / `category_kind_tags` / `digest_score` / topic dedup 全部同步收紧
- `disabled_kinds: ["press_release"]` / `["volume_spike"]` / `["buyback"]` 这种 YAML 配置启动会被 `first_invalid_kind_tag` 拒绝，请删除

外部 sink / integration 如果 match 过这三个变体，需要去掉对应分支。

### 4. NL tool action 改名：`set_digest_windows` → `set_digest_slots`（**breaking**）

`notification_prefs` 工具的 `set_digest_windows` action 已删除，对应能力迁到 `set_digest_slots`：

```jsonc
// v0.4.x
{ "action": "set_digest_windows", "value": ["19:00", "09:00"] }

// v0.5.0（参数兼容，HH:MM 字符串数组，每条自动建一条 default-id slot）
{ "action": "set_digest_slots", "value": ["19:00", "09:00"] }
```

`get` action 的 JSON 输出字段从 `digest_windows` 换成 `digest_slots`。skill 文档与错误消息提示同步更新。

### 5. Unified digest scheduler（来自 v0.4.2 之后合入的 commit 1–6）

v0.4.2 cut 之后合并的 `feat(unified_digest)` 6-commit 系列：旧的"持仓事件 digest"和"全球要闻 digest"两条独立链路合并为单一 `UnifiedDigestScheduler`，buffer / synth / global news 在 per-actor fan-out 阶段合流，每个 slot 触发渲染单条 message。actor 没自定义 `prefs.digest_slots` 时回退到 `event_engine.digest.default_slots`。

### 6. Public 端持久化 contact email（来自 2b54a27）

`crates/hone-web-api` 的 public 接口新增 contact email 持久化，方便用户后续找回账号 / 接收升级提醒。运营在公开端面向真实用户的实例可考虑接入。

### 7. Bug 修复批次

- **Heartbeat near-threshold false alerts**（1ad822d）—— 接近阈值时心跳告警频繁误触，已修。
- **Multi-agent local-state direct replies**（97d2279）—— 多 agent local-state 直回的边界条件收紧。
- **Desktop effective config reuse**（4144fa4）—— Tauri desktop 启动时 effective config 复用路径修复。
- **Heartbeat / cron run ledger**（bc0a0ad、f435e2b）—— scheduler heartbeat & 任务 ledger 多个边界条件。
- **Empty-answer fallback false success**（5a3ef29）—— LLM 空 answer 兜底逻辑误判为成功。
- **Active bug burn-down reliability gaps**（12f6e30）—— bug 巡检台账可靠性补漏。

## 谁应该尽快更新

- **所有自托管部署方**：本 release 是**强 breaking 升级**——`config.yaml` 必须改字段才能启动；忽略升级直接拉新版会反序列化失败。
- **NL tool 集成方**：`set_digest_windows` 工具消费方需切到 `set_digest_slots`，否则 action 名会报"未知 action"。
- **依赖 `EventKind::PressRelease/VolumeSpike/Buyback` 的外部 sink / integration 作者**：match 分支需更新；运行时不会有事件流入这三个变体（本来就没人 emit），但编译会失败。
- **Discord / Feishu / Telegram digest 用户**：调度行为改善（unified scheduler）+ 一处 multi-agent direct reply 边界 fix。

## 升级后会有什么变化

- **`config.yaml` 必须改**：`event_engine.digest.{pre_market, post_market}` → `default_slots: [...]`；`event_engine.global_digest.schedules` 整段删除。示例见上文与 `config.example.yaml`。
- **per-actor prefs JSON**：如有 `digest_windows: [...]` 非空值，需迁移到 `digest_slots: [...]`；本 release 提供了 SQL 与 jq 检查命令。
- **NL tool 调用方**：`set_digest_windows` → `set_digest_slots`，参数兼容。
- **EventStore 入库**：`disabled_kinds` 不能再放 `press_release` / `volume_spike` / `buyback`，否则 `first_invalid_kind_tag` 启动校验报错。
- **Scheduler 行为不变**：actor 已自定义 `prefs.digest_slots` 的部署，升级后触发时刻完全一致；只在 actor 没自定义时才会用到新的 `default_slots` 字段。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本。**注意先按上文修改 `config.yaml`**，否则启动会反序列化失败。
- Homebrew 用户：`brew update && brew upgrade honeclaw`。同样先改配置。
- **prefs JSON 迁移**：升级前跑 `jq -r '... select(.digest_windows != null and .digest_windows != [])' data/notif_prefs/*.json`，非空命中按上文示例改成 `digest_slots`。

## 已知注意事项

- 本 release 以 hard-break 而非 deprecation 形式落地。我们没保留 `pre_market` / `post_market` / `digest_windows` 的兼容反序列化路径——部署方升级前必须改 config 与 prefs。
- `EventKind::PressRelease` / `VolumeSpike` / `Buyback` 数据库零历史行已确认；如自托管实例怀疑有外部上游真的写入过这三个 kind，请先 `SELECT kind FROM events WHERE kind IN ('press_release','volume_spike','buyback')` 自查，再升级。
- tag 推送只代表 GitHub release workflow 被触发；最终 tarball / Homebrew formula 是否发布，请以 GitHub Actions 结果为准。

## English

Release date: 2026-04-29

> Breaking-change cleanup release after v0.4.2. Headline: **digest scheduling unification + dead-config / dead-type pruning**. The three parallel digest-trigger paths (`pre_market` + `post_market` named slots, `global_digest.schedules` list, and `prefs.digest_windows`) collapse into a single `Vec<DigestSlot>` N-ary array. Three `EventKind` variants that were never emitted by any poller (`PressRelease` / `VolumeSpike` / `Buyback`) are removed wholesale. NL tool action `set_digest_windows` renames to `set_digest_slots`. Bundles all post-v0.4.2 scheduler / heartbeat / multi-agent / desktop / empty-answer fixes plus public-endpoint contact email persistence.

### What Users Should Know

#### 1. Digest Config Schema Refactor (**breaking**)

The hard-coded `event_engine.digest.{pre_market, post_market}` pair is replaced by `default_slots: Vec<DefaultDigestSlot>`, each slot carrying `time` (local `HH:MM`) plus optional `label` for header rendering. The deprecated `event_engine.global_digest.schedules` field is fully removed.

```yaml
# v0.4.x
event_engine:
  digest:
    pre_market: "08:30"
    post_market: "09:00"
  global_digest:
    schedules: ["07:30", "21:00"]

# v0.5.0
event_engine:
  digest:
    default_slots:
      - {time: "08:30", label: "盘前摘要"}
      - {time: "09:00", label: "晨间摘要"}
  # global_digest.schedules removed — triggering moved to per-actor prefs.digest_slots
```

Actors fall back to these defaults when no `prefs.digest_slots` is set. As a side effect, `SourceSchedule::CronAligned` switches from `{pre_prefetch, post_prefetch}` to `prefetch_at: Vec<String>`, so cron-aligned pollers can prefetch ahead of any number of default slots.

#### 2. Hard-delete `prefs.digest_windows` (**breaking**, only impacts non-null legacy data)

`NotificationPrefs.digest_windows: Option<Vec<String>>` is removed from the struct. Serde will silently ignore the field in legacy JSON (no errors), but **values will be dropped**. Audit `data/notif_prefs/*.json` before upgrading:

```bash
jq -r 'select(.digest_windows != null and .digest_windows != []) | .digest_windows' data/notif_prefs/*.json
```

Migrate non-empty hits to `digest_slots`: `["07:00", "19:00"]` → `[{"id":"s0","time":"07:00"}, {"id":"s1","time":"19:00"}]`.

#### 3. Drop `EventKind::PressRelease` / `VolumeSpike` / `Buyback` (**breaking**, zero runtime impact)

These three variants existed in the enum but were never produced by any poller. v0.5.0 removes them outright:

- `events.sqlite3` confirmed zero historical rows for these kinds (verifiable via `SELECT kind FROM events WHERE kind IN (...)`)
- All match arms, `ALL_KIND_TAGS`, `kind_label` / `kind_short` / `event_category` / `KindBucket` / `category_kind_tags` / `digest_score` / topic dedup updated accordingly
- YAML configs with `disabled_kinds: ["press_release"]` / `["volume_spike"]` / `["buyback"]` now fail startup validation via `first_invalid_kind_tag`

External sinks / integrations that match these variants must drop the corresponding arms.

#### 4. NL Tool Action Rename: `set_digest_windows` → `set_digest_slots` (**breaking**)

The `notification_prefs` tool's `set_digest_windows` action is removed; its capability moves to `set_digest_slots`:

```jsonc
// v0.4.x
{ "action": "set_digest_windows", "value": ["19:00", "09:00"] }

// v0.5.0 (signature-compatible — HH:MM array; each entry auto-creates a default-id slot)
{ "action": "set_digest_slots", "value": ["19:00", "09:00"] }
```

The `get` action's JSON output renames `digest_windows` to `digest_slots`. Skill docs and error messages updated.

#### 5. Unified Digest Scheduler (bundled from post-v0.4.2 commits 1–6)

The `feat(unified_digest)` 6-commit series merged after v0.4.2 cut: previously-separate "portfolio event digest" and "global news digest" pipelines collapse into one `UnifiedDigestScheduler`. Buffer / synth / global news merge during per-actor fan-out, with one rendered message per slot trigger. Actors fall back to `event_engine.digest.default_slots` when their `prefs.digest_slots` is unset.

#### 6. Public Endpoint Contact Email Persistence (from 2b54a27)

`crates/hone-web-api` public surface gains contact email persistence so users can recover access / receive upgrade notices. Recommended for deployments running the public surface against real users.

#### 7. Bug Fix Batch

- **Heartbeat near-threshold false alerts** (1ad822d) — heartbeat alarms triggering spuriously near the threshold.
- **Multi-agent local-state direct replies** (97d2279) — edge-case tightening for multi-agent local-state direct reply path.
- **Desktop effective config reuse** (4144fa4) — Tauri desktop startup effective-config reuse fix.
- **Heartbeat / cron run ledger** (bc0a0ad, f435e2b) — scheduler heartbeat & cron job ledger edge cases.
- **Empty-answer fallback false success** (5a3ef29) — LLM empty-answer fallback no longer reports false success.
- **Active bug burn-down reliability gaps** (12f6e30) — bug-tracking ledger reliability fixes.

### Who Should Update Soon

- **All self-hosted deployments**: this is a **hard-break upgrade** — `config.yaml` must be edited or startup fails on deserialization.
- **NL tool integrators**: callers of `set_digest_windows` must switch to `set_digest_slots`, otherwise action returns "unknown action".
- **External sinks / integrations matching `EventKind::PressRelease / VolumeSpike / Buyback`**: match arms must be dropped; runtime never produced these (no poller emits them), but compile will fail.
- **Discord / Feishu / Telegram digest users**: improved scheduling (unified scheduler) plus a multi-agent direct-reply edge fix.

### What Will Change After Upgrading

- **`config.yaml` requires edits**: `event_engine.digest.{pre_market, post_market}` → `default_slots: [...]`; `event_engine.global_digest.schedules` removed entirely. See examples above and `config.example.yaml`.
- **Per-actor prefs JSON**: if any `digest_windows: [...]` is non-empty, migrate to `digest_slots: [...]`; this release provides SQL and jq checks.
- **NL tool callers**: `set_digest_windows` → `set_digest_slots`, signature compatible.
- **EventStore ingestion**: `disabled_kinds` cannot include `press_release` / `volume_spike` / `buyback` anymore — `first_invalid_kind_tag` startup validation rejects.
- **Scheduler behavior unchanged**: deployments where actors already use `prefs.digest_slots` see identical trigger times after upgrade; the new `default_slots` field is only consulted for actors without custom slots.

### Install and Update

- `curl | bash` users: rerun the install script. **Edit `config.yaml` first per the examples above** or startup will fail on deserialization.
- Homebrew users: `brew update && brew upgrade honeclaw`. Same — edit config first.
- **prefs JSON migration**: before upgrading run `jq -r '... select(.digest_windows != null and .digest_windows != [])' data/notif_prefs/*.json`; migrate non-empty hits to `digest_slots`.

### Known Notes

- This release lands as a hard-break, not a deprecation cycle. We did **not** keep compatibility deserialization paths for `pre_market` / `post_market` / `digest_windows` — operators must edit configs and prefs ahead of upgrade.
- The `EventKind::PressRelease` / `VolumeSpike` / `Buyback` zero-history-rows result was confirmed against this repo's `events.sqlite3`. If your self-hosted instance suspects an external upstream actually wrote these kinds, run `SELECT kind FROM events WHERE kind IN ('press_release','volume_spike','buyback')` before upgrading.
- Pushing the tag only triggers the GitHub release workflow; whether the tarball / Homebrew formula publish must be confirmed via GitHub Actions.
