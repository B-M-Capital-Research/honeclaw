# v0.12.1

发布日期：2026-05-14

English readers: please scroll down for the English version below.

## 用户应该知道的

- 公开用户端 `/chat` 的桌面布局恢复为左侧导航 + 右侧对话的工作台形态。聊天区不再被顶部导航挤压，左侧菜单可收起，GitHub stars、路线图、账号入口、语言 / 偏好设置和「联系我们」集中在侧栏里。
- 移动端聊天体验继续稳定：输入框聚焦、键盘弹出、流式输出结束时的滚动锚定更稳，减少突然跳到消息顶部、页面放大后无法缩回、登录按钮被键盘挤出屏幕等问题。
- 公开站联系信息进一步收敛：B 站、YouTube、邮箱、微信社群统一放在「联系我们」区域，页面上不再暴露剩余额度、用量或不友好的 `web-*` 用户名。
- 公开短信登录链路已按生产反馈补齐：手机号 + 验证码登录、阿里云短信配置和图形验证码守门经过排障，减少「短信服务未配置」和短信接口被脚本刷爆的风险。
- 生产运行脚本和 web actor 沙箱做了稳定性修复，降低本地仓库数据被 web actor 污染、服务启动检查误判、渠道进程拉起不完整的风险。

## 谁应该尽快更新

- 正在对外开放公开用户端 `/chat` 的部署者，尤其是近期遇到移动端滚动跳顶、输入框缩放、PC 页面回到上下排布的人。
- 依赖手机号验证码登录的公开用户端运营者。
- 同时运行 web、Feishu / Lark、iMessage、Discord、Telegram 或 desktop sidecar 的生产部署。

## 升级后会有什么变化

- 桌面端 `/chat` 首屏变为左右排布：左侧为可收起导航与联系入口，右侧为完整高度聊天区；移动端继续保留顶部 header，不显示桌面侧栏。
- 登录态展示更友好：侧栏底部显示「已登录 / 账号中心」，邮箱、社群和视频平台入口收在「联系我们」里，不再占用顶部横向空间。
- 发送验证码前的验证链路更完整，生产配置缺失时更容易暴露并修正；配置正确后短信发送路径与图形验证码守门可一起工作。
- 本次 release 不改变配置文件格式、数据库结构或渠道协议。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 阿里云短信和图形验证码仍依赖生产环境正确配置访问密钥、签名、模板和 captcha 场景参数；升级不会替你自动创建云端资源。
- 公开 `/chat` 的 GitHub stars 依赖 GitHub 接口可访问；接口失败时会显示兜底值而不是加载省略号。
- 架构 SVG 不需要更新：本次 release 没有新增 channel、runner、poller、EventKind 或存储布局；变动集中在公开 web 前端体验、短信验证码守门和运维脚本稳态修复。
- 远端 release workflow 在 tag 推送后由 GitHub Actions 触发；本地只能确认 tag 已推送，不能替代远端产物发布结果。

## English

Release date: 2026-05-14

### What Users Should Know

- Public `/chat` on desktop is back to a workbench layout with a left navigation rail and the conversation on the right. The chat area is no longer squeezed by the top nav, the rail can collapse, and GitHub stars, roadmap, account, language / preference controls, and contact links live in the sidebar.
- Mobile chat interaction is more stable around input focus, keyboard resize, streaming completion, and scroll anchoring. This reduces sudden jumps to the top of the conversation, viewport zoom getting stuck, and login buttons being pushed off-screen.
- Public contact information is now grouped under "Contact us": Bilibili, YouTube, email, and WeChat group details no longer stretch across multiple page headers. Remaining quota, usage, and unfriendly `web-*` identifiers are no longer surfaced in the public chat chrome.
- The public SMS login path has been hardened after production feedback. Phone + code login, Aliyun SMS configuration, and the captcha gate were checked together to reduce "SMS service not configured" failures and lower the risk of scripted SMS abuse.
- Production scripts and web actor sandboxing received stability fixes, reducing the risk of repo data pollution, false startup checks, and incomplete channel restarts.

### Who Should Update Soon

- Operators exposing the public `/chat` entry point, especially if users have seen mobile scroll jumps, input zoom issues, or the desktop page reverting to a stacked layout.
- Operators relying on phone-code login for public users.
- Production deployments running web together with Feishu / Lark, iMessage, Discord, Telegram, or the desktop sidecar.

### What Will Change After Upgrading

- Desktop `/chat` uses a left-right layout: collapsible navigation and contact links on the left, full-height chat on the right. Mobile keeps the top header and does not show the desktop sidebar.
- Logged-in UI is friendlier: the sidebar footer shows "Signed in / Account center", while email, community, and video-channel links are tucked under "Contact us" instead of occupying top horizontal space.
- The SMS-code send path is clearer to operate: missing production configuration is easier to identify, and a correctly configured deployment can run SMS delivery behind the captcha gate.
- This release does not change configuration file shape, database schema, or channel protocols.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- Aliyun SMS and captcha still require production credentials, signatures, templates, and captcha scene settings to be configured correctly. Upgrading does not create cloud-side resources automatically.
- Public `/chat` GitHub stars depend on GitHub API availability; when the API fails, the UI now falls back instead of showing a loading ellipsis.
- Architecture SVG does not need an update: this release does not add channels, runners, pollers, EventKinds, or storage layout changes. Changes are concentrated in public web UX, SMS captcha gating, and operational script hardening.
- The remote release workflow is triggered by the pushed tag in GitHub Actions. Local verification can confirm that the tag was pushed, but not that remote release assets finished publishing.
