# v0.1.8

发布日期：2026-04-12

## 用户应该知道的

- 公司画像能力继续升级，长期档案现在更接近专业投研研究底稿，而不只是保存一段结论
- 新增了更明确的画像框架、事件模板和研究路径保留方式，方便把“结论为什么成立、证据是什么、这轮是怎么研究出来的”一起沉淀下来

## 谁应该尽快更新

- 正在用 Hone 持续跟踪公司、维护长期 thesis 和事件时间线的用户
- 希望把财报、公告、管理层变化后的判断沉淀成可持续复用研究档案的用户

## 升级后会有什么变化

- 公司画像默认结构现在更强调 `Thesis`、关键经营指标、风险台账等长期研究要点
- 追加事件时，推荐同时记录“为什么重要”“证据与来源”“本轮研究路径”，后续回看更容易理解当时的判断依据
- `company_portrait` skill 已改成更清晰的轻主文档 + references 结构，后续让 agent 帮你建档、补事件、更新画像时会更稳定

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 当前前端页面仍然主要提供只读查看、时间线浏览和删除；创建画像、追加事件、更新 section 仍主要通过 agent / tool 路径完成
