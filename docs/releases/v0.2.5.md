# v0.2.5

发布日期：2026-04-22

English readers: please scroll down for the English version below.

## 用户应该知道的

- 修复管理端前端启动时可能一直停在“正在连接后端...”的问题。后端其实已经在 `8077` 正常响应，但持仓上下文初始化时提前读取了尚未创建的 `portfolioData`，导致浏览器端 JavaScript 抛错并阻断首屏渲染。
- 这个修复对 `http://127.0.0.1:3000` 的本地开发管理端和桌面内嵌管理端都适用；配置文件和后端端口不需要迁移。
- 本次版本也带入了最新 bug patrol / Feishu partial reply 证据文档，方便后续排障延续上下文。

## 谁应该尽快更新

- 打开管理端后看到“正在连接后端...”但 `/api/meta` 实际返回正常的用户。
- 使用持仓 / 关注列表、管理端设置、任务中心等前端页面的桌面或本地开发用户。
- 正在追踪 Feishu 部分回复、scheduler heartbeat 或 bug patrol 台账的维护者。

## 升级后会有什么变化

- 管理端初始化不再因为持仓数据派生列表的声明顺序问题卡住。
- 如果后端已启动并且 `/api/meta` 正常，前端会继续进入开始页或上次访问的管理端页面。
- 现有 `config.yaml`、effective config、`8077` 管理端端口和 release 启动方式保持不变。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 如果页面仍停在连接状态，请先强制刷新浏览器缓存，并确认 `http://127.0.0.1:8077/api/meta` 能返回 `desktop-v1`。

## English

Release date: 2026-04-22

### What Users Should Know

- Fixed a web console startup failure where the UI could stay on "connecting to backend" even though the backend was already responding on `8077`. The portfolio context was reading `portfolioData` before the resource was initialized, which raised a browser-side JavaScript error and blocked the first render.
- The fix applies to both the local development console at `http://127.0.0.1:3000` and the desktop-embedded admin console. No config or backend port migration is required.
- This release also carries the latest bug patrol and Feishu partial-reply evidence docs so follow-up debugging keeps its context.

### Who Should Update Soon

- Users who see the admin console stuck on "connecting to backend" while `/api/meta` responds normally.
- Desktop or local development users who rely on portfolio, watchlist, settings, task center, or other admin console pages.
- Maintainers tracking Feishu partial replies, scheduler heartbeat behavior, or the bug patrol ledger.

### What Will Change After Upgrading

- The admin console no longer gets blocked by the portfolio derived-list declaration order during initialization.
- When the backend is running and `/api/meta` is healthy, the frontend proceeds to the start page or the previously visited admin page.
- Existing `config.yaml`, effective config, the `8077` admin backend port, and release startup paths remain unchanged.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- If the page still stays on the connecting screen, hard-refresh the browser cache and confirm `http://127.0.0.1:8077/api/meta` returns `desktop-v1`.
