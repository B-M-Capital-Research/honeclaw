# v0.13.0

发布日期：2026-07-11

English readers: please scroll down for the English version below.

## 用户应该知道的

- HONE 用户端完成统一品牌升级：网站、公开聊天、登录页、macOS App 与新 iOS App 现在共享同一套 HONE 名称、Logo、暖白视觉语言和更精致的桌面/移动菜单交互，不再展示 `OPEN FINANCIAL CONSOLE` 或 `Hone Financial` 用户品牌。
- GitHub Release 新增独立 Apple 用户端资产：Universal macOS DMG 同时支持 Apple Silicon 与 Intel；iOS 提供可验证的 Simulator App 与完整 Xcode 工程。两个客户端默认进入 `https://hone-claw.com/chat`，不打包 ACP、MCP、飞书监听或本地数据目录。
- Web 定时任务结果升级为持久化推送中心：聊天中显示摘要卡片，可点开完整内容，服务端记录已读状态；左侧栏与移动端导航聚合全部推送，读到最新消息后清除红点，Feishu 等其它渠道保持原输出。
- 用户端财经日历支持按月查看宏观事件和持仓/关注公司财报日期、生成并发送图片，并完成移动端全屏预览、缩放与弹层层级修复。
- Cloud PG / OSS 运行时覆盖 sessions、Web auth、quota、cron、skill registry、通知偏好、portfolio、LLM audit、company profiles 与公开文件等持久化路径，并持续收紧本地回退、迁移和运行诊断。
- ACP、heartbeat、scheduler、Feishu/Discord 投递和用户可见输出经过集中可靠性与脱敏修复，包括瞬时失败重试、子进程清理、重复/过期提醒抑制、执行超时恢复和内部状态文案过滤。

## 谁应该尽快更新

- 使用公开 Web 对话、定时任务推送或财经日历的用户。
- 希望通过独立 macOS 或 iOS 客户端使用 HONE 的 Apple 用户。
- 使用 cloud mode、ACP runner、Feishu/Discord 定时投递或自托管多进程运行时的维护者。

## 升级后会有什么变化

- 公开用户界面统一显示 `HONE` 品牌；导航、菜单、登录与聊天侧栏的布局和交互会发生视觉变化，但账号、会话和推送数据不迁移也不清空。
- macOS 用户端是远程薄壳，安装包内不包含本地 HONE runtime；登录态保存在系统 WebKit 数据中。
- iOS Release 资产中的 `HONE-iOS-Simulator-v0.13.0.zip` 仅用于 Simulator 验证；真机/App Store 分发需要使用随附 Xcode 工程配置 Apple Developer 签名。
- 定时推送已读状态由后端持久化，前端只呈现聚合红点，不显示逐条“已读/未读”文字。

## 安装与更新

- macOS 用户端：从 GitHub Release 下载 `HONE-macOS-v0.13.0-universal.dmg`。
- iOS 开发/验证：下载 `HONE-iOS-Simulator-v0.13.0.zip` 或 `HONE-iOS-Xcode-v0.13.0.zip`；真机分发需在 Xcode 中配置签名后 Archive。
- `curl | bash` 用户：重新运行安装脚本即可切到新版本。
- Homebrew 用户：`brew update && brew upgrade honeclaw`。

## 已知注意事项

- 当前仓库没有 Apple Developer 签名与 notarization 凭证。macOS DMG 为 ad-hoc 签名；iOS Release 不包含可安装到真机的 IPA，也不会误标为 App Store 成品。
- 独立 Apple 用户端依赖 `hone-claw.com` 网络可用性；离线时提供恢复界面，但无法离线完成对话。
- 本版本新增 Apple 客户端边界并刷新 `resources/architecture.svg` / `resources/architecture.html`。

## English

Release date: 2026-07-11

### What Users Should Know

- The HONE user experience now has one brand system across the public site, chat, login, macOS app, and new iOS app. They share the HONE name, mark, warm visual language, and refined desktop/mobile navigation, with legacy `OPEN FINANCIAL CONSOLE` and `Hone Financial` user branding removed.
- GitHub Releases now include dedicated Apple client assets: a Universal macOS DMG for Apple Silicon and Intel, plus an iOS Simulator app and complete Xcode project. Both clients open `https://hone-claw.com/chat` and bundle no ACP, MCP, Feishu listener, or local data runtime.
- Scheduled Web results now use a durable push center. Chat shows summary cards with full-content drill-down, server-owned read state, an aggregate sidebar/mobile inbox, and an unread dot that clears after the latest known push is read. Other channels keep their current output.
- The public finance calendar can browse monthly macro events and holding/watchlist earnings dates, render a shareable image, and now has corrected mobile full-screen zoom and overlay behavior.
- Cloud PG / OSS runtime coverage now includes sessions, Web auth, quota, cron, skill registry, notification preferences, portfolio, LLM audit, company profiles, and public files, with stricter migration, fallback, and diagnostic behavior.
- ACP, heartbeat, scheduler, Feishu/Discord delivery, and user-visible output received a broad reliability and redaction pass covering transient retries, child-process cleanup, stale/duplicate suppression, timeout recovery, and internal-state filtering.

### Who Should Update Soon

- Users of public Web chat, scheduled pushes, or the finance calendar.
- Apple users who want a focused macOS or iOS HONE client.
- Operators using cloud mode, ACP runners, Feishu/Discord scheduled delivery, or self-hosted multi-process runtimes.

### What Will Change After Upgrading

- Public surfaces consistently display `HONE`; navigation, menus, login, and chat rail visuals change without migrating or clearing accounts, sessions, or pushes.
- The macOS user app is a remote-only shell with no local HONE runtime, and sign-in state persists in the system WebKit store.
- `HONE-iOS-Simulator-v0.13.0.zip` is for Simulator verification only. Device/App Store distribution requires the included Xcode project and an Apple Developer signing setup.
- Scheduled-push read state is persisted by the backend; the UI exposes only the aggregate unread dot, not per-item read labels.

### Install and Update

- macOS user app: download `HONE-macOS-v0.13.0-universal.dmg` from the GitHub Release.
- iOS development/verification: download `HONE-iOS-Simulator-v0.13.0.zip` or `HONE-iOS-Xcode-v0.13.0.zip`; configure signing in Xcode before a device archive.
- `curl | bash` users: rerun the install script to move to this version.
- Homebrew users: `brew update && brew upgrade honeclaw`.

### Known Notes

- The repository currently has no Apple Developer signing or notarization credentials. The macOS DMG is ad-hoc signed, and the iOS Release intentionally contains no device-installable IPA.
- The focused Apple clients require network access to `hone-claw.com`; they provide offline recovery UI but not offline chat.
- This release adds a new Apple-client boundary and refreshes `resources/architecture.svg` / `resources/architecture.html`.
