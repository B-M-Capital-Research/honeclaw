# v0.1.11

发布日期：2026-04-13

English readers: please scroll down for the English version below.

## 用户应该知道的

- Hone 不再依赖 `runtime config` 这条旧配置链路；启动和运行时配置来源更收敛，减少“同一项配置有多份真相源”的问题
- 旧的 `data/reports/`、`data/x_drafts/` 和 X 发布 skill 已从代码主路径移除，运行时不再继续围绕这些目录组织数据
- 新发起的系统性公司研究，默认会沉淀长期公司画像；同时画像内容默认跟随当前对话语言维护

## 谁应该尽快更新

- 维护 Hone 部署、启动脚本或本地运行环境的用户
- 依赖历史 `runtime config`、`data/reports/`、`data/x_drafts/` 目录约定的用户
- 使用公司研究画像工作流，希望默认沉淀长期研究资产的用户

## 升级后会有什么变化

- `launch.sh` 和相关运行时路径不再要求额外的 `runtime config` 文件
- 文件服务与运行时代码不再暴露 `reports`、`x_drafts` 这些旧目录入口
- 如果你发起的是新的系统性公司调研，Hone 会默认创建并更新长期画像；只有在你明确表示不要沉淀，或场景明显只是一次性短问答时才跳过

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 如果你本地还有依赖旧目录名或旧 `runtime config` 文件的外部脚本，需要一并清理或改到当前配置入口
- 这次发布移除了 X 发布相关 skill；如果你之前有手工串接这条流程，需要改用当前保留的发布方式

## English

Release date: 2026-04-13

### What Users Should Know

- Hone no longer depends on the old `runtime config` chain; startup and runtime config sources are now more tightly unified, reducing “multiple sources of truth for the same setting”
- The legacy `data/reports/`, `data/x_drafts/`, and X publishing skill have been removed from the main code path
- New systematic company research now defaults to being written into a long-term company portrait, and portrait content follows the current conversation language by default

### Who Should Update Soon

- Users who maintain Hone deployments, startup scripts, or local runtime environments
- Users who still depend on the old `runtime config`, `data/reports/`, or `data/x_drafts` directory conventions
- Users who use the company research workflow and want long-term research assets to be persisted by default

### What Will Change After Upgrading

- `launch.sh` and related runtime paths no longer require an extra `runtime config` file
- The file service and runtime code no longer expose the legacy `reports` and `x_drafts` directory entry points
- If you start a new systematic company research flow, Hone will create and update a long-term portrait by default; it will skip only when you clearly say not to persist it or when the turn is obviously just a one-off short Q&A

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- If you still have local scripts depending on the old directory names or old `runtime config` file, you need to migrate them to the current config entry points
- This release removes the X publishing skill; if you had a manual workflow around it, you need to switch to the currently supported publishing path
