# v0.2.6

发布日期：2026-04-23

English readers: please scroll down for the English version below.

## 用户应该知道的

- Event Engine 这一轮把推送质量清单完整收口了：digest 现在会做 topic memory、相似标题去重、source/domain/symbol 限流、窗口最小间隔和分类预算，减少同一批主题在相邻窗口里反复出现。
- 价格、宏观和财报类推送的默认路由更保守也更可解释了。新增方向性价格阈值、大仓位例外、macro immediate 时窗、quiet mode、portfolio-first 默认行为，以及 `earnings_call_transcript` 独立 kind。
- ACP 长会话的 compact 外泄问题这次一并修掉：`codex_acp` / `opencode_acp` 会拦掉 `Context compacted` 或 opencode 自动总结块，不再把这些内部压缩痕迹直接发给用户；Hone 自己也不会再对自管上下文的 ACP runner 做自动 compact。
- `gemini_acp` 现在被明确禁用。如果你还在用 `agent.runner: gemini_acp`，升级后需要切到 `codex_acp`、`opencode_acp`、`multi-agent` 或 `function_calling`。
- 用户端公开 roadmap 页面已同步到 `v0.2.6`，并更新为 19 个公开 Skill、最新 multi-runner 说明和当前稳定能力。

## 谁应该尽快更新

- 依赖 Event Engine 主动推送，尤其在意摘要重复、新闻噪声、价格阈值和 delivery observability 的用户或维护者。
- 长时间使用 `codex_acp` / `opencode_acp` 会话，且之前见过 `Context compacted`、自动总结片段或 compact 后回复异常的用户。
- 还在使用 `gemini_acp` 的部署。

## 升级后会有什么变化

- 同一 actor 的 digest 会更少重复、排序更稳，社交源和低质量来源更难挤占摘要头部。
- 配置层新增了 event-engine 的 digest min-gap、news upgrade cap、价格直推最小阈值、大仓位权重阈值、macro immediate lookahead/grace，以及默认不确定来源新闻分类模型 `amazon/nova-lite-v1`；这些都是增量字段，现有配置无需迁移。
- `config.example.yaml` 的推荐 runner 默认值改为 `multi-agent`；现有已部署实例不会被自动改写，但新配置和新安装路径会按这个推荐值展示。
- Web / public roadmap 页面会显示本次 release 版本号与最新能力说明，无需额外前端配置迁移。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 如果你当前配置仍是 `agent.runner: gemini_acp`，服务启动后的 agent 执行会直接报迁移提示；这不是兼容模式，需要手动改 runner。
- Event Engine 的不确定来源新闻分类基线依然建议保留手工抽检；真实渠道送达质量仍取决于各渠道凭据和上游 API 健康度。

## English

Release date: 2026-04-23

### What Users Should Know

- This release finishes the Event Engine push-quality cleanup pass. Digests now apply topic memory, similar-title dedupe, source/domain/symbol caps, minimum spacing between digest windows, and per-category budgets, so adjacent windows stop repeating the same cluster of stories.
- Default routing for price, macro, and earnings events is more conservative and easier to reason about. New controls include directional price thresholds, large-position overrides, macro immediate windows, quiet mode, portfolio-first defaults, and a dedicated `earnings_call_transcript` kind.
- ACP long-session compact leakage is fixed in this release. `codex_acp` and `opencode_acp` now suppress literal compact markers and opencode auto-summary blocks instead of forwarding them to end users, and Hone no longer auto-compacts sessions for runners that manage their own context.
- `gemini_acp` is now explicitly disabled. If you still run `agent.runner: gemini_acp`, you need to move to `codex_acp`, `opencode_acp`, `multi-agent`, or `function_calling`.
- The public roadmap page has been refreshed for `v0.2.6`, including 19 public skills, the latest multi-runner wording, and the current stable capability set.

### Who Should Update Soon

- Users or operators who rely on Event Engine proactive pushes and care about digest repetition, news noise, price thresholds, and delivery observability.
- Users running long `codex_acp` or `opencode_acp` sessions who have seen `Context compacted`, injected auto-summaries, or post-compact reply corruption.
- Deployments still configured with `gemini_acp`.

### What Will Change After Upgrading

- Digests for the same actor should repeat less often, rank higher-signal items more consistently, and keep social / low-quality sources from crowding the top of the batch.
- Event-engine config adds incremental fields for digest min-gap, news-upgrade caps, minimum direct-push price thresholds, large-position weight thresholds, macro immediate lookahead/grace windows, and the default uncertain-source classifier model `amazon/nova-lite-v1`. Existing configs do not require a migration.
- `config.example.yaml` now recommends `multi-agent` as the default runner. Existing deployed configs are not rewritten automatically, but new setups and fresh configs will show that recommendation.
- The web/public roadmap page now reflects this release version and the latest capability wording without requiring a frontend migration.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- If your config still uses `agent.runner: gemini_acp`, agent execution will now fail fast with a migration message. This is not a compatibility path; you must switch runners manually.
- The Event Engine uncertain-source classifier still benefits from manual spot checks, and real delivery quality still depends on channel credentials plus upstream API health.
