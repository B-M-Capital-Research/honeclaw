# v0.1.41

发布日期：2026-04-20

English readers: please scroll down for the English version below.

## 用户应该知道的

- 公开站点根层静态图片（`/company_profile.png`、`/hone_channels.jpg`、`/hone_introduction.jpg`、`/hone_page.jpg`、`/hone_solution.jpg`）在 v0.1.40 部署后 404 的 bug 已修复
- 公开站与管理台都改用 `ServeDir` 直接服务 `dist` / `dist-public` 根层文件，不再走自定义 SPA fallback handler
- 站点首页的外部图片卡片（公司画像、渠道、解决方案截图）现在可以正常显示
- 首页 `repo` / footer 里指向仓库文档的链接更新为真实的 GitHub 路径（README_ZH.md、AGENTS.md、CASES_ZH.md、CONTRIBUTING.md 等）

## 谁应该尽快更新

- 已部署 v0.1.40 公开站点的运维：图片加载是当前最直接的用户可见 bug，建议立即升级并重新发布

## 升级后会有什么变化

- `/` 下任何在 `dist-public/` 实际存在的文件都会被直接服务，无需在路由里显式声明
- `/chat`、`/me`、`/roadmap` 这类 SPA deep-link 依然会命中 `index.html`（由 `ServeDir::not_found_service` 处理）
- `/api/*`、`/api/public/*` 路径上的 404 仍显式返回空 body 404，不会被 SPA fallback 吞掉
- `crates/hone-web-api` 内部移除了 `handle_spa_index`、`handle_public_spa_index`、`serve_spa_index`、`web_index_path`、`public_web_index_path` 等冗余辅助函数

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- `crates/hone-web-api` 在 `cargo check --all-targets` 下仍存在 `auth.rs:111` 测试期 `public_auth_limiter` 字段缺失的预存编译告警，不影响发布构建

## English

Release date: 2026-04-20

### What Users Should Know

- Root-level public-site images (`/company_profile.png`, `/hone_channels.jpg`, `/hone_introduction.jpg`, `/hone_page.jpg`, `/hone_solution.jpg`) returned 404 after the v0.1.40 deploy — fixed
- Both the public site and admin console now serve `dist` / `dist-public` root files directly through `ServeDir`; the custom SPA fallback handler is gone
- Homepage image cards (company profile, channels, solution screenshots) render correctly again
- Repo / footer links on the homepage now point to real GitHub documents (README_ZH.md, AGENTS.md, CASES_ZH.md, CONTRIBUTING.md, etc.)

### Who Should Update Soon

- Anyone running the v0.1.40 public deployment — broken images are the most visible regression; upgrade and redeploy

### What Will Change After Upgrading

- Any file that actually exists in `dist-public/` is served directly from `/`, no explicit route required
- SPA deep links (`/chat`, `/me`, `/roadmap`) still resolve to `index.html` via `ServeDir::not_found_service`
- 404s on `/api/*` and `/api/public/*` are still returned explicitly with an empty body — never swallowed by the SPA fallback
- `handle_spa_index`, `handle_public_spa_index`, `serve_spa_index`, `web_index_path`, and `public_web_index_path` helpers were removed from `crates/hone-web-api`

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- `crates/hone-web-api` still has a pre-existing `auth.rs:111` test-build warning (`public_auth_limiter` missing) under `cargo check --all-targets`; release build is clean
