# v0.1.27

发布日期：2026-04-16

English readers: please scroll down for the English version below.

## 用户应该知道的

- Desktop release 运行链路现在正式收口到打包后的 `.app` 形态；仓库内的 release helper 不再把裸 `target/release/hone-desktop` 当作正式运行结果
- `launch.sh --release`、`scripts/build_desktop.sh` 与 runbook 现在统一使用 `~/Library/Caches/honeclaw/target` 作为 cache target，减少“重建了错误 target 树但线上仍在跑旧二进制”的误操作
- release supervisor 现在会持续写入 `data/runtime/current.pid`，后台重启链路可以先停旧 supervisor 再接管新实例，降低 split-brain、端口冲突和中途替换的风险
- 最近一轮高优先级缺陷修复也一并纳入：Feishu timeout 无回复、Feishu 附件 transcript 泄漏、会话 compact 伪摘要、Desktop release runtime supervision gap 等问题已收口

## 谁应该尽快更新

- 使用 macOS Desktop release app 作为长期运行入口的用户
- 需要通过 `launch.sh --release`、`restart_hone` 或 runbook 流程维护本地稳定运行态的维护者
- 最近遇到 Feishu 直聊无回复、Desktop 掉线、runner 设置切换异常或会话历史污染的用户

## 升级后会有什么变化

- 新的 release 构建产物会直接落到 `~/Library/Caches/honeclaw/target/release/bundle/...`
- macOS 上的正式桌面产物名会与本次版本一致，并继续以 `.app/Contents/MacOS/hone-desktop` 作为标准启动入口
- `restart_hone` 相关链路现在有稳定的 supervisor pid 契约可依赖，旧实例清理路径更可预测
- 直聊和定时任务失败分支的用户态兜底更完整，不再那么容易出现“卡住但没有最终回复”的静默失败

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`
- 本地仓库用户：重新执行 runbook 中的 release build 命令，并使用打包后的 `.app` 运行

## 已知注意事项

- 本次构建前已清理 `hone-desktop` 的一个非阻塞编译 warning；当前 release 打包链路应保持干净
- `docs/releases/v0.1.23.md` 到 `v0.1.26.md` 已存在，但最近实际 git tag 最高仍是 `v0.1.22`；本次 compare 范围因此按最新已存在 tag 计算
- `Heartbeat` 的 `JsonUnknownStatus` 静默跳过问题目前仍在活跃队列，尚未包含在这次“全部 bug 清零”的范围里

## English

Release date: 2026-04-16

### What Users Should Know

- The Desktop release runtime path now explicitly converges on the packaged `.app` form; repo helpers no longer treat the naked `target/release/hone-desktop` binary as the formal release runtime
- `launch.sh --release`, `scripts/build_desktop.sh`, and the runbook now consistently use `~/Library/Caches/honeclaw/target` as the cache target, which reduces cases where you rebuild the wrong target tree while the live runtime still serves stale binaries
- The release supervisor now keeps writing `data/runtime/current.pid`, so background restart flows can stop the old supervisor before taking over with the new instance
- This release also bundles the latest high-priority fixes, including the Feishu timeout no-reply issue, Feishu attachment transcript leakage, compact-summary hallucination, and the Desktop release runtime supervision gap

### Who Should Update Soon

- Users who rely on the macOS Desktop release app as their long-running runtime
- Maintainers who operate local stable runtime sessions via `launch.sh --release`, `restart_hone`, or the release runbook
- Users who recently hit Feishu no-reply failures, Desktop disconnects, runner-switch issues, or polluted session history

### What Will Change After Upgrading

- New release bundles now land under `~/Library/Caches/honeclaw/target/release/bundle/...`
- On macOS, the formal desktop artifact will keep matching the release version and continue to use `.app/Contents/MacOS/hone-desktop` as the standard runtime entrypoint
- `restart_hone` now has a stable supervisor-pid contract, so old-instance cleanup is more predictable
- User-facing failure fallbacks are more complete in both direct-chat and scheduler paths, reducing silent “stuck but no final reply” outcomes

### Install and Update

- `curl | bash` users: rerun the installer to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`
- Local repo users: rerun the release build command from the runbook and launch the packaged `.app`

### Known Notes

- A non-blocking `hone-desktop` warning was removed before the final packaging pass, so the current release build should stay clean
- `docs/releases/v0.1.23.md` through `v0.1.26.md` already exist, but the latest actual git tag is still `v0.1.22`; this release therefore uses the latest real tag as the compare base
- The `Heartbeat` `JsonUnknownStatus` silent-skip issue is still active and is not part of this release's “all bugs fixed” story
