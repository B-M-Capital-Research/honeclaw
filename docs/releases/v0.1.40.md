# v0.1.40

发布日期：2026-04-20

English readers: please scroll down for the English version below.

## 用户应该知道的

- 公开站点正式上线：开放控制台 (`/`、`/chat`、`/me`、`/roadmap`) 现在可生产可用，CORS、SPA 回退、Cookie 鉴权全部打通
- 路线图与文档页重写：新增快速开始、能力矩阵、内置渠道与技能清单、架构图、FAQ 等十个区块，所有数据按真实仓库能力填写
- 公开聊天页移动端布局与刷新后会话恢复状态修复
- 报表桥接模式与 validate code 入口的修复让多渠道流程更稳定
- 技能体系合并：Hone 财务技能面收敛、移除 `sync-open-source` 项目技能、仓库级 Codex 技能迁入 `.agents/`

## 谁应该尽快更新

- 运维公开站点的部署：本次包含 CORS 与 SPA fallback 的关键修复，缺失会导致刷新 404 与跨域失败
- 使用财务/项目技能的用户：技能清单已重组，旧的 `sync-open-source` 入口已移除
- 依赖报表桥接或邀请码登录链路的渠道接入方

## 升级后会有什么变化

- 公开站点新增 `/roadmap` 完整文档中心，含 curl/brew/source 安装指引
- API surface 不变；公开端 `/api/public/*` 仍走 HttpOnly + SameSite=Strict Cookie 鉴权
- `.gitignore` 收敛了 `packages/app/dist-public/` 的目录规则，构建产物不会再泄漏成未跟踪文件
- Tauri 桌面端版本号同步至 0.1.40，与 workspace 对齐

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- `crates/hone-web-api` 在 `cargo check --all-targets` 下存在 `auth.rs:111` 测试期 `public_auth_limiter` 字段缺失的预存编译告警，不影响发布构建（`cargo check --workspace` 通过）

## English

Release date: 2026-04-20

### What Users Should Know

- The open public site is live: `/`, `/chat`, `/me`, and `/roadmap` are production-ready with CORS, SPA fallback, and cookie auth wired through
- The roadmap/docs page was rebuilt: ten new sections covering quick start, capability matrix, built-in channels and skills, architecture, FAQ — every value sourced from the real repository
- Mobile layout for the public chat and post-refresh session restore are fixed
- Report bridge mode and the validate-code gate were stabilized for multi-channel flows
- Skills surface consolidated: Hone finance skills regrouped, `sync-open-source` removed, repo-level Codex skills moved into `.agents/`

### Who Should Update Soon

- Operators of the public deployment — this release includes the critical CORS + SPA fallback fixes; without them, deep links 404 and cross-origin requests fail
- Users of finance/project skills — the skill list has been reorganized and the old `sync-open-source` entry is gone
- Channel integrators relying on report bridge mode or invite-code login

### What Will Change After Upgrading

- The public site gains a full `/roadmap` documentation hub with curl/brew/source install guides
- API surface unchanged; `/api/public/*` still uses HttpOnly + SameSite=Strict cookie auth
- `.gitignore` now uses a single `packages/app/dist-public/` directory rule, so build artifacts no longer leak as untracked files
- Tauri desktop version bumped to 0.1.40 to match the workspace

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- `crates/hone-web-api` has a pre-existing `auth.rs:111` test-build warning (`public_auth_limiter` missing) under `cargo check --all-targets`; release build (`cargo check --workspace`) is clean
