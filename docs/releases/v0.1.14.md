# v0.1.14

发布日期：2026-04-13

English readers: please scroll down for the English version below.

## 用户应该知道的

- `codex_acp`、`gemini_acp`、`opencode_acp` 的 `session/prompt` 不再是固定 300 秒总超时；现在改为“5 分钟无进展超时 + 20 分钟总超时”
- 渠道侧用户可见的运行进度不再原样透出 actor sandbox 绝对路径；sandbox 内路径会改写成相对路径，sandbox 外绝对路径会被遮罩
- 默认示例配置和当前仓库的 `config.yaml` 都已补上新的 ACP timeout 字段，便于直接对齐运行时行为

## 谁应该尽快更新

- 经常发起长研究、长写作或多工具链路任务，容易在 300 秒附近被误判超时的用户
- 关注运行日志、进度回显或工具状态文案，不希望绝对路径泄漏本机目录结构的用户
- 维护本仓库部署配置，希望 canonical config 与运行时行为保持一致的用户

## 升级后会有什么变化

- 只要 ACP 在 5 分钟内持续有 chunk、tool update、usage update 或其它事件，prompt 轮次会继续执行，不会因为固定 300 秒墙钟超时被截断
- 如果 ACP 连续 300 秒没有任何输出，会明确报 `idle timeout`；如果整轮超过 1200 秒，则报 `overall timeout`
- 运行进度中的 `company_profiles/...`、`data/...` 之类 sandbox 内路径会以相对路径展示，sandbox 外路径只显示 `<absolute-path>/basename`

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 如果你在自己的部署配置里手动覆盖了 ACP timeout，仍以你的显式配置为准
- 这次 tag 会包含当前工作区内的全部已保存变更，而不只是一处 timeout 修复

## English

Release date: 2026-04-13

### What Users Should Know

- `session/prompt` in `codex_acp`, `gemini_acp`, and `opencode_acp` no longer uses a fixed 300-second wall-clock timeout; it now uses “5-minute idle timeout + 20-minute overall timeout”
- User-visible runtime progress on channels no longer exposes raw absolute actor-sandbox paths; paths inside the sandbox are rewritten as relative paths and paths outside it are masked
- Both the default sample config and the current repo’s `config.yaml` now include the new ACP timeout fields so runtime behavior is easier to align

### Who Should Update Soon

- Users who frequently run long research, long writing, or multi-tool workflows and were getting incorrectly cut off around 300 seconds
- Users who care about runtime logs, progress echo, or tool-status wording and do not want absolute local paths leaked
- Users maintaining deployment config in this repo and wanting canonical config plus runtime behavior to stay aligned

### What Will Change After Upgrading

- As long as ACP continues emitting chunks, tool updates, usage updates, or other events within 5 minutes, the prompt turn keeps running instead of being cut off by a fixed 300-second wall clock
- If ACP is silent for 300 seconds, it now reports `idle timeout`; if the whole turn exceeds 1200 seconds, it reports `overall timeout`
- Sandbox-internal paths such as `company_profiles/...` and `data/...` are now shown as relative paths in user-visible progress, while paths outside the sandbox appear only as `<absolute-path>/basename`

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- If you override ACP timeout values explicitly in your own deployment config, your explicit config still wins
- This tag includes all saved worktree changes present at release time, not just the timeout fix
