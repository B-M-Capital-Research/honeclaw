# v0.1.16

发布日期：2026-04-13

English readers: please scroll down for the English version below.

## 用户应该知道的

- 飞书渠道现在会更稳地处理表格内容：标准 Markdown 表格仍会正常渲染成飞书表格
- 如果模型误输出了损坏的飞书原始 `<table .../>` 标签，渠道不再把坏标签原样发给用户
- 飞书提示词也已收紧，模型会被明确要求只写标准 Markdown 表格，不要手写飞书原始卡片标签

## 谁应该尽快更新

- 经常在飞书里接收包含表格的研究结论、监控结果或股票列表的用户
- 遇到过飞书消息里直接出现 `<table columns=... data=.../>` 这类未渲染源码的用户
- 需要依赖飞书渠道稳定展示结构化列表的日常使用者

## 升级后会有什么变化

- 标准 Markdown 表格会继续自动转换成飞书兼容的表格卡片
- 非法、截断或字段错误的 raw table 会安全降级为普通文本，而不是直接破坏消息渲染
- 流式更新、占位消息更新和最终消息发送都共享这层保护

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 当前工程护栏主要覆盖 raw `<table .../>`；其它飞书原始卡片标签仍然不建议由模型直接手写
- 如果表格本身过宽、过长或超出飞书卡片限制，仍可能需要改写成列表或更短的表格

## English

Release date: 2026-04-13

### What Users Should Know

- The Feishu channel now handles table content more safely: standard Markdown tables still render as Feishu tables
- If the model accidentally emits a broken raw Feishu `<table .../>` tag, the channel no longer sends the broken tag directly to users
- Feishu prompt guidance has also been tightened so the model is explicitly told to write only standard Markdown tables instead of hand-authoring raw Feishu card tags

### Who Should Update Soon

- Users who often receive research conclusions, monitoring results, or stock lists with tables in Feishu
- Users who have seen raw `<table columns=... data=.../>` source text appear unrendered in Feishu messages
- Users who depend on stable structured rendering in the Feishu channel

### What Will Change After Upgrading

- Standard Markdown tables will continue to be converted automatically into Feishu-compatible table cards
- Invalid, truncated, or schema-broken raw table tags now degrade safely into plain text instead of breaking message rendering
- Stream updates, placeholder updates, and final message sends all share this guardrail

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- The current guardrail mainly covers raw `<table .../>` tags; other raw Feishu card tags are still not recommended for model-authored output
- If a table is too wide, too long, or exceeds Feishu card limits, it may still need to be rewritten as a list or a smaller table
