# v0.1.25

发布日期：2026-04-15

English readers: please scroll down for the English version below.

## 用户应该知道的

- `codex_acp`、`opencode_acp`、`gemini_cli`、`codex_cli` 与 `function_calling` 现在都会把本轮 assistant/tool 尾部转成可恢复 transcript 持久化下来，下一轮反序列化时不再只剩“最终一句话”
- ACP 与 reasoning runner 现在会把执行中的工具渲染成更接近用户视角的状态文案，例如 `正在执行：python - <<'PY' [truncated]`、`data_fetch quote AAOI,COHR`、`read uploads`
- `hone-cli` 新增 `probe` 命令，可以直接指定 identity 和 query，观察一次真实 runner 的 `progress / tool / done` 事件
- multi-agent 搜索阶段不再把“我先查一下”“先核实两个点”这类工作中提示误判成可直接返回给用户的最终答案

## 谁应该尽快更新

- 依赖长会话上下文恢复、希望 tool-call 历史在下一轮仍能稳定重建的用户
- 正在排查 `codex_acp` / `opencode_acp` / `gemini_cli` / `codex_cli` 链路问题的开发者
- 需要在本机快速复现指定 actor、runner 与工具链路的维护者

## 升级后会有什么变化

- session restore 现在可以恢复 assistant 的 tool-calls 元数据，以及对应的 tool message；下一轮上下文更接近真实执行序列
- runner 对外展示的工具状态会带上裁剪后的命令、路径或参数摘要，而不是只有泛化的 tool 名称
- `codex_cli` 非交互执行改为使用当前版本支持的 `--full-auto` 组合，不再依赖已失效的 `--ask-for-approval never`
- 可以用 `hone-cli probe --channel ... --user-id ... --query ...` 直接验证某个 runner 是否真实调用了工具，并查看过程中发出的事件

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次发布主要收敛的是 transcript 持久化、工具状态渲染和本地 probe 能力，不包含所有 runner 的统一 prompt/history budget 收口
- `gemini_acp` 的真实 prompt 阶段联通性问题仍依赖上游 CLI / ACP 行为，当前发布没有把它变成已验证通过的默认链路

## English

Release date: 2026-04-15

### What Users Should Know

- `codex_acp`, `opencode_acp`, `gemini_cli`, `codex_cli`, and `function_calling` now persist the assistant/tool tail from the current turn as a restorable transcript instead of collapsing everything into only the final answer
- ACP and reasoning runners now expose tool activity with user-facing labels such as `正在执行：python - <<'PY' [truncated]`, `data_fetch quote AAOI,COHR`, and `read uploads`
- `hone-cli` now includes a `probe` command so you can run one real query against a chosen identity and inspect the emitted `progress / tool / done` events
- The multi-agent search stage no longer treats short working notes like “let me check” as a final answer that can skip the answer stage

### Who Should Update Soon

- Users who depend on stable long-session restoration and want tool-call history to survive into the next turn
- Developers debugging `codex_acp`, `opencode_acp`, `gemini_cli`, or `codex_cli` execution chains
- Maintainers who need a fast local way to probe a specific actor, runner, and tool path

### What Will Change After Upgrading

- Session restoration now rehydrates assistant tool-call metadata together with the matching tool messages, so the next-turn context stays closer to the real execution transcript
- User-visible tool status lines now include truncated commands, paths, or argument summaries instead of only a generic tool name
- Non-interactive `codex_cli` runs now use the currently supported `--full-auto` path instead of the removed `--ask-for-approval never` flags
- You can run `hone-cli probe --channel ... --user-id ... --query ...` to verify that a runner really called tools and to inspect the emitted events

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release focuses on transcript persistence, tool-status rendering, and local probe capability; it does not yet unify prompt/history budgeting across every runner
- `gemini_acp` prompt-phase connectivity still depends on upstream CLI / ACP behavior, so this release does not promote it as a fully validated default path
