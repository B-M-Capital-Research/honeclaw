# v0.9.0

发布日期：2026-05-11

English readers: please scroll down for the English version below.

## 用户应该知道的

### 1. 渠道配置与定时推送路由更完整

Hone 现在把 Feishu、Discord、Telegram、iMessage 的入站配置和定时任务投递目标收口到同一套配置面：

- CLI / Web Settings 都可以配置渠道启用状态、`chat_scope`、allowlist，以及 iMessage 的 `target_handle`。
- 定时任务会优先沿用“创建任务的原渠道”投递，不再引入单独的 `home_channel` 默认投递概念。
- 缺少投递目标时会确定性拒绝或记录，不再默默发错渠道。
- 新增 `hone-cli channels targets`，可以查看 cron 可用的 typed channel target directory。

### 2. 源码目录启动路径改为 CLI 驱动

源码 checkout 现在推荐直接用 CLI 启动本地 Web 体验：

- `cargo run -p hone-cli -- start --build web`
- 发行版 / Homebrew / 其它安装方式继续通过 `hone-cli start` 进入同一条启动路径。
- `launch.sh` 仅保留兼容入口，不再作为 README 中的主要启动方式。

### 3. 推送质量与 event-engine 稳定性增强

本 release 合并了一组主动推送质量修复：

- earnings surprise 事件增加 earnings quality review，减少低质量财报推送。
- poller tick 加入上限，单个卡住的数据源不会长期压住下一轮轮询。
- SEC filing 摘要改为摘抄重试并限制 OpenRouter token 预算，降低 402 / prompt overflow 风险。
- mainline distill 预算收紧，避免一次蒸馏消耗过高。
- heartbeat JSON 恢复、重复抑制、watchlist hit-zone context、商品因果 guard、stale market data fallback 等路径已加固。

### 4. 会话与调度可靠性修复

- 修复 session SQLite shadow-write / shadow backfill，使 CLI、Web、channel 恢复路径更一致。
- 修复 Feishu scheduler 卡在 pending、被中断后未 finalize、空回复误判成功等问题。
- 修复 active scheduler / desktop startup 中一批会影响后台任务继续运行的边界问题。

## 谁应该尽快更新

- 使用 Feishu / Discord / Telegram / iMessage 渠道和定时任务推送的用户。
- 从源码目录启动 Hone 的用户，尤其是希望用同一条 CLI 路径验证本地构建的人。
- 依赖 event-engine 主动推送、财报事件、SEC filing 摘要或每日 digest 的用户。
- 遇到 scheduler pending、heartbeat 重复/丢失、session 恢复不一致的用户。

## 升级后会有什么变化

- Web Settings 的渠道页会读取真实 `config.yaml` / effective config；浏览器模式下也不再把 SPA HTML 当成 JSON 响应。
- 定时任务投递会更严格依赖创建任务时的渠道来源；缺少渠道目标时会显式失败或留下记录。
- 源码启动命令统一为 `cargo run -p hone-cli -- start --build web`；发行版用户使用 `hone-cli start`。
- event-engine 的财报、SEC、heartbeat、digest 输出会更保守，推送量可能降低但信号质量更高。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`
- 源码用户：拉取最新代码后运行 `cargo run -p hone-cli -- start --build web`

## 已知注意事项

- 已运行的本地 Hone 进程需要重启后才会读取新版二进制和配置路径。
- 渠道 Settings 只管理现有平台，不新增平台；具体 token / secret 仍需按各平台要求配置。
- 架构 SVG 已随本次发版同步刷新（v0.9.0、`/channel-settings`、earnings quality review、近期里程碑）。

## English

Release date: 2026-05-11

### What Users Should Know

#### 1. More complete channel configuration and scheduled delivery routing

Hone now brings Feishu, Discord, Telegram, and iMessage channel configuration together with scheduled delivery targets:

- CLI and Web Settings can configure channel enabled state, `chat_scope`, allowlists, and the iMessage `target_handle`.
- Scheduled jobs prefer delivering back to the channel where the job was created, instead of introducing a separate `home_channel` default.
- Missing delivery targets now fail or get recorded deterministically instead of silently routing to the wrong place.
- `hone-cli channels targets` exposes the typed channel target directory available to cron delivery.

#### 2. Source checkout startup is now CLI-driven

For source checkouts, the recommended local Web startup path is now:

- `cargo run -p hone-cli -- start --build web`
- Release, Homebrew, and other installed setups continue to use the same `hone-cli start` path.
- `launch.sh` remains only as a compatibility entrypoint and is no longer the primary README startup path.

#### 3. Push quality and event-engine stability improvements

This release includes a focused batch of proactive push hardening:

- Earnings surprise events now run through an earnings quality review gate to reduce low-signal earnings pushes.
- Poller ticks are bounded so a stuck source no longer suppresses future polling rounds indefinitely.
- SEC filing summarization now retries with selected excerpts and bounded OpenRouter token budgets, reducing 402 / prompt-overflow failures.
- Mainline distillation uses a tighter token budget.
- Heartbeat JSON recovery, duplicate suppression, watchlist hit-zone context, commodity causality guards, and stale market data fallback paths were hardened.

#### 4. Session and scheduler reliability fixes

- Fixed session SQLite shadow-write / shadow backfill so CLI, Web, and channel recovery paths stay more consistent.
- Fixed Feishu scheduler runs getting stuck in pending, interrupted runs not finalizing, and empty replies being treated as successful sends.
- Fixed active scheduler and desktop startup edge cases that could prevent background tasks from continuing reliably.

### Who Should Update Soon

- Users relying on Feishu, Discord, Telegram, or iMessage channels and scheduled push delivery.
- Users starting Hone from a source checkout and wanting one CLI-driven build/start path.
- Users relying on event-engine proactive pushes, earnings events, SEC filing summaries, or daily digests.
- Users affected by scheduler pending states, heartbeat duplicate/loss issues, or inconsistent session recovery.

### What Will Change After Upgrading

- The Web Settings channel page reads the real `config.yaml` / effective config; browser mode no longer treats SPA HTML as JSON.
- Scheduled delivery is stricter about using the originating channel target; missing targets fail visibly or leave a record.
- Source startup is unified under `cargo run -p hone-cli -- start --build web`; installed users use `hone-cli start`.
- Event-engine earnings, SEC, heartbeat, and digest output is more conservative; volume may drop while signal quality improves.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`
- Source users: pull the latest code and run `cargo run -p hone-cli -- start --build web`

### Known Notes

- Already-running local Hone processes need a restart before the new binaries and config paths take effect.
- Channel Settings manages existing platforms only; tokens and secrets still need to be configured according to each platform's requirements.
- The architecture SVG was refreshed for this release (v0.9.0, `/channel-settings`, earnings quality review, and recent milestones).
