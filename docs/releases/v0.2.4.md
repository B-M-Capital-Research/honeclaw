# v0.2.4

发布日期：2026-04-22

English readers: please scroll down for the English version below.

## 用户应该知道的

- Event Engine 的盘前 / 盘中 / 晨间摘要现在会在有原文 URL 的条目后追加一个 `🔗` 入口。Telegram、Discord 和 Feishu 会把它渲染成可点击的图标链接，不再把整段 URL 展开到正文里。
- 新闻源的文章级 URL 会继续从结构化字段读取，例如 FMP `stock_news.url` 会进入 `MarketEvent.url`；社交源也会从 Telegram preview / Truth Social API 的帖子链接字段读取原文地址。
- Telegram 公开频道和 Truth Social 的社交事件标题上限从 80 字符放宽到 240 字符，摘要里不再轻易把 `JUST IN: ...` 标题截成半句话。
- 已经进入 digest buffer 的社交事件如果带有 `payload.raw_text`，渲染时也会优先恢复原始首行，减少旧队列里标题被截断的情况。
- 本次 release 还吸收了最新 bug 台账证据更新，便于继续排查 Feishu 部分回复、油价地缘事件误报和 scheduler heartbeat 异常。

## 谁应该尽快更新

- 依赖 Telegram / Discord / Feishu 接收市场摘要、需要点击回原文核验新闻来源的用户。
- 订阅 Telegram 公开频道或 Truth Social 账号，并希望摘要标题保留完整语义的用户。
- 正在排查 event-engine、scheduler、Feishu 出站质量问题的部署维护者。

## 升级后会有什么变化

- 富文本渠道会显示紧凑的 `🔗` 图标；点击后打开该条事件的结构化原文 URL。
- 纯文本渠道无法隐藏超链接，因此只显示 `🔗 domain.com` 作为来源提示，不展开完整长链接。
- Feishu Event Engine 出站会优先发送 `post` 富文本消息；如果传入的内容不是 Feishu post JSON，仍会回退为普通 text 消息。
- Telegram Event Engine 出站现在使用 HTML parse mode；摘要里的 HTML 会由 renderer 统一转义，避免链接或标题中的特殊字符破坏消息格式。
- 对 High 事件的 LLM polish 仍只用于纯文本格式；富文本渠道保留 renderer 的结构化输出，避免润色文本破坏可点击链接。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 能否打开到具体原文取决于上游是否提供文章级 URL。当前 FMP 新闻、Telegram preview 和 Truth Social API 都会优先写入结构化 URL。
- Telegram 默认禁用网页预览，避免摘要被链接卡片撑开；原文仍可通过 `🔗` 打开。

## English

Release date: 2026-04-22

### What Users Should Know

- Event Engine pre-market, intraday, and morning digests now append a compact `🔗` source entry when an event has a source URL. Telegram, Discord, and Feishu render it as a clickable icon link instead of expanding long URLs into the message body.
- News article URLs continue to come from structured fields. For example, FMP `stock_news.url` is stored as `MarketEvent.url`; social sources also read post URLs from Telegram preview pages and the Truth Social API.
- Telegram public channel and Truth Social social-event title limits increased from 80 to 240 characters, so digest items are less likely to cut `JUST IN: ...` headlines in the middle.
- Social events that were already queued can recover the original first line during digest rendering when `payload.raw_text` is present.
- This release also includes the latest bug ledger evidence updates for Feishu partial replies, oil-price geopolitical false positives, and scheduler heartbeat issues.

### Who Should Update Soon

- Users who receive market digests through Telegram, Discord, or Feishu and want one-tap access to the original source.
- Users subscribed to Telegram public channels or Truth Social accounts who need digest titles to preserve full context.
- Operators investigating event-engine, scheduler, or Feishu outbound quality issues.

### What Will Change After Upgrading

- Rich-text channels show a compact `🔗` icon. Clicking it opens the event's structured source URL.
- Plain-text channels cannot hide hyperlinks, so they show only `🔗 domain.com` as a compact source hint instead of expanding the full URL.
- Feishu Event Engine outbound messages now prefer `post` rich text. Non-post payloads still fall back to normal text messages.
- Telegram Event Engine outbound messages now use HTML parse mode. The renderer escapes titles and links before sending.
- LLM polishing for High events remains limited to plain-text output. Rich-text channels keep renderer-structured messages to avoid breaking clickable links.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- Opening the exact source depends on the upstream source providing an article-level URL. FMP news, Telegram preview, and the Truth Social API currently populate structured URLs when available.
- Telegram web previews are disabled by default so digests stay compact; the source remains available through the `🔗` icon.
