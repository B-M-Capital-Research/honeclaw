# v0.2.1

发布日期:2026-04-22

English readers: please scroll down for the English version below.

> v0.2.0 把 OutboundSink 接通后,bug 巡逻立刻暴露 3 条质量问题:律所广告被打 High 直推 Telegram、heartbeat 写盘失败仍报"健康"、新闻窗口收敛 burst 污染 digest。v0.2.1 把这三条一次修干净,并把 news 严重性判定从"关键字一刀切"升级为 **来源信誉分类 + 反模板 + per-actor LLM 仲裁** 的级联。

## 用户应该知道的

### 律所广告 / 财经博客 不再冒充重大事件

之前 `pollers/news.rs` 只看关键字 (`fraud`/`lawsuit`/`class action`...) 命中即升 High,导致 globenewswire 的 SHAREHOLDER ALERT、Bernstein Liebhard / Rosen Law 系列模板,加上 fool.com / zacks.com / youtube.com / defenseworld.net 这类博客内容,大量挤进 immediate sink,挤掉 daily high cap。

v0.2.1 引入**来源信誉分类**:

- `Trusted`(reuters / bloomberg / wsj / sec.gov / fda.gov 等):关键字命中才升 High
- `PrWire`(globenewswire / prnewswire / businesswire / newsfilecorp 等):整域降 Low,关键字命中也不升
- `OpinionBlog`(fool / zacks / youtube / cnet / 247wallst / etftrends / defenseworld / finbold / schaeffersresearch 等 18 个域):整域降 Low,**也不进 LLM 仲裁**
- `Uncertain`:其它源,severity 维持 Low,但允许进 LLM 仲裁

外加**律所模板标题反模板**(`shareholder alert` / `class action lawsuit has been filed` / `Bernstein Liebhard` / `Rosen Law` 等):无论来源都强制 Low,且不可被 LLM 复活。

### `Uncertain` 来源走 LLM 仲裁,可按 actor 配置重要性

对来源不确定但标题命中关键字的新闻,router 阶段会对每个 actor 单独唤起 OpenRouter `google/gemini-3-flash-preview` 做"是否符合本 actor 重要性标准"的判定,被判 important 时把 severity 升到 Medium 进 digest。新增 `notif_prefs/<actor>.json` 字段:

```json
{
  "news_importance_prompt": "公司或潜在影响公司长期逻辑和宏观叙事的重大事件"
}
```

未配置时使用 `event_engine.news_importance_prompt` 全局默认(同上)。LLM 调用按 `(event_id, prompt_hash)` + `(title_norm, symbols_norm, prompt_hash)` 二级缓存,避免 N 个 actor × M 条新闻把 OpenRouter 配额打爆。

实测对 12 股持仓 + 5 股新测试集(JPM/XOM/PFE/META/TSLA),LLM 候选数下降 ~37%,真信号(CEO 退任 / 重大合同 / 集体诉讼 / 监管调查)100% 召回。

### Heartbeat 写盘失败不再"伪装健康"

之前 `crates/hone-core/src/heartbeat.rs::write_snapshot` 写失败仅 `tracing::warn!`,磁盘里仍是上一次成功 snapshot;`web-api/meta.rs::heartbeat_is_fresh` 只比对 `updated_at` 与 `Utc::now()`,75s 内即视 fresh。结果:磁盘满几秒到几分钟内,进程已经丢心跳 tick,但 status API 仍报"healthy"。

v0.2.1 双通道并行:

1. **内存计数器** `HeartbeatMetrics`:`AtomicU32` 累计连续写失败次数 + 最近一条错误信息
2. **磁盘 sidecar** `*.heartbeat.error`:每次写失败原子覆盖,记录 pid / 失败次数 / 错误文本

消费方 `web-api/meta.rs::heartbeat_health` 综合两个信号:即使 `*.heartbeat.json` mtime 仍 < 75s,只要 sidecar 里有同 pid 的活跃错误记录,status 直接报 `degraded`。

### News digest burst 不再被同 symbol 升级压塌

之前 `router.rs::maybe_upgrade_news` 在 `[news_ts - 1d, news_ts + 2d]` 窗口内只要命中 `earnings_upcoming` / `sec_filing` 等 hard signal 就升级,无 per-symbol 上限,导致单秒 15-27 条同 symbol news 全被升 Medium 挤进 digest 顶端,把别的 symbol 挤掉。

v0.2.1 引入 **per-symbol per-tick 升级上限**(配置项 `event_engine.thresholds.news_upgrade_per_symbol_per_tick`,默认 3,设为 0 表示无限制)。`process_events` 在 tick 边界调 `router.reset_tick_counters()` 清零。

### 来源域名扩展(向后兼容)

只是在内置三类列表里加了几条:`PR_WIRE_DOMAINS` 加 `newsfilecorp.com`,`OPINION_BLOG_DOMAINS` 全新引入 18 个域。**marketbeat.com 故意不加**——该域偶有真合同/研报新闻(如 "Broadcom & Meta Extend AI Pact Into 2029"),整域降级会丢真信号;`tests/marketbeat_kept_as_uncertain_to_preserve_real_signals` 守护这一行为。

## 已知限制

- LLM 仲裁结果存在 ±1 条边界漂移(同公司同主题不同标题在不同源上偶尔会判定不一致),L2 缓存只能 dedupe 同标题。要彻底消漂移需要 OpenRouter 暴露 `temperature=0` + `seed`,本次未做
- `OPINION_BLOG_DOMAINS` 是硬编码内置列表,目前不可通过 config 覆盖。如果碰到新的高噪音域,需要发 patch
- FMP 偶尔把无关公司新闻关联到 ticker(例如 Tesla 字段下出现 "Solis lithium with Rio Tinto" 这种小公司新闻),目前 prompt 会把这种"主语不是目标公司"的判 no,但仍会浪费一次 LLM 调用;router 层 ticker-relevance hard 校验未做
- `news_upgrade_per_symbol_per_tick` 是全局上限,不分 symbol 重要性。后续若发现核心持仓被压制,再拆 per-symbol override

# v0.2.1 (English)

Release date: 2026-04-22

> v0.2.0 wired up `OutboundSink`s; bug patrol immediately surfaced three quality issues: legal-ad noise pushed as High to Telegram, heartbeat write failures still reporting "healthy", and news convergence bursts polluting digests. v0.2.1 fixes all three, and upgrades news severity classification from "keyword-only" to a **source-reputation + anti-template + per-actor LLM arbitration** cascade.

## What users should know

### Legal ads / financial blogs no longer impersonate major events

Previously `pollers/news.rs` upgraded any keyword hit (`fraud` / `lawsuit` / `class action`...) to High, allowing globenewswire's SHAREHOLDER ALERT and Bernstein Liebhard / Rosen Law templates, plus fool.com / zacks.com / youtube.com / defenseworld.net blog content, to flood the immediate sink and consume the daily high cap.

v0.2.1 introduces **source reputation classification**:

- `Trusted` (reuters / bloomberg / wsj / sec.gov / fda.gov ...): keyword hit allowed to upgrade to High
- `PrWire` (globenewswire / prnewswire / businesswire / newsfilecorp ...): whole-domain Low, no upgrade
- `OpinionBlog` (fool / zacks / youtube / cnet / 247wallst / etftrends / defenseworld / finbold / schaeffersresearch ... 18 domains): whole-domain Low, **also skipped by LLM arbitration**
- `Uncertain`: other sources, severity stays Low but eligible for LLM arbitration

Plus a **legal-ad title anti-template** (`shareholder alert` / `class action lawsuit has been filed` / `Bernstein Liebhard` / `Rosen Law` ...): forced to Low regardless of source, and cannot be resurrected by LLM.

### `Uncertain` sources go through per-actor LLM arbitration

For uncertain-source news with keyword hits, the router invokes OpenRouter `google/gemini-3-flash-preview` per actor to judge "does this match the actor's importance standard"; if yes, severity is bumped to Medium for the digest. New per-actor `notif_prefs/<actor>.json` field:

```json
{
  "news_importance_prompt": "Major events affecting the company's long-term thesis or macro narrative"
}
```

Without it, falls back to `event_engine.news_importance_prompt` global default. LLM calls are cached with two layers: `(event_id, prompt_hash)` + `(title_norm, symbols_norm, prompt_hash)`, to avoid N actors × M news blowing OpenRouter quota.

Real-world e2e against 12-stock portfolio + 5-stock new test set (JPM/XOM/PFE/META/TSLA): LLM call volume reduced ~37%, real signal recall (CEO step-down / major contracts / class action / regulatory probes) at 100%.

### Heartbeat write failures no longer fake "healthy"

Previously `crates/hone-core/src/heartbeat.rs::write_snapshot` only `tracing::warn!`-ed on failure; on disk the previous successful snapshot remained, and `web-api/meta.rs::heartbeat_is_fresh` only compared `updated_at` to `Utc::now()`, calling anything within 75s fresh. Net result: during disk-full windows the process had already lost ticks, but the status API kept saying "healthy".

v0.2.1 runs two parallel channels:

1. **In-memory counter** `HeartbeatMetrics`: `AtomicU32` of consecutive write failures + last error string
2. **On-disk sidecar** `*.heartbeat.error`: atomically rewritten on every failure with pid / failure count / error text

The consumer `web-api/meta.rs::heartbeat_health` combines both: even if `*.heartbeat.json` mtime is still < 75s, an active sidecar entry for the same pid flips status to `degraded`.

### News digest bursts no longer crowded out by single-symbol upgrades

Previously `router.rs::maybe_upgrade_news` upgraded any news within the `[news_ts - 1d, news_ts + 2d]` window that overlapped a hard signal (`earnings_upcoming` / `sec_filing`), with no per-symbol cap. Single-second bursts of 15–27 same-symbol news could all get bumped to Medium and crowd the digest top, evicting other symbols.

v0.2.1 adds a **per-symbol per-tick upgrade cap** (config `event_engine.thresholds.news_upgrade_per_symbol_per_tick`, default 3, 0 = unlimited). `process_events` calls `router.reset_tick_counters()` at tick boundaries.

### Domain list expansions (backward-compatible)

Just additions to the built-in classification: `PR_WIRE_DOMAINS` adds `newsfilecorp.com`, and `OPINION_BLOG_DOMAINS` is brand-new with 18 domains. **marketbeat.com is intentionally excluded** — it carries real contract / research news (e.g. "Broadcom & Meta Extend AI Pact Into 2029"); whole-domain demotion would lose real signal. Test `marketbeat_kept_as_uncertain_to_preserve_real_signals` guards this behavior.

## Known limits

- LLM arbitration has ±1 boundary drift (same company / same topic with different titles on different sources may classify inconsistently); L2 cache only dedupes identical titles. Fully deterministic results require `temperature=0` + `seed` exposure from OpenRouter, not done in this release
- `OPINION_BLOG_DOMAINS` is hardcoded; not currently configurable. New high-noise domains require a patch
- FMP occasionally associates unrelated company news to a ticker (e.g. "Solis lithium with Rio Tinto" appearing under Tesla); the prompt currently catches such off-topic items as no, but still wastes one LLM call. A router-level ticker-relevance hard check is not yet implemented
- `news_upgrade_per_symbol_per_tick` is a global cap; doesn't differentiate symbol importance. If core holdings get suppressed in practice, we'll add per-symbol override later
