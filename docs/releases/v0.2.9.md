# v0.2.9

发布日期：2026-04-24

English readers: please scroll down for the English version below.

## 用户应该知道的

- 定时任务与心跳解析更稳。模型输出前带 `<think>` 等内部 reasoning 块时，调度器会先剥离内部块再解析 JSON，避免因为 `status=unknown` 静默跳过应执行的 heartbeat。
- 长耗时 runner 不再长时间无进展提示。Hone 现在把 runner 进度 tick 抽成可复用 watchdog，长模型调用期间会持续产生活跃信号，降低 Feishu / scheduled task 看起来“卡住”的概率。
- 金融一致性 smoke 覆盖更贴近真实场景。新增的 MiniMax 路径会用冲突油价样本验证模型拒绝广播并标记 `data-consistency` 异常，减少错误价格被推送出去的风险。
- Codex ACP 的模型与推理强度配置拆开了。`agent.codex_acp.model` 只作为模型名传入，`agent.codex_acp.variant` 会作为 Codex CLI 的 `model_reasoning_effort` 传入；历史写成 `model/variant` 的配置会回退成基础模型名。

## 谁应该尽快更新

- 使用 Feishu / Telegram 定时任务、heartbeat 或 scheduler 自动推送的用户。
- 遇到过 scheduled task 生成成功但后续静默、placeholder 长时间不更新、或心跳任务被跳过的用户。
- 使用 `codex_acp` 并配置了 `variant: medium|high|xhigh` 的用户。
- 维护真实 LLM smoke / bug 复现链路的开发者。

## 升级后会有什么变化

- scheduler 对模型 reasoning 前缀更宽容，内部思考块不再干扰 heartbeat JSON 判断。
- 长模型调用期间的进度 tick 可通过 `HONE_AGENT_RUN_PROGRESS_TICK_SECS` 在 smoke 场景中缩短，用于验证 watchdog 行为；生产默认节奏不需要用户配置。
- Codex ACP 启动参数会包含 `-c model_reasoning_effort="..."`，不会再把 `medium/high/xhigh` 拼进模型 id。
- `config.example.yaml` 已同步说明 Codex ACP `variant` 的新语义。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 新增的真实 LLM smoke 依赖本机有效模型配置和外部服务可用性，不属于默认 CI 门禁。
- tag 推送会触发远端 release workflow；本地只能确认 tag 已推送，最终产物发布结果仍以 GitHub Actions 为准。

## English

Release date: 2026-04-24

### What Users Should Know

- Scheduled tasks and heartbeat parsing are more robust. When a model response starts with internal reasoning blocks such as `<think>`, the scheduler strips those blocks before parsing JSON, avoiding silent `status=unknown` heartbeat skips.
- Long-running runners no longer look idle for as long. Hone now has a reusable progress watchdog for runner calls, so long model calls keep emitting activity signals and are less likely to appear stuck in Feishu or scheduled-task flows.
- Finance consistency smokes now cover a more realistic failure mode. The new MiniMax path validates that conflicting oil-price samples are rejected for broadcast and tagged as `data-consistency` anomalies.
- Codex ACP now separates model id from reasoning effort. `agent.codex_acp.model` is passed as the model name, while `agent.codex_acp.variant` is passed to Codex CLI as `model_reasoning_effort`; legacy `model/variant` values are normalized back to the base model.

### Who Should Update Soon

- Users relying on Feishu / Telegram scheduled tasks, heartbeat checks, or scheduler-driven pushes.
- Users who have seen scheduled tasks generate successfully but then go quiet, placeholders stay unchanged, or heartbeat jobs get skipped.
- Users running `codex_acp` with `variant: medium|high|xhigh`.
- Maintainers of live LLM smoke tests and bug reproduction paths.

### What Will Change After Upgrading

- The scheduler is more tolerant of reasoning prefixes, so internal model-thought blocks no longer interfere with heartbeat JSON classification.
- The long-call progress tick interval can be shortened in smoke tests with `HONE_AGENT_RUN_PROGRESS_TICK_SECS`; normal production use does not require extra configuration.
- Codex ACP startup args now include `-c model_reasoning_effort="..."` instead of appending `medium/high/xhigh` to the model id.
- `config.example.yaml` documents the new Codex ACP `variant` meaning.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- The new live LLM smoke examples require valid local model configuration and external service availability; they are not part of the default CI gate.
- Pushing the tag triggers the remote release workflow; local verification can confirm the tag push, while final artifact publication still depends on GitHub Actions.
