# v0.12.3

发布日期：2026-05-26

English readers: please scroll down for the English version below.

## 用户应该知道的

- **公开 Web 体验更完整**：新增双语 Blog 模块、Rust 技术文章、首页/导航入口和更稳定的分享预览，公开 `/chat` 的移动端布局、分享卡、链接恢复和 stale asset 自动恢复都经过了一轮加固。
- **桌面与运行时配置更稳**：desktop release smoke mode、config runtime 文档、runner 生效链路和安装包归档检查继续收口，便于 packaged app、remote backend 和源码运行路径之间排障。
- **消息渠道与调度修复**：Feishu 私聊、cron 联系人解析、partial reply fallback、心跳结构化状态解析和最大迭代耗尽等问题得到修复，减少“看起来运行但没有正确回复/推送”的情况。
- **事件引擎更可靠**：替换已下线/废弃的 Grok 默认模型，隐藏不稳定 The Fly 链接，并收窄 market review 的 commodity guard 误判，降低误报和坏链接对投资提醒的干扰。
- **工具与安全边界更清晰**：本地路径输出继续做可见性清理，PDF 文本抽取 panic 被捕获，runner 诊断细节和外部错误输出进一步脱敏。

## 谁应该尽快更新

- 正在使用公开 `/chat`、分享卡、Blog/官网入口或 Cloudflare Pages public surface 的部署者
- 使用 desktop packaged app、remote backend 或本机 channel sidecar 的用户
- 依赖 Feishu / Telegram / Discord / scheduler 主动推送的用户
- 使用 event-engine 市场提醒、market review、SEC/news 链接或 LLM polish 的部署

## 升级后会有什么变化

- 公开页面和移动端聊天更不容易因旧资源、分享选择、滚动恢复或布局细节出错。
- 桌面 release app 与 backend/runtime 之间的诊断路径更清楚，remote backend 模式仍要求 `/api/meta` 返回兼容的 `desktop-v1`。
- Feishu 与 scheduler 相关回复/推送路径更少出现空回复、误跳过和状态残留。
- 事件提醒会避开更多不稳定链接和误判分支，默认模型配置跟随当前可用服务。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 架构 SVG 不需要更新：本次核对未发现 pollers / EventKind / agent runners / tools / channels / storage layout 的结构性数量变化需要刷新海报。
- Desktop remote backend 模式需要可访问的管理端 `/api/meta`，并在云端启用 Bearer token 时配置对应 token；公开 `/api/public/*` 健康不等同于 desktop admin API 健康。

## English

Release date: 2026-05-26

### What Users Should Know

- **A fuller public Web experience**: this release adds the bilingual Blog module, the Rust article, homepage/navigation entry points, and a more reliable share-preview path. Public `/chat` mobile layout, share cards, link recovery, and stale asset recovery were all hardened.
- **More stable desktop and runtime configuration**: desktop release smoke mode, config runtime docs, runner apply behavior, and install archive checks were tightened to make packaged app, remote backend, and source-runtime diagnostics easier.
- **Channel and scheduler fixes**: Feishu direct chat, cron contact resolution, partial reply fallback, heartbeat structured status parsing, and max-iteration exhaustion handling were fixed to reduce “running but not actually replying/pushing” failures.
- **More reliable event engine behavior**: deprecated Grok defaults were replaced, unstable The Fly links are hidden, and the market-review commodity guard is narrower, reducing noisy false positives and bad links.
- **Clearer tool and safety boundaries**: user-visible local paths are redacted more consistently, PDF text extraction panics are caught, and runner diagnostics / external errors leak fewer sensitive details.

### Who Should Update Soon

- Operators running public `/chat`, share cards, Blog/site pages, or the Cloudflare Pages public surface
- Users running the desktop packaged app, remote backend, or local channel sidecars
- Users relying on Feishu / Telegram / Discord / scheduler proactive pushes
- Deployments using event-engine market alerts, market review, SEC/news links, or LLM polish

### What Will Change After Upgrading

- Public pages and mobile chat are less likely to fail on stale assets, share selection, scroll restoration, or layout edge cases.
- Desktop release app and backend/runtime diagnostics are clearer; remote backend mode still requires `/api/meta` to return compatible `desktop-v1` metadata.
- Feishu and scheduler reply/push paths are less prone to empty replies, false skips, and stale status rows.
- Event alerts avoid more unstable links and false-positive branches, while default model config follows currently available providers.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- Architecture SVG was not updated: the checklist did not find structural count changes for pollers / EventKind / agent runners / tools / channels / storage layout.
- Desktop remote backend mode needs a reachable admin `/api/meta` endpoint, plus the matching bearer token when cloud auth is enabled; a healthy public `/api/public/*` route does not by itself prove the desktop admin API is healthy.
