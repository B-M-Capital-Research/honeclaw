# v0.4.2

发布日期：2026-04-28

English readers: please scroll down for the English version below.

> v0.4.1 当天的第二次发版。主线是 **digest 推送富文本化**——盘前/盘后/quiet 合集这些聚合推送从纯 markdown bullet 升级成 Discord embed / Feishu interactive card / Telegram bucket-grouped HTML，按 6 大类分组、按 severity 三色变色、抑制自动 URL 预览,顺手把"AAPL earnings on 2026-04-30 + in 2 days (2026-04-30)"这种同语义重复折叠掉。同时吸收了 v0.4.1 之后合入 main 的 quiet hours / shelf-life 推送 + public auth 会话加固。

## 用户应该知道的

### 1. Digest 推送：4 个渠道齐换富文本

之前盘前/盘后摘要在 Discord 上是这样：14 条 bullet 摞在一起、新闻链接被 Discord 自动展开成大卡片把主消息推下去、AAPL/SNDK earnings 同时显示 "on 2026-04-30" 和 "in 2 days (2026-04-30)" 重复占位。本次彻底升级:

- **Discord**：单 message 一个 embed,按 6 大类分 fields(💹 价格异动 / 📅 财报 / 📰 新闻公告 / 🏢 公司行动 / 🌐 宏观 / 🗣 社交)。embed 主色按本批最高 severity 三档:High=红 / Medium=黄 / Low=蓝。消息加 `flags=4 (SUPPRESS_EMBEDS)` 抑制 Discord 自动 URL 预览,链接全部走 `[→](url)` 短锚文本不再霸屏。
- **Feishu**：interactive card v2,header.template 跟着 severity 走 red/yellow/blue。每个 bucket 一个 div header + markdown body,bucket 间 hr 分割线,`/missed` 提示走 note element。
- **Telegram**：保留 HTML parse mode,但 digest 路径不再扁平 bullet——按 bucket 分组,标题前面挂 🔴/🟡/🔵 severity 球,链接收成 `🔗` emoji 锚。`disable_web_page_preview=true` 避免长链接撑大消息。
- **iMessage**：未升级(系统层无富文本能力),继续走原有 markdown 字符串路径。
- **Discord/Feishu/Telegram 字符串 fallback 仍存在**:`OutboundSink::send_digest` 的默认实现会回退到 `send(actor, body)` 字符串路径,任何尚未升级 override 的 sink 都能保持旧行为。

### 2. Digest 同语义重复折叠

之前 earnings 倒计时的 T-2 / T-1 / "on date" 三条会同时进同一批 digest,SocialPost 的多源转载也常常重复成 2-3 条。本次在 digest 渲染前做一道结构化去重:

- `EarningsUpcoming` 按 `payload.report_date + symbol` 折叠——同一公司同一日只留一条;
- `NewsCritical` / `PressRelease` 按 url 的 `host+path` 归一化——同篇文章不同 `?utm_source=` 链接合并;
- 其他 kind 默认按 `event.id`,稳定保留 scheduler 排好的顺序。

折叠是显示层的,**不影响 delivery_log 落库**:被折叠的事件仍然单独写 `digest_item:omitted`,`/missed` 仍能查到完整清单。

### 3. quiet hours + shelf-life-aware push（来自 v0.4.1 之后合入的 9290594）

接 v0.4.1 之后才合入 main 的能力:

- `prefs.quiet_hours`:per-actor 勿扰区间,router immediate sink hold + digest scheduler 在 to 时刻 quiet_flush,把 hold 住的 High + buffer 里 Medium/Low 合并成一条早间合集。
- `EventKind::shelf_life`:仅盘中事件过期(PriceAlert/VolumeSpike 2h, Weekly52 8h, SocialPost 12h, PortfolioPre/Post 6h),财报/SEC/分红/分析师评级/宏观/新闻这类事实性事件 None 永不过期。过保鲜期事件在 quiet_flush 阶段 drop 写 `quiet_dropped` 审计。
- `cron_job.bypass_quiet_hours`:per-job 豁免严守时刻的任务(06:55 盘前复盘等)。
- `notification_prefs` 新增 3 个 NL action:`set_quiet_hours` / `clear_quiet_hours` / `get_overview`(按 actor.channel 自动选渲染:Discord ``` monospace 代码块 / Telegram `<pre>` / Feishu+iMessage 项目符号)。
- `/api/admin/schedule` + `/schedule` 前端页:把 4 个数据源(per-actor prefs / cron jobs / 全局 digest / 即时推阈值)拍平成同一份事实源。

### 4. Public auth session handling 加固（来自 v0.4.1 之后合入的 180679d）

`crates/hone-web-api/src/routes/public.rs` + `memory/src/web_auth.rs` 的会话/token 处理收紧。`api.test.ts` 加了对应的单元测试。如果有部署在公开端面向真实用户的实例,建议升级。

## 谁应该尽快更新

- **使用 Discord/Feishu/Telegram 接收 digest 推送的用户**:盘前/盘后/quiet 合集的视觉体验大幅改善,且不再被 Discord 自动卡片预览顶下去。
- **在 v0.4.1 之后还没拉到 main 的部署**:本次顺手吸收了 quiet hours + shelf-life + public auth fix 三块,统一升级一次最划算。
- **有大量重复 earnings 倒计时被吐槽的用户**:digest 折叠后同一财报只保留一条。

## 升级后会有什么变化

- **Discord/Feishu/Telegram 的 digest 推送外观完全不同**——bucket 分组 + severity 配色 + 链接锚文本化,不再是一长串扁平 bullet。
- **iMessage digest 不变**(系统层无富文本能力)。
- **`/missed` 命令仍然返回完整未展示清单**:dedup 折叠的不是删除,只是显示层合并。
- **`OutboundSink` trait 新增 `send_digest(actor, payload, fallback_body)` 默认方法**:第三方 sink 实现者**无需任何改动**——默认走旧字符串 fallback;想用富文本 override 即可。
- **`MultiChannelSink::send_digest` 必须正确转发到内层 sink**(本 release 已加 unit test 锁死),否则富文本 override 拿不到 payload。

## 安装与更新

- `curl | bash` 用户:重新运行安装脚本即可切到新版本。
- Homebrew 用户:`brew update && brew upgrade honeclaw`。
- **不需要改 config.yaml**:本次升级零配置变更,所有渠道开箱即用新视觉。

## 已知注意事项

- 本次 digest 富文本只覆盖 `digest_window` 与 `quiet_flush` 两条主链路;**immediate 单事件推送仍走原有字符串路径**(本就单条事件,embed 化收益不大,后续视用户反馈再评估)。
- Discord embed 单 field value 上限 1024 字符、单 embed 总和 6000 字符——超长 bucket 会在末尾追加"…还有 N 条",不会发第二条 message。`/missed` 拿完整清单。
- tag 推送只代表 GitHub release workflow 被触发;最终 tarball / Homebrew formula 是否发布,请以 GitHub Actions 结果为准。

## English

Release date: 2026-04-28

> Same-day v0.4.1 follow-up. Headline: **digest pushes get rich-text on every channel that supports it** — pre-market / post-market / quiet-flush summaries upgrade from flat markdown bullets to Discord embeds / Feishu interactive cards / Telegram bucket-grouped HTML, grouped into 6 buckets with severity-based three-tone coloring, with auto-URL-preview suppressed. Same-language duplicates ("AAPL earnings on 2026-04-30" + "in 2 days (2026-04-30)") collapse. Bundles two changes that landed on main after v0.4.1 cut: quiet-hours + shelf-life-aware push, and public auth session hardening.

### What Users Should Know

#### 1. Digest Pushes: Rich Text on All 4 Channels

Before, a Discord pre-market summary was 14 bullets stacked, news links auto-unfurled into giant preview cards that pushed the main message offscreen, and AAPL/SNDK earnings showed up twice ("on 2026-04-30" and "in 2 days (2026-04-30)"). This release rewrites the rendering:

- **Discord**: single message, single embed, fields by 6 buckets (💹 price moves / 📅 earnings / 📰 news+filings / 🏢 corporate actions / 🌐 macro / 🗣 social). Embed color follows the batch's max severity: High=red / Medium=yellow / Low=blurple. Message carries `flags=4 (SUPPRESS_EMBEDS)` to kill Discord's auto-URL-preview; links use `[→](url)` short anchors instead of full URLs.
- **Feishu**: interactive card v2, `header.template` follows severity → red/yellow/blue. One div header + markdown body per bucket, `hr` dividers between, `/missed` hint goes into a `note` element.
- **Telegram**: keeps HTML parse mode, but the digest path is no longer flat bullets — grouped by bucket, title prefixed with a 🔴/🟡/🔵 severity dot, links collapse to a `🔗` anchor. `disable_web_page_preview=true` keeps the message tight.
- **iMessage**: not upgraded (no rich text at the OS layer); keeps the existing markdown string path.
- **String fallback preserved**: `OutboundSink::send_digest`'s default implementation falls back to `send(actor, body)`, so any sink that doesn't override stays on the old path.

#### 2. Digest Same-Language Dedup

Before, earnings countdown T-2 / T-1 / "on date" all entered the same digest batch; SocialPost re-shares often duplicated 2–3x. v0.4.2 adds a structured dedup pass before render:

- `EarningsUpcoming` collapses by `payload.report_date + symbol` — one row per company per report date.
- `NewsCritical` / `PressRelease` normalize URL to `host + path`, merging same article across different `?utm_source=` links.
- Other kinds fall back to `event.id`, preserving scheduler ordering.

Collapse is **display-layer only** — collapsed events still get their own `digest_item:omitted` row in `delivery_log`, and `/missed` returns the full list.

#### 3. quiet hours + shelf-life-aware push (bundled from 9290594)

Capabilities that landed on main after the v0.4.1 cut:

- `prefs.quiet_hours`: per-actor do-not-disturb window. Router immediate sink holds, digest scheduler runs `quiet_flush` at the `to` minute combining held High + buffered Medium/Low into one morning roll-up.
- `EventKind::shelf_life`: only intraday events expire (PriceAlert/VolumeSpike 2h, Weekly52 8h, SocialPost 12h, PortfolioPre/Post 6h); factual events (earnings/SEC/dividend/analyst grade/macro/news) keep None=never. Stale held events drop with `quiet_dropped` audit.
- `cron_job.bypass_quiet_hours`: per-job opt-out for time-sensitive tasks (06:55 pre-market thesis review, etc.).
- `notification_prefs` adds three NL actions: `set_quiet_hours` / `clear_quiet_hours` / `get_overview` (auto-renders by actor.channel: Discord ``` code block / Telegram `<pre>` / Feishu+iMessage bullet list).
- `/api/admin/schedule` + `/schedule` admin page: flattens 4 data sources (per-actor prefs / cron jobs / global digest / immediate push thresholds) into one source of truth.

#### 4. Public Auth Session Hardening (bundled from 180679d)

`crates/hone-web-api/src/routes/public.rs` + `memory/src/web_auth.rs` tighten session/token handling. `api.test.ts` adds matching unit tests. If you run the public surface against real users, upgrade.

### Who Should Update Soon

- **Anyone receiving digest pushes via Discord/Feishu/Telegram**: visual experience materially better, and Discord auto-preview no longer pushes the digest offscreen.
- **Deployments still on v0.4.1**: this release also bundles quiet-hours + shelf-life + public auth fix, so a single upgrade catches everything.
- **Users who complained about repeated earnings countdowns**: dedup collapses to one row per report date.

### What Will Change After Upgrading

- **Discord/Feishu/Telegram digest visuals look entirely different** — bucketed grouping + severity-based color + link anchor text, no more long flat bullet lists.
- **iMessage digests unchanged** (no OS-level rich text).
- **`/missed` still returns the full unfilltered list**: dedup is collapse, not delete.
- **`OutboundSink` trait gains a `send_digest(actor, payload, fallback_body)` default method**: third-party sink implementations need **zero changes** — default falls back to the existing string path.
- **`MultiChannelSink::send_digest` now correctly forwards to the inner sink** (locked in via a regression test) — without it, rich-text overrides never see the payload.

### Install and Update

- `curl | bash` users: rerun the install script.
- Homebrew users: `brew update && brew upgrade honeclaw`.
- **No `config.yaml` changes required**: zero-config upgrade; new visuals work out of the box.

### Known Notes

- This release's rich-text upgrade only covers `digest_window` and `quiet_flush` paths. **Immediate single-event pushes still use the original string path** (single events get little embed benefit; will revisit based on feedback).
- Discord embed limits: 1024 chars per field value, 6000 chars per embed. Over-long buckets append "…还有 N 条" tail; we do **not** spawn a second message. `/missed` retains the full list.
- Pushing the tag only triggers the GitHub release workflow; whether the tarball / Homebrew formula publish must be confirmed via GitHub Actions.
