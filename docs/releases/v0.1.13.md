# v0.1.13

发布日期：2026-04-13

## 用户应该知道的

- `register-admin` 不再依赖源码里的硬编码口令；现在必须在 `config.yaml` 或环境变量里显式配置 secret，运行时才允许升级管理员
- `/report` 的本地 workflow validate code 也改为配置项，不再把验证码写死在代码里
- 附件链路补上了真正的持久化 manifest，并修复了多处附件/解压清理缺失导致的泄漏问题

## 谁应该尽快更新

- 在生产或多人环境里运行 Hone，担心 `/register-admin` 暗门提权的用户
- 经常通过 Feishu / Discord / Telegram 上传附件的用户
- 依赖 heartbeat 定时任务或 Web bearer token 鉴权的用户

## 升级后会有什么变化

- 只有同时满足“当前 identity 已在 `admins` 白名单中”且“`/register-admin <secret>` 提供的 secret 与配置匹配”时，运行时管理员升级才会成功
- heartbeat 后台提示词不再把内部 noop sentinel 错写进兼容示例，模型收到的规则更准确
- Feishu 附件不再先落 channel 临时目录再做准入；超限/拒绝场景会更干净，压缩包解压失败时也会回收临时目录
- Web bearer token 比较改成常量时间，降低 timing oracle 风险

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 如果你要继续使用运行时管理员升级，需要显式配置 `admins.runtime_admin_registration_passphrase`，或设置 `HONE_ADMIN_REGISTER_PASSPHRASE`
- 如果你使用跨渠道 `/report` 快捷入口，也需要配置 `web.local_workflow_validate_code` 或 `HONE_REPORT_VALIDATE_CODE`
