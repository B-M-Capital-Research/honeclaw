# v0.1.31

发布日期：2026-04-16

English readers: please scroll down for the English version below.

## 用户应该知道的

- `scheduled_task -> cron_job` 这条链路现在终于能正确创建“每周几”的 weekly 任务了。之前 skill 明明知道 weekly 需要 `weekday`，但工具 schema 没把这个字段暴露出来，结果会在真正落库时失败。
- weekly 任务现在不止能创建，也能正确修改星期；把 weekly 改回 daily 之类的非 weekly 类型时，旧的 `weekday` 也会被一起清掉，不再留下脏 schedule。
- Feishu 失败兜底消息的持久化也补上了，出错场景下的 fallback 回复不会再只出现在即时链路里、却丢失在后续可追踪记录之外。

## 谁应该尽快更新

- 通过 Telegram、Feishu 或其它支持 cron 的渠道，让 Hone 帮你创建“每周日 12:00”这类 weekly 提醒的用户。
- 之前已经碰到过“weekly 类型须指定 weekday (0-6)”但当前接口又传不进去的维护者。
- 需要排查 Feishu 异常回复链路，并依赖会话/日志持久化做复盘的维护者。

## 升级后会有什么变化

- `cron_job` 对外参数 schema 新增 `weekday`，weekly 任务终于可以被 runner / model 合法调用。
- `cron_job update` 改成走共享存储层的 schedule 更新逻辑，weekly 的 weekday 校验和非 weekly 的清理语义都会一致。
- Feishu 失败 fallback 文本会和主链路其它输出一样进入持久化记录，后续排障更容易对上真实用户看到的内容。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次修复的是工具接口层和更新逻辑；已经运行着的旧二进制仍需要一次有意图的重启，才会真正带上新的 `weekday` 参数 schema。
- `docs/current-plan.md` 同时做了一轮清理，把 6 个长期失焦或只剩占位语义的计划移出活跃列表；这不影响运行时行为。

## English

Release date: 2026-04-16

### What Users Should Know

- The `scheduled_task -> cron_job` path can now create real weekly jobs correctly. Previously the skill knew that weekly schedules required `weekday`, but the tool schema did not expose that field, so creation still failed at persistence time.
- Weekly jobs can now be updated correctly as well, and switching a job back from weekly to a non-weekly repeat clears the stale `weekday` instead of leaving behind a dirty schedule.
- Feishu failure fallback messages are now persisted too, so an error reply no longer appears only in the live path while disappearing from later debugging records.

### Who Should Update Soon

- Users who rely on Hone to create weekly reminders such as “every Sunday at 12:00” through Telegram, Feishu, or other cron-enabled channels.
- Maintainers who have already hit `weekly 类型须指定 weekday (0-6)` while the current interface still gave them no way to pass that field.
- Maintainers who debug Feishu failure paths and depend on persisted records to reconstruct what the user actually saw.

### What Will Change After Upgrading

- The public `cron_job` parameter schema now includes `weekday`, so weekly jobs can finally be called legally through the runner/model layer.
- `cron_job update` now reuses the shared storage-layer schedule update logic, which keeps weekly weekday validation and non-weekly cleanup semantics consistent.
- Feishu failure fallback text is now persisted alongside the rest of the runtime conversation trail, making debugging much easier.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This fixes the tool interface layer and update logic; already-running old binaries still need an intentional restart before they expose the new `weekday` parameter schema.
- `docs/current-plan.md` was also cleaned up to remove six long-stale placeholder plans from the active list; that does not affect runtime behavior.
