# v0.1.15

发布日期：2026-04-13

English readers: please scroll down for the English version below.

## 用户应该知道的

- 记忆页里的“公司画像”不再要求画像文件具备结构化 frontmatter；页面现在会直接读取当前 actor 空间下现有的 `company_profiles/*/profile.md`
- 画像空间选择器按 `渠道 / 用户` 展示，选中后会直接列出该空间下已有的画像目录
- 画像详情页现在直接渲染原始 `profile.md` 和 `events/*.md` 内容，不再因为结构化解析失败而整页显示为空

## 谁应该尽快更新

- 已经在 actor sandbox 里沉淀了公司画像 Markdown，但此前 Web 页面看不到任何画像的用户
- 用 agent 手工维护 `company_profiles/` 文档，不希望被固定 schema 约束的用户
- 需要从 Web 侧按渠道和用户快速查看现有画像文档的运营或排障人员

## 升级后会有什么变化

- 只要目录里存在 `profile.md`，前端就会把它当成一份可见画像列出来
- 事件目录里的 Markdown 文件会按文件直接展示，不再依赖事件 frontmatter 才能被看见
- 页面语义从“结构化画像面板”收口为“文件系统直显视图”

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 当前页面仍然是只读展示；建档、修改和追加事件仍应通过 agent 或直接文件操作完成
- 删除画像仍然按目录删除，因此选择空间和画像时要确认无误

## English

Release date: 2026-04-13

### What Users Should Know

- The Company Portrait view in Memory no longer requires structured frontmatter; the page now reads existing `company_profiles/*/profile.md` files directly from the current actor space
- The portrait-space selector is now displayed by `channel / user`, and selecting a space lists the portrait directories already present there
- The portrait detail page now renders raw `profile.md` and `events/*.md` content directly instead of showing an empty page when structured parsing fails

### Who Should Update Soon

- Users who already have company portrait Markdown files in actor sandboxes but previously saw nothing in the Web page
- Users who maintain `company_profiles/` manually through the agent and do not want to be constrained by a fixed schema
- Operators or support engineers who need to inspect portrait documents quickly by channel and user from the Web side

### What Will Change After Upgrading

- As long as a directory contains `profile.md`, the frontend will list it as a visible portrait
- Markdown files under the events directory are displayed directly and no longer depend on event frontmatter to appear
- The page semantics are now closer to a “filesystem direct-view surface” than a “strict structured portrait panel”

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- The page is still read-only; portrait creation, edits, and event append should still happen through the agent or direct file operations
- Deleting a portrait still removes the whole directory, so confirm the selected space and portrait before deleting
