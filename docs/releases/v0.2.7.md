# v0.2.7

发布日期：2026-04-24

English readers: please scroll down for the English version below.

## 用户应该知道的

- Feishu 每日动态监控现在会正确识别“按规则可跳过正式推送”“不触发正式推送”等结论，并把这类轮次收口为静默 `noop`，不再把一边声明跳过、一边继续外发的长文推给用户。
- Heartbeat 价格阈值的默认口径这次被收紧了。除非任务文案明确写的是日内高点、低点或振幅，否则“盘中涨跌幅超过 X%”统一按最新价格相对昨收判断，避免把区间振幅或日内高点误当成触发条件。
- Desktop release 打包链路补了一个实际阻塞项：runner CLI 自检不再因为未定义变量而让 `hone-desktop` release build 失败。
- 文档侧已经同步回写 bug 台账和 release note，`v0.2.7` 覆盖的是 `v0.2.6` tag 之后这轮真实可交付修复。

## 谁应该尽快更新

- 依赖 Feishu 每日动态监控、且在意“无新增催化时不要打扰”的用户。
- 依赖 heartbeat 价格/事件提醒、且之前见过“当前涨幅没过线却被提醒触发”的用户。
- 需要本地重打 desktop release app 或依赖 Tauri release 构建的维护者。

## 升级后会有什么变化

- 每日动态监控类任务在模型正文已经明确判定“跳过正式推送”时，会落成 `noop + skipped_noop`，不再继续送出“无新增长文”。
- Heartbeat prompt 现在会明确禁止把日内高点相对昨收、日内低点相对昨收或高低点振幅，默认当作“盘中涨跌幅阈值”命中依据。
- 如果你确实希望按“日内高点 / 低点 / 振幅”触发提醒，需要在任务文案中显式写出对应口径；否则系统会按最新价格相对昨收处理。
- Desktop release 构建恢复可用，新的 `.app` 包可以正常打出并按 release runtime 模式重启。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- heartbeat 的价格阈值这次仍主要依赖 prompt 约束，而不是完全结构化校验；如果任务文案本身口径含糊，仍建议写清楚“最新价 / 日内高点 / 日内振幅”。
- 若本地 runtime 配置长期漂移到非 `multi-agent` runner，后续重启前仍建议复核当前运行态配置。

## English

Release date: 2026-04-24

### What Users Should Know

- Feishu daily monitoring jobs now correctly suppress deliveries when the model concludes with phrases like “skip formal delivery” or “do not send a formal update,” instead of pushing a long message that contradicts its own conclusion.
- Heartbeat price-threshold semantics are now tighter by default. Unless the task explicitly asks for intraday highs, lows, or range, “intraday move above X%” is evaluated as latest price versus previous close, which prevents false triggers caused by intraday highs or range swings.
- The desktop release packaging lane also gets a practical unblock: runner CLI health checks no longer fail the `hone-desktop` release build because of an undefined variable.
- Bug docs and release assets have been synchronized, and `v0.2.7` captures the actual deliverable fixes that landed after the `v0.2.6` tag.

### Who Should Update Soon

- Users relying on Feishu daily monitoring and expecting silence when there is no new catalyst.
- Users relying on heartbeat price/event alerts who have seen false triggers even though the current move never crossed the threshold.
- Maintainers who need to rebuild the desktop release app locally or depend on the Tauri release packaging flow.

### What Will Change After Upgrading

- Daily monitoring jobs now resolve to `noop + skipped_noop` when the model explicitly concludes that formal delivery should be skipped, so “no-update long-form pushes” stop reaching users.
- Heartbeat prompts now explicitly forbid substituting intraday highs vs previous close, lows vs previous close, or intraday range for a generic “intraday move above X%” threshold unless the task text asks for that metric.
- If you really want alerts based on intraday high/low/range, spell that metric out in the scheduled-task text; otherwise Hone will treat the threshold as latest price versus previous close.
- Desktop release packaging is healthy again, and the new `.app` bundle can be rebuilt and restarted through the release runtime flow.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- Heartbeat price-threshold tightening in this release is still prompt-driven rather than fully structurally validated; if your task wording is ambiguous, be explicit about whether you mean latest price, intraday high/low, or range.
- If your local runtime configuration drifts back to a non-`multi-agent` runner, re-check the live runner setting before the next manual restart.
