# v0.5.1

发布日期：2026-05-01

English readers: please scroll down for the English version below.

> v0.5.0 之后的非破坏性 patch release。主线：**全局 digest 抓原文加 Jina Reader 兜底层**，让 Reuters / WSJ 这类反爬 / 付费墙站的全文重新进得来；附带一批 scheduler / heartbeat / Feishu / OpenRouter / web 路径的 bug 修复。无 config schema 变更；旧 `config.yaml` 直接启动即可，新增字段保持可选。

## 用户应该知道的

### 1. Global digest fetcher 加 Jina Reader fallback（新功能）

`event_engine.global_digest` 的 Pass 2 抓原文链路从两段式扩到三段式：

1. 直抓原文（沿用现有 reqwest 客户端）
2. **新增**：直抓非 2xx / 空 body 时,带 Bearer key 走 `https://r.jina.ai/<url>` 二次抓取；Jina 后端跑无头浏览器渲染 + 抽正文
3. Jina 也失败 / 上游返回 error → 落到事件本体的 FMP `text` 摘要

实测效果：

- **Reuters** 整片域名解锁(Jina 共享 IP 池被滥用导致的 451 abuse-block 在带 key 时消失),拿到完整正文 / 引语 / 财务数据
- **WSJ** 拿到 lede + 前两段(撞付费墙后停)
- **Barron's / NYT / FT** 站点本身屏蔽 Jina 的无头浏览器,继续走 FMP fallback —— 这部分没办法

新增配置(可选,留空跳过 Jina 层,行为同 v0.5.0):

```yaml
event_engine:
  global_digest:
    jina_api_key: "jina_xxx"  # https://jina.ai 邮箱注册,免费层 1M tokens/月
```

`ArticleSource` 枚举新增 `JinaFallback` variant —— 外部 sink / 调度器如果对该 enum 做了 exhaustive match 需要补一个分支(本仓内已处理)。

### 2. Paywall 域名直抓非 2xx 的日志降级

Reuters / WSJ / Barron's / NYT / Bloomberg / FT / Economist 这 7 个域名直抓返回 401/403 是常态(反爬层硬墙),原本每次 Pass 2 都打 `WARN`,日志噪音很大。本 release 把这一组域名的非 2xx 降到 `INFO`,真实失败语义由 Jina + FMP fallback 兜底。其他域名继续 `WARN`,行为不变。

## English

Release date: 2026-05-01

> Non-breaking patch release after v0.5.0. Headline: **Jina Reader fallback for global-digest article fetch**, which brings full-text retrieval back for anti-bot / paywall sites like Reuters and WSJ. Bundles a batch of scheduler / heartbeat / Feishu / OpenRouter / web-path bug fixes. No config schema changes — existing `config.yaml` boots as-is; the new field is optional.

### What Users Should Know

#### 1. Global Digest Fetcher Gains Jina Reader Fallback (new feature)

The `event_engine.global_digest` Pass-2 article fetch grows from two-stage to three-stage:

1. Direct fetch (existing reqwest client)
2. **New**: on direct non-2xx / empty body, retry through `https://r.jina.ai/<url>` with Bearer key; Jina runs a headless-browser render + body extraction
3. Jina also fails / upstream-error → fall back to the event's FMP `text` summary

Empirical results:

- **Reuters** is fully unblocked (Jina's anonymous shared-IP-pool 451 abuse-block disappears with a key) — full body, quotes, and figures land
- **WSJ** returns lede + first 2 paragraphs (paywall cuts in after that)
- **Barron's / NYT / FT** are blocked at Jina's headless browser too — these continue to fall back to FMP, no path forward

New config field (optional — leave empty to skip Jina, behavior matches v0.5.0):

```yaml
event_engine:
  global_digest:
    jina_api_key: "jina_xxx"  # register at https://jina.ai (email only); free tier 1M tokens/month
```

The `ArticleSource` enum gains a `JinaFallback` variant — external sinks / schedulers doing exhaustive matches must add an arm (in-tree call sites are already handled).

#### 2. Paywall-Domain Direct-Fetch Non-2xx Log Demotion

For Reuters / WSJ / Barron's / NYT / Bloomberg / FT / Economist, direct-fetch 401/403 is the steady state (anti-bot edge). v0.5.0 logged these as `WARN` every Pass-2 cycle, drowning the log. v0.5.1 demotes that domain group to `INFO`; real failure semantics are now carried by Jina + FMP fallback. Other domains continue at `WARN` — behavior unchanged there.

### Who Should Update Soon

- **Deployments running `event_engine.global_digest`**: especially instances that care about Reuters-class news — without upgrading you'll keep seeing 401 noise and Pass-2 stuck on the weak FMP summary. Registering a Jina key produces a substantial improvement.
- **Feishu / Discord / Telegram digest users**: scheduler heartbeat, Feishu open_id fallback, and OpenRouter error preservation fixes all directly improve delivery observability.
- **Tauri desktop users**: startup errors no longer block the process — even with missing config, the app reaches the main UI.

### What Will Change After Upgrading

- `event_engine.global_digest.jina_api_key` is a new optional field — leaving it empty exactly matches v0.5.0 behavior (direct fetch + FMP fallback only).
- Direct-fetch failures on paywall domains drop from `WARN` to `INFO`; alerting that greps `WARN` will quiet down.
- `ArticleSource` gains `JinaFallback` — in-tree call sites are handled; external code doing full enum matches needs to add an arm.
- All other bug fixes are behavior-tightening with no config changes.

### Install and Update

- `curl | bash` users: rerun the install script.
- Homebrew users: `brew update && brew upgrade honeclaw`.
- To enable Jina fallback: register at https://jina.ai (email only, no credit card), put the key into `config.yaml` under `event_engine.global_digest.jina_api_key`, restart the web-api process.

### Known Notes

- Jina's free tier is 1M tokens/month; this repo's hourly digest is estimated at ~0.5–2M tokens/month, generally enough. Beyond the limit Jina returns 429 and the code transparently falls back to FMP — no panics.
- Barron's / NYT / FT are blocked at Jina's renderer too (they refuse Jina's nodes). No further path here without a paid API or subscription cookies.
- Pushing the tag only triggers the GitHub release workflow; whether the tarball / Homebrew formula actually publishes must be confirmed via GitHub Actions.
