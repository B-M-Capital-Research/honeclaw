# v0.4.0

发布日期：2026-04-26

English readers: please scroll down for the English version below.

> 本次的主线是“全局精读 digest + 自动 thesis 蒸馏”：在原有 per-actor digest 之外，新增一条独立管道，用 trusted-source 候选池 + 二阶段 LLM 聚类 + 强模型精读，每日按时把最重要的几条市场叙事广播给所有开了订阅的 direct actor；并且不再让用户手动维护 `investment_theses`，系统会从公司画像自动蒸馏。配套：admin console 重构成 entity-centric IA，移除 Truth Social 数据源，event-engine 完成 EventSource 迁移收官，feishu / scheduler / desktop / codex acp 的若干 P1 缺陷修复。

## 用户应该知道的

### 1. Global digest：每日定时广播 + grok 二阶段去重 + 自动 thesis 蒸馏

v0.3.0 的 digest 是 per-actor 的“你订阅了什么、过去 N 分钟有什么”。本次新增**独立的全局精读管道**，跟 per-actor digest 完全并行：

- **候选池 → Pass1 → Pass2 → 广播**：trusted-source 范围内的 High/Medium news 进候选池，先用便宜模型做评分 / 聚类 / takeaway，再用强模型抓原文做精读，最后把单条结果按 `GlobalDigestConfig.schedules` 每天 N 次广播给所有 `global_digest_enabled = true` 的 direct actor。`delivery_log.channel = "global_digest"`，可以独立审计。
- **配套通用 RSS poller**：新增 `pollers/rss.rs`，默认接 Bloomberg Markets / SpaceNews / STAT News，弥补 FMP 漏掉的料。`events.source = "rss:{handle}"`，跟 FMP news 一起进候选池。
- **event 级 grok 二阶段去重**：上线第一天的全局 digest 出现“7 条里 5 条是同一 Hormuz 故事”——根因是 nova-lite 一次调用里跨 item 自洽生成 cluster id，是 LLM 已知不擅长的事。换成 grok 二阶段（先 induce 再 assign），17–236 候选量级稳定保守，同事件 11 条精准合并、不同主题事件不会被强行归类。失败降级是直接透传，不阻塞推送。`event_dedupe_enabled` / `event_dedupe_model` 可关 / 可换。
- **自动 thesis 蒸馏**：原本 `investment_theses` 需要用户用 NL tool 手动维护，但用户其实已经在公司画像里写了 100–400 行 thesis-grade 内容——系统不该再让用户重写一份。新增 `thesis_distill`，每天扫一遍画像，用 grok 蒸馏 1–2 句 thesis + 整体投资风格，持久化到 `NotificationPrefs`；缺画像/已蒸馏靠 `last_thesis_distilled_at` + 7 天周更新去节流，避免空 ticker 死循环。修复了一个画像里 `（U+FF08）` 全角括号 ticker 解析失败的小坑。
- **NL tool 收敛**：删除了 5 个 thesis 相关 action（`set_investment_style` / `upsert_thesis` / `delete_thesis` / `clear_all_theses` / `get_theses`），改成提示 agent “thesis 由系统从公司画像蒸馏，改画像即可”。`set_global_digest_enabled` / `set_macro_floor_picks` 等其它 4 个新 action 仍可用。
- **公开 portfolio 页**：新增 `/portfolio`（SolidJS），展示蒸馏出来的 thesis 卡片、整体风格、画像 inventory，并给一个“立即刷新”按钮；点入卡片是 markdown 渲染弹窗。session 锁定为自己。
- **管理端 Users 页 thesis tab**：admin console 用户中心多了一个 thesis tab，可以查任意 actor 的蒸馏结果，也能立即触发蒸馏，方便排查“为什么这个用户的 digest 没收到 X 信号”。

### 2. Admin console：entity-centric 重构

旧的 admin 是 module-centric 的（一个 module 一个页面），跨模块查“这个用户在干什么”要在 sidebar 里跳很多次。本次彻底重构：

- **侧栏分四组**：工作台 / 用户视角 / 研究 / 系统。
- **新增 `/dashboard` 落地页**：进 admin 的第一眼。
- **新增 `/users/:actorKey?/:tab?` 用户中心**：把 portfolio + 公司画像 + 会话 + 研究 + thesis 全部合并到一个页面，按 tab 切。
- **SymbolDrawer**：右侧 420px 抽屉，URL 里 `?symbol=` 同步，点任何 ticker cell 都能拉出来——展示匹配画像 / 研究任务 / 最近会话，并带快捷动作（加自选 / 起研究）。
- **EntityRefLink**：`/logs` 和 `/llm-audit` 的错误条目里加了反向跳转 chip，可以直接点回到出错的用户 / 会话 / 任务。
- **AdminChatShell**：会话视图加了 actor 工具条，可以直接跳到该会话所属用户的 portfolio / 公司画像 / 推送任务，不用回去重新选人。
- **路径兼容**：`/start` / `/memory` / `/portfolio/:userId?` 走 `<Navigate>` 透传，不破坏旧链接。
- **Public surface 完全不受影响**：chat-view 核心未动，只有 admin 侧 wrap 了一层。

### 3. Event engine：EventSource 迁移收官 + Truth Social 退役

- **EventSource 迁移收官**（PR #18 / #20 / #21）：earnings / news / macro / analyst_grade / earnings_surprise / corp_action+sec_filings / price 七个 poller 全部迁到通用 `EventSource` trait + `spawn_event_source` 单一入口。`spawner.rs` 从 ~390 行削到 ~110 行；`pipeline::cron_aligned_loop` 已无外部调用。行为不变（cron-aligned / FixedInterval / 冷启动立即拉一次都由 `spawn_event_source` 通用层保证），但调度路径终于一致。
- **新增 `task_observer`**：统一周期任务的 tracing 字段（`task=` / `outcome=` / `items=` / `degraded=`），方便后续 grafana / alert。约定文档 `docs/conventions/periodic_tasks.md` 一并落库。
- **Truth Social 数据源退役**：历史 403 断流不再追，`TruthSocialAccountConfig` / `Sources.truth_social_accounts` / `TruthSocialPoller` / engine 装配 / 配置块 / digest 低质量社交源过滤分支全部删除；watcherguru 仍保留。bug ticket 标 Closed。
- **配置开关补齐**：global digest / event_dedupe / thesis 等新行为的开关已经在 `hone-core::config::event_engine` 里暴露，并在 `config.example.yaml` 有完整示例。

### 4. Channel / Scheduler / Desktop 修复批

主要面向 Feishu 用户和 desktop 用户：

- **Feishu P1 投递批修复**（#af3f3a3）：feishu out-bound short retry、scheduler 起始台账、空回复假成功、tenant access token transport 失败、send_message transport 失败 等几条 P1 缺陷修复。
- **Scheduler 修复批**（#0e917fe）：feishu direct timeout 无回复、compact retry 之后仍无法答新主题、空回复假成功、scheduler 卡住不触发 cron、地缘价格幻觉、minimax HTTP transport 不重试、scheduler heartbeat 未知状态静默跳过、session compact summary 报告幻觉……一批分散修复，集中收口在 `bins/hone-feishu` + `crates/hone-channels`。
- **Desktop agent 切换端口冲突**（#f9ad66e）：切换 agent 时旧 web server 端口未释放，新进程启不来——已经修。
- **Codex ACP permission string id**（#9a4b27c）：ACP runner 的 permission id 类型对齐字符串。
- **Public checkbox 链接吞掉**（#3a93c66）：v0.3.0 引入的 `PublicCheckbox` 在 `<label>` 上 preventDefault，把内嵌 ToS / Privacy `<a>` 链接的点击吞了——现在 `<a>` 内点击会跳过 toggle，链接正常导航。

### 5. CLI onboard：通知默认值告知

`hone-cli onboard` 加了 step 5 “Notifications”，纯告知部署者：新用户默认会订阅 global digest + per-event 通知，thesis 在后台自动蒸馏。同时指明终端用户可以用自然语言关闭，部署者改默认值要去 `prefs.rs::NotificationPrefs::default()`。**不引入新配置项，也不写 mutation**——这一步只是把行为讲清楚。

- 任何对 trusted-source 市场叙事感兴趣的部署：global digest 是本版本的主特性，建议升级后开 `global_digest_enabled = true` 试一两天。

- 任何 Feishu / Desktop / ACP runner 的部署：含若干 P1 修复。

- **`investment_theses` 自动维护**：用户不再需要手动 upsert thesis；改公司画像即可，系统每日自动蒸馏。`get_theses` / `upsert_thesis` 等 5 个旧 NL action 已下线，agent 会被引导改画像。
- **公开 web 多了 `/portfolio` 页**：展示蒸馏 thesis + 整体风格 + 画像 inventory，session 锁定为自己。
- **Admin console IA 重构**：sidebar 分四组，新 `/dashboard` 与 `/users/:actorKey?/:tab?` 是主要入口；旧路径走 redirect。
- **Truth Social source 已删**：config 里残留的 `truth_social_accounts` 不再生效；watcherguru 仍保留为社交源。
- **`hone-cli onboard` 多一步 Notifications**：纯告知，不会改 config。
- **公开 API 路径稳定**：所有 `hone_event_engine::*` / `hone_channels::*` / `hone_core::config::*` 的 re-export 仍保留，外部 binary 无需改。

## 安装与更新

## 已知注意事项

- Global digest 与 thesis distill 的 schedule / model / RSS feeds 等 engine-level 配置改动后**需要重启 service**才会生效（admin API 会返回 `needs_restart=true`）。这是有意为之：scheduler / poller 的 spawn 在 start time 一次性完成，hot-reload 改动率太低不值得维护。
- Event-level dedup 失败时是降级透传（不去重，原样推送），不会阻塞 broadcast。`audit log` 里有完整 dedup stats 与每簇合并明细。
- `thesis_distill` 第一次上线建议跑一次 `examples/distill_first_wave.rs` 触发所有 actor 立即蒸馏，避免等 7 天周更新。
- tag 推送只代表 GitHub release workflow 被触发；最终 tarball / Homebrew formula 是否发布，请以 GitHub Actions 结果为准。

## English

Release date: 2026-04-26

> The headline of v0.4.0 is **global-broadcast digest + auto thesis distillation**: a brand-new pipeline that lives alongside the per-actor digest, taking trusted-source candidates through a two-pass LLM (cluster + deep-read) and broadcasting the few most important market narratives to every direct actor that opted in. Investment theses are no longer maintained by hand — the system distills them from your company profiles. Around it: an entity-centric admin console rewrite, the Truth Social source retired, the EventSource migration finished end-to-end, and a batch of P1 fixes across Feishu / scheduler / desktop / codex ACP.

### What Users Should Know

#### 1. Global Digest — Daily Broadcast + Grok Two-Pass Dedup + Auto Thesis Distill

v0.3.0's digest was per-actor: "what you subscribe to, what happened in the last N minutes." v0.4.0 adds an **independent global broadcast pipeline** that runs in parallel:

- **Candidate pool → Pass1 → Pass2 → broadcast**: trusted-source High/Medium news enters a candidate pool; a cheap model scores / clusters / extracts takeaways; a stronger model then fetches and reads the original article; the result is broadcast to every direct actor with `global_digest_enabled = true`, N times per day per `GlobalDigestConfig.schedules`. `delivery_log.channel = "global_digest"` so it can be audited independently.
- **General-purpose RSS poller**: new `pollers/rss.rs`, ships defaults for Bloomberg Markets / SpaceNews / STAT News to fill what FMP misses. `events.source = "rss:{handle}"`, fed into the candidate pool alongside FMP news.
- **Event-level grok two-pass dedup**: launch day showed "5 of 7 items were the same Hormuz story repackaged" — root cause was nova-lite generating cluster ids in a single self-consistent call across items, a known LLM weakness. Replaced with grok two-pass (induce-then-assign), which stays conservative across 17–236 candidates: 11 same-event variants merged precisely, no cross-topic forced grouping. Failure mode is pass-through (no merge, no block). `event_dedupe_enabled` / `event_dedupe_model` are configurable.
- **Auto thesis distillation**: `investment_theses` used to require manual upserts via the NL tool. But users had already written 100–400 lines of thesis-grade content per holding in their company profiles — the system shouldn't make them duplicate it. The new `thesis_distill` scans profiles daily and uses grok to extract 1–2 sentence theses plus an overall investment style, persisted into `NotificationPrefs`. Missing-profile and already-distilled cases throttle via `last_thesis_distilled_at` + a 7-day refresh window to avoid loops on tickers with no profile. Also fixes a small parse bug where full-width parens (U+FF08/FF09) around tickers in profiles were dropped.
- **NL tool consolidation**: removed five thesis-related actions (`set_investment_style` / `upsert_thesis` / `delete_thesis` / `clear_all_theses` / `get_theses`); the tool description now nudges the agent to "edit the company profile — the system distills the thesis." Four other new actions (`set_global_digest_enabled`, `set_macro_floor_picks`, etc.) remain.
- **Public `/portfolio` page**: new SolidJS page that shows distilled thesis cards, overall style, profile inventory, and a "refresh now" button; cards open into a markdown modal. The session is locked to the current user.
- **Admin "thesis" tab on Users page**: the user center now has a thesis tab so admins can inspect any actor's distilled thesis and trigger a re-distill on demand — useful when investigating "why didn't user X receive signal Y in their digest."

#### 2. Admin Console — Entity-Centric Rewrite

The old admin was module-centric (one page per module), and answering "what is this user doing?" required several sidebar jumps. v0.4.0 reorganizes:

- **Sidebar in four groups**: 工作台 / 用户视角 / 研究 / 系统.
- **New `/dashboard` landing page**.
- **New `/users/:actorKey?/:tab?` user center**: portfolio + company profiles + sessions + research + thesis merged into one page, switched by tab.
- **SymbolDrawer**: 420px right-side drawer, URL-synced via `?symbol=`, opens from any ticker cell — surfaces matched profiles / research tasks / recent sessions / quick actions (add to watchlist, start research).
- **EntityRefLink chips**: error rows in `/logs` and `/llm-audit` carry reverse-jump chips back to the user / session / task that produced them.
- **AdminChatShell**: chat view gained an actor toolbar that links straight to the owning user's portfolio / profiles / push tasks, no re-selection needed.
- **Backward-compatible paths**: `/start` / `/memory` / `/portfolio/:userId?` redirect via `<Navigate>` so old links still work.
- **Public surface unchanged**: the shared `chat-view` core was not touched; admin only wraps around it.

#### 3. Event Engine — EventSource Migration Finished + Truth Social Retired

- **EventSource migration finished** (PRs #18 / #20 / #21): earnings / news / macro / analyst_grade / earnings_surprise / corp_action+sec_filings / price all moved to the unified `EventSource` trait + `spawn_event_source` single entry point. `spawner.rs` shrank from ~390 → ~110 lines; `pipeline::cron_aligned_loop` no longer has external callers. Behavior is preserved (cron-aligned / FixedInterval / cold-start immediate-poll all live in the common layer), but the scheduling path is finally consistent.
- **New `task_observer`**: standardizes periodic-task tracing fields (`task=` / `outcome=` / `items=` / `degraded=`) for downstream Grafana / alerting. The conventions doc is `docs/conventions/periodic_tasks.md`.
- **Truth Social retired**: the long-running 403 outage isn't worth chasing. `TruthSocialAccountConfig` / `Sources.truth_social_accounts` / `TruthSocialPoller` / engine wiring / config block / the digest social-source low-quality filter branch are all removed; watcherguru remains. The bug ticket is closed.
- **New config knobs**: global digest / event_dedupe / thesis switches are exposed via `hone-core::config::event_engine` and have full examples in `config.example.yaml`.

#### 4. Channel / Scheduler / Desktop Fixes

Aimed primarily at Feishu and desktop users:

- **Feishu P1 delivery batch** (`af3f3a3`): outbound short retry, scheduler running/pending bookkeeping, false-success on empty replies, tenant access token transport failure, send_message transport failure.
- **Scheduler bug batch** (`0e917fe`): feishu direct idle timeout no-reply, compact retry still cannot answer a new topic, false-success on empty replies, scheduler stuck without cron job run, geopolitical price hallucination, minimax HTTP transport without retry, scheduler heartbeat unknown-status silent skip, session compact summary report hallucination — converged into `bins/hone-feishu` + `crates/hone-channels`.
- **Desktop agent switch port conflict** (`f9ad66e`): switching agents left the old web server port held; the new process couldn't bind. Fixed.
- **Codex ACP permission string ids** (`9a4b27c`): the ACP runner's permission id type aligned to strings.
- **Public checkbox swallowing inner links** (`3a93c66`): the v0.3.0 `PublicCheckbox` ran `preventDefault` on the wrapping `<label>`, swallowing clicks on the embedded ToS / Privacy `<a>` links — clicking inside an `<a>` now skips the toggle and the link navigates normally.

#### 5. CLI onboard — Notification Defaults Surfaced

`hone-cli onboard` gained a step 5 "Notifications" — purely informational — telling the operator: new users default to subscribing to the global digest + per-event notifications; thesis is auto-distilled in the background. It also points at the natural-language path for end users to opt out, and at `prefs.rs::NotificationPrefs::default()` for operators who want to change defaults. **No new config field, no mutation** — this step only documents existing behavior.

### Who Should Update Soon

- Any deployment interested in trusted-source market narratives: global digest is the headline feature; recommend turning `global_digest_enabled = true` on for a day or two after upgrading.
- Deployments already on per-actor digest: a parallel global-digest channel will appear; behavior is fully independent and does not affect existing pushes.

- Deployments still listing Truth Social as a source: the source is gone; leftover `truth_social_accounts` blocks will be ignored.
- Any Feishu / Desktop / ACP runner deployment: includes a batch of P1 fixes.

### What Will Change After Upgrading

- **One extra global-digest push**: default `global_digest_enabled = false` (per `NotificationPrefs::default()`); users opt in. Once enabled, recipients get one broadcast per `GlobalDigestConfig.schedules` slot per day.
- **`investment_theses` is auto-maintained**: users no longer manually upsert theses — edit company profiles, the system distills daily. Five legacy NL actions are gone; the agent is nudged toward editing profiles instead.
- **Public web has a new `/portfolio` page**: shows distilled theses + style + profile inventory; session is locked to the current user.
- **Admin console IA changed**: sidebar in four groups, `/dashboard` and `/users/:actorKey?/:tab?` are the main entry points; legacy paths redirect.
- **Truth Social source removed**: any leftover `truth_social_accounts` config is inert; watcherguru remains.
- **`hone-cli onboard` adds a Notifications step**: informational only, does not write config.
- **Public API paths are stable**: every `hone_event_engine::*` / `hone_channels::*` / `hone_core::config::*` re-export was preserved.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version.
- Homebrew users: `brew update && brew upgrade honeclaw`.
- Admin console users: refresh the page; `/start` / `/memory` / `/portfolio/:userId?` redirect automatically.

### Known Notes

- Engine-level config changes (global-digest schedules / models / RSS feeds, etc.) **require a service restart** to take effect (the admin API returns `needs_restart=true`). This is intentional: scheduler / poller spawn happens once at start time, and these knobs change a handful of times per year — hot-reload isn't worth the cost.
- Event-level dedup failure is pass-through (no merge, no block), so it can never stall a broadcast. The audit log carries full dedup stats and per-cluster merge details.
- For first-time `thesis_distill` rollout, run `examples/distill_first_wave.rs` once to fan out an immediate distill across all actors instead of waiting for the 7-day refresh window.
- Pushing the tag only triggers the GitHub release workflow; whether the tarball / Homebrew formula publish must be confirmed via GitHub Actions.
