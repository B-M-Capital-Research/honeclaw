# v0.1.28

发布日期：2026-04-16

English readers: please scroll down for the English version below.

## 用户应该知道的

- Feishu 直聊最近两轮 busy / placeholder 修复已经并入这一版，直聊在启动窗口和忙态切换附近更不容易出现“占位消息挂着但 agent 实际没跑起来”或“卡住却不给最终回复”的假成功状态。
- Multi-agent 搜索阶段现在会正确清洗历史 `tool_calls` 上下文，不再因为残留的 assistant tool-call 壳和被删掉的 tool result 失配，触发 provider 侧 `tool call result does not follow tool call` 这类协议错误。
- Skill runtime 现在开始严格执行“看得见就应该能用”的契约：当前 stage 缺少 `cron_job` 或其它 required tool 的 skill 不会再被继续暴露给模型；`HONE_SKILLS_DIR` 也会稳定透传到 `hone-mcp`，避免 prompt 里能看到、真正加载时却失败。
- Telegram 群聊现在会优先参考“同一发言者最近一次与机器人的往返”作为 continuation anchor，减少 compact summary 里的群全局旧话题把当前追问带偏的情况。

## 谁应该尽快更新

- 最近在 Feishu 直聊里遇到 busy / placeholder 卡住、没有最终回复或误报成功的用户。
- 依赖 multi-agent 搜索阶段做联网检索，且最近见过 `tool call` 协议错误、搜索阶段统一失败提示或搜索结果异常中断的用户。
- 大量使用 skill、尤其是 `scheduled_task` 这类依赖特定工具的 skill，并且希望“会话里显示出来的 skill 就一定能执行”的维护者。
- 在 Telegram 群里高频连续追问、希望机器人更稳地承接“同一人上一轮上下文”的用户。

## 升级后会有什么变化

- Feishu 直聊的 placeholder / busy 分支会更保守地只在真实 agent run 启动后维持占位，不再那么容易留下误导性的中间态。
- Multi-agent 搜索链路对旧上下文的清洗更完整，OpenAI-compatible provider 上的 tool-call 协议报错会明显减少。
- 若某个 skill 在当前会话阶段实际上不能运行，它会直接从可见 surface 消失；即便走到底层调用路径，也会收到明确的“当前阶段缺少工具”错误，而不是一个看起来像随机故障的失败。
- Telegram 群聊里同一发言者的续问更容易沿着自己上一轮问答继续，而不是被 compact summary 里的别人的旧话题抢走上下文。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 更完整的 Claude Code skill runtime 对齐工作仍在继续；本次发布主要收口的是 visible-vs-usable 契约、stage tool gating 和 `HONE_SKILLS_DIR` 透传，不代表 hook 执行和 watcher 热重载已经全部完成。
- `Heartbeat` 相关的部分已知问题仍在 bug 台账中持续跟踪，不属于这次 release 的主收口范围。

## English

Release date: 2026-04-16

### What Users Should Know

- This release includes the latest Feishu direct-chat busy / placeholder fixes, making it much less likely to get stuck in a fake-success state where a placeholder is shown but no real agent run actually started, or where a busy path hangs without a final reply.
- The multi-agent search stage now sanitizes historical `tool_calls` correctly, so it no longer leaves behind assistant tool-call shells after dropping old tool results, which was triggering provider-side protocol errors such as `tool call result does not follow tool call`.
- The skill runtime now enforces a stricter “if it is visible, it should be usable” contract: skills that require blocked tools such as `cron_job` disappear from the current stage instead of being shown and then failing later, and `HONE_SKILLS_DIR` is now forwarded into `hone-mcp` so prompt-time discovery and real loading see the same skill root.
- Telegram group follow-ups now prefer the same speaker's most recent exchange with the bot as a continuation anchor, which reduces cases where old group-level compact-summary context hijacks a user's current follow-up.

### Who Should Update Soon

- Users who recently hit Feishu direct-chat busy / placeholder hangs, missing final replies, or false-success states.
- Users who rely on multi-agent web research and recently saw tool-call protocol errors or search-stage failures.
- Maintainers who use skills heavily, especially tool-gated skills such as `scheduled_task`, and expect every visible skill to be actually executable in the current session.
- Users who ask frequent follow-up questions in Telegram group chats and want the bot to preserve same-speaker continuity more reliably.

### What Will Change After Upgrading

- Feishu direct-chat placeholder and busy branches now behave more conservatively around real agent-run startup, reducing misleading intermediate states.
- The multi-agent search pipeline now performs more complete context cleanup, which should reduce tool-call protocol failures on OpenAI-compatible providers.
- If a skill cannot run in the current stage, it now disappears from visible surfaces and returns an explicit missing-tool error on lower-level invocation paths instead of looking like a random runtime failure.
- In Telegram groups, same-speaker follow-ups are more likely to continue from that speaker's own last exchange instead of being pulled toward unrelated compact-summary context.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- The broader Claude Code skill-runtime alignment is still in progress; this release closes the visible-vs-usable contract, stage tool gating, and `HONE_SKILLS_DIR` forwarding, but it does not mean hook execution and watcher hot reload are fully done.
- Some `Heartbeat` issues remain under active tracking and are not the main scope of this release.
