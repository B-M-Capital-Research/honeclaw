# v0.11.0

发布日期：2026-05-11

English readers: please scroll down for the English version below.

## 用户应该知道的

- 新增 `hone-cli web admin-ui` 和 `hone-cli web user-ui`，安装用户可以通过 CLI 启动或定位管理后台与用户页面，不需要记住 Bun / Vite 的底层命令。
- `curl | bash` 和 Homebrew 安装包现在会同时携带管理端 `web` 与用户端 `web-public` 静态资源，用户端页面不再依赖源码 checkout 才能被本地 runtime 服务。
- 源码开发仍保留 `bun run dev:web` 和 `bun run dev:web:public`；推荐日常使用 `cargo run -p hone-cli -- web admin-ui --dev` / `cargo run -p hone-cli -- web user-ui --dev`，让端口和后端 URL 默认值与安装路径一致。
- `hone-cli start` 启动后会明确打印管理端和用户端本地 URL，减少“后端启动了但前端去哪开”的困惑。

## 谁应该尽快更新

- 通过 Homebrew 或安装脚本使用 Hone CLI，并希望从命令行直接启动本地 Web 管理台或用户页面的用户。
- 需要验证 release 包是否完整包含管理端和用户端 Web 资源的维护者。
- 做前端或安装路径开发的贡献者。

## 升级后会有什么变化

- 安装包 wrapper 会额外设置 `HONE_PUBLIC_WEB_DIST_DIR`，默认指向 `share/honeclaw/web-public`。
- release workflow 会构建并打包 `packages/app/dist-public` 到 `share/honeclaw/web-public`。
- `hone-cli web admin-ui` / `hone-cli web user-ui` 在源码 checkout 中会走 Bun/Vite；在安装路径中会使用已打包的静态资源并按需启动 `hone-console-page` 的 Web-only 前台进程。
- 自 `v0.10.0` 以来的巡检清理也已包含在本版本中，主要覆盖配置示例一致性、错误文案、测试可维护性、复杂度热点和前端设置状态整理。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- `--dev` / `--no-build` 是源码开发模式，仍要求本机有 Bun 和仓库源码。
- 完整渠道监听仍由 `hone-cli start` 管理；`hone-cli web ...` 适合在另一个终端启动或定位 Web UI。
- 架构 SVG 不需要更新：本次没有改变 admin / public 双前端结构，只收敛了 CLI 启动入口和 release bundle 打包路径。

## English

Release date: 2026-05-11

### What Users Should Know

- Added `hone-cli web admin-ui` and `hone-cli web user-ui`, so installed CLI users can start or locate the admin console and user-facing page through the CLI without memorizing Bun / Vite commands.
- `curl | bash` and Homebrew bundles now ship both admin `web` assets and user `web-public` assets, so the local runtime can serve the user-facing page without a source checkout.
- Source development still supports `bun run dev:web` and `bun run dev:web:public`; daily development should prefer `cargo run -p hone-cli -- web admin-ui --dev` / `cargo run -p hone-cli -- web user-ui --dev` so ports and backend URL defaults match the installed path.
- `hone-cli start` now prints the local admin and user Web URLs after startup.

### Who Should Update Soon

- Users who installed Hone CLI through Homebrew or the install script and want command-line managed local Web startup.
- Maintainers who need release bundles to contain both admin and user Web assets.
- Contributors working on frontend or install-path behavior.

### What Will Change After Upgrading

- Installed wrappers now export `HONE_PUBLIC_WEB_DIST_DIR`, pointing to `share/honeclaw/web-public` by default.
- The release workflow builds and packages `packages/app/dist-public` into `share/honeclaw/web-public`.
- `hone-cli web admin-ui` / `hone-cli web user-ui` use Bun/Vite in a source checkout; in an installed bundle they use packaged static assets and can start a Web-only foreground `hone-console-page`.
- Maintenance patrol fixes since `v0.10.0` are included, covering config-example drift, error copy, test maintainability, complexity hotspots, and frontend settings state cleanup.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- `--dev` / `--no-build` are source-development modes and still require Bun plus a source checkout.
- Full channel listeners are still managed by `hone-cli start`; `hone-cli web ...` is best used in another terminal to start or locate the Web UI.
- Architecture SVG does not need an update: this release does not change the admin / public dual-surface architecture, only the CLI startup entrypoints and release bundle layout.
