# v0.2.8

发布日期：2026-04-24

English readers: please scroll down for the English version below.

## 用户应该知道的

- Event engine 的 Telegram 摘要降噪更严格。Cramer、earnings preview、估值观点、Zacks/SeekingAlpha/Benzinga/247WallSt 等低信号股票新闻不再挤占摘要前排。
- 摘要省略项现在有可审计记录。后续 `…… 另 N 条已省略` 不再只能靠历史 buffer 反推，诊断导出会同时展示 omitted items。
- WatcherGuru / Truth Social 的低优先级社交消息更保守。只有 LLM 判定为重要后升到 Medium 的社交事件才会进入 digest，带 `$TSLA`、`$AAPL`、`$SPX` 等 cashtag 但判定不重要的帖子会被过滤。
- 收盘大幅波动不再等到下一个摘要。超过 high 阈值的 `price_close` 会走即时提醒，普通收盘波动仍留在 digest。
- Truth Social poller 的失败日志补齐了 HTTP status、content type 和响应前缀，后续真实断流更容易定位。

## 谁应该尽快更新

- 依赖 Telegram event-engine 摘要做盘中 / 晨间扫盘的用户。
- 之前觉得摘要里新闻太多、社交噪声太多，或者看不到“省略了什么”的用户。
- 依赖收盘大幅波动提醒、尤其是希望 AAOI / RKLB / TEM 这类超过 6% 的 close move 及时触达的用户。
- 需要继续排查 Truth Social / WatcherGuru 社交源质量的维护者。

## 升级后会有什么变化

- Low stock news、opinion / PR wire news、no-op analyst hold / reiterate、远期低优宏观、低优先级 WatcherGuru / Truth Social social 会更早被过滤。
- `thenewswire.com` 这类微盘 PR wire 不再进入 uncertain-source LLM 仲裁，避免被模型误升进 digest。
- Analyst grade 如果只是 `Outperform → Outperform`、`Buy → Buy` 这类无评级变化，不会再因为 `immediate_kinds=["analyst_grade"]` 被强制即时推送。
- 批量校准已用 FMP 499 条、WatcherGuru 220 条真实样本跑过；补丁后同批 WatcherGuru 保留项从 26 条降到 11 条，只保留 LLM yes 的 Medium 社交。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 历史 digest 在本版本之前没有 `digest_item omitted` 行，旧批次仍需要从 `data/digest_buffer/*.flushed-*` 反推。
- 本版本没有把所有低强度非新闻事件都过滤掉；小幅价格波动等非新闻项仍受现有用户偏好和阈值控制。
- tag 推送会触发远端 release workflow；本地只能确认 tag 已推送，最终产物发布结果仍以 GitHub Actions 为准。

## English

Release date: 2026-04-24

### What Users Should Know

- Event-engine Telegram digests are now stricter about low-signal content. Cramer pieces, earnings previews, valuation commentary, and low-value Zacks / SeekingAlpha / Benzinga / 247WallSt style articles no longer crowd the digest.
- Omitted digest items are now auditable. Future “N more items omitted” footers will have `digest_item omitted` delivery-log records and diagnostic exports.
- WatcherGuru / Truth Social social posts are more conservative. Only posts promoted to Medium by the LLM importance check stay in digest; cashtag-only low-priority posts that the model says are not important are filtered.
- Large closing-price moves no longer wait for the next digest. `price_close` events above the high threshold can be delivered immediately, while normal close moves remain digest-only.
- Truth Social poller diagnostics now include HTTP status, content type, and a response-body prefix, making live source failures easier to debug.

### Who Should Update Soon

- Users relying on Telegram event-engine digests for intraday or morning market scanning.
- Users who found the digest too noisy or could not inspect omitted items.
- Users relying on large close-move alerts, especially moves like AAOI / RKLB / TEM above roughly 6%.
- Maintainers continuing to tune Truth Social / WatcherGuru source quality.

### What Will Change After Upgrading

- Low stock news, opinion / PR-wire news, no-op analyst hold / reiterate items, far-future low-priority macro events, and low-priority WatcherGuru / Truth Social social posts are filtered earlier.
- `thenewswire.com` is treated as PR wire, so microcap press releases no longer enter uncertain-source LLM arbitration and get accidentally promoted.
- Analyst grade items with no actual rating change, such as `Outperform → Outperform` or `Buy → Buy`, are no longer forced into immediate delivery by `immediate_kinds=["analyst_grade"]`.
- Batch calibration covered 499 FMP news items and 220 WatcherGuru posts. On the same sample set, WatcherGuru retained items dropped from 26 to 11, keeping only LLM-yes Medium social posts.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- Historical digests created before this version still do not have `digest_item omitted` rows; old batches must still be reconstructed from `data/digest_buffer/*.flushed-*`.
- This release does not filter every low-intensity non-news event. Small price moves and similar non-news items remain governed by existing user preferences and thresholds.
- Pushing the tag triggers the remote release workflow; local verification can confirm the tag push, while final artifact publication still depends on GitHub Actions.
