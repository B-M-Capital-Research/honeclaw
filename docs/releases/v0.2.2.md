# v0.2.2

发布日期：2026-04-22

English readers: please scroll down for the English version below.

## 用户应该知道的

- 每个用户 / 渠道 actor 现在可以单独配置推送节奏：`timezone` 使用 IANA 时区名，`digest_windows` 使用本地 `HH:MM` 列表。留空时沿用全局 digest 设置，传空列表时只关闭该 actor 的 digest，不影响 immediate 推送。
- 每个 actor 现在可以单独降低价格异动即时推阈值：`price_high_pct_override` 命中后会把对应 `PriceAlert` 升为 High 并走 immediate sink，但仍保留每日 High 上限和 cooldown。
- 每个 actor 现在可以把指定事件类型强制即时推：`immediate_kinds` 支持用 kind tag 把如 `weekly52_high`、`weekly52_low`、`analyst_grade` 等事件升为 High。
- Web 控制台的通知偏好卡片已加入这些 per-actor 控制项；`notification_prefs` tool 和 Web API 也同步支持，并会校验 IANA 时区、`HH:MM`、百分比范围和 kind tag。
- Web runtime 日志改为每日滚动并保留最近 15 天；`acp-events.log` 也会按日归档和清理，避免长期运行后日志文件无限增长。
- 如果你是从上一个正式 tag `v0.2.0` 升级，这个版本也包含 `v0.2.1` 已落库的修复：新闻来源信誉分类与 LLM 仲裁、heartbeat 写盘失败降级、以及 news digest 单 symbol burst 限流。

## 谁应该尽快更新

- 需要给不同 Telegram / Feishu / Web 用户设置不同 digest 时区、摘要时刻或关闭 digest 的使用者。
- 希望某些用户对价格异动更敏感，或希望把 52 周新高/新低、分析师评级等事件即时推送的人。
- 长期运行桌面 / Web runtime，并依赖 `data/runtime/logs` 排障的人。
- 仍在 `v0.2.0` 官方 tag 的用户；本次 release 汇总了 `v0.2.1` 和 `v0.2.2` 的正式发布内容。

## 升级后会有什么变化

- 旧的 `data/notif_prefs/<actor>.json` 继续兼容；新字段缺失时会回到全局默认，不需要迁移。
- `digest_windows: []` 是显式关闭 digest；`digest_windows: null` 或字段缺失才是沿用全局窗口。
- `price_high_pct_override` 只影响该 actor 的路由结果，不会改写原始事件 severity。
- `immediate_kinds` 只接受内置 kind tag；未知 tag 会在 tool / API 层被拒绝。
- `web.log` 文件名会进入 rolling appender 形态，保留最近 15 天。依赖旧日志路径的脚本应确认自己的 glob 规则能匹配新的日滚动文件。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 新增的 `per_actor_override_e2e` example 会读取真实 `config.yaml` 和 `data/notif_prefs` 并可能触发真实 Telegram 推送；它是手工验证入口，不属于默认 CI。

## English

Release date: 2026-04-22

### What Users Should Know

- Each user / channel actor can now define its own delivery cadence: `timezone` accepts an IANA time zone name, and `digest_windows` accepts local `HH:MM` times. Missing values keep the global digest settings; an empty list disables digest only for that actor while leaving immediate delivery available.
- Each actor can now lower its own price-alert immediate threshold: `price_high_pct_override` promotes matching `PriceAlert` events to High for that actor, while still honoring the daily High cap and cooldown.
- Each actor can force selected event kinds into immediate delivery through `immediate_kinds`, such as `weekly52_high`, `weekly52_low`, and `analyst_grade`.
- The Web console notification preferences card now exposes these per-actor controls. The `notification_prefs` tool and Web API support the same fields and validate IANA time zones, `HH:MM` windows, percentage ranges, and kind tags.
- Web runtime logs now roll daily and retain the latest 15 days. `acp-events.log` is also rotated and cleaned up by day, preventing long-running installs from growing a single log file indefinitely.
- If you are upgrading from the previous official tag `v0.2.0`, this release also includes the fixes already recorded for `v0.2.1`: news source reputation plus LLM arbitration, degraded heartbeat reporting on write failures, and single-symbol burst limiting for news digests.

### Who Should Update Soon

- Users who need different digest time zones, summary times, or digest-off behavior for different Telegram / Feishu / Web actors.
- Users who want some actors to receive more sensitive price alerts, or to immediately receive 52-week high/low and analyst-grade events.
- Operators who keep the desktop / Web runtime running for long periods and rely on `data/runtime/logs` for debugging.
- Anyone still on the official `v0.2.0` tag; this release packages the formal `v0.2.1` and `v0.2.2` changes.

### What Will Change After Upgrading

- Existing `data/notif_prefs/<actor>.json` files remain compatible. Missing new fields fall back to global defaults.
- `digest_windows: []` explicitly disables digest; `digest_windows: null` or a missing field means "use the global windows".
- `price_high_pct_override` only changes routing for that actor. It does not rewrite the original event severity.
- `immediate_kinds` accepts only built-in kind tags; unknown tags are rejected by the tool / API validation layer.
- `web.log` moves to a daily rolling appender shape with a 15-day retention window. Scripts that depend on old log paths should confirm their glob patterns still match the rolled files.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- The repository already contains `docs/releases/v0.2.1.md`, but the latest real remote tag is still `v0.2.0`; therefore this `v0.2.2` compare range starts from `v0.2.0`.
- The new `per_actor_override_e2e` example reads real `config.yaml` and `data/notif_prefs` and may send real Telegram messages. It is a manual verification entry point, not part of default CI.
