# v0.7.0

发布日期：2026-05-03

English readers: please scroll down for the English version below.

> v0.6.0 之后的功能 release。两条主线：**整套 admin 控制台 + CLI onboard 双语化（中文 / English）**、**前端"thesis → 投资主线 / mainline" 整体改名**。无破坏性变更；旧 `config.yaml` 直接启动即可，新增 `language` 字段缺省为 `zh`，行为与旧版一致。架构 SVG 已经在本次发布里随 `resources/architecture.svg` 一并刷新。

## 用户应该知道的

### 1. 整套界面 + CLI 走进双语时代（中文 / English）

之前 admin 控制台 12 个页面 + ~30 个组件全部硬编码中文，海外用户和合作伙伴只能借机翻；CLI `onboard` 也只有中文。本 release 把双语支持从公共站延伸到全栈：

**a) 后端持久化 + API 暴露**
- `HoneConfig.language: Locale`（`zh | en`，默认 `zh`），写到 `data/canonical/config.yaml`，是设备维度的 canonical 选择。
- `MetaInfo` 里 `/api/meta` 多了 `language` 字段；前端首次加载若 `localStorage` 没设置过本地 override，按这个值 boot。
- 新 endpoint `PUT /api/language`（bearer 鉴权），admin Settings 页一键切换会原子地写回 YAML 并触发轻量刷新。

**b) 前端整套翻译**
- 新增 `packages/app/src/lib/admin-content/` 目录，按 admin 页面切成 15 个模块（dashboard / sessions / tasks / users / research / portfolio / company-profiles / notifications / schedule / task-health / llm-audit / logs / skills / settings + shared 公用 nav / 按钮 / toast）。
- `lib/i18n.ts` 抽出公共 `makeContentProxy(zh, en)`、`tpl({name})` 占位符插值、`formatDate / formatNumber` 区域感知格式化、`pluralRule` 工具，与公共站共享。
- 侧边栏底部新增 `中文 | EN` 分段控件：`localStorage` 单设备覆盖，不会写回 YAML，方便某台机器临时切语言。
- Settings 页顶部新增 "Language / 语言" 卡片：写 YAML 同时调 `setLocale()` 让进程内信号立刻翻新。
- 新增 `lib/admin-content/structure.test.ts` 双语 key shape parity guard：任一文件 ZH / EN 树形不一致会立刻 CI 报错，避免漏翻。

**c) CLI `onboard` 全流程双语化**
- 步数从 6 步扩到 7 步，新加 **第 1 步 Language**（默认从 `LC_ALL` / `LANG` 嗅探）。
- 新 `bins/hone-cli/src/i18n.rs` 模块，集中维护 ~170 个翻译 key，覆盖 runner / channel / admin / provider / notifications / recovery / discord_token / yaml.apply / apply / lang / step / banner 共 12 个 namespace。
- 之后所有 prompt / validation / success / fail 行都走 `t!(lang, "...")`，缺 key 时 fallback 到 key 字面量（CI 一眼看出来）。
- `language` 选择被原子地预置进 onboard 的 `ConfigMutation::Set` 列表，与其余字段一起写入。

**d) 非 onboard 子命令的语言解析**
- `Lang::from_locale(hone_core::config::Locale)` 把已持久化的 `language` 字段桥到 CLI 显示语言。
- `resolve_lang(Option<&Path>)` 在执行 `apply` / `recovery` 等子命令时按优先级：CLI `--lang` flag > config 文件 > 系统 locale > 中文 fallback。

### 2. 前端 "thesis → mainline / 投资主线" 改名

旧 admin tab "投资观点 / Thesis" 与公共站 `/thesis` 路径在投资语境里词不达意（thesis 在英文里是"论文"也是"假设"，与中文"投资主线"语境对不上），本 release 全栈统一：

- admin tab + 路由 `/thesis` → `/mainline`（旧路径会 redirect，链接不会断）。
- 公共站对应入口同步改名。
- 文案上中文统一为"投资主线"，英文统一为 "Mainline"。
- 数据库表 / API path 在后端层保持稳定（仅前端路由 + 显示文案改），不需要数据迁移。

### 3. 其他改动

- **task-health 列表展示失败时间戳与恢复状态**（7802496）：之前只看到当前 health 状态，无法判断"上次掉是什么时候 / 是否已经自愈"，现在两栏都能看到。
- **Feishu clarification fallback 不再被静默吞掉**（8f84da0）：之前 fallback 路径在某些情况下会 suppress 给用户的澄清提问，已修复。
- **Star history 链接走 timeline 视图**（431c1df）：README 上的 star history 链接更直观。
- 架构 SVG `resources/architecture.svg` 随本次双语前端结构调整同步刷新。

## 谁应该尽快更新

- **海外用户 / 英语优先的合作伙伴**：本 release 后整套 admin + CLI 都能用英文跑，再也不需要靠浏览器机翻或对照截图工作。
- **多人共用一台 Hone 实例的小团队**：YAML 里 `language` 是设备 canonical 默认，每个人可以用 sidebar `中文 | EN` 在自己设备上 override，不会互相影响。
- **跑 `hone-cli onboard` 引导新用户的运营方**：onboard 第 1 步会先问语言，比之前直接进运行模式选择更友好。
- **关心 mainline 路由的前端二开**：旧路径 `/thesis` 还在 redirect，但请把外链 / 文档逐步改到 `/mainline`。

## 升级后会有什么变化

- `HoneConfig` 新增 `language` 字段（默认 `zh`）。旧 config 不需要改，行为完全等价。
- `/api/meta` 响应多 `language` 字段；前端首次加载会按它 boot 语言。
- `/api/language` 是新 endpoint（PUT，bearer 鉴权），仅 admin Settings 卡片会调；外部脚本如需切语言可以直接 `curl` 上去。
- admin 控制台首次加载到 v0.7.0 时，如果 `localStorage` 没存过 `hone-public-locale` key，会按 YAML `language` 显示（迁移前默认 `zh`，与旧版完全一致）；如果之前手动通过公共站 sidebar 切过，那个本地 override 会保留。
- `hone-cli onboard` 流程从 6 步变 7 步；新加的第 1 步默认按系统 `LANG` 自动选。已经 onboard 过的部署不会被重新触发。
- 公共站 / admin 路由 `/thesis` 自动 redirect 到 `/mainline`；外部链接、Slack 收藏、Discord 嵌入等不会断，但建议手动更新到新路径。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本。
- Homebrew 用户：`brew update && brew upgrade honeclaw`。
- 切语言的两条路径：
  - **canonical**：admin Settings → "Language / 语言" 卡片，写 YAML，所有从这台实例打开浏览器、且未手动 override 过的人都按这个。
  - **per-device**：sidebar 底部 `中文 | EN`，只写 `localStorage["hone-public-locale"]`，单设备覆盖。

## 已知注意事项

- 双语翻译覆盖的是 admin 用户可见 string；后端 Rust 日志、错误堆栈、channel（Telegram / Discord / Feishu / iMessage）发给终端用户的消息仍按原有 per-user / per-channel 偏好渲染，不会跟随 admin 语言切换。
- CLI `onboard` 的 banner / step header / 验证消息已全双语，但 module-level `//!` doc 注释仍是中文（developer-facing，本次有意保留）。
- 翻译 parity guard (`structure.test.ts`) 走的是 key shape 比较，不会校验"翻译是否准确"。如果发现某个英文翻译别扭，可以直接改 `lib/admin-content/<page>.ts` 里的 EN 分支，无需后端动作。
- tag 推送只代表 GitHub release workflow 被触发，最终 tarball / Homebrew formula 是否发布以 GitHub Actions 结果为准。

## English

Release date: 2026-05-03

> Feature release after v0.6.0. Two headline themes: **full bilingual (Chinese / English) support across the admin console and CLI onboard**, plus the **frontend "thesis → mainline / 投资主线" rename**. No breaking changes; existing `config.yaml` boots as-is — the new `language` field defaults to `zh`, preserving prior behavior. The architecture SVG (`resources/architecture.svg`) was refreshed alongside the bilingual restructure.

### What Users Should Know

#### 1. Bilingual Admin Console + CLI (Chinese / English)

Previously the admin console (12 pages + ~30 components) was hardcoded Chinese, and `hone-cli onboard` only spoke Chinese — overseas users and partners had to lean on browser machine-translation. This release extends bilingual support from the public site into the entire stack:

**a) Backend persistence + API exposure**
- `HoneConfig.language: Locale` (`zh | en`, defaults to `zh`) is persisted to `data/canonical/config.yaml` as the canonical per-instance choice.
- `/api/meta` gains a `language` field; the frontend boots from it on first load when no `localStorage` override is set.
- New endpoint `PUT /api/language` (bearer-authenticated): the admin Settings page uses it to atomically write back YAML and broadcast to other devices on next refresh.

**b) Full frontend translation**
- New directory `packages/app/src/lib/admin-content/` carved into 15 page-scoped modules (dashboard / sessions / tasks / users / research / portfolio / company-profiles / notifications / schedule / task-health / llm-audit / logs / skills / settings + a shared module for nav / buttons / toasts).
- `lib/i18n.ts` exposes shared helpers: `makeContentProxy(zh, en)`, `tpl({name})` placeholder interpolation, locale-aware `formatDate / formatNumber`, and a pluralization rule helper — reused by the public site.
- A `中文 | EN` segmented switcher at the bottom of the sidebar provides per-device override via `localStorage` only — does not write back to YAML.
- A "Language / 语言" card at the top of the Settings page writes YAML and calls `setLocale()` so the in-process signal flips immediately.
- New `lib/admin-content/structure.test.ts` parity guard: if any module's ZH / EN trees diverge in shape, CI fails immediately — catches missing translations early.

**c) CLI `onboard` is now fully bilingual**
- Step count goes from 6 to 7 with a new **Step 1: Language** (default sniffed from `LC_ALL` / `LANG`).
- New `bins/hone-cli/src/i18n.rs` module centralizes ~170 translation keys spanning 12 namespaces: runner / channel / admin / provider / notifications / recovery / discord_token / yaml.apply / apply / lang / step / banner.
- Every prompt / validation / success / failure line now flows through `t!(lang, "...")`. Missing keys fall back to the literal key string — visible to CI at a glance.
- The chosen `language` is atomically prepended to the onboard `ConfigMutation::Set` list and written alongside the rest.

**d) Language resolution for non-onboard subcommands**
- `Lang::from_locale(hone_core::config::Locale)` bridges the persisted `language` field to the CLI's display language.
- `resolve_lang(Option<&Path>)` follows priority: CLI `--lang` flag > config file > system locale > Chinese fallback.

#### 2. Frontend "thesis → mainline / 投资主线" Rename

The old admin tab "投资观点 / Thesis" and public path `/thesis` didn't translate cleanly (thesis evokes "academic paper" or "hypothesis" in English — neither matches the Chinese "投资主线" investing context). This release unifies the surface:

- admin tab + route `/thesis` → `/mainline` (the old path redirects, so existing links don't break).
- Public site entry renamed in lockstep.
- Copy: Chinese standardizes on "投资主线", English on "Mainline".
- Database tables / backend API paths are unchanged (only frontend routes + displayed copy moved) — no data migration required.

#### 3. Other Changes

- **task-health list shows failure timestamp + recovery status** (7802496): you now see both "when it last broke" and "has it self-healed" instead of just the current health state.
- **Feishu clarification fallback no longer silently swallowed** (8f84da0): the fallback path was suppressing user-facing clarification questions in some cases — fixed.
- **Star history link uses the timeline view** (431c1df): the README's star history link is more readable.
- The architecture SVG (`resources/architecture.svg`) was refreshed alongside the bilingual frontend restructure.

### Who Should Update Soon

- **Overseas users / English-first partners**: post-upgrade, the entire admin + CLI runs in English. No more leaning on browser machine-translation or comparing screenshots.
- **Small teams sharing a single Hone instance**: the YAML `language` is the canonical per-device default; each user can override on their own device via the sidebar `中文 | EN` switcher without affecting others.
- **Operators running `hone-cli onboard` for new users**: the onboard wizard now asks for language as Step 1, friendlier than diving straight into runner selection.
- **Frontend integrators referencing the mainline route**: the old `/thesis` path still redirects, but please migrate external links / docs to `/mainline` over time.

### What Will Change After Upgrading

- `HoneConfig` gains a `language` field (default `zh`). Existing configs need no changes — behavior is identical.
- `/api/meta` responses include `language`; frontend uses it for first-load boot.
- `/api/language` is a new PUT endpoint (bearer-authenticated). Only the admin Settings card calls it; external scripts can `curl` it directly to flip language.
- On first load against v0.7.0, if `localStorage["hone-public-locale"]` is unset, the admin console boots according to the YAML `language` (default `zh`, fully equivalent to the old version); if you previously set a sidebar override on the public site, that local override is preserved.
- `hone-cli onboard` flow goes from 6 to 7 steps; the new Step 1 defaults to your system `LANG`. Already-onboarded deployments are not re-prompted.
- Public site / admin route `/thesis` automatically redirects to `/mainline` — external links, Slack bookmarks, Discord embeds will not break, but updating them is recommended.

### Install and Update

- `curl | bash` users: rerun the install script.
- Homebrew users: `brew update && brew upgrade honeclaw`.
- Two ways to flip language:
  - **Canonical**: admin Settings → "Language / 语言" card writes YAML; everyone opening this instance who hasn't manually overridden will follow.
  - **Per-device**: sidebar `中文 | EN` writes `localStorage["hone-public-locale"]` only — single-device override.

### Known Notes

- Bilingual translation covers admin-visible strings only. Backend Rust logs, error stacks, and channel-side messages to end users (Telegram / Discord / Feishu / iMessage) still render per-user / per-channel preferences and do not follow the admin language switch.
- The CLI `onboard` banner / step headers / validation messages are fully bilingual, but module-level `//!` doc comments remain in Chinese (developer-facing — intentionally left for now).
- The parity guard (`structure.test.ts`) compares key shape, not translation accuracy. If you spot an awkward English translation, edit the EN branch in the corresponding `lib/admin-content/<page>.ts` directly — no backend change needed.
- Pushing the tag only triggers the GitHub release workflow; whether the tarball / Homebrew formula actually publishes must be confirmed via GitHub Actions.
