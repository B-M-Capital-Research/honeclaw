# v0.6.0

发布日期：2026-05-02

English readers: please scroll down for the English version below.

> v0.5.1 之后的功能 release。三大主线:**ExtendedHoursPoller 美股盘前/盘后价格异动捕捉**、**单规则价格分档推送**(完全替换原 cap+gap+bypass 三旋钮机制)、**SEC filings whitelist 扩到 5 类 form + LLM thesis-investor 摘要 enrichment**。无破坏性变更;旧 `config.yaml` 直接启动即可,新增字段全部带默认值。

## 用户应该知道的

### 1. 美股盘前 / 盘后价格异动即时捕捉(新通道)

`event_engine.sources.extended_hours = true`(默认开启)。新建 `ExtendedHoursPoller` 在 ET 04:00–09:30 / 16:00–20:00 窗口每 30 分钟拉一次 FMP `/v3/historical-chart/1min?extended=true`,按持仓逐只算 30min 振幅 (`max_high - min_low / prev_close`)。振幅 ≥ 用户 `price_high_pct_override`(回退全局 `event_engine.thresholds.price_alert_high_pct`)即生成 Severity::High `PriceAlert`,自动走现有 dispatch / quiet_hours / sink 全链路。

事件 id 形如 `extended:{SYM}:{ET_DATE}:{pre|post}`,SQLite `INSERT IGNORE` 保证同股票同 session 全天最多一条触达 sink。`PriceAlert.window` 取值扩到 `"day" | "close" | "pre" | "post"`,`policy.rs::is_intraday_price_band_alert` 不变(`window != "close"` 已正确识别为非收盘),quiet 内自动 hold 并由 quiet_to flush 转 "🌙 凌晨曾过" 隔夜 recap(沿用 v0.5.0 fix #2)。

**修复的真实场景**:GOOGL 2026-04-29 Q1 财报夜 ET 16:00 披露后盘后三个 30min 窗振幅 +4.80% / +4.30% / +3.12%,旧版 `PricePoller` 走 `/v3/quote` 在 pre/post 不更新 timestamp 被 `is_us_regular_close_quote` + 15min stale 滤掉,用户隔 16h 直到次日 ET 09:30 开盘才看到累计 +6%。新通道当晚就能推。

Renderer header 加盘前 ☀️ / 盘后 🌃 emoji,与盘中 🔔 区分。

### 2. 价格分档推送从 3 旋钮简化为 1 个 advance 阈值

旧机制:`price_symbol_direction_daily_cap`(N 条 / 日)+ `price_dispatch_band_pct_gap`(分档间距)+ bypass 例外。三旋钮交互复杂,真实场景 AAOI 单日 +6/+8/+10/+12/+14/+16% 序列里被 `cap=2` 截到只发 6/8 两条,后面 4 条全 `price_capped`,用户错过第二天关键卖点。

新机制:**单规则** `price_band_min_advance_pct` (默认 2.0)。下一条 `PriceAlert` 必须比该 ticker+方向的上一次成功推送在绝对值上至少前进这么多 pct,否则 cooldown drop。无每日次数上限、无 bypass 例外。

POC 在 AAOI 上验证:6 → 8 → 10 → 12 → 14 → 16% 全部推送(每档前进 ≥ 2%),哑火序列 6 → 6.5 → 6.3 → 6.7 全部 dropped。

行为变化:`event_engine.thresholds.price_symbol_direction_daily_cap` 与 `price_dispatch_band_pct_gap` **完全删除**,旧 config 仍可启动(serde 忽略未知字段)。

### 3. SEC filings 通道扩展 + LLM thesis-investor 摘要

之前 `SecFilingsPoller` 硬编码只拉 8-K,把 10-Q / 10-K / S-1 / DEF 14A 全漏掉 —— 而 10-Q 季报里的 backlog / 大客户变化、10-K 风险因子新增、S-1 增发稀释信号、DEF 14A 治理变化对 long-thesis 投资者价值并不亚于 8-K。本 release:

**a) Form whitelist 扩到 5 类**(默认覆盖 8-K / 10-Q / 10-K / S-1 / DEF 14A,可在 `event_engine.sec_filings.forms` 自定义)。Severity 按 form 业务影响硬映射:8-K / S-1 → High,10-Q / 10-K → Medium,DEF 14A → Low。

**b) LLM 摘要 enrichment**(默认开启,可在 `event_engine.sec_filings.enrichment.enabled` 关闭):每条 SecFiling 事件构造完成后,新建 `LlmSecFilingSummarizer` 抓 SEC.gov HTML → 纯文本抽取(scraper crate)→ 截到 300k 字符 → 调 OpenRouter `x-ai/grok-4.1-fast`(可改 `enrichment.model`)写一段 ~200 字中文摘要到 `payload.llm_summary`。

System prompt 锚定 thesis-investor 视角(POC 在 11 持仓 × 5 form 上验证):**跳过 GAAP 数字**(用户已从财报新闻知道)、抓 backlog / 大客户 / 资本配置 / 风险因子**新增**条目。Renderer 在 immediate 与 digest 行都优先渲染该字段,fallback 到原 form/link body。

POC 实测年成本:11 持仓 × ~70 条 filings/年 ≈ **$0.82/年**(grok-4.1-fast 在 11 持仓 × 5 form 上质量、成本、延迟均第一)。SEC EDGAR 强制要求 User-Agent 含联系邮箱,默认占位 `honeclaw event-engine ops@honeclaw.local` 务必改成自己的。

**enrichment 永远 best-effort**:fetch / extract / LLM 任一失败 → 静默跳过 llm_summary,filing 推送本身不阻塞。

### 4. 其他 bug 修复

- **quiet_flush 隔夜 recap 不再 drop 过期 PriceAlert**(0e3ed6a):shelf_life 过期的 PriceAlert 进 quiet_held 后,quiet_to flush 时改成 "🌙 凌晨曾过" recap 提示而不是直接丢。
- **digest_slots / quiet_hours 互斥校验**(8fe1713):prefs 写入时拒绝两窗互相覆盖的非法配置。
- **App 通知面板渲染 digest_slots / quiet_hours**(3601fb9):前端真实显示用户当前的两组配置,之前是空白。
- **Analyst grade cooldown 按 gradingCompany 而非 symbol**(557e136):多家投行同日给同股升降级不再被互相 cooldown 屏蔽。
- **Feishu quota 拒绝回复保留**(b791c32):quota 拒绝消息不再丢失。
- **Heartbeat 重复事件抑制**(f82179d)、**ACP prompt echo stream leak**(ad37506)、**desktop isolated prompt asset setup**(5bf2ccb)。

## 升级后会有什么变化

- `event_engine.sources.extended_hours` 新增字段,默认 `true`;不想要美股盘前/盘后推送可设 `false`。

- `EventKind::PriceAlert.window` 取值扩到 `"day" | "close" | "pre" | "post"`,enum 本身仍是 String,不破坏序列化。
- 不需要改 sink / channel 代码,所有变更在 event-engine 内闭环。

## English

Release date: 2026-05-02

> Feature release after v0.5.1. Three headline changes: **ExtendedHoursPoller for US pre/post-market price alerts**, **single-rule price-band dispatch** (fully replacing the old cap+gap+bypass three-knob mechanism), and **SEC filings whitelist expanded to 5 form types + LLM thesis-investor summary enrichment**. No breaking changes; existing `config.yaml` boots as-is; all new fields ship with defaults.

### What Users Should Know

#### 1. US Pre/Post-Market Price Alerts (new channel)

`event_engine.sources.extended_hours = true` (default on). New `ExtendedHoursPoller` polls FMP `/v3/historical-chart/1min?extended=true` every 30 minutes during ET 04:00–09:30 / 16:00–20:00 windows, computing per-symbol 30-min amplitude (`max_high - min_low / prev_close`). When amplitude ≥ user's `price_high_pct_override` (falling back to global `event_engine.thresholds.price_alert_high_pct`), it produces a Severity::High `PriceAlert` that flows through the existing dispatch / quiet_hours / sink chain.

Event ids look like `extended:{SYM}:{ET_DATE}:{pre|post}`. SQLite `INSERT IGNORE` guarantees at most one delivery per symbol per session per day. `PriceAlert.window` now allows `"day" | "close" | "pre" | "post"`; `policy.rs::is_intraday_price_band_alert` already reads `window != "close"` as non-close, so quiet-mode behavior carries over (held in quiet, flushed at quiet_to as "🌙 overnight recap" inheriting v0.5.0 fix #2).

**Real scenario fixed**: GOOGL Q1 earnings ET 16:00 on 2026-04-29 produced three post-market 30-min windows with +4.80% / +4.30% / +3.12% amplitude. The old `PricePoller` hits `/v3/quote` which doesn't update timestamps in pre/post and got filtered by `is_us_regular_close_quote` + 15min stale guard, leaving the user blind for 16h until next-day ET 09:30 open showed cumulative +6%. The new channel pushes that night.

Renderer header gains pre ☀️ / post 🌃 emojis, distinct from regular-session 🔔.

#### 2. Price-Band Dispatch Simplified from 3 Knobs to 1 Advance Threshold

Old mechanism: `price_symbol_direction_daily_cap` (N alerts/day) + `price_dispatch_band_pct_gap` (band spacing) + bypass exceptions. Three knobs interacted complexly. Real case: AAOI's single-day +6/+8/+10/+12/+14/+16% sequence got truncated at `cap=2` to only 6/8, with the next 4 marked `price_capped` — user missed key second-day exit signals.

New mechanism: **single rule** `price_band_min_advance_pct` (default 2.0). The next `PriceAlert` must advance at least that many pct (absolute) past the last successful push for the same ticker+direction, else cooldown drop. No daily count cap, no bypass.

POC validated on AAOI: 6 → 8 → 10 → 12 → 14 → 16% all push (each advances ≥ 2%); flat sequence 6 → 6.5 → 6.3 → 6.7 all dropped.

Behavior change: `event_engine.thresholds.price_symbol_direction_daily_cap` and `price_dispatch_band_pct_gap` **fully removed**. Existing config still boots (serde ignores unknown fields) but behavior switches to the new rule.

#### 3. SEC Filings Channel Expansion + LLM Thesis-Investor Summary

Previously `SecFilingsPoller` hardcoded only 8-K, missing 10-Q / 10-K / S-1 / DEF 14A entirely — yet 10-Q backlog/customer shifts, 10-K newly-added risk factors, S-1 dilution signals, and DEF 14A governance changes are no less valuable to long-thesis investors than 8-K. This release:

**a) Form whitelist expanded to 5 types** (default covers 8-K / 10-Q / 10-K / S-1 / DEF 14A; configurable via `event_engine.sec_filings.forms`). Severity hardcoded by form business impact: 8-K / S-1 → High, 10-Q / 10-K → Medium, DEF 14A → Low.

**b) LLM summary enrichment** (default on; togglable via `event_engine.sec_filings.enrichment.enabled`): for each SecFiling event after construction, the new `LlmSecFilingSummarizer` fetches the SEC.gov HTML → extracts text via the `scraper` crate → caps at 300k chars → calls OpenRouter `x-ai/grok-4.1-fast` (configurable via `enrichment.model`) to produce a ~200-character Chinese summary written to `payload.llm_summary`.

The system prompt is anchored to a thesis-investor view (validated by POC across 11 holdings × 5 forms): **skip GAAP numbers** (users already see those in earnings news), capture backlog / large customers / capital allocation / **newly added** risk-factor entries. The renderer prefers this field in both immediate and digest lines, falling back to the original form/link body.

POC measured annual cost: 11 holdings × ~70 filings/year ≈ **$0.82/year** (grok-4.1-fast won quality / cost / latency among 5 candidate models tested). SEC EDGAR mandates a contact email in the User-Agent; the default placeholder `honeclaw event-engine ops@honeclaw.local` **must** be changed to your own.

**Enrichment is always best-effort**: any failure (fetch / extract / LLM) silently skips llm_summary; filing dispatch itself never blocks.

#### 4. Other Bug Fixes

- **quiet_flush no longer drops expired PriceAlerts during overnight recap** (0e3ed6a): shelf-life-expired PriceAlerts held during quiet hours now flush as "🌙 overnight recap" reminders instead of being silently dropped.
- **digest_slots / quiet_hours mutual-mask validation** (8fe1713): prefs writes reject configurations where the two windows fully overshadow each other.
- **App notification panel renders digest_slots / quiet_hours** (3601fb9): the frontend now actually displays these two sections (previously blank).
- **Analyst-grade cooldown keyed by gradingCompany not symbol** (557e136): multiple firms upgrading/downgrading the same stock on the same day no longer mutually cooldown each other.
- **Feishu quota rejection replies preserved** (b791c32): quota-rejection messages no longer disappear.
- **Heartbeat duplicate event suppression** (f82179d), **ACP prompt echo stream leak** (ad37506), **desktop isolated prompt asset setup** (5bf2ccb).

### Who Should Update Soon

- **Deployments running `event_engine`**: the post-market channel and new price-band rule are visible behavior improvements; SEC enrichment is on by default — confirm an OpenRouter key is wired and `sec_filings.enrichment.user_agent` reflects your own email.
- **Users tracking earnings-aftermath disclosures**: 10-Q / 10-K / S-1 / DEF 14A enter the push channel for the first time and ship with LLM summaries — first upgrade will produce a wave of "previously unseen" filing alerts; remove unwanted forms via `event_engine.sec_filings.forms`.
- **Telegram / Discord / Feishu digest users**: pre ☀️ / post 🌃 emojis surface in the header — no channel-side changes required.

### What Will Change After Upgrading

- `event_engine.sources.extended_hours` is a new field, default `true`; set `false` to opt out of US pre/post-market push.
- `event_engine.sec_filings { forms, enrichment }` is a new section, defaults shown in `config.example.yaml`. **Enrichment is default-on**; if no LlmProvider is wired, you'll see a `sec_filings.enrichment.enabled=true 但未注入 LlmProvider — filing 摘要降级到 form/link body` warn line, and filings still push but only show form / link.
- `event_engine.thresholds.price_symbol_direction_daily_cap` and `price_dispatch_band_pct_gap` are **fully removed**; the new field `price_band_min_advance_pct` (default 2.0) takes over. Old configs don't error (serde ignores unknown fields) but behavior switches.
- `EventKind::PriceAlert.window` allows `"day" | "close" | "pre" | "post"`; the underlying enum is still String — serialization unchanged.
- No sink / channel code changes required; all changes are contained inside event-engine.

### Install and Update

- `curl | bash` users: rerun the install script.
- Homebrew users: `brew update && brew upgrade honeclaw`.
- Enable SEC enrichment: confirm `event_engine.sec_filings.enrichment.enabled=true` (default) and an OpenRouter key is injected via `with_global_digest_provider` (hone-web-api wires this automatically; no binary change). Change `sec_filings.enrichment.user_agent` to your own contact email — SEC EDGAR mandates the format.

### Known Notes

- SEC EDGAR throttles or 403s requests with non-compliant User-Agent. The default placeholder UA carries long-term throttling risk — **always change it to your own real email**.
- LLM summary calls go through OpenRouter; grok-4.1-fast was estimated at $0.82/year for 11 holdings × ~70 filings/year, but pricing may change — actual billing is on the OpenRouter dashboard.
- ExtendedHoursPoller does not consult US market holidays. Weekends / federal holidays still fire the 30-min cron, but the FMP endpoint returns empty arrays, so no events are produced — just a small wasted API quota.
- Pushing the tag only triggers the GitHub release workflow; whether the tarball / Homebrew formula actually publishes must be confirmed via GitHub Actions.
