# v0.1.35

发布日期：2026-04-17

English readers: please scroll down for the English version below.

## 用户应该知道的

- Hone 现在新增了 `chart_visualization` skill：当回答趋势、对比、分布这类问题时，模型可以调用本地 Python 渲染器产出 PNG 图表，而不再只能用纯文本描述。
- Web、Feishu、Telegram、Discord 现在对同一类“本地图表”输出有统一契约：Web 会内联显示图表，外部通道会按 `文本 -> 图片 -> 文本` 的顺序发送真实图片，不再把本地 `file://` 路径直接暴露给用户。
- 群聊场景不再像以前那样长时间静默等待。Telegram、Discord、Feishu 群聊现在会显示更克制的 compact 进度提示，让用户知道任务仍在处理中，但不会把 query、命令行或目录路径这类过细执行细节发到群里。
- 会话恢复、历史提取和图片标记解析也一起补强了：除了裸 `file:///...png`，现在连 HTML anchor、Markdown link、紧跟 `<br>` 的本地图片 marker 都能识别；如果图片文件已经失效，系统会退化成友好的提示文案，而不是继续泄露坏掉的绝对路径。

## 谁应该尽快更新

- 依赖 Hone 做股票、估值、黄金或市场分析，并希望答案里直接附带图表的用户。
- 在 Telegram、Discord 或 Feishu 群聊里使用 Hone，希望长任务至少能给出处理中信号、但又不希望暴露底层执行细节的团队。
- 近期排查过本地图片路径漏发、历史回放看不到图表，或 session 恢复后继续引用失效 `file://` 路径的维护者。

## 升级后会有什么变化

- 当相关 skill 产出图表时，运行时会保留结构化 artifact 信息，并把可见图片输出统一成稳定的本地图片 marker；Web 会继续走内联预览，Feishu / Telegram / Discord 会改为上传真实图片。
- 群聊默认会看到 compact 级别的处理中提示，而不是完全隐藏中间进度；私聊仍保持更完整的进度可见性。
- 会话压缩和历史恢复不再把旧的本地图片路径直接喂回模型；已经不存在的图表文件会显示为“请重新生成”的回退提示，减少跨轮引用坏路径的概率。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- `chart_visualization` 当前依赖运行环境里的 `python3 + matplotlib`；如果依赖缺失，skill 会结构化失败并退回纯文本回答，不会自动安装依赖。
- 这次 release 已补齐代码级自动化验证，但真实 Feishu / Telegram / Discord 账号链路图片发送仍主要依赖手工回归；已经运行中的旧 bot 进程也需要重启到新二进制后，才会真正带上新的图片发送逻辑。
- compare 范围里也包含若干活跃 bug ledger / 计划文档更新；它们用于记录修复状态，不应解读成所有相关问题都已完全关闭。

## English

Release date: 2026-04-17

### What Users Should Know

- Hone now includes a `chart_visualization` skill. For trend, comparison, and distribution-style answers, the model can call a local Python renderer to produce PNG charts instead of relying on text only.
- Web, Feishu, Telegram, and Discord now share one consistent contract for local chart outputs: Web renders them inline, while external channels deliver real images in `text -> image -> text` order instead of exposing raw local `file://` paths to users.
- Group chats no longer stay completely silent during longer runs. Telegram, Discord, and Feishu groups now show compact progress hints so users can see the task is still running without leaking raw queries, shell commands, or directory paths into the chat.
- Session restore, history extraction, and local-image marker parsing were also hardened. The runtime now recognizes bare `file:///...png` markers, HTML anchors, Markdown links, and markers followed by tags like `<br>`. If the image file is gone, the runtime degrades to a friendly note instead of leaking a dead absolute path.

### Who Should Update Soon

- Users who rely on Hone for stock, valuation, gold, or market analysis and want the answer to include charts directly.
- Teams using Hone in Telegram, Discord, or Feishu group chats who want visible progress for long tasks without exposing low-level execution details.
- Maintainers who have recently debugged leaked local image paths, missing chart playback in history, or stale `file://` markers being reused after session restore.

### What Will Change After Upgrading

- When a relevant skill produces a chart, the runtime now preserves structured artifact metadata and converts the visible output into a stable local-image marker. Web keeps inline rendering, while Feishu, Telegram, and Discord now upload and send real images.
- Group chats now show compact progress updates by default instead of hiding progress entirely, while direct chats keep fuller progress visibility.
- Session compaction and history restore no longer feed stale local image paths back into the model. Missing chart files now degrade into a “please regenerate” style fallback instead of a broken absolute path.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- `chart_visualization` currently depends on `python3 + matplotlib` in the runtime environment. If that dependency is missing, the skill fails in a structured way and falls back to a text-only answer; it does not auto-install dependencies.
- This release includes code-level automated verification, but real Feishu / Telegram / Discord account-path image delivery still depends mainly on manual regression. Already-running bot processes also need a restart onto the new binary before they pick up the new image-delivery behavior.
- The compare range also includes active bug-ledger and planning doc updates. Those documents record repair status and should not be read as proof that every related issue is fully closed.
