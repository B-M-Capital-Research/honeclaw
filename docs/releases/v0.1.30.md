# v0.1.30

发布日期：2026-04-16

English readers: please scroll down for the English version below.

## 用户应该知道的

- `telegram` 现在正式支持管理员白名单，不再只有 iMessage、Feishu、Discord 能走共享管理员判定。
- 这意味着 Telegram 私聊身份可以被直接识别为管理员，后续像管理员专属 skill、管理员提示词和 `/register-admin` 这类依赖 allowlist 的链路终于能走通。
- 本地配置示例也已经补齐 `admins.telegram_user_ids`，不需要再靠发一条“ADMIN LOCATOR”文本去人工确认 identity 是否被系统识别。

## 谁应该尽快更新

- 通过 Telegram 私聊使用 Hone，并且希望拥有管理员能力的维护者。
- 之前在 Telegram 里已经能看到部分 admin 相关行为，但始终进不了真正管理员链路的用户。
- 需要跨渠道统一管理员配置，不想为 Telegram 单独维护旁路逻辑的仓库维护者。

## 升级后会有什么变化

- 管理员配置结构新增 `admins.telegram_user_ids`，可以直接按 Telegram 数字 user ID 做白名单。
- 共享 `is_admin` / `is_admin_actor` 判定现在会覆盖 Telegram，因此权限判断不再出现“配置层没入口、运行层没分支”的缺口。
- `config.example.yaml` 已暴露 Telegram admin 示例字段，新的部署不会再因为示例配置缺失而误以为 Telegram 不支持管理员。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次 release 补齐的是代码和配置契约；已经在运行的旧进程仍需要一次有意图的重启，才能加载新的 Telegram 管理员判定逻辑。
- `admins.telegram_user_ids` 只解决 allowlist 入口问题；如果你还依赖 `/register-admin <secret>`，对应的 runtime passphrase 仍然需要单独配置。

## English

Release date: 2026-04-16

### What Users Should Know

- `telegram` now has first-class admin allowlist support instead of being the one channel missing from the shared admin identity path.
- Telegram direct-chat identities can now be recognized as admins, which unblocks admin-only skills, admin prompt behavior, and allowlist-gated flows such as `/register-admin`.
- The sample config now exposes `admins.telegram_user_ids`, so maintainers no longer need to rely on ad hoc locator messages just to confirm whether a Telegram identity can participate in the admin flow.

### Who Should Update Soon

- Maintainers who use Hone from Telegram direct chats and need admin privileges there.
- Users who previously saw partial admin-related behavior in Telegram but could not enter the actual admin path.
- Repository maintainers who want one unified admin config model across channels instead of Telegram-specific workarounds.

### What Will Change After Upgrading

- The admin config schema now includes `admins.telegram_user_ids` for Telegram numeric user IDs.
- Shared `is_admin` / `is_admin_actor` checks now cover Telegram, closing the gap where the config surface and runtime behavior were inconsistent.
- `config.example.yaml` now documents the Telegram admin field, so new setups no longer imply that Telegram lacks admin support.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release updates the code and config contract; any already-running old process still needs an intentional restart before the new Telegram admin path becomes active.
- `admins.telegram_user_ids` solves the allowlist entry point only; if you also rely on `/register-admin <secret>`, the runtime passphrase still needs to be configured separately.
