# v0.1.42

发布日期：2026-04-21

English readers: please scroll down for the English version below.

## 用户应该知道的

- 公开聊天登录面完成安全加固：邀请码尝试现在同时按 IP 与手机号限流，邀请码熵提升到 80 bits，并恢复 workflow runner `validateCode` 鉴权字段
- 公开聊天的 Session Cookie 支持通过 `HONE_PUBLIC_SECURE_COOKIE` 显式控制 `Secure` 标志，错误配置会优先保持安全默认，避免被伪造请求头降级
- 公开聊天恢复登录态时会先加载历史消息再进入 ready 状态，减少空白聊天界面闪烁
- 聊天界面现在能展示工具执行步骤与工具调用状态，长回答和工具链路的可见性更好
- 公开站点继续完善中英文内容与导航，新增/更新首页、产品页、路线图、公开聊天相关内容
- Feishu、heartbeat 与 scheduler 链路补齐多项失败收口：运行中重启补偿、上下文溢出恢复、重复提醒控制、跳过信号识别、传输错误重试与发送失败台账更新
- 金融类 skill 与运行时提示进一步收敛，减少实体歧义、旧上下文漂移、重复风险段落和不合规选股措辞

## 谁应该尽快更新

- 使用公开聊天或公开站点的部署方
- 依赖 Feishu 定时任务、heartbeat 监控、公司研报 workflow runner 的用户
- 正在使用金融研究、股票池、估值或公告监控相关 skill 的用户

## 升级后会有什么变化

- 公开登录的失败尝试会更稳定地被限流；攻击者伪造 `X-Forwarded-For` 不再能单独绕过同一手机号的尝试限制
- 如果部署设置了 `HONE_PUBLIC_SECURE_COOKIE=true/false/yes/no/1/0`，Cookie `Secure` 行为会以该配置为准；未设置或空值时仍回退到请求头检测
- 工作流研报启动请求会重新携带 `validateCode`，本地 runner 若启用校验将恢复预期鉴权
- 聊天前端会保留更稳定的历史消息 ID，新增历史行时不再轻易重建已有消息
- 定时任务和 Feishu 出站失败会更倾向于给出可追踪状态，而不是静默跳过或把内部错误直接暴露给用户

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- Tag 推送后仍需等待远端 release workflow 完成二进制包、GitHub Release 与 Homebrew formula 发布

## English

Release date: 2026-04-21

### What Users Should Know

- Public chat authentication is hardened: invite attempts are now rate-limited by both IP and phone number, invite code entropy is raised to 80 bits, and workflow runner requests include the `validateCode` field again
- Public chat session cookies can now use `HONE_PUBLIC_SECURE_COOKIE` to control the `Secure` flag explicitly; invalid values keep a secure default instead of downgrading through spoofable headers
- Public chat restores history before entering the ready state, reducing the empty-chat flash after session restore
- The chat UI now shows tool execution steps and tool-call status, making long answers and tool-heavy runs easier to follow
- The public site continues to improve bilingual content, navigation, homepage/product/roadmap pages, and public chat content
- Feishu, heartbeat, and scheduler paths include several failure-handling fixes: runtime-restart compensation, context-overflow recovery, duplicate alert control, skip-signal detection, transport retry, and clearer send-failure tracking
- Finance skills and runtime prompts were tightened to reduce ambiguous-entity guessing, stale ticker drift, repeated risk sections, and non-compliant stock-picking wording

### Who Should Update Soon

- Operators running the public chat or public site
- Users depending on Feishu scheduled jobs, heartbeat monitors, or the company-report workflow runner
- Users relying on finance research, watchlist, valuation, or alerting skills

### What Will Change After Upgrading

- Failed public-login attempts are throttled more reliably; spoofing `X-Forwarded-For` no longer bypasses per-phone attempt limits by itself
- If `HONE_PUBLIC_SECURE_COOKIE=true/false/yes/no/1/0` is set, cookie `Secure` behavior follows that configuration; unset or empty values still fall back to request-header detection
- Company-report workflow runner requests include `validateCode` again, restoring expected authorization when local runner validation is enabled
- The chat frontend keeps more stable history message IDs, so appended history no longer rebuilds existing rows unnecessarily
- Scheduler and Feishu outbound failures are more likely to produce traceable states instead of silently skipping or exposing internal errors directly to users

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- After the tag is pushed, the remote release workflow still needs to finish binary packaging, GitHub Release publication, and Homebrew formula publication
