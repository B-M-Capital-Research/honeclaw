# v0.11.1

发布日期：2026-05-11

English readers: please scroll down for the English version below.

## 用户应该知道的

- 公开站点的“路线图与文档”页面现在跟随当前 release 版本，显示 `v0.11.1`，并用当前代码能力重写了能力矩阵、渠道接入、架构说明、内置 Skill、开源边界、文档入口、贡献入口和 FAQ。
- 公开 Skill 口径已校准为仓库当前的 16 个公开 Skill。已删除的独立 `valuation` / `stock_selection` / `major_alert` / `one_sentence_memory` 不再作为单独能力展示；估值和筛选入口归入 `stock_research`，事件漏推回查通过 `missed` / `missed_events` 说明。
- 用户可见文案继续收敛：CLI 首装/配置提示、管理后台 LLM audit、日志、通知、持仓、会话、设置、任务健康和任务页的措辞更贴近实际行为。
- SQLite session runtime backfill 增加了回归证明，避免旧 JSON session 在切换 SQLite 索引/运行时存储时被遗漏。

## 谁应该尽快更新

- 依赖公开站点 Roadmap / Docs 页面向用户说明能力范围、安装方式或贡献入口的部署者。
- 使用 SQLite session runtime backend，或正在验证 JSON 到 SQLite session backfill 行为的维护者。
- 维护 CLI 配置向导、事件 digest、missed events、用户文案或公开文档的贡献者。

## 升级后会有什么变化

- `/roadmap` 页面不再只展示少量概览卡片，而是完整渲染 quick start、Now / Next / Later、能力矩阵、渠道、架构、Skill、开源边界、文档、贡献和 FAQ 内容；中英文内容结构保持同构。
- 公开文档会明确当前代码里的 7 个接入渠道、16 个公开 Skill、public/admin user-ui 路由、runner 层、事件与任务边界，以及私域能力边界。
- 这次 release 没有改变 CLI 启动命令、配置文件结构、channel 二进制、runner 类型、存储路径或事件引擎推送策略。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 架构 SVG 不需要更新：本次 release 没有改变 admin / public dual-surface 结构、顶层 Web API 路由、runner、channel、poller、EventKind 或存储布局；只是让公开 Roadmap 页面更完整地渲染已有内容。
- 远端 release workflow 在 tag 推送后由 GitHub Actions 触发；本地只能确认 tag 已推送，不能替代远端产物发布结果。

## English

Release date: 2026-05-11

### What Users Should Know

- The public “Roadmap & Docs” page now follows the current release version, shows `v0.11.1`, and has been rewritten around the capabilities that are actually present in the code: capability matrix, channels, architecture notes, built-in skills, open-source boundary, docs, contribution paths, and FAQ.
- The public skill count now matches the current repository: 16 public skills. Deleted standalone `valuation`, `stock_selection`, `major_alert`, and `one_sentence_memory` entries are no longer advertised as separate skills; valuation and screening live under `stock_research`, while missed-event recovery is documented through `missed` / `missed_events`.
- User-facing copy was tightened across CLI onboarding/configuration prompts and admin surfaces such as LLM audit, logs, notifications, portfolio, sessions, settings, task health, and tasks.
- SQLite session runtime backfill now has a regression test so old JSON sessions do not get missed when the SQLite index/runtime path is enabled.

### Who Should Update Soon

- Operators who rely on the public Roadmap / Docs page to explain product scope, install paths, or contribution entrypoints.
- Maintainers using the SQLite session runtime backend or validating JSON-to-SQLite session backfill behavior.
- Contributors working on CLI configuration copy, event digests, missed events, user-facing copy, or public docs.

### What Will Change After Upgrading

- `/roadmap` now renders quick start, Now / Next / Later, capability matrix, channels, architecture, skills, open-source boundary, docs, contributing, and FAQ content instead of a small overview-only page; Chinese and English content keep the same shape.
- Public docs now explicitly describe the current 7 channels, 16 public skills, public/admin user-ui routes, runner layer, event/task boundaries, and private-feature boundary.
- This release does not change CLI startup commands, configuration shape, channel binaries, runner kinds, storage paths, or event-engine push policy.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- Architecture SVG does not need an update: this release does not change the admin / public dual-surface structure, top-level Web API routes, runners, channels, pollers, EventKinds, or storage layout; it only makes the public Roadmap page render existing content more completely.
- The remote release workflow is triggered by the pushed tag in GitHub Actions. Local verification can confirm that the tag was pushed, but not that remote release assets finished publishing.
