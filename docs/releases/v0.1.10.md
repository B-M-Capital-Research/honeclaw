# v0.1.10

发布日期：2026-04-12

English readers: please scroll down for the English version below.

## 用户应该知道的

- `hone-cli onboard` 现在会把 `FMP` 和 `Tavily` API key 纳入首装向导，不再需要用户自己回忆去哪里补配置
- 对这两个 provider，向导都会要求你明确选择“现在填写”或“这次跳过”，减少首次安装后功能不可用但原因不清楚的情况
- 如果你当场填写 `FMP` key，Hone 会直接写入多 key 配置，并清理旧的单 key 兼容字段

## 谁应该尽快更新

- 准备第一次安装 Hone CLI，或者经常帮别人做首装配置的用户
- 依赖 `data_fetch`、`web_search`、研报/研究工作流，需要尽快把金融数据和联网搜索能力配齐的用户

## 升级后会有什么变化

- onboarding 在 runner 和 channel 之后，会继续询问 `FMP` 和 `Tavily` 配置
- 如果选择配置但输入为空，向导会继续要求你重试或明确跳过，而不是静默结束
- `FMP` 配置会优先写入 `fmp.api_keys`，并把旧的 `fmp.api_key` 清空，避免运行时来源混杂

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次补的是 `hone-cli onboard` 的 provider 步骤；如果你之前已经手动配好 `fmp.api_keys` 或 `search.api_keys`，向导会允许直接保留现有值
- `opencode_acp` 的 provider / model 真相源仍然优先在本机 `opencode` 配置里管理；这次发布没有改变那条约定

## English

Release date: 2026-04-12

### What Users Should Know

- `hone-cli onboard` now includes `FMP` and `Tavily` API keys in the first-run setup flow instead of leaving users to remember where to configure them later
- For both providers, the wizard now requires an explicit choice between “configure now” and “skip for now”, reducing the chance that features stay unavailable without a clear reason
- If you enter an `FMP` key during onboarding, Hone will write it into the multi-key config and clean up the legacy single-key compatibility field

### Who Should Update Soon

- Users installing Hone CLI for the first time, or users who frequently help others with first-run setup
- Users who rely on `data_fetch`, `web_search`, or research/report workflows and want financial data plus network search configured early

### What Will Change After Upgrading

- After the runner and channel steps, onboarding will continue with `FMP` and `Tavily` configuration
- If you choose to configure a provider but submit an empty value, the wizard will ask you to retry or explicitly skip instead of silently finishing
- `FMP` config now prefers `fmp.api_keys` and clears the legacy `fmp.api_key` field to avoid mixed runtime sources

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release only improves the provider step inside `hone-cli onboard`; if you already configured `fmp.api_keys` or `search.api_keys` manually, the wizard will let you keep those values
- The provider/model source of truth for `opencode_acp` still lives primarily in the local `opencode` config; this release does not change that rule
