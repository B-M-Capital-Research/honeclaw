# v0.9.1

发布日期：2026-05-11

English readers: please scroll down for the English version below.

## 用户应该知道的

### 1. LLM 配置现在支持 provider + profile symbol

Hone 现在可以把 LLM 配置收敛成两层：

- `llm.providers` 负责 provider 类型、Base URL、API Key / env、多 Key fallback、timeout 等连接信息。
- `llm.profiles` 负责模型名、`max_tokens`、`temperature`、`top_p`、`reasoning`、JSON 输出格式和 provider-specific 参数。
- event-engine 的 news classifier、SEC filing 摘要、earnings 质检、digest pass1/pass2、去重和 mainline distill 都可以直接引用 profile symbol。
- 旧的 `llm.openrouter` / `llm.auxiliary` 配置仍然保留 fallback，不要求现有用户马上迁移配置。

### 2. Settings 页面可以配置 LLM Profile 路由

桌面 Settings 的 OpenAI-compatible 配置区新增 `LLM Profile 路由`：

- 可以按用途选择 `main`、`aux`、`news_classifier`、`filing_summary`、`earnings_quality`、`digest_fast`、`digest_strong`、`mainline_short` 等 symbol。
- 可以直接编辑每个 profile 的 provider、model、token budget、temperature、top_p、reasoning effort / tokens 和 JSON 输出开关。
- 保存后写入本地 `config.yaml` / runtime overlay，并在 bundled backend 模式下触发重启生效。

### 3. 主动推送 LLM 路由更容易控成本和质量

这次改动把原来散落在 event-engine 里的模型和 token budget 选择集中到 profile 上。你可以更明确地把便宜快速模型用于分类 / digest 初筛，把更强模型用于公告摘要、财报质检或 digest 个性化，不需要在多个配置段重复写同一套参数。

## 谁应该尽快更新

- 依赖 event-engine 主动推送、digest、SEC filing 摘要或 earnings quality review 的用户。
- 想把不同 LLM 用途拆成清晰 profile，并允许为每个 profile 配置 reasoning / JSON 输出 / token budget 的用户。
- 使用桌面 Settings 管理本地配置，希望在 Web UI 中直接调整 LLM 路由的用户。

## 升级后会有什么变化

- 默认示例配置新增 `llm.providers.openrouter` 和一组 `llm.profiles.*`。
- event-engine 优先使用新 profile 字段；如果没有配置 profile，会继续回落到旧 `llm.openrouter` / `llm.auxiliary` 字段。
- Settings 页面会显示 LLM profile 路由和参数配置块。
- 已运行的本地进程需要重启后才会读取新二进制和新配置。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`
- 源码用户：拉取最新代码后运行 `cargo run -p hone-cli -- start --build web`

## 已知注意事项

- runner 自身配置（`codex_acp`、`opencode`、`gemini_cli`、`hone_cloud`）仍保持独立，不纳入本次 LLM profile registry。
- Streaming 路径仍沿用原 SDK 请求形态；完整 profile request options 目前覆盖非 streaming 的 runtime LLM 调用路径。
- Settings UI 先覆盖默认 profile 集；手写的自定义 profile 可以被保留，但暂时没有新增 / 删除任意 profile 的按钮。
- 架构 SVG 已随本次发版同步刷新（v0.9.1、LLM profile registry、Settings profile routing、近期里程碑）。

## English

Release date: 2026-05-11

### What Users Should Know

#### 1. LLM configuration now supports provider + profile symbols

Hone can now organize LLM configuration into two layers:

- `llm.providers` owns provider type, Base URL, API key / env, multi-key fallback, timeout, and other connection details.
- `llm.profiles` owns model name, `max_tokens`, `temperature`, `top_p`, `reasoning`, JSON response format, and provider-specific parameters.
- Event-engine news classification, SEC filing summaries, earnings review, digest pass1/pass2, dedupe, and mainline distillation can reference profile symbols directly.
- Existing `llm.openrouter` / `llm.auxiliary` settings remain supported as fallbacks, so existing users do not need to migrate immediately.

#### 2. Settings can configure LLM profile routing

The desktop Settings OpenAI-compatible area now includes `LLM Profile Routing`:

- Select symbols such as `main`, `aux`, `news_classifier`, `filing_summary`, `earnings_quality`, `digest_fast`, `digest_strong`, and `mainline_short` per use case.
- Edit each profile's provider, model, token budget, temperature, top_p, reasoning effort / tokens, and JSON output toggle.
- Saving writes to local `config.yaml` / runtime overlay and restarts the bundled backend when applicable.

#### 3. Proactive-push LLM routing is easier to tune for cost and quality

Model and token-budget choices that were previously spread across event-engine config are now centralized in profiles. You can use cheaper fast models for classification and digest pass1, while reserving stronger models for filing summaries, earnings review, or digest personalization without duplicating the same parameters across multiple sections.

### Who Should Update Soon

- Users relying on event-engine proactive pushes, digest, SEC filing summaries, or earnings quality review.
- Users who want clear LLM profiles per use case, including reasoning / JSON output / token budget controls.
- Desktop Settings users who want to adjust local LLM routing directly from the Web UI.

### What Will Change After Upgrading

- The example config now includes `llm.providers.openrouter` and a default set of `llm.profiles.*`.
- Event-engine prefers new profile fields and falls back to existing `llm.openrouter` / `llm.auxiliary` fields when profiles are unset.
- Settings shows LLM profile routing and parameter controls.
- Already-running local processes need a restart before reading the new binaries and config.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`
- Source users: pull the latest code and run `cargo run -p hone-cli -- start --build web`

### Known Notes

- Runner configuration (`codex_acp`, `opencode`, `gemini_cli`, `hone_cloud`) remains separate from this LLM profile registry.
- Streaming paths still use the existing SDK request shape; full profile request options currently apply to non-streaming runtime LLM calls.
- Settings covers the default profile set first; custom hand-written profiles can be preserved, but there is not yet an add/remove UI for arbitrary profiles.
- The architecture SVG was refreshed for this release (v0.9.1, LLM profile registry, Settings profile routing, and recent milestones).
