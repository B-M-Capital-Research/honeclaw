# v0.8.0

发布日期：2026-05-07

English readers: please scroll down for the English version below.

## 用户应该知道的

### 1. Hone Cloud Runner — 接入托管 API 密钥

新增 `HoneCloudRunner` agent runner，支持通过托管 API 密钥接入 Hone Cloud 服务。配合新增的 Web API（`/public/v1/cloud-runner/*`）以及 Admin Settings 中的 API Key 管理面板，您现在可以在 Hone Cloud 上托管运行器，而无需在本地环境维护 LLM 提供商凭据。

- `config.yaml` 新增 `hone_cloud` 字段；不配置时行为与旧版完全一致。
- Admin Settings 新增 "Cloud Runner API Keys" 卡片，支持生成和撤销密钥。

### 2. TOS 2.0 — 法律实体迁移至 Snowdrift Capital LLC

本 release 将服务运营主体从杭州巴芒科技有限责任公司正式迁移至 **Snowdrift Capital LLC**（美国怀俄明州注册的有限责任公司）。

- 适用法律变更为美国怀俄明州法律。
- 隐私政策新增 CCPA（加州居民）与 GDPR / UK GDPR（欧洲/英国用户）权利条款。
- 数据处理地点明确为美国。
- 联系方式新增电子邮件 `bm@hone-claw.com` 及注册地址。
- **所有已登录用户下次登录时需重新勾选接受新版协议（TOS 版本 1.0 → 2.0）。**

### 3. Scheduler 稳定性专项修复

本 release 集中处理了一批 scheduler 相关 bug：

- 修复 legacy cron prompt 时区/时间错位（旧 `config.yaml` 写法下晨报时间不匹配）
- 修复 heartbeat 跨 job 误判去重导致任务静默跳过
- 限制 heartbeat completion budget 避免无限回执
- 修复 watchlist hit zone scheduler context 未正确传递
- 抑制 scheduler 内部 idle timeout 日志（减少日志噪音）
- 修复 skip signal 在特定 runner variant 下未被正确处理

### 4. Feishu 信道修复

- 修复 Feishu token 过期后未刷新缓存（下次消息报 401）
- 修复 live session update 向 Feishu card 输出原始 ACP delta 文本
- 加固 Feishu interactive card 的 live update 与 target lookup 逻辑
- 修复 clarification reply 在 fallback 路径下被吞掉

### 5. 其他修复

- 修复 Codex ACP session replay leak（旧 session 内容混入新会话）
- 修复内部 workflow prelude 被透传给用户（用户不应看到的 `<context>` 块）
- 修复 raw tool status 泄漏到用户消息流
- 收紧 search topic drift guard，避免多轮后查询主题漂移
- 修复 FMP quote 批量拉取 domain boundary bug
- OpenRouter sec enrichment 请求加 max_tokens 上限（避免 402）
- 升级 OpenSSL 0.10.78 → 0.10.79（安全依赖更新）
- 公共 Web session 持久化修复（多 session 切换后 auth 状态丢失）

## 谁应该尽快更新

- **所有用户**：TOS 2.0 变更生效，下次登录须重新接受协议。
- **使用 Feishu channel 的用户**：本 release 修复了多个影响 Feishu 消息正确性的问题，建议尽快升级。
- **依赖 scheduler 定期任务的用户**：多个 scheduler bug 已修复，旧版本可能存在任务静默跳过或时间错位。
- **希望接入 Hone Cloud 托管运行器的用户**：本 release 开放 API Key 管理入口。

## 升级后会有什么变化

- 已登录用户下次打开 Web 控制台时会看到 TOS 2.0 重新确认弹窗，确认后恢复正常。
- `config.yaml` 不需要任何修改；新增的 `hone_cloud` 字段为可选，不填时功能与旧版一致。
- Scheduler 行为更稳定：旧版晨报时间错位的场景会被修正。
- Feishu card 不再出现 ACP 原始 delta 文本。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- `HoneCloudRunner` 需要 `config.yaml` 中配置有效的 `hone_cloud.api_key`，否则调用会返回错误；其他 runner 不受影响。
- TOS 2.0 重新确认弹窗只影响通过公共 Web 登录的用户；通过 channel（Telegram / Feishu / Discord）交互的用户不受影响。
- 架构 SVG 已随本次发版同步刷新（HoneCloud Runner 替换已禁用的 Gemini ACP，版本号 / 里程碑更新）。

## English

Release date: 2026-05-07

### What Users Should Know

#### 1. Hone Cloud Runner — managed API key access

A new `HoneCloudRunner` agent runner is now available, enabling access to Hone Cloud services via managed API keys. Combined with the new Web API (`/public/v1/cloud-runner/*`) and the "Cloud Runner API Keys" management panel in Admin Settings, you can now run the agent through Hone Cloud without maintaining LLM provider credentials locally.

- `config.yaml` gains an optional `hone_cloud` field; omitting it leaves all behavior identical to before.
- Admin Settings gains a "Cloud Runner API Keys" card for generating and revoking keys.

#### 2. TOS 2.0 — legal entity migration to Snowdrift Capital LLC

This release formally migrates the service operator from Hangzhou Bamang Technology Co., Ltd. to **Snowdrift Capital LLC** (a limited liability company organized under the laws of the State of Wyoming, United States).

- Governing law changes to the laws of the State of Wyoming, USA.
- Privacy Policy adds CCPA rights (California residents) and GDPR / UK GDPR rights (EEA / UK users).
- Data processing location is now explicitly the United States.
- Contact information adds `bm@hone-claw.com` email and registered address.
- **All existing logged-in users will be prompted to re-accept the new terms (TOS version 1.0 → 2.0) at next login.**

#### 3. Scheduler stability hardening

A concentrated batch of scheduler bugs was addressed in this release:

- Fixed legacy cron prompt time zone / time mismatch (morning briefing times were wrong for configs using the legacy `config.yaml` format)
- Fixed heartbeat cross-job deduplication false-skip silently dropping tasks
- Capped heartbeat completion budget to prevent unbounded receipts
- Fixed watchlist hit-zone scheduler context not being propagated correctly
- Suppressed internal scheduler idle-timeout log spam
- Fixed skip signal not being handled correctly for certain runner variants

#### 4. Feishu channel fixes

- Fixed Feishu token cache not refreshing after token expiry (next message returned 401)
- Fixed live session update writing raw ACP delta text to Feishu cards
- Hardened Feishu interactive card live-update and target-lookup logic
- Fixed clarification replies being swallowed on the fallback path

#### 5. Other fixes

- Fixed Codex ACP session replay leak (old session content bleeding into new sessions)
- Fixed internal workflow prelude leaking to users (users were seeing `<context>` blocks they should not)
- Fixed raw tool status leaking into the user message stream
- Tightened search topic drift guard to prevent query drift across multiple turns
- Fixed FMP quote batch polling domain boundary bug
- Added max_tokens cap on OpenRouter sec enrichment requests (prevents 402 errors)
- Upgraded OpenSSL 0.10.78 → 0.10.79 (security dependency update)
- Fixed public Web session persistence (auth state was lost when switching between multiple sessions)

### Who Should Update Soon

- **All users**: TOS 2.0 takes effect; re-acceptance is required at next login.
- **Feishu channel users**: multiple correctness bugs fixed — upgrade recommended.
- **Users relying on scheduler jobs**: several bugs caused silent task drops or time mismatches — now fixed.
- **Users interested in Hone Cloud managed runner**: the API key management entry point is now open.

### What Will Change After Upgrading

- Logged-in Web console users will see a TOS 2.0 re-confirmation dialog on next visit; accepting it returns to normal.
- `config.yaml` requires no changes; the new `hone_cloud` field is optional.
- Scheduler behavior is more stable: legacy morning-briefing time mismatches are corrected.
- Feishu cards will no longer display raw ACP delta text.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- `HoneCloudRunner` requires a valid `hone_cloud.api_key` in `config.yaml`; calls will return an error without it. Other runners are unaffected.
- The TOS 2.0 re-confirmation dialog only affects users logged in via the public Web console; channel users (Telegram / Feishu / Discord) are not affected.
- The architecture SVG was refreshed alongside this release (HoneCloud Runner replaces the disabled Gemini ACP; version and milestone cards updated).
