# v0.3.0

发布日期：2026-04-25

English readers: please scroll down for the English version below.

> 本次是一个比较大的 minor 版本：公开 web 端拿到了完整的密码登录与服务条款流程、digest 推送的"多出的 N 条"被重新拆成"噪音"与"上限"两件事并配套了 `/missed` 工具、Price Event Lane 完整改写，再加上 hone-cli onboard 重写、event-engine / hone-channels / memory / hone-core 的多轮模块化重构。下面只列**用户应该感知到的东西**，重构细节集中在最后一节。

## 用户应该知道的

### 1. 公开 web 端：密码登录 + 服务条款 + 修改密码

之前公开 web 端只能凭邀请码激活账号，session cookie 永远续期，没有"密码"这个概念，也没有 ToS。本次补齐了一整套：

- **首次登录强制设置个人密码**：登录 `/me` 或 `/chat` 后，如果账号还没设置过密码，会被 `PasswordSetupGuard` 拦在一个全屏表单后面，必须先勾选服务条款与隐私政策（v1.0，2026-04-25 生效）并设置密码，才能继续使用聊天。
- **密码登录 = 默认入口，邀请码 = 新用户激活**：`/me` 与 `/chat` 共享同一份 `PublicLoginForm`，分两个 tab——「密码登录」走手机号 + 个人密码，「邀请码激活」用一次性激活码。两个 tab 下都有一行灰色引导文案，避免用户猜该选哪个。
- **保持登录 = 30 天，否则 = 1 天**：登录表单默认勾选「保持登录」，session cookie TTL 为 30 天；取消勾选则只签发 1 天的短 session。短 session 不再因为访问而隐式续期（防止短 TTL 被悄悄拉长）。
- **修改密码**：`/me` 在已设置密码的状态下会出现「修改密码」按钮，弹出 modal 收集 current/new/confirm，校验当前密码、要求新密码与当前不同，并强制满足强度规则（8–128 位，含数字与字母）。
- **服务条款 / 隐私政策页面**：`/terms` 和 `/privacy` 是 11 节的 ToS / Privacy v1.0（中英双语），登录与设置密码时强制勾选；首页 footer、登录表单、强制设密码引导上都有可点击入口。运营主体为 **杭州巴芒科技有限责任公司**；条款明确禁止涉黄、赌博、危害国家安全、煽动颠覆国家政权、民族仇恨、暴力恐怖主义、以及通过提示词诱导本服务输出反华 / 政治敏感 / 虚假信息 / 仇恨言论；联系方式统一走 GitHub issue。
- **完整 zh/en i18n**：登录、密码字段、密码强度提示、修改密码、ToS / Privacy 全部双语。顺便修了 `public-nav` 的一个 bug：mount 时刻的 locale 会"冻住"，导致切换语言时导航栏文案不更新。

后端这一侧用 **argon2id + OsRng salt** 哈希密码，登录 endpoint 同时按 IP 和手机号双键限速，失败统一返回不区分用户名 / 密码错误的提示，set-password 成功后会**轮换 session token** 防止 fixation。

### 2. 公开 chat：文件 / 图片附件端到端打通

`/chat` 现在支持在发消息时附带图片或文件（最多 4 个 / 总计 10MB）：

- 输入框新增 attachment 选择器、缩略图预览、拖拽与粘贴支持、上传进度显示。
- 后端新增 `POST /upload`、`GET /image`、`GET /file` 公开路由；上传响应通过 `attachments` 字段一起带进 SSE chat 流。
- 配套 e2e（`public-chat-upload.spec.ts`）覆盖上传 → 助手回复带图。

### 3. Event engine：digest "省略 N 条"被拆成两件事 + `/missed` 工具

旧 digest footer 的「另 138 条已省略（优先展示高优先级/最新）」把两件性质完全不同的事情混在了一起，并且用户没有任何途径回查。这次拆开：

- **噪音过滤静默化**：per-symbol/source/domain 上限、jaccard topic dedup、opinion blog / PR-wire 抑制器属于"按设计去噪"，对用户没有信息价值，footer 不再提及。
- **真实截断有了 footer**：只有当 `max_items_per_batch` 把合法事件砍掉时，才会显示「…… 另 N 条因数量上限未展示，发送 `/missed` 查看完整清单」。
- **新 `/missed` slash command + `missed_events` tool**：用户可以查最近 24h（最长 7d）内被丢弃的事件，按状态分组（omitted / capped / cooled_down / price_band_cool / price_close_dedup / filtered …），构造函数绑定 actor，**只能查自己**的漏推。
- **三处 digest 去重 bug 修复**（拿 2026-04-24 的 3406 条事件回放出来的）：
  - 同一支股票的 intraday 价格档位与日终收盘不再重复占两个槽位（AMD 之前会同时显示「跨过 +12% 档」和「+13.91%」）。
  - 宏观事件（如 `[CA] Retail Sales MoM` / `[CA] Retail Sales MoM (Mar)` / `[CA] Retail Sales Ex Autos MoM (Feb)`）现在也参与 jaccard topic dedup，不再各占一行。
  - 同一日合成 earnings countdown 不再因为 08:30 / 17:00 两次 flush 而被重新列出来。

### 4. Event engine：Price Event Lane 完整改写

intraday band crossing 与 close-price 走全新的 lane 进入 router → digest 流水线，去重、路由、digest flush 全部重写过；新行为可以通过 `hone-core::config::event_engine` 暴露的开关在 `config.example.yaml` 里调整。

### 5. hone-cli `onboard`：可视化重写 + 安全/可达性补齐

之前 `hone-cli onboard` 是一面纯白色 `println!` 墙，每段看起来都一样；并且有几个**装完就没法发消息**的硬伤。这次：

- **TUI 视觉结构**：banner / step header（`━━ Step 3/5: Admins ━━━━…`）/ subsection（`▸ Feishu prerequisites`）/ bullet / ✓ ok / ⚠ warn / ✗ fail / dim hint，每一步都看得出现在站在哪。
- **P0：补全两个把新装挡死的硬伤**
  - `multi-agent` runner kind 现在出现在 picker 里（之前 `config.example.yaml` 默认就是 multi-agent，但 onboard 根本不让你选）。
  - **OpenRouter API key 现在第一步就问**。之前只问 FMP 和 Tavily，每个非 opencode_acp 用户走完 wizard 后第一条消息就报「openrouter.api_key 为空」。
- **P1：安全收口**
  - 每开一个 channel，都强制问一个 admin（手机号 / Telegram / Discord 数字 ID / Feishu email / mobile / open_id），否则 `/register-admin`、`/report`、restart-tool 全都打不开。
  - channel enable 时打印 `⚠ allow_*` 默认开放警告，并指向 `hone-cli configure --section channels`。
- **P2：iMessage 在非 macOS 自动跳过**，不会再把 Linux 用户拉进一个永远跑不通的 channel。
- **细节打磨**：欢迎页加 3–5 分钟预估 + Ctrl+C 安全提示；runner 选错时的「Continue anyway?」措辞改写得更明确；`opencode_acp` 会先问「已 `/connect` 了吗？」；apply 阶段汇报写入了多少个 config 路径。
- **修了一个 `--help` 回归**：phase 1–4 重构在每个子命令的 Args 上写了 rustdoc，clap derive 把它当成了 subcommand `about` 文案，导致 `hone-cli onboard --help` 开始显示「保留结构以便将来扩展…」这种内部说明。改成在 `Commands` enum variant 上放面向用户的一句话，help 输出回到稳定状态。

### 6. 安全 / 依赖

- 新增 argon2 + password-hash 依赖；密码使用 argon2id + OsRng salt。
- `openssl` 0.10.75 → 0.10.78（dependabot，间接依赖安全升级）。

### 7. 内部架构（用户看不见，但值得说一下）

本次累计完成了多个早就该做的拆分；公开 API 路径全部保留，外部 binary 不需要改任何代码：

- `crates/hone-event-engine`：A/B/C 三阶段拆分。`lib.rs` 2144 行 → `engine` / `pipeline` / `spawner` / `tests`；`digest.rs` 2024 行 → `digest/{buffer,time_window,curation,render,scheduler,tests}`；`router.rs` 3047 行 → `router/{sink,stats,config,classify,policy,dispatch,tests}`。
- `crates/hone-channels`：`agent_session.rs` 3728 行、`core.rs` 1635 行、`runners/acp_common.rs` 1767 行、`core_traits.rs` 462 行全部按 concern 拆成目录；`HoneBotCore` 的"上帝对象"职责被拆成 6 个 trait（`AuditRecorder` / `AdminIntercept` / `PathResolver` / `RunnerFactory` / `ToolRegistryFactory` / `LlmProviderBundle`）；`AgentSession::run` 的 quota 释放收敛到 `QuotaReservationGuard` RAII；各 channel 的 `hard_max` 常量收敛到 `PlatformMessageSplitter` trait。
- `bins/hone-cli`：`main.rs` 3218 行 → 586 行（cleanup / discord_token / reports / mutations / yaml_io / prompts / onboard / configure 等模块）。
- `memory`：`cron_job.rs` 1624 行 → `cron_job/{types,schedule,storage,history,mod}`；`SessionStorage` 引入 `SessionIndex` trait，SQLite mirror 不再硬耦合。
- `crates/hone-core`：`config.rs` 2117 行 → `config/{yaml,materialize,mutation,mod,tests}`；6 处 `truncate_chars` 实现合并到 `hone_core::text`。

- 第一次安装 honeclaw 的人：onboard 修复了多处会让首次发消息直接失败的问题，**强烈建议**新装直接用 v0.3.0。
- 所有依赖 openssl 的部署：含一个间接安全升级。

## 升级后会有什么变化

- **公开 web 端老用户**：第一次访问 `/me` 或 `/chat` 时会被强制走一次「同意 ToS + 设置密码」；之后既可以继续用邀请码登录，也可以用手机号 + 密码登录。
- **session cookie**：勾「保持登录」=30 天；不勾 = 1 天，且短 session 不再因访问续期。
- **digest footer**：「另 N 条已省略」措辞和触发条件都变了。看到 footer 时一定意味着真的有事件被 `max_items_per_batch` 截掉，并且可以通过 `/missed`（或自然语言「digest 漏了什么」）查回。
- **`/missed`** 是新的 user-invocable skill，需要 `missed_events` tool；该 tool 已在 `HoneBotCore::create_tool_registry` 里默认注册到 `data_dir` 下的 `events.sqlite3`。
- **公开 chat** 现在可以拖入 / 粘贴附件；上传上限 4 个文件 / 10MB。
- **`hone-cli onboard`** 流程改了：会多问一次 OpenRouter API key、每个 channel 多问一个 admin、iMessage 在 Linux 上直接跳过。已有用户跑 `onboard` 时这些 prompt 会按"已配置"的判断结果显示 yes/no 默认值。
- **公开 API 路径稳定**：所有 `hone_event_engine::*`、`hone_channels::*`、`hone_core::config::*`、`hone_memory::cron_job::*` 的 re-export 都保留，外部代码无需改动。

## English

Release date: 2026-04-25

> This is a sizable minor release: the public web surface gained a full password-login + Terms-of-Service flow, the digest "N items omitted" footer was split into "noise" vs "cap overflow" with a new `/missed` tool to recover what was actually dropped, the Price Event Lane was rewritten end-to-end, and `hone-cli onboard` was rebuilt — alongside multi-phase refactors across `event-engine`, `hone-channels`, `memory`, and `hone-core`. The list below only covers **user-visible changes**; refactor details live in the last section.

### What Users Should Know

#### 1. Public Web — Password Login, ToS, Change-Password

Previously, the public web surface only had invite-code activation, sliding session cookies, no real "password" concept, and no Terms of Service. v0.3.0 ships the whole story:

- **Forced first-login set-password**: when you visit `/me` or `/chat` and your account hasn't set a password yet, `PasswordSetupGuard` blocks the page until you accept the Terms / Privacy (v1.0, effective 2026-04-25) and set a personal password.
- **Password login is the default; invite code is for new-user activation**: `/me` and `/chat` share the same `PublicLoginForm` with two tabs. The "Password Login" tab takes phone + personal password; the "Invite Activation" tab takes a one-time code. Each tab has a one-line gray hint so users don't have to guess.
- **"Stay signed in" = 30 days; otherwise = 1 day**: the login form defaults to "Stay signed in", which issues a 30-day session cookie. Unticking it issues a 1-day short session. Short sessions no longer extend on access (so a short TTL stays short).
- **Change password**: when the user already has a password, `/me` shows a "Change Password" action that opens a modal collecting current / new / confirm. The backend verifies the current password, requires the new one to differ, and enforces strength rules (8–128 chars, including a digit and a letter).
- **Terms / Privacy pages**: `/terms` and `/privacy` render an 11-section bilingual (zh/en) ToS / Privacy v1.0 and are required by the login + set-password flows; reachable from the public footer, the login form, and the set-password guard. The operating entity is **Hangzhou Bamang Technology Co., Ltd.**; the Terms explicitly prohibit pornography, gambling, threats to national security, incitement to subvert state power, ethnic hatred, terrorism / extremism / violence, and using prompts to induce the service to produce anti-China / politically sensitive / disinformation / hateful content. Contact is consolidated into GitHub issues.
- **Full zh/en i18n**: login, password field, strength hints, change-password, ToS/Privacy are all bilingual. Also fixes a `public-nav` bug where the locale "froze" at mount time and stale labels persisted across language switches.

The backend uses **argon2id + OsRng salt** for password hashes; the login endpoint rate-limits by **both IP and phone**, returns a unified failure message that doesn't distinguish wrong-username vs wrong-password, and **rotates the session token on set-password** to prevent fixation.

#### 2. Public Chat — File / Image Attachments End-to-End

`/chat` now supports attaching images or files alongside a message (max 4 files / 10MB total):

- The composer gained an attachment picker, thumbnail preview, drag-and-drop and paste support, and an upload progress indicator.
- New public routes: `POST /upload`, `GET /image`, `GET /file`. The upload response is fed back into the SSE chat stream via the `attachments` field.
- A new e2e (`public-chat-upload.spec.ts`) covers upload → assistant reply with the returned image.

#### 3. Event Engine — Digest Footer Split + New `/missed`

The old digest footer "另 N 条已省略（优先展示高优先级/最新）" mixed two completely different cuts and gave the user no recourse. This release separates them:

- **Curation noise is now silent**: per-symbol/source/domain caps, jaccard topic dedup, opinion-blog / PR-wire suppressors are by-design noise filtering. They no longer surface in the footer — telling the user "130 PR-wire reposts were collapsed" only creates anxiety.
- **Real truncation gets a footer**: only when `max_items_per_batch` actually drops valid events does the digest emit "…… 另 N 条因数量上限未展示，发送 `/missed` 查看完整清单".
- **New `/missed` slash command + `missed_events` tool**: queries the last 24h (max 7d) of dropped items, grouped by reason (omitted / capped / cooled_down / price_band_cool / price_close_dedup / filtered …). The tool's constructor binds the actor, so users can **only see their own** missed events.
- **Three digest dedup bugs fixed**, surfaced by replaying 2026-04-24's 3406 events through the post-refactor router → digest pipeline:
  - The intraday price band and the daily close for the same symbol no longer occupy two slots (AMD used to show both "跨过 +12% 档" and "+13.91%" telling the same story).
  - Macro events (`[CA] Retail Sales MoM` / `(Mar)` / `Ex Autos MoM (Feb)`) now participate in jaccard topic dedup instead of each eating its own line.
  - Same-day synth earnings countdowns no longer re-fire across the 08:30 and 17:00 flush windows.

#### 4. Event Engine — Price Event Lane Rewrite

Intraday band crossings and close-price handling now flow through a fresh lane into the router → digest pipeline. Dedup, routing, and digest flush were rewritten end-to-end; the new behavior is configurable via `hone-core::config::event_engine` knobs surfaced in `config.example.yaml`.

#### 5. `hone-cli onboard` — Visual Rewrite + Reachability / Safety Fixes

`hone-cli onboard` used to be a wall of identical-looking white `println!`s, with several **install-breaking** gaps. v0.3.0 fixes both:

- **TUI structure**: banner / step header (`━━ Step 3/5: Admins ━━━━…`) / subsection (`▸ Feishu prerequisites`) / bullet / ✓ ok / ⚠ warn / ✗ fail / dim hint. Users always know where they are in the flow.
- **P0 — install-breaking**
  - `multi-agent` runner kind now appears in the picker (it was the default in `config.example.yaml` but literally unreachable from onboard).
  - **OpenRouter API keys are prompted first**. Previously only FMP and Tavily were asked, so every non-opencode_acp user finished the wizard and immediately hit "openrouter.api_key 为空" on their first message.
- **P1 — security exposure**
  - Each enabled channel now requires at least one admin id (phone, Telegram / Discord numeric id, Feishu email / mobile / open_id). Without this, `/register-admin`, `/report`, and the restart tool were all unreachable post-install.
  - Each channel-enable path prints a `⚠` warning that `allow_*` defaults to "open to everyone", and points at `hone-cli configure --section channels`.
- **P2**: iMessage is silently skipped on non-macOS hosts instead of dragging Linux users through a channel that can never work.
- **Polish**: welcome page shows a 3–5 minute estimate and a Ctrl+C safety reminder; runner "Continue anyway?" wording explains both branches; `opencode_acp` asks "已 /connect 了吗?"; `apply` reports the number of config paths written.
- **Stable `--help`**: phase 1–4 refactor docstrings on per-module Args structs accidentally surfaced as subcommand `about` text. The user-facing one-liners moved onto the `Commands` enum variants, so `hone-cli onboard --help` is back to a stable, user-friendly description.

#### 6. Security / Dependencies

- Adds `argon2` + `password-hash` dependencies; passwords are stored as argon2id with OsRng-generated salts.
- `openssl` 0.10.75 → 0.10.78 (dependabot, indirect security bump).

#### 7. Internal Architecture (Invisible to Users, Worth Mentioning)

A long-overdue wave of module splits landed this cycle. All public API paths are preserved — external binaries don't need any change:

- `crates/hone-event-engine`: three-phase split. `lib.rs` 2144 lines → `engine` / `pipeline` / `spawner` / `tests`; `digest.rs` 2024 lines → `digest/{buffer,time_window,curation,render,scheduler,tests}`; `router.rs` 3047 lines → `router/{sink,stats,config,classify,policy,dispatch,tests}`.
- `crates/hone-channels`: `agent_session.rs` (3728), `core.rs` (1635), `runners/acp_common.rs` (1767), and `core_traits.rs` (462) split per concern; `HoneBotCore`'s god-object responsibilities split into six traits (`AuditRecorder` / `AdminIntercept` / `PathResolver` / `RunnerFactory` / `ToolRegistryFactory` / `LlmProviderBundle`); `AgentSession::run` quota release converged into a single `QuotaReservationGuard` RAII; each channel's `hard_max` constants converged behind a `PlatformMessageSplitter` trait.
- `bins/hone-cli`: `main.rs` 3218 lines → 586 lines (extracted into cleanup / discord_token / reports / mutations / yaml_io / prompts / onboard / configure modules).
- `memory`: `cron_job.rs` 1624 lines → `cron_job/{types,schedule,storage,history,mod}`; `SessionStorage` no longer hard-depends on the SQLite mirror — it goes through a new `SessionIndex` trait.
- `crates/hone-core`: `config.rs` 2117 lines → `config/{yaml,materialize,mutation,mod,tests}`; six divergent `truncate_chars` implementations consolidated into `hone_core::text`.

### Who Should Update Soon

- Any deployment running the public web surface (`/me` / `/chat`): this is the first release with the password + ToS flow. Existing sessions will be guided through a one-time set-password.
- Users receiving digest pushes via Telegram / Feishu / etc.: footer behavior changed and `/missed` is new.
- Users on the Price Event Lane (intraday band / close): the lane was rewritten — keep an eye on it for a day or two after upgrading.
- Anyone installing honeclaw for the first time: onboard fixed several gaps that previously broke first-message delivery. **Strongly recommended** to install v0.3.0 directly.
- All deployments depending on openssl: includes an indirect security bump.

### What Will Change After Upgrading

- **Existing public-web users**: the first visit to `/me` or `/chat` triggers a one-time "accept ToS + set password" flow. After that, both invite-code login and phone+password login work.
- **Session cookies**: "Stay signed in" = 30 days; otherwise = 1 day, and short sessions no longer extend on access.
- **Digest footer**: wording and trigger condition both changed. Seeing the footer now always means events were truly cut by `max_items_per_batch`, and you can see them via `/missed` (or natural language like "digest 漏了什么").
- **`/missed`** is a new user-invocable skill backed by the `missed_events` tool, registered by `HoneBotCore::create_tool_registry` against `events.sqlite3` under `data_dir`.
- **Public chat** now accepts drag/paste attachments (4 files / 10MB total).
- **`hone-cli onboard`** asks one extra question for OpenRouter and one extra question per channel (admin id), and skips iMessage on non-macOS. Existing users running `onboard` will see those new prompts default-answered based on whether config already looks seeded.
- **Public API paths are stable**: every `hone_event_engine::*`, `hone_channels::*`, `hone_core::config::*`, `hone_memory::cron_job::*` re-export was preserved. External code does not need to change.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version.
- Homebrew users: `brew update && brew upgrade honeclaw`.
