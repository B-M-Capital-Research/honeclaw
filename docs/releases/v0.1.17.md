# v0.1.17

发布日期：2026-04-13

## 用户应该知道的

- Agent 超时配置现在只保留两档：单步超时 `3` 分钟、整轮超时 `20` 分钟
- `codex_acp`、`gemini_acp`、`opencode_acp` 的 prompt 超时语义已统一，不再需要分别调各 runner 的私有 timeout
- `session/load timeout` 不再直接打断整轮请求，而是会自动回退到新建 session，降低长时间未使用或恢复较慢时的失败率

## 谁应该尽快更新

- 通过 `launch.sh --web` 或消息渠道长期运行 honeclaw 的用户
- 最近遇到 `codex acp session/prompt timeout`、`session/load timeout`、长研究轮次被提前打断的用户
- 需要在不同 runner 间复用统一配置、避免单独调参的维护者

## 升级后会有什么变化

- `agent.step_timeout_seconds` 统一控制单步阶段时限，默认 `180` 秒
- `agent.overall_timeout_seconds` 统一控制整轮处理上限，默认 `1200` 秒
- ACP runner 的 `initialize`、`session/load`、`session/new` 等阶段使用单步超时；`session/prompt` 使用“单步 idle + 整轮 overall”的双超时
- `gemini_cli` 的逐行输出等待和整轮等待也改为同一套顶层配置
- Web、Feishu、Discord、iMessage 等入口在运行时会共享同一套整体超时上限

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次只收敛了 ACP、`gemini_cli` 和多 agent answer 路径；其它 runner 的阶段细分仍按各自实现保留
- 单步超时现在覆盖更多启动/恢复阶段；如果本地 CLI 或网络环境异常，仍可能在 `3` 分钟内失败并重试或回退
