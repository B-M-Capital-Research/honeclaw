# v0.1.22

发布日期：2026-04-14

English readers: please scroll down for the English version below.

## 用户应该知道的

- macOS 桌面端的正式运行方式现在明确收敛到 release app bundle 路径，减少了桌面白屏和资源加载异常的风险
- 桌面 release 模式下，前端资源会以适合 Tauri 打包环境的相对路径构建，不再依赖 `file://` 下容易失效的绝对 `/assets/...` 引用
- 团队现在有一份可直接执行的桌面 release app 运行手册，可用于稳定启动、验证数据目录与确认不会被代码改动打断

## 谁应该尽快更新

- 在 macOS 上长期运行 Hone 桌面端的用户
- 需要让桌面持续运行，而不希望被 Rust 或前端代码改动自动干扰的开发者
- 负责桌面打包、重启服务、夜间自动修复或线上切换流程的维护者

## 升级后会有什么变化

- 打包阶段会使用桌面专用的前端构建入口，确保 release app 里的资源路径适合 Tauri shell 加载
- `launch.sh` 与 Tauri 构建流程对 release 桌面模式的说明更清晰，减少“跑起来了但其实不是正式形态”的歧义
- 仓库新增了专门的 release app runbook，明确规定了 `.app` 运行、repo-local `data`/`skills`/runtime config 固定方式，以及无 watch 干扰的验证步骤

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次更新主要集中在桌面 release 运行路径、构建方式和操作手册，不包含新的产品功能入口
- 在 macOS 上，正式长期运行建议优先使用打包后的 `.app`，而不是 `tauri dev` 或裸 `target/release/hone-desktop`
- 启动初期若偶发看到远程 `/api/meta` probe 噪音日志，只要窗口正常渲染且应用稳定保持运行，可先视为启动瞬时噪音

## English

Release date: 2026-04-14

### What Users Should Know

- The macOS desktop production runtime is now explicitly centered on the packaged release app bundle path, reducing white-screen and asset-loading failures
- In release mode, frontend assets are now built with relative paths that work correctly inside the Tauri packaged runtime instead of relying on absolute `/assets/...` references under `file://`
- The repo now includes a concrete desktop release app runbook for stable startup, repo-local data validation, and verifying that code edits do not disturb the running app

### Who Should Update Soon

- Users who keep Hone desktop running for long sessions on macOS
- Developers who want the desktop to stay up without being disrupted by Rust or frontend source edits
- Maintainers responsible for desktop packaging, service restarts, overnight auto-fix flows, or production cutover steps

### What Will Change After Upgrading

- Packaging now uses a desktop-specific frontend build path so release app assets resolve correctly inside the Tauri shell
- `launch.sh` and the Tauri build flow now describe the release desktop lane more clearly, reducing ambiguity around what counts as the proper production shape
- A dedicated release app runbook now documents `.app` startup, repo-local `data`/`skills`/runtime config pinning, and no-watch verification steps

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release focuses on desktop release runtime stability, packaging, and operating guidance rather than new product-facing functionality
- On macOS, the recommended long-running production path is the packaged `.app`, not `tauri dev` and not the raw `target/release/hone-desktop` binary
- If a brief remote `/api/meta` probe warning appears during startup but the window renders and the app remains stable, treat it as transient startup noise unless it persists
