# v0.1.12

发布日期：2026-04-13

English readers: please scroll down for the English version below.

## 用户应该知道的

- 本地 `pre-push` 现在会对“即将推送的 Rust 提交”强制执行 `rustfmt --check`，减少代码已经 push 但 CI 才发现格式问题的情况
- `v0.1.11` 恢复为不可变发布；本轮包含 hook 收口的更新改为单独发布在 `v0.1.12`
- `v0.1.11` 中移除旧 `runtime config`、`data/reports/`、`data/x_drafts/` 与 X 发布 skill 的变化仍然保留在当前版本中

## 谁应该尽快更新

- 经常直接向 `main` push Rust 代码、希望在本地先被格式门禁拦住的开发者
- 依赖 GitHub Release 产物或 Homebrew 安装最新 Hone CLI 的用户

## 升级后会有什么变化

- 如果即将推送的提交里有未通过 `rustfmt` 的 Rust 文件，`git push` 会先被本地 hook 拦截，并提示先 `cargo fmt --all` 后重新提交
- `pre-push` 仍然会继续执行 `gitleaks`，secret 扫描门禁没有变化
- `v0.1.12` 对应的是当前包含 pre-push 格式门禁的正式发布；不会再复用 `v0.1.11` 覆盖内容

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这个 hook 只检查“即将推送的提交”里的 Rust 文件，不会自动改写 commit；如果被拦截，需要手动格式化并重新提交
- 如果本机没有安装 `rustfmt`，hook 会直接提示先执行 `rustup component add rustfmt`

## English

Release date: 2026-04-13

### What Users Should Know

- Local `pre-push` now enforces `rustfmt --check` on the Rust commits that are about to be pushed, so formatting problems are more likely to be caught before CI
- `v0.1.11` is restored as an immutable release; the hook-related update is now published separately as `v0.1.12`
- The changes shipped in `v0.1.11` around removing the old `runtime config`, `data/reports/`, `data/x_drafts`, and the X publishing skill are still included in the current version

### Who Should Update Soon

- Developers who frequently push Rust code directly to `main` and want local formatting checks before the push goes out
- Users who install the latest Hone CLI from GitHub Releases or Homebrew

### What Will Change After Upgrading

- If the commits you are pushing contain Rust files that do not pass `rustfmt`, `git push` will now be blocked locally and will ask you to run `cargo fmt --all` and recommit
- The `pre-push` hook still runs `gitleaks`; the secret-scanning gate is unchanged
- `v0.1.12` is the formal release that includes the pre-push formatting gate; `v0.1.11` will not be reused or overwritten

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- The hook checks only Rust files in the commits about to be pushed; it does not rewrite commits automatically. If it blocks your push, you need to format and recommit manually
- If `rustfmt` is not installed locally, the hook will tell you to run `rustup component add rustfmt`
