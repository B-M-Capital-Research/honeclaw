# v0.1.44

发布日期:2026-04-21

English readers: please scroll down for the English version below.

> 紧跟 v0.1.43 事件引擎首发的两处打磨:把 `event_engine.sources` 配置从"按 API 口"改成"按 poller 口"一对一映射(breaking change),以及让 Hone 在用户首次打招呼时主动做一次能力自我介绍。

## 用户应该知道的

### Breaking change:`event_engine.sources` 字段重构

v0.1.43 的 `sources` 是 `fmp_news / fmp_press_release / fmp_economic_calendar / fmp_sec_filings` 四个字段,和代码里实际的 7 个 poller 不是 1:1 映射 —— `fmp_press_release` 并没有对应 poller(死字段),`fmp_news` 同时压制 News 与 PressRelease,price / earnings_calendar / analyst_grade / earnings_surprise 根本没有开关。

v0.1.44 把 `sources` 替换成 8 个与 `spawn_*_poller` 严格 1:1 的字段:

```yaml
event_engine:
  sources:
    news: true               # /v3/stock_news → NewsCritical
    price: true              # /v3/quote 按 watch pool → PriceAlert / 52W / VolumeSpike
    earnings_calendar: true  # /v3/earning_calendar → EarningsUpcoming
    corp_action: true        # 拆股 / 分红日历 → Split / Dividend
    sec_filings: true        # SEC 8-K per-ticker → SecFiling
    macro_calendar: true     # /v3/economic_calendar → MacroEvent
    analyst_grade: true      # 按 watch pool → AnalystGrade
    earnings_surprise: true  # 按 watch pool → EarningsReleased
```

- 每个字段默认 `true`,未在 YAML 里出现的旧字段直接继承默认(已存在的 `event_engine` 块也不会被意外关停)
- `corp_action_poller` 内部的分红 / 拆股全局日历分支和 SEC 8-K per-ticker 分支分别由 `corp_action` 与 `sec_filings` 独立控制,关掉其一另一侧仍会 tick
- `fmp_*` 旧字段**全部移除**,不做兼容 shim;若旧 `config.yaml` 写了它们,解析阶段会被 serde 静默忽略(用默认),升级时请重命名为新字段
- 对应的 `hone-event-engine` 内部 gate 也从"无条件 spawn"改为"按字段 skip",关掉一个源真正省 FMP 配额与数据库写入,而不仅仅是 router 层丢弃

### 首次问候时 Hone 会主动介绍能力

`soul.md` 新增「能力全景与首次问候响应」一节(第十节):

- 明确列出当前八块核心能力:事件引擎 / 公司画像 / 组合管理 / 定时任务 / 多渠道协同 / 数据与检索 / 发稿与分享 / 管理员能力
- 指令 Hone 在用户首次打招呼、问"你能做什么"、或明显不熟悉能力时,先扼要列出八块能力,再引导:"具体想深入哪一块都可以直接问我 —— 例如事件引擎阈值怎么调、怎么建公司画像、怎么设定时任务、某类推送想关掉等,都能直接告诉我"
- 同时提醒:绝大多数可调参数(推送强度、偏好 kinds、定时节奏、渠道选择、画像内容)用户可通过自然语言调整,不需要改配置文件
- 仅当用户明显只是轻量寒暄("在吗""嗨")且本会话已非首轮,才回到"简短回复即可"的默认

这个改动解决了一个真实体感问题:此前新用户第一次接上 Hone,不知道除了问股票还能做什么;现在 Hone 自己会引导用户发现事件引擎、定时任务、公司画像这些能力面,并告知这些都可以自然语言调整。

## 开发者 / 运维视角

- `crates/hone-core/src/config/event_engine.rs` 的 `Sources` 字段已完全替换,8 字段全 `#[serde(default = "default_true")]`
- `crates/hone-event-engine/src/lib.rs` 的 `EventEngine::start` 在每个 `spawn_*_poller` 前加 `if sources.{field}` gate,关闭时打 info log 便于排查
- `spawn_corp_action_poller` 签名扩展为 `(corp_action_enabled: bool, sec_filings_enabled: bool)`,两个 flag 控制内部两条分支
- `config.example.yaml` 同步为新 schema 并附注释;用户自建的 `config.yaml` 若此前没有 `event_engine.sources` 块,升级无感

## 升级指南

1. 拉取最新 `main`,跑 `cargo check --workspace --exclude hone-desktop` 确认编译通过
2. 打开你的 `config.yaml`:
   - 若**没有** `event_engine.sources` 段:**无需改动**,默认全开
   - 若**写了** `fmp_news / fmp_press_release / fmp_economic_calendar / fmp_sec_filings`:按上面 8 字段重命名;`fmp_press_release` 没有对应新字段(原本就是死字段),直接删掉
3. 重启 Hone(或 `launch.sh --web`);关掉某个 source 时在日志里能看到形如 `news poller disabled by config.sources.news=false` 的提示
4. 新接入的终端用户第一次在 Telegram / iMessage / Feishu / Discord 上跟 Hone 打招呼时,会自动收到能力介绍

# v0.1.44 (English)

Release date: 2026-04-21

> Two polish items riding on top of the v0.1.43 event-engine debut: a breaking-change refactor of `event_engine.sources` so it maps 1:1 to the actual pollers, plus an on-greeting capability tour in `soul.md`.

## What users should know

### Breaking change: `event_engine.sources` schema refactor

v0.1.43's `sources` block had four fields — `fmp_news / fmp_press_release / fmp_economic_calendar / fmp_sec_filings` — which did **not** line up with the seven pollers actually wired in:

- `fmp_press_release` was dead weight (no poller produces `PressRelease` today)
- `fmp_news` gated both News and PressRelease together (can't split)
- `price / earnings_calendar / analyst_grade / earnings_surprise` had **no toggle at all**

v0.1.44 replaces the block with eight fields that map 1:1 to each `spawn_*_poller`:

```yaml
event_engine:
  sources:
    news: true
    price: true
    earnings_calendar: true
    corp_action: true
    sec_filings: true
    macro_calendar: true
    analyst_grade: true
    earnings_surprise: true
```

- All fields default to `true`; users without a `sources` block are unaffected
- `corp_action_poller`'s two internal branches (dividend/split global calendar vs. SEC 8-K per-ticker) are now independently gated by `corp_action` and `sec_filings`
- Old `fmp_*` fields are **fully removed**, no back-compat shim; if your YAML still contains them, serde silently falls back to defaults — rename to the new fields during upgrade
- The engine-side spawn gate moved from "unconditional spawn" to "skip per flag", so turning a source off actually saves FMP quota and DB writes, not just router-level drops

### Hone now self-introduces on greeting

`soul.md` gained a new Section X, "Capability overview and first-greeting response":

- Explicitly lists the eight capability surfaces: event engine / company profiles / portfolio / scheduled tasks / multi-channel / data & retrieval / publishing / admin
- Tells Hone: on a user's first greeting, a "what can you do?" style probe, or any signal the user doesn't know the feature surface, briefly summarize all eight capabilities and then invite: "tell me which one you want to dig into — event-engine thresholds, building a company profile, scheduling tasks, silencing a push category, etc."
- Also reminds Hone that most tunable parameters (push severity, kind preferences, schedule cadence, channel routing, profile content) are adjustable via natural language — no config-file editing required
- Only when the user is clearly doing light small talk ("hi", "are you there") **and** it's not the first turn does Hone fall back to the "keep it short" default

This addresses a real UX gap: previously, a new user meeting Hone for the first time had no discovery path for the event engine, cron tasks, or profiles. Now Hone surfaces those capabilities itself and tells users they can tune them in plain language.

## Developer / ops notes

- `crates/hone-core/src/config/event_engine.rs` — `Sources` fully replaced; eight fields all `#[serde(default = "default_true")]`
- `crates/hone-event-engine/src/lib.rs` — `EventEngine::start` gates every `spawn_*_poller` with `if sources.{field}`; disabled pollers emit an info log
- `spawn_corp_action_poller` signature widened to `(corp_action_enabled, sec_filings_enabled)` so the dividend/split and SEC branches can be toggled independently
- `config.example.yaml` updated with the new schema and inline comments; existing `config.yaml` files without an `event_engine.sources` block upgrade silently

## Upgrade guide

1. Pull latest `main`, run `cargo check --workspace --exclude hone-desktop` to confirm the build
2. Inspect your `config.yaml`:
   - **No** `event_engine.sources` block → nothing to change, defaults stay on
   - Has `fmp_news / fmp_press_release / fmp_economic_calendar / fmp_sec_filings` → rename per the mapping above; drop `fmp_press_release` (no counterpart, was already dead)
3. Restart Hone (or `launch.sh --web`); when a source is disabled you'll see a log like `news poller disabled by config.sources.news=false`
4. New users greeting Hone on Telegram / iMessage / Feishu / Discord will now automatically receive the capability tour
