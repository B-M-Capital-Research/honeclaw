# v0.14.0

发布日期：2026-07-12

English readers: please scroll down for the English version below.

## 用户应该知道的

- HONE 对话现已使用真实模型流式输出：回复会在原有思考卡片中逐步出现，工具调用前的临时文字可原位撤回，内部推理不会暴露；完成后数据库仍只保存一条最终消息。
- 公开聊天启动和恢复改为更接近即时通信应用的体验：首次加载最近 20 条消息，默认定位到最新一条，向上滚动再加载更早历史；刷新期间使用同一恢复界面，图片加载不再动态撑开聊天区域。
- 普通用户继续使用 actor 隔离的安全工具 runner，不能通过 ACP/CLI 读取服务器仓库、配置、数据库或其他用户数据；明确的管理员仍可使用 Codex/OpenCode ACP。
- 财经日历桌面图和移动图现在作为后端消息元数据持久化，服务端按设备直接返回唯一图片。历史消息不再在浏览器重新请求数据、Canvas 重绘、生成 Blob 或动态换图，重复打开还有私有长期缓存。
- 移动端交互继续收口：分享图中的用户问题正确居中，意外双指页面缩放被限制，财经日历保留独立、受控的缩放/拖动/保存能力。
- macOS 与 iOS 用户端继续使用统一 HONE 品牌和远程薄壳边界，本版本重新发布 Universal DMG、iOS Simulator App 与完整 Xcode 工程，默认进入 `https://hone-claw.com/chat`。

## 谁应该尽快更新

- 使用 HONE macOS App 或 iOS App 的用户。
- 经常恢复长对话、使用工具查询、查看财经日历或在移动端分享对话的用户。
- 维护公开 Web、ACP runner、PG/OSS 会话或多用户部署的自托管运营者。

## 升级后会有什么变化

- 已登录账号、历史会话、持仓、推送已读状态和财经日历图片不会被清空或迁移。
- 普通用户的回复由安全 function-calling runner 原生流式返回；管理员 ACP 继续使用 ACP 自身的消息 chunk，二者共享同一用户端流式卡片协议。
- 旧财经日历消息仍可读取：有双图 marker 时服务端按设备选择，只有桌面图时安全回退到该图，不再由前端重建。
- Apple 客户端仍不打包 Codex ACP、`hone-mcp`、Feishu 监听、服务端数据库或本地用户目录，Web 能力更新由 HONE 站点统一提供。

## 安装与更新

- macOS 用户端：从 GitHub Release 下载 `HONE-macOS-v0.14.0-universal.dmg`。
- iOS 开发/验证：下载 `HONE-iOS-Simulator-v0.14.0.zip` 或 `HONE-iOS-Xcode-v0.14.0.zip`；真机分发需在 Xcode 中配置签名后 Archive。
- `curl | bash` 用户：重新运行安装脚本即可切到新版本。
- Homebrew 用户：`brew update && brew upgrade honeclaw`。

## 已知注意事项

- 仓库仍未配置 Apple Developer 签名与 notarization 凭证。macOS DMG 为 ad-hoc 签名；iOS Simulator zip 不能安装到真机，也不包含 IPA/TestFlight 成品。
- Apple 用户端依赖 `hone-claw.com` 网络可用性；离线界面可重试，但不支持离线对话。
- 远端 Rust CI 仍可能命中既有的 Linux `hone-mcp` 测试二进制发现/进程组时序问题；正式 release 的 Apple 客户端构建与本地定向测试独立验证。
- 本版本触达 runner、LLM provider、公开聊天与 Apple 客户端发布面，已同步刷新 `resources/architecture.svg` / `resources/architecture.html`。

## English

Release date: 2026-07-12

### What Users Should Know

- HONE chat now streams from the actual model response. Text appears progressively inside the existing thinking card, temporary tool preambles can reset in place, internal reasoning stays hidden, and storage still records one final assistant message.
- Public chat startup and recovery now behave more like an IM client: the newest 20 messages load first, the viewport lands on the latest message, and older history is fetched as the user scrolls upward. Recovery uses one consistent screen, while reserved media layout prevents images from expanding the conversation after restore.
- Ordinary users remain on an actor-isolated safe tool runner and cannot use ACP/CLI access to inspect server repositories, configuration, databases, or another user's data. Explicit administrators can still use Codex/OpenCode ACP.
- Desktop and mobile finance-calendar images are now persisted as backend message metadata, and the server returns exactly one device-appropriate image. History no longer refetches calendar data, repaints Canvas, creates replacement blobs, or swaps image sources in the browser; repeat views also use private long-lived caching.
- Mobile interaction received another reliability pass: user queries are centered in shared images, accidental page-level pinch zoom is constrained, and the finance calendar retains its own bounded zoom, pan, save, and share controls.
- The macOS and iOS clients retain the unified HONE brand and remote-only boundary. This release republishes a Universal DMG, iOS Simulator app, and complete Xcode project, all opening `https://hone-claw.com/chat` by default.

### Who Should Update Soon

- Users of the HONE macOS or iOS apps.
- Users who restore long conversations, run tool-backed queries, view finance calendars, or share chats on mobile.
- Self-hosted operators maintaining the public Web surface, ACP runners, PG/OSS sessions, or multi-user deployments.

### What Will Change After Upgrading

- Existing accounts, conversation history, portfolios, push read state, and calendar images are retained without a destructive migration.
- Ordinary-user replies stream natively from the safe function-calling runner, while administrator ACP sessions keep native ACP chunks; both feed the same in-thread client protocol.
- Legacy finance calendars remain readable. The server selects a mobile marker when one exists and safely falls back to the desktop image when it does not, without client-side rebuilding.
- Apple clients still bundle no Codex ACP, `hone-mcp`, Feishu listener, server database, or local user-data directory. HONE Web remains the shared feature source of truth.

### Install and Update

- macOS user app: download `HONE-macOS-v0.14.0-universal.dmg` from the GitHub Release.
- iOS development/verification: download `HONE-iOS-Simulator-v0.14.0.zip` or `HONE-iOS-Xcode-v0.14.0.zip`; configure signing in Xcode before a device archive.
- `curl | bash` users: rerun the install script to move to this version.
- Homebrew users: `brew update && brew upgrade honeclaw`.

### Known Notes

- Apple Developer signing and notarization credentials are still not configured. The macOS DMG is ad-hoc signed; the iOS Simulator zip is not device-installable and is not an IPA or TestFlight build.
- Apple clients require `hone-claw.com` network access. Their offline surface can retry loading but does not provide offline chat.
- Remote Rust CI may still encounter the pre-existing Linux `hone-mcp` test-binary discovery/process-group timing issue; Apple release builds and targeted local tests are verified independently.
- This release changes runner, LLM-provider, public-chat, and Apple release surfaces, so `resources/architecture.svg` / `resources/architecture.html` were refreshed.
