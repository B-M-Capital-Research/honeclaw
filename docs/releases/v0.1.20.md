# v0.1.20

发布日期：2026-04-14

English readers: please scroll down for the English version below.

## 用户应该知道的

- Hone 现在会更严格地清理内部工作稿、伪工具调用和思考残留，避免把 `<think>`、`<tool_call>` 一类内部内容直接发给用户
- 当会话上下文超限时，系统会优先自动压缩并重试，而不是直接把底层 provider 报错原样抛给用户
- 桌面端升级时，对 legacy runtime 配置的迁移补得更完整，OpenRouter 相关设置更不容易在升级后静默丢失
- 系统提示词新增了更直接的表达约束，减少“不是……而是……”这类空转对比句式

## 谁应该尽快更新

- 使用多渠道聊天入口，尤其在长会话里依赖稳定输出质量的用户
- 使用桌面端并从旧版本升级、依赖 OpenRouter 配置继续可用的用户
- 维护会话恢复、运行时输出净化、桌面配置迁移或系统提示词规则的开发者

## 升级后会有什么变化

- multi-agent 和渠道输出链路会更严格地区分“内部执行内容”和“用户可见回复”；即使历史会话里混入过内部残渣，也会在恢复和 compact 过程中继续清理，减少后续污染
- 命中 context overflow 时，`AgentSession` 会先强制 compact 当前 session，再自动重试一次；如果仍失败，用户看到的是稳定、友好的提示，而不是 provider 的原始错误细节
- 桌面端在把 legacy runtime 配置迁到 canonical config 时，会更完整保留 OpenRouter 相关用户设置，降低升级后默认对话链路因密钥配置缺失而直接失效的风险
- 回复风格会更直接，少用空转式对比句，判断和原因会更直说

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次发布主要是稳定性、错误卫生和升级迁移修复，不包含新的桌面功能入口或新的模型能力
- context overflow 的自动恢复当前只会执行一次强制 compact；如果后续还需要更细粒度的恢复策略，会继续补充

## English

Release date: 2026-04-14

### What Users Should Know

- Hone now strips internal working notes, pseudo tool calls, and reasoning residue more aggressively so `<think>` or `<tool_call>` style content is less likely to leak into user-visible replies
- When a session hits the context limit, the system now prefers an automatic compact-and-retry path instead of surfacing raw provider errors directly to the user
- Desktop upgrades now preserve legacy runtime configuration more completely, reducing the chance that OpenRouter-related settings silently disappear after migration
- The system prompt now enforces more direct wording and avoids empty contrast phrasing such as "not this, but that"

### Who Should Update Soon

- Users relying on stable output quality across multi-channel chat entrypoints, especially in long-running sessions
- Desktop users upgrading from older versions and expecting existing OpenRouter configuration to keep working
- Developers maintaining session recovery, runtime output sanitization, desktop config migration, or system prompt policy rules

### What Will Change After Upgrading

- The multi-agent and channel output pipeline now separates internal execution text from user-visible answers more aggressively; even if older sessions were polluted, restore and compaction paths now keep cleaning that residue
- On context overflow, `AgentSession` now force-compacts the current session and retries once automatically; if that still fails, the user sees a stable, friendly message instead of raw provider error details
- Desktop migration from legacy runtime config to canonical config now keeps OpenRouter-related user settings more reliably, lowering the risk that the default chat path breaks after an upgrade because keys appear missing
- Reply wording is now more direct, with less empty contrast phrasing and more explicit statements of judgment and reason

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release is mainly about stability, error hygiene, and upgrade migration fixes; it does not introduce a new desktop feature surface or new model capabilities
- The automatic context-overflow recovery currently does one forced compaction pass; more granular fallback strategies may be added later if needed
