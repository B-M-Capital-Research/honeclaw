# v0.14.1

发布日期：2026-07-12

English readers: please scroll down for the English version below.

## 用户应该知道的

- HONE macOS 用户端现在显式使用稳定的 WebKit 持久数据仓。macOS 14 及以上版本会在 App 重启和正常版本升级后继续使用同一份登录 Cookie 与 Web 存储；macOS 12/13 继续安全回退到系统默认持久仓。
- PC 用户端的财经日历弹窗已移到页面顶层，不再被聊天输入区、顶部菜单或窗口边界遮挡。在接近 App 最小窗口高度时，日历内容会在弹窗内部滚动，月份选择、状态和发送按钮都可完整访问。
- 中文输入法候选确认的回车不再误发送消息；普通 Enter 发送和 Shift+Enter 换行保持不变。
- 用户端移动导航、对话气泡、快捷入口与浅色菜单进一步统一为简洁的 HONE 视觉语言；PC 顶栏、侧栏退出、持仓分析和财经日历也使用一致字体。
- GitHub Star 展示会读取最新仓库数据，并使用带有效期的缓存，不再长期显示旧数量。

## 谁应该尽快更新

- 使用 HONE macOS App、遇到重启后需要重新登录的用户。
- 在 PC 或 macOS App 中使用财经日历、中文输入法或长对话的用户。
- 需要重新分发当前 HONE Universal macOS DMG 的维护者。

## 升级后会有什么变化

- macOS 14+ 从本版本开始固定使用命名 WebKit 数据仓。首次从旧版切换时可能需要再登录一次；此后正常重启和升级会继续复用该登录仓。
- 财经日历弹窗始终限制在当前视口内，较矮窗口通过弹窗内部滚动访问底部操作，不再裁切页面。
- 账号、历史会话、持仓、推送已读状态和服务端财经日历数据不做迁移或清空。
- Apple 用户端仍是远程薄壳，不包含 Codex ACP、`hone-mcp`、Feishu 监听、服务端数据库或本地用户数据目录。

## 安装与更新

- macOS 用户端：从 GitHub Release 下载 `HONE-macOS-v0.14.1-universal.dmg`。
- iOS 开发/验证：下载 `HONE-iOS-Simulator-v0.14.1.zip` 或 `HONE-iOS-Xcode-v0.14.1.zip`；真机分发需在 Xcode 中配置签名后 Archive。
- `curl | bash` 用户：重新运行安装脚本即可切到新版本。
- Homebrew 用户：`brew update && brew upgrade honeclaw`。

## 已知注意事项

- 仓库仍未配置 Apple Developer ID 与 notarization 凭证。macOS DMG 为 ad-hoc 签名，另一台 Mac 仍可能显示 Gatekeeper 提示。
- macOS 14+ 用户从未命名的旧 WebKit 默认仓切换到本版本固定仓时，可能需要进行一次登录；本版本保证后续发布保持该仓标识不变。
- Apple 用户端依赖 `hone-claw.com` 网络可用性，不支持离线对话。
- 架构 SVG 不需要更新：本版本没有改变 remote-only Apple 客户端边界、runner、channel、tool、数据库结构或存储布局。
- 远端 release workflow 在 tag 推送后生成多平台与 Apple 产物；tag 创建本身不代表所有资产已经完成上传。

## English

Release date: 2026-07-12

### What Users Should Know

- The HONE macOS client now explicitly uses a stable persistent WebKit data store. macOS 14 and newer reuse the same login cookie and Web storage across app restarts and normal upgrades; macOS 12/13 safely retain WebKit's default persistent-store fallback.
- The desktop finance-calendar dialog now renders at the page root instead of inside the chat composer stack. It stays within the current window and scrolls internally near the app's minimum height, keeping month controls, status, and the send action reachable.
- Enter no longer sends a message while a Chinese IME candidate is being committed. Normal Enter-to-send and Shift+Enter line breaks are unchanged.
- Mobile navigation, chat bubbles, quick actions, and the light menu now share a cleaner HONE visual language. Desktop navigation, sidebar logout, portfolio analysis, and finance calendar controls also use one typography contract.
- GitHub Star counts now refresh from the repository and use an expiring cache instead of retaining stale values indefinitely.

### Who Should Update Soon

- HONE macOS App users who were asked to sign in again after restarting the app.
- Desktop or macOS users who use the finance calendar, a Chinese IME, or long conversations.
- Maintainers redistributing the current Universal HONE macOS DMG.

### What Will Change After Upgrading

- Starting with this version, macOS 14+ uses a fixed named WebKit data store. One sign-in may be required when moving from the previous unnamed default store; subsequent normal restarts and releases reuse the fixed store.
- The finance-calendar dialog remains bounded by the viewport, with internal scrolling used in shorter windows instead of clipping the bottom half.
- Accounts, conversation history, portfolios, push read state, and server-side calendar data are not migrated or cleared.
- Apple clients remain remote-only shells and still bundle no Codex ACP, `hone-mcp`, Feishu listener, server database, or local user-data directory.

### Install and Update

- macOS user app: download `HONE-macOS-v0.14.1-universal.dmg` from the GitHub Release.
- iOS development/verification: download `HONE-iOS-Simulator-v0.14.1.zip` or `HONE-iOS-Xcode-v0.14.1.zip`; configure signing in Xcode before a device archive.
- `curl | bash` users: rerun the install script to move to this version.
- Homebrew users: `brew update && brew upgrade honeclaw`.

### Known Notes

- Apple Developer ID and notarization credentials are not configured. The macOS DMG is ad-hoc signed and may still trigger Gatekeeper on another Mac.
- macOS 14+ users may need to sign in once when moving from the old unnamed default WebKit store to this fixed store. The identifier is now a release invariant for future versions.
- Apple clients require `hone-claw.com` network access and do not provide offline chat.
- Architecture SVG does not need an update: this release does not change the remote-only Apple boundary, runners, channels, tools, database schema, or storage layout.
- The remote release workflow builds the multi-platform and Apple assets after the tag is pushed; tag creation alone does not mean every asset has finished uploading.
