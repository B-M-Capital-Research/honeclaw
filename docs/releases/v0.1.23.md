# v0.1.23

发布日期：2026-04-14

English readers: please scroll down for the English version below.

## 用户应该知道的

- 宏观、行业和市场分析现在更强调“区分主线与噪音”，不会因为几天涨跌、单条新闻或一句 `capex / 需求` 表态就在冲突叙事之间来回切换
- 在下一次 `compact` 之前，Hone 不会再因为自身过早裁剪恢复窗口、把本轮相关 skill 提示写进静态 system prompt，或 `codex_cli` 额外只取最近 20 条历史，而提前制造可避免的 cache miss
- `compact` 之后前缀变化仍然是正常边界，但 `compact` 之前的前缀稳定性现在有了更明确的实现约束和回归测试

## 谁应该尽快更新

- 依赖 Hone 做长会话研究、希望在一次 `compact` 之前尽量稳定复用上下文的用户
- 频繁使用宏观、行业或市场叙事分析，希望减少前后逻辑跳变的用户
- 维护 prompt 规则、session restore / compaction 或 `codex_cli` runner 行为的开发者

## 升级后会有什么变化

- direct session 的 pre-compact 恢复窗口现在至少保持到 20 条消息，group session 则不会早于 `compress_threshold_messages` 滚动
- 当前用户输入相关的 `related skills` 提示改为注入到本轮输入，而不是写入静态 system prefix，从而减少不必要的前缀变化
- `codex_cli` 现在直接消费 Hone 已准备好的上下文窗口，不会再在 runner 内额外收缩到最近 20 条
- 仓库新增了对应的长期约束、归档计划与 handoff，便于后续继续收口 runner budget 或更强的 cache contract

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次更新收敛的是 Hone 自己可控的 pre-compact cache miss 来源，不保证所有底层 provider / ACP runner 都一定具备相同的跨 turn KV cache 复用能力
- `gemini_cli` 仍然受总 prompt 字节预算约束；如果未来固定前缀继续膨胀，可能还需要继续联动收口 compression threshold 和 runner budget

## English

Release date: 2026-04-14

### What Users Should Know

- Macro, sector, and market analysis now more explicitly separates the main thesis from noise instead of flipping between conflicting narratives on a few days of price action or a single `capex / demand` headline
- Before the next `compact`, Hone no longer creates avoidable cache misses by shrinking its own restore window too early, writing turn-specific related-skill hints into the static system prompt, or letting `codex_cli` re-trim history to the latest 20 messages
- A prefix change after `compact` is still treated as a normal boundary, but prefix stability before `compact` now has an explicit implementation contract plus regression coverage

### Who Should Update Soon

- Users who rely on long research sessions and want context reuse to stay as stable as possible before the next `compact`
- Users who frequently ask for macro, sector, or market narrative analysis and want fewer abrupt logic shifts between turns
- Developers maintaining prompt policy, session restore / compaction behavior, or the `codex_cli` runner

### What Will Change After Upgrading

- The pre-compact restore window for direct sessions now stays at least 20 messages, while group sessions no longer roll earlier than `compress_threshold_messages`
- Related-skill hints that depend on the current user turn now move into the current turn input instead of the static system prefix, reducing unnecessary prefix churn
- `codex_cli` now consumes the full context window prepared by Hone instead of re-shrinking it to the latest 20 messages inside the runner
- The repo now includes the matching long-lived invariant, archived plan, and handoff so future runner-budget or stronger cache-contract work has a clear entry point

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release fixes the pre-compact cache-miss sources that Hone itself controls; it does not guarantee identical cross-turn KV-cache reuse behavior across every underlying provider or ACP runner
- `gemini_cli` is still constrained by the total prompt byte budget; if the fixed prefix grows further, the compression threshold and runner budget may need another coordinated pass
