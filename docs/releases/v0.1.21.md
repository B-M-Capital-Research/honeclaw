# v0.1.21

发布日期：2026-04-14

English readers: please scroll down for the English version below.

## 用户应该知道的

- 公司分析在命中已有画像时，会更明确参考用户已经沉淀的风险偏好、既有看法、估值口味和约束条件
- 系统不再顺着极端风险偏好给建议；遇到满仓、满融、梭哈或单一催化重注这类倾向时，会主动把建议收敛到更可执行的仓位和风险框架
- 单股分析被进一步约束为第一性原理导向，优先看商业模式、竞争优势、盈利质量、估值与产业周期，而不是只盯 K 线和短期价格波动

## 谁应该尽快更新

- 在 Hone 里长期跟踪公司，并已经积累了公司画像、个人偏好或估值框架的用户
- 容易在高波动环境下把讨论滑向重仓、满融或梭哈决策的用户
- 维护公司画像、系统提示词或公司分析输出质量的开发者

## 升级后会有什么变化

- 当系统找到对应公司的画像时，回答会更主动对齐用户自己的长期框架，而不是退回成泛化的标准答案
- 如果画像或当前对话显示用户风险偏好过激，系统会直接降温，强调仓位、节奏、触发条件和证伪条件，而不是继续放大情绪
- 公司分析会更少围绕短线价格动作打转，更明确回到价值、竞争格局、盈利能力和周期位置

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这是一次提示词与分析纪律增强，不包含新的画像存储结构或新的前端功能入口
- 第一性原理分析仍然依赖底层事实校验；如果实时数据未校验完成，系统仍只应给分析框架，不应编造数字

## English

Release date: 2026-04-14

### What Users Should Know

- Company analysis now aligns more explicitly with the user's recorded risk preference, existing views, valuation taste, and constraints when a company profile already exists
- The system is less willing to echo extreme risk appetite; when the user leans toward all-in, leveraged, or single-catalyst concentrated bets, the answer now pulls back toward a more executable risk framework
- Single-company analysis is now more explicitly first-principles driven, prioritizing business model, competitive advantage, earnings quality, valuation, and cycle position instead of over-focusing on charts and short-term price moves

### Who Should Update Soon

- Users who do long-term company tracking in Hone and already have stored company profiles, personal preferences, or valuation frameworks
- Users who tend to push discussions toward concentrated, leveraged, or all-in positioning in volatile setups
- Developers maintaining company profile behavior, system prompt policy, or company-analysis output quality

### What Will Change After Upgrading

- When Hone finds an existing company profile, answers will align more actively with the user's long-term framework instead of drifting back to generic analysis
- If the stored profile or current discussion shows overly aggressive risk preference, the system now cools the answer down and emphasizes position sizing, pacing, triggers, and falsification conditions
- Company analysis is less likely to revolve around short-term price action alone and more likely to return to value, competitive structure, earnings power, and cycle context

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This is a prompt and analysis-discipline update; it does not introduce a new profile storage structure or a new frontend surface
- First-principles analysis still depends on factual verification; if real-time data is not verified yet, the system should stay at the framework level instead of inventing numbers
