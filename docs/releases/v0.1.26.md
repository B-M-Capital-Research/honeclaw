# v0.1.26

发布日期：2026-04-15

English readers: please scroll down for the English version below.

## 用户应该知道的

- 会话存储现在正式切到统一的 `user/assistant + content[] + status` 模型；新写入不再依赖旧的扁平字符串 `content`
- `codex_acp`、`opencode_acp`、`codex_cli` 与 `multi-agent` 现在共享同一套可恢复 transcript 语义：assistant 过程态、tool call、tool result、final answer 会作为同一轮 assistant 内容的一部分持久化
- `codex_acp` 现在会对 MCP permission 采用 `approved-for-session`，不再把 Hone MCP 工具统一打成 `user cancelled MCP tool call`
- `hone-cli probe` 现在会直接打印 `delta / thought / segment`，排查 runner 事件流更直接

## 谁应该尽快更新

- 依赖跨 turn 历史恢复、希望不同 runner 可以互换而不丢过程态的用户
- 正在排查 `codex_acp`、`opencode_acp`、`codex_cli` 或 `multi-agent` transcript / tool message 重建问题的开发者
- 需要用本机回归脚本验证 Codex ACP 事件流和 Hone MCP 执行链路的维护者

## 升级后会有什么变化

- top-level 历史恢复会优先消费统一的标准问答模型，而不是某个 runner 私有的原始事件流
- `codex_acp` execute 完成态现在会把 `rawOutput` 补成可持久化的 `tool_result`，恢复后不再只剩半截 tool call
- `multi-agent` 顶层 runner 不再只保留最终答案；search-stage 和 answer-stage transcript 会一起写回共享 session 模型
- Web / API 读取历史时会看到新的标准化结构，和 runner 间的 transcript 互换性更一致
- 新增了手工回归脚本 `tests/regression/manual/test_codex_acp_event_stream.sh`，可直接检查 Codex ACP 事件和 Hone MCP 执行结果

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次发布允许 session 存储发生 breaking 变更；新写入已经使用统一模型，旧数据仍保留兼容反序列化
- 统一历史模型已经接入 `codex_acp`、`opencode_acp`、`codex_cli` 与 `multi-agent`，但不同 ACP runner 的原始事件细节仍由各自 runner 私有字段负责，不会被硬塞进公共 schema
- 本次随上游同步带入了最新 `docs/bugs/README.md` 台账更新，但那两个远端提交本身不是产品代码修复

## English

Release date: 2026-04-15

### What Users Should Know

- Session storage now writes the unified `user/assistant + content[] + status` model instead of relying on the old flat string `content`
- `codex_acp`, `opencode_acp`, `codex_cli`, and `multi-agent` now share the same restorable transcript semantics: assistant progress, tool calls, tool results, and final answers persist as parts of a single assistant turn
- `codex_acp` now answers MCP permission requests with `approved-for-session`, so Hone MCP calls no longer collapse into `user cancelled MCP tool call`
- `hone-cli probe` now prints `delta / thought / segment` events directly, which makes runner event-stream debugging easier

### Who Should Update Soon

- Users who rely on cross-turn history restoration and want different runners to stay interchangeable without losing process-state transcript
- Developers debugging transcript and tool-message reconstruction in `codex_acp`, `opencode_acp`, `codex_cli`, or `multi-agent`
- Maintainers who need a local regression script for the Codex ACP event stream and Hone MCP execution chain

### What Will Change After Upgrading

- Top-level history restoration now prefers the shared normalized conversation model instead of any single runner's raw event stream
- `codex_acp` execute completions now patch `rawOutput` into persistable `tool_result` parts, so restored sessions no longer stop at a partial tool-call record
- The top-level `multi-agent` runner no longer stores only the final answer; search-stage and answer-stage transcript now persist together into the shared session model
- Web / API history consumers now see the normalized structure with more consistent transcript interchangeability across runners
- A new manual regression script, `tests/regression/manual/test_codex_acp_event_stream.sh`, can validate Codex ACP events and Hone MCP execution results directly

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release intentionally allows a breaking session-storage write format; new writes use the unified model while old data remains readable through compatibility deserialization
- The unified history model is now wired into `codex_acp`, `opencode_acp`, `codex_cli`, and `multi-agent`, but raw ACP event details still remain runner-owned instead of being forced into a shared schema
- The latest upstream `docs/bugs/README.md` updates are included after pull, but those remote commits themselves were documentation updates rather than product-code fixes
