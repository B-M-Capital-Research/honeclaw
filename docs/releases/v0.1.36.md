# v0.1.36

发布日期：2026-04-19

English readers: please scroll down for the English version below.

## 用户应该知道的

- Hone 现在把 Web 管理端和面向终端用户的聊天端彻底拆开了：管理端继续保留完整后台能力，新的公网用户端只暴露 `public` API 和单会话聊天页，邀请码用户可以直接进入 `/chat` 开聊，而不需要接触后台控制台。
- 管理端新增了邀请码管理能力。你现在可以生成、复制并跟踪邀请码，系统会为每个邀请码创建对应的 Web 用户，并沿用现有 quota 约束记录剩余次数和最近登录时间。
- Memory 里的公司画像现在支持 bundle zip 导入导出、自动扫描和冲突审阅；历史上没有 frontmatter 的 `profile.md` / `events/*.md` 以及 plain Markdown 画像包也能继续 preview、导入和替换前备份，不会因为旧格式直接失败。
- Web 端交互也做了整理：开始页改成更清晰的 pill 式 channel 选择器，Memory 页面把“目标用户空间”和“当前公司切换”拆到更合理的位置，移动端的阅读和操作也比之前顺手。

## 谁应该尽快更新

- 想把 Hone 的 Web 聊天能力通过反向代理或公网入口开放给终端用户、但又不希望顺手暴露后台管理 API 的团队。
- 需要在不同 actor / 用户空间之间迁移、备份或分享公司画像资料的研究团队。
- 还保留着历史 plain Markdown 公司画像文件，或者过去导入画像包时经常被 `缺少 frontmatter` 卡住的维护者。

## 升级后会有什么变化

- Web 部署会变成“管理端”和“用户端”两套 surface：管理端继续走原有后台入口，用户端走独立的公开路由；两个 surface 之间互相访问对方 API 会返回 `404`，降低把后台一起暴露出去的概率。
- 生成邀请码后，用户可以直接用邀请码登录轻量聊天页，看到按轮次组织的 thinking / tool / final 卡片，不再需要从管理端侧边栏进入完整后台。
- 公司画像导入会先自动扫描包内容并识别冲突；没有冲突时可以直接导入，需要替换时会先给出预览并自动产出备份下载入口。
- 对旧版公司画像数据的兼容性更强了。没有 frontmatter 的本地画像和 bundle 会按最小 metadata 推断继续参与读取、列表、匹配和 transfer，而不是整包中断。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次隔离的是“用户端公网 surface”和“管理端后台 surface”，不是在默认配置下自动把管理端变成安全公网服务。如果你要暴露管理端，仍然必须单独配置 `web.auth_token` 并做好反向代理保护。
- 邀请码管理目前支持生成、列表和复制，还不支持停用、删除、重置或设备级会话管理。
- 公司画像导入仍然是目录级 `skip / replace` 语义，当前不提供逐文件 merge 编辑器；如果你希望更细粒度合并，需要后续单独设计。

## English

Release date: 2026-04-19

### What Users Should Know

- Hone now separates the admin web console from the end-user chat surface. The admin side keeps the full management UI, while the new public-facing web surface only exposes `public` APIs and a single-session `/chat` page, so invited users can start chatting without touching the admin console.
- The admin side now includes invite-code management. You can generate, copy, and track invite codes, and each code creates a corresponding web user that reuses the existing quota model, including remaining uses and last-login visibility.
- Company profiles in Memory now support bundle zip import/export, automatic scanning, and conflict review. Legacy `profile.md` / `events/*.md` files without frontmatter, as well as plain Markdown bundles, can now still be previewed, imported, and backed up before replacement instead of failing immediately on old format assumptions.
- The web UX was also cleaned up: the start page now uses a clearer pill-style channel selector, and the Memory layout now separates “target user space” from “current company” navigation in a way that works better on mobile too.

### Who Should Update Soon

- Teams that want to expose Hone web chat to end users through a reverse proxy or public entrypoint without accidentally exposing the admin API at the same time.
- Research teams that need to migrate, back up, or share company-profile data across different actors or user spaces.
- Maintainers who still keep legacy plain Markdown company-profile files or who were previously blocked by `missing frontmatter` failures during bundle import.

### What Will Change After Upgrading

- Web deployment now becomes two separate surfaces: an admin surface and a user-facing surface. The admin side keeps the original management entrypoints, while the public side uses dedicated routes; probing the other side's APIs now returns `404`, which reduces accidental admin exposure.
- After generating an invite code, users can log into a lightweight chat page directly and see per-turn thinking / tool / final cards without going through the full admin shell.
- Company-profile import now scans the bundle first and detects conflicts automatically. Clean imports can proceed directly, while replacement flows now show a preview and generate a downloadable backup before applying changes.
- Compatibility for older company-profile data is stronger. Local profiles and bundles without frontmatter now keep participating in read/list/match/transfer flows through minimal metadata inference instead of aborting the whole flow.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release isolates the public user-facing surface from the admin surface; it does not automatically turn the admin side into a safe public service. If you expose the admin side, you still need `web.auth_token` and proper reverse-proxy protection.
- Invite-code management currently supports create/list/copy, but not disable, delete, reset, or device-level session management yet.
- Company-profile import still uses directory-level `skip / replace` semantics. There is no per-file merge editor in this release.
