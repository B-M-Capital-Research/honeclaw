# v0.1.39

发布日期：2026-04-20

English readers: please scroll down for the English version below.

## 用户应该知道的

- 修复了用户端公开聊天入口的一处后端编译回归。此前如果按 `./launch.sh --web` 启动，`hone-web-api` 会因为路由层调用参数没有跟上内部接口变更而直接编译失败，Web 服务无法拉起。
- 这次发布把公开聊天路由和当前的 SSE 构建接口重新对齐，恢复了 Web 模式下的正常构建与启动流程。

## 谁应该尽快更新

- 依赖 `./launch.sh --web` 启动 Hone Web 管理端或公网聊天端的用户。
- 正在从源码构建 Hone，且最近遇到 `build_chat_sse` 参数数量不匹配编译错误的维护者。

## 升级后会有什么变化

- 从当前源码拉起 Web 模式时，`hone-web-api` 不会再因为公开聊天路由的过期函数调用而中断编译。
- 用户端公开聊天入口会继续走现有的 SSE 消息流，不涉及新的配置项、端口调整或使用方式变化。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次发布是一次针对 Web 启动链路的修复发布，重点解决源码构建时的编译失败；如果你的环境还存在独立的端口占用、反向代理或鉴权配置问题，仍需按原有部署方式排查。

## English

Release date: 2026-04-20

### What Users Should Know

- This release fixes a backend build regression in the public web chat entrypoint. Before this fix, starting Hone with `./launch.sh --web` could fail while compiling `hone-web-api` because a route still called an internal SSE helper with an outdated argument list.
- The public chat route now matches the current SSE builder signature again, so the web stack can build and start normally from source.

### Who Should Update Soon

- Users who rely on `./launch.sh --web` to start Hone's admin web surface or public chat surface.
- Maintainers who recently hit a `build_chat_sse` argument-count mismatch while compiling from source.

### What Will Change After Upgrading

- Building the web mode from the current source tree no longer stops at the outdated public-chat route call in `hone-web-api`.
- The public chat entrypoint keeps using the existing SSE flow; there are no new config knobs, port changes, or workflow changes in this release.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This is a targeted web-startup fix release. It resolves the source-build failure, but any separate environment issues around occupied ports, reverse proxies, or auth configuration still need to be handled through the usual deployment checks.
