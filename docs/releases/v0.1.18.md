# v0.1.18

发布日期：2026-04-13

English readers: please scroll down for the English version below.

## 用户应该知道的

- multi-agent 的 search 阶段现在可以直接读取当前 actor sandbox 内的本地文件，不再只会做网络搜索
- 当用户问题依赖本地持久化状态时，search 阶段可以先检查 `company_profiles/`、`uploads/`、`runtime/` 等目录，而不是默认回答“没有记忆”或“无法访问文件”
- 新增的本地文件能力是严格只读的，只允许访问当前 actor 自己的 sandbox，相对路径越界、绝对路径和符号链接逃逸都会被拒绝

## 谁应该尽快更新

- 已经在 actor sandbox 里沉淀了 `company_profiles/`、上传文件或其它本地研究材料的用户
- 经常问“之前研究过哪些公司”“本地档案里有哪些内容”“我上传过什么资料”这类依赖本地持久化状态问题的用户
- 维护 multi-agent 搜索链路、MCP 工具暴露或 actor sandbox 权限边界的开发者

## 升级后会有什么变化

- search 阶段新增 `local_list_files`、`local_search_files`、`local_read_file` 三个只读工具，和 `web_search` / `data_fetch` 一样可以被模型自由调用
- 这些工具只会暴露当前 actor sandbox 内的相对路径，并且输出会按结果数、行数和字符数做截断，避免把大文件整段塞进上下文
- search 阶段一旦调用了本地只读工具，就会像调用其它工具一样自然进入 answer 阶段；零工具时仍保留 direct return 逻辑
- README 与 README_ZH 也补充了公司画像长期沉淀能力说明

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次新增的是 actor sandbox 内的只读文件能力，不开放写入，也不读取仓库目录或其它用户的 sandbox
- 当前修复的是“search 阶段读不到本地持久化状态”这个问题，不包含 search direct return 的 thinking 泄漏治理

## English

Release date: 2026-04-13

### What Users Should Know

- The multi-agent search stage can now read local files directly from the current actor sandbox instead of relying only on network search
- When a user question depends on local persisted state, the search stage can now inspect directories such as `company_profiles/`, `uploads/`, and `runtime/` before defaulting to “I have no memory” or “I cannot access files”
- This new local file capability is strictly read-only and limited to the current actor’s sandbox; path traversal, absolute-path access, and symlink escapes are rejected

### Who Should Update Soon

- Users who already have `company_profiles/`, uploaded files, or other local research material stored in their actor sandbox
- Users who often ask questions like “which companies did I research before”, “what is in my local archive”, or “what files did I upload”
- Developers maintaining the multi-agent search path, MCP tool exposure, or actor-sandbox permission boundaries

### What Will Change After Upgrading

- The search stage now has three read-only local tools: `local_list_files`, `local_search_files`, and `local_read_file`, available alongside `web_search` and `data_fetch`
- These tools expose only relative paths inside the current actor sandbox, and their output is truncated by result count, line count, and character count so large files do not flood context
- Once the search stage calls a local read-only tool, it continues into the answer stage just like other tool use; zero-tool turns still keep the direct-return behavior
- README and README_ZH were also updated to mention the long-term company portrait capability

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release adds only read-only file access inside the actor sandbox; it does not allow writes and it does not read the repo directory or other users’ sandboxes
- It fixes the “search stage cannot read local persisted state” problem only; it does not include the search direct-return thinking leak cleanup
