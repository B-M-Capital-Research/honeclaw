# v0.10.0

发布日期：2026-05-11

English readers: please scroll down for the English version below.

## 用户应该知道的

### 1. LLM 凭证统一进入 config.yaml

Hone 的 LLM provider、profile、auxiliary LLM 和 Gemini ACP 用户配置不再从父进程环境变量兜底读取 API key。OpenRouter、辅助 LLM、Gemini ACP 等凭证需要直接写在 `config.yaml` 中：

- 首选 OpenRouter 路径是 `llm.providers.openrouter.api_key` / `llm.providers.openrouter.api_keys`。
- 旧的 `llm.openrouter.api_key` / `llm.openrouter.api_keys` 仍可作为 config-only 迁移 fallback 读取，但 CLI 和 Desktop 保存时会写入新 provider 路径。
- `llm.auxiliary.api_key` 和 `agent.gemini_acp.api_key` 改为配置内字段；`MINIMAX_API_KEY`、`GEMINI_API_KEY`、`OPENROUTER_API_KEY` 不再作为 Hone 运行时 LLM 凭证真相源。

### 2. CLI 和 Settings 会按新路径写入并脱敏展示

`hone-cli onboard`、`hone-cli configure`、`hone-cli config set/get` 和 Desktop Settings 的 OpenRouter 配置会写入 `llm.providers.openrouter.api_keys`。展示层继续遮罩 API key，`status` 只报告是否已配置，不打印明文凭证。

### 3. 主动推送默认示例切到 Grok 快速模型

示例配置和 event-engine 手工基线脚本中的默认 news classifier / global digest 模型从 `amazon/nova-lite-v1` 调整为 `x-ai/grok-4.1-fast`，和当前 OpenRouter/Grok 配置保持一致。

### 4. Scheduler 和 quota 可见性修复也包含在本次发布中

自 `v0.9.1` 以来，watchlist hit-zone scheduler 在任务正文没有显式 ticker 时的恢复路径被修复，quota 拒绝也会在 transcript 中保留可见回复，便于用户和排障者理解为什么请求被拒绝。

## 谁应该尽快更新

- 只依赖 `OPENROUTER_API_KEY`、`MINIMAX_API_KEY` 或 `GEMINI_API_KEY` 环境变量运行 LLM 能力的用户。
- 使用 Desktop Settings 或 `hone-cli configure/onboard` 管理模型配置的用户。
- 依赖 event-engine 主动推送、watchlist hit-zone、digest 或 quota 可见反馈的用户。

## 升级后会有什么变化

- 如果 LLM key 只存在于环境变量中，相关 LLM 路径会报错并提示把 key 迁移到 `config.yaml`。
- `hone-cli config get`、`hone-cli status` 和 Settings 仍会遮罩敏感字段；token / secret 展示策略不变。
- `opencode_acp` 仍会把 config-owned OpenRouter key 注入子进程环境，这是对子进程 CLI 的兼容桥接，不是 Hone 从父进程 env 读 key。
- 已运行的本地进程需要重启后才会读取新二进制和新配置。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`
- 源码用户：拉取最新代码后运行 `cargo run -p hone-cli -- start --build web`

## 已知注意事项

- 本次只收敛 LLM / provider / agent runner 相关凭证，不改变 Feishu、Telegram、Discord、iMessage 等渠道 bot token 的存储策略。
- 旧配置中的 `api_key_env` 字段会被忽略；请把实际 key 写入 `config.yaml`。
- 架构 SVG 已随本次发版同步刷新（v0.10.0、config-only LLM credentials、近期里程碑）。

## English

Release date: 2026-05-11

### What Users Should Know

#### 1. LLM credentials now live in config.yaml

Hone no longer falls back to parent-process environment variables for LLM provider, profile, auxiliary LLM, or Gemini ACP user credentials. OpenRouter, auxiliary LLM, Gemini ACP, and related keys should be stored directly in `config.yaml`:

- The preferred OpenRouter path is `llm.providers.openrouter.api_key` / `llm.providers.openrouter.api_keys`.
- Legacy `llm.openrouter.api_key` / `llm.openrouter.api_keys` remains readable as a config-only migration fallback, but CLI and Desktop saves now write the new provider path.
- `llm.auxiliary.api_key` and `agent.gemini_acp.api_key` are config fields; `MINIMAX_API_KEY`, `GEMINI_API_KEY`, and `OPENROUTER_API_KEY` are no longer Hone runtime LLM credential sources.

#### 2. CLI and Settings write the new path and keep keys masked

`hone-cli onboard`, `hone-cli configure`, `hone-cli config set/get`, and Desktop Settings OpenRouter configuration now write `llm.providers.openrouter.api_keys`. Display surfaces continue to mask API keys, and `status` only reports whether credentials are configured.

#### 3. Proactive-push examples now use a fast Grok model

Example config and event-engine manual baseline defaults moved from `amazon/nova-lite-v1` to `x-ai/grok-4.1-fast` for news classification and global digest examples, matching the current OpenRouter/Grok setup.

#### 4. Scheduler and quota visibility fixes are included

Since `v0.9.1`, watchlist hit-zone scheduler recovery has been fixed for jobs whose prompt body does not explicitly list tickers, and quota rejections now preserve visible transcript replies so users and operators can understand why a request was rejected.

### Who Should Update Soon

- Users who rely only on `OPENROUTER_API_KEY`, `MINIMAX_API_KEY`, or `GEMINI_API_KEY` environment variables for LLM features.
- Users managing model configuration through Desktop Settings or `hone-cli configure/onboard`.
- Users relying on event-engine proactive pushes, watchlist hit-zone jobs, digest, or visible quota feedback.

### What Will Change After Upgrading

- If an LLM key only exists in the environment, the affected LLM path will fail with a migration message asking you to move the key into `config.yaml`.
- `hone-cli config get`, `hone-cli status`, and Settings still mask sensitive fields; token / secret display policy is unchanged.
- `opencode_acp` still injects a config-owned OpenRouter key into the child process environment for CLI compatibility; this is not a parent-env fallback.
- Already-running local processes need a restart before reading the new binaries and config.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`
- Source users: pull the latest code and run `cargo run -p hone-cli -- start --build web`

### Known Notes

- This release only changes LLM / provider / agent runner credential handling. Feishu, Telegram, Discord, iMessage, and other channel bot-token storage is unchanged.
- Old `api_key_env` fields are ignored; move actual keys into `config.yaml`.
- The architecture SVG was refreshed for this release (v0.10.0, config-only LLM credentials, and recent milestones).
