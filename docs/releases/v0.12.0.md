# v0.12.0

发布日期：2026-05-13

English readers: please scroll down for the English version below.

## 用户应该知道的

- 公开 `/chat` 增加了消息分享：每条助手回答右下角出现复制 + 分享按钮（桌面 hover、移动常驻），分享面板里可多选要包含的消息，离屏生成一张手机竖屏比例的品牌长图，提供下载图片、复制图片（写入剪贴板，可直接粘到微信 / 飞书）、复制文字、系统分享（在支持的设备上调用原生分享面板）四种出口，长图底部带二维码指向 `https://hone-claw.com/chat`。
- 公开 `/chat` 做了一轮密度 / 渲染 / 视觉整体打磨：列表 bullet 与序号回来了、`**bold**` 标签提升为块级小标题、连续助手回复合并 HONE 品牌片、行内代码块去掉双层背景压成一张柔和 chip 且长公式自动换行不再产生内嵌横向滚动；同时修复了 marked 代码渲染器把 Promise 字符串化为 `[object Promise]` 直接出现在正文里的 bug。
- 公开 `/chat` 流式回答、移动输入框视口、滚动锚定 / 回到底部按钮在多种机型上更稳定，发送消息不再触发整页跳动。
- 公开站文案与排版整体重写：`/portfolio`、`/me`、`/home` 视频卡片与轮播、`/roadmap` 移动卡片、`/terms` 与 `/privacy` 增加目录 + 回到顶部，移动 header 收紧、`GitHub Stars` 在移动端仍可见、整套公开文案剥离内部术语并完成本地化。
- 公开 SMS 登录前置阿里云验证码守门：触发短信发送前先走阿里云行为验证，避免 `/api/public/auth/sms/send` 被滥用。

## 谁应该尽快更新

- 正在对外开放 `hone-claw.com/chat` 或其它 public web 用户入口的部署者。
- 希望终端用户能把单条 / 整段对话以图片形式分享到微信、钉钉、飞书等 IM 的团队。
- 需要为公开 SMS 登录加阿里云行为验证码（避免短信被刷）的运营者。
- 维护公开站文案、移动布局、`/terms` `/privacy` 的人。

## 升级后会有什么变化

- 每条助手消息底部多了复制 + 分享两个图标；点击分享会弹出底部抽屉（移动）或居中对话框（桌面），默认勾选触发分享的那条消息，可一键全选或只保留这一条。
- 分享导出的图片宽度为 420px、走手机竖屏审美：顶部 Hone 标识 + tagline，中部按真实 markdown 渲染（标题 / 列表 / 加粗 / 表格 / 代码 chip 全部保留），底部品牌 + 二维码。Logo 使用内联 SVG 以保证 html2canvas 能正确栅格化。
- 公开 SMS 登录流程：用户输入手机号 → 通过阿里云行为验证 → 后端再向阿里云短信网关下发验证码；阿里云行为验证密钥沿用 `ALIBABA_CLOUD_ACCESS_KEY_ID` / `ALIBABA_CLOUD_ACCESS_KEY_SECRET` 体系，同样兼容 `ALIYUN_*` 和 `HONE_ALIYUN_*` 前缀。
- 公开 `/chat` 在 iOS / Android 浏览器上不再因为 `visualViewport` 抖动或键盘弹出导致整页位置跳动；流式输出过程中滚动锚定更稳，长答案不会被新 token 顶飞。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 分享图片是通过 `html2canvas` 在浏览器端栅格化的。整段对话很长时图片高度可能达到上万像素；移动端低端机器请考虑只勾选关键消息以控制内存占用。
- 「复制图片」依赖 `navigator.clipboard.write` + `ClipboardItem`，在不支持的浏览器上按钮会自动隐藏，可改用「下载图片」或「系统分享」。
- 「系统分享」依赖 `navigator.share` 与 `canShare({ files })`；桌面 Chromium 下通常不可用，按钮也会自动隐藏。
- 二维码硬编码指向 `https://hone-claw.com/chat`。在私有部署环境下需要修改 `chat-share-modal.tsx` 中的 `qrUrl` 才能用自己的入口。
- 架构 SVG 不需要更新：本次 release 没有新增 channel、runner、poller、EventKind 或存储布局；变动集中在公开 web 前端 + 公开 SMS 守门，沿用既有 admin / public dual-surface 架构。
- 远端 release workflow 在 tag 推送后由 GitHub Actions 触发；本地只能确认 tag 已推送，不能替代远端产物发布结果。

## English

Release date: 2026-05-13

### What Users Should Know

- Public `/chat` now supports message sharing. Every assistant reply has copy + share icons in the bottom-right corner (desktop hover, always-on for mobile). The share sheet lets users multi-select which messages to include, then renders a portrait-shaped branded card off-screen with four outputs: download image, copy image (writes a PNG to the clipboard so it can be pasted directly into WeChat / Feishu), copy text, and system share (native share sheet where supported). The card footer carries a QR code pointing to `https://hone-claw.com/chat`.
- Public `/chat` received a broad density / rendering / visual pass: list bullets and numbers are back, `**bold**` labels are promoted to block-level mini-headings, consecutive assistant replies collapse the HONE brand chip, and code blocks are now a single soft chip without the previous double background — long formulas wrap inside the bubble instead of spawning a nested horizontal scrollbar. The bug where marked's code renderer stringified its promise into a literal `[object Promise]` is fixed.
- Streaming output, mobile composer viewport, scroll anchoring, and the new scroll-to-bottom affordance are noticeably more stable across devices. Sending no longer causes a full-page jump.
- Public-site copy and layout were rewritten end to end: `/portfolio`, `/me`, the `/home` video card and carousel, `/roadmap` mobile cards, table of contents + back-to-top on `/terms` and `/privacy`. The mobile header is tighter, `GitHub Stars` stays visible on mobile, and the entire public copy was stripped of internal jargon and localized.
- Public SMS login is now gated by Aliyun behavior captcha: a verification token must be issued before `/api/public/auth/sms/send` can deliver a real code, blocking SMS abuse.

### Who Should Update Soon

- Operators exposing `hone-claw.com/chat` or another public web user entry point.
- Teams whose end users want to share single replies or full conversations as images into WeChat, DingTalk, Feishu, or similar IM.
- Operators who need Aliyun behavior captcha in front of public SMS login.
- Maintainers of public-site copy, mobile layout, and the `/terms` / `/privacy` pages.

### What Will Change After Upgrading

- Every assistant message gains a copy + share icon pair. Clicking share opens a bottom sheet (mobile) or centered dialog (desktop) pre-selecting the message that triggered it; one tap can also select all or keep only this one.
- The exported image is 420px wide and built for the mobile-portrait aesthetic: Hone mark + tagline at top, full markdown rendering (headings, lists, bold, tables, code chips preserved) in the middle, and brand + QR at the bottom. The logo is inlined as real SVG so html2canvas can rasterize it reliably.
- Public SMS login flow: phone number → Aliyun behavior captcha → backend issues the Aliyun SMS code. The Aliyun credential mechanism still uses `ALIBABA_CLOUD_ACCESS_KEY_ID` / `ALIBABA_CLOUD_ACCESS_KEY_SECRET`; the `ALIYUN_*` and `HONE_ALIYUN_*` prefixes are also recognized.
- Public `/chat` no longer jitters when the iOS / Android `visualViewport` resizes around the keyboard, and streaming replies keep their scroll anchor — long answers are not bumped down by each new token.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- Share images are rasterized in-browser via `html2canvas`. Selecting the entire conversation can produce an image tens of thousands of pixels tall; on low-end mobile, prefer selecting only the key messages to keep memory bounded.
- "Copy image" requires `navigator.clipboard.write` + `ClipboardItem`; the button is hidden automatically where unsupported. Users can fall back to "Download" or "System share".
- "System share" requires `navigator.share` and `canShare({ files })`; it is normally unavailable on desktop Chromium, so the button is hidden there.
- The QR code is hard-coded to `https://hone-claw.com/chat`. Self-hosted deployments need to edit `qrUrl` in `chat-share-modal.tsx` to point at their own entry.
- Architecture SVG does not need an update: this release does not add channels, runners, pollers, EventKinds, or storage layout changes. Changes are concentrated in the public web frontend and the public SMS captcha gate, both layered on top of the existing admin / public dual-surface architecture.
- The remote release workflow is triggered by the pushed tag in GitHub Actions. Local verification can confirm that the tag was pushed, but not that remote release assets finished publishing.
