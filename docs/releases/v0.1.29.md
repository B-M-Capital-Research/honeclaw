# v0.1.29

发布日期：2026-04-16

English readers: please scroll down for the English version below.

## 用户应该知道的

- Desktop 设置页现在会正确显示 `codex_acp` runner，之前那种“实际可用但 UI 看不见 / 不能选”的状态已经收口，首次配置和后续切换都更一致。
- `telegram + codex_acp` 链路下，`scheduled_task` 这类依赖 `cron_job` 的能力现在能真正透传给 Codex；之前会出现 skill 能看到、能加载，但 `cron_job` 在 runner 层被静默丢掉，最后只剩“工具没暴露”的假判断。
- 仓库内现在新增了 `formal-release` skill，维护者可以用一条统一流程完成版本号更新、release note、验证、`main` 推送和 tag 推送，减少正式发版时的人肉漏项。

## 谁应该尽快更新

- 在 Desktop 里使用或切换 `codex_acp` runner，但最近在设置页看不到它、无法稳定选中它的用户。
- 在 Telegram 私聊里通过 `codex_acp` 配置定时任务、周期提醒或其它依赖 `cron_job` 的维护者。
- 需要频繁发正式版本、希望 release 流程更标准化的仓库维护者。

## 升级后会有什么变化

- Desktop runner 可见性和实际后端支持会更一致，减少“后端支持但前端配置入口缺失”的配置落差。
- `codex_acp` 在 MCP schema 转换阶段不再因为 `cron_job` 参数定义不合法而丢弃整把工具；cron 打开时，`scheduled_task -> cron_job` 的真实调用链路现在已用手工回归跑通。
- 正式发版时可以直接复用仓库内置的 `formal-release` skill 和 checklist，而不是每次临时拼 release 步骤。

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`

## 已知注意事项

- 这次主要收口的是 `codex_acp` 可见性、`codex_acp + cron_job` 真实链路，以及 release 工作流标准化；更广泛的 runner / skill runtime 对齐工作仍在继续。
- bug ledger 的文档整理也包含在 compare 范围内，但不属于本次 release 的主用户价值。

## English

Release date: 2026-04-16

### What Users Should Know

- The Desktop settings UI now shows the `codex_acp` runner correctly, closing the gap where the backend supported it but the UI could hide or fail to expose it consistently.
- On the `telegram + codex_acp` path, capabilities that depend on `cron_job`, such as `scheduled_task`, now make it all the way through to Codex. Previously the skill could be visible and loadable while `cron_job` was silently dropped at the runner layer, leading to the false conclusion that the tool was not exposed.
- The repo now includes a `formal-release` skill so maintainers can follow one consistent flow for version bumps, release notes, validation, `main` push, and tag push.

### Who Should Update Soon

- Users who rely on the Desktop app and recently could not see or select the `codex_acp` runner from settings.
- Maintainers who configure scheduled tasks or recurring reminders in Telegram direct chats through `codex_acp`.
- Repository maintainers who cut releases frequently and want a more standardized release workflow.

### What Will Change After Upgrading

- Desktop runner visibility is now aligned more closely with actual backend support, reducing cases where a supported runner had no reliable configuration entry in the UI.
- `codex_acp` no longer drops the entire `cron_job` tool because of an invalid MCP schema fragment; the cron-enabled `scheduled_task -> cron_job` execution chain is now covered by a real manual regression test.
- Maintainers can reuse the new in-repo `formal-release` skill and checklist instead of rebuilding the release flow manually each time.

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`

### Known Notes

- This release mainly closes the `codex_acp` visibility gap, the real `codex_acp + cron_job` execution path, and the standardized release workflow; broader runner and skill-runtime alignment work is still ongoing.
- The compare range also includes internal bug-ledger documentation maintenance, but that is not the main user-facing value of this release.
