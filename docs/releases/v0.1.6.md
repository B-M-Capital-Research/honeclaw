## English

Release date: {{RELEASE_DATE_UTC}}

### What Users Should Know

- `Codex ACP` is no longer pinned to exactly `codex-acp == 0.9.5`
- The requirement is now `codex >= 0.115.0` and `codex-acp >= 0.9.5`
- If you already have a newer `codex-acp` such as `0.11.1`, this version will pass validation directly

### Who Should Update Soon

- Users running `Codex ACP` who are blocked by errors like “found 0.11.1 / only supports 0.9.5”
- Users who want `hone-cli onboard` to show runner install commands and official entry points directly

### What Will Change After Upgrading

- `hone-cli onboard` will keep showing install commands and official links for `Codex CLI`, `Codex ACP`, and `OpenCode ACP`
- `Codex ACP` errors will now describe the minimum required version instead of misleading users into thinking only one exact version is supported
- `codex-acp` versions below the minimum requirement will still be rejected with an upgrade hint

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`
- If you update `codex-acp` separately, the recommended command is `npm install -g @zed-industries/codex-acp@latest`

### Known Notes

- This release relaxes the `codex-acp` version gate only; it does not change the minimum required `codex` version
- The current minimum requirement for `codex` remains `0.115.0`
