## 用户应该知道的

- `Codex ACP` 不再被硬性锁死在 `codex-acp == 0.9.5`
- 现在只要求 `codex >= 0.115.0` 且 `codex-acp >= 0.9.5`
- 如果你本机已经装了较新的 `codex-acp`，例如 `0.11.1`，升级到这个版本后可以直接通过版本校验

## 谁应该尽快更新

- 正在使用 `Codex ACP`，并被 “found 0.11.1 / only supports 0.9.5” 这类报错卡住的用户
- 希望在 `hone-cli onboard` 里直接看到各 runner 安装方式和官方入口的用户

## 升级后会有什么变化

- `hone-cli onboard` 会继续展示 `Codex CLI` / `Codex ACP` / `OpenCode ACP` 的安装命令和官方链接
- `Codex ACP` 的错误提示会改成“最低要求版本”，而不是误导成“只能安装某一个精确版本”
- 低于最低要求的 `codex-acp` 仍会被拒绝，并给出升级命令

## 安装与更新

- `curl | bash` 用户：重新运行安装脚本即可切到新版本
- Homebrew 用户：`brew update && brew upgrade honeclaw`
- 如果你单独更新 `codex-acp`，推荐命令：`npm install -g @zed-industries/codex-acp@latest`

## 已知注意事项

- 这次放宽的是 `codex-acp` 版本门槛，不改变 `codex` 自身的最低要求
- 当前最低要求仍是 `codex >= 0.115.0`
