## English

Release date: {{RELEASE_DATE_UTC}}

### What Users Should Know

- The memory page now includes Company Portraits, with long-term company profiles, event timelines, and tracking status by company
- `hone-cli cleanup` can now interactively remove local config, runtime data, and downloaded bundles before uninstall
- Both Homebrew and GitHub release install paths now clearly separate “uninstall the program” from “clean user data”

### Who Should Update Soon

- Users who want long-term company research archives instead of scattered KB records
- Users who need to uninstall Hone without manually guessing which directories are safe to delete
- Users who currently manage company research entries through the knowledge-memory table

### What Will Change After Upgrading

- The Web memory page now has a Company Portrait list and detail view for profile content, event timelines, and tracking status
- You can open a company portrait directly from the knowledge-memory table; if no portrait exists yet, the UI will prompt you to create one through the agent
- `hone-cli cleanup` will ask whether to delete:
  - `~/.honeclaw/data`
  - `~/.honeclaw/releases`
  - `~/.honeclaw/current`
  - and optionally `~/.honeclaw/config.yaml` / `soul.md`
- `brew uninstall honeclaw` still removes only the program binary; if you also want to remove user data, run `hone-cli cleanup` first

### Install and Update

- `curl | bash` users: rerun the install script to move to this version
- Homebrew users: `brew update && brew upgrade honeclaw`
- If you want to clean local data before uninstalling:
  - Interactive: `hone-cli cleanup`
  - Full non-interactive cleanup: `hone-cli cleanup --all --yes`

### Known Notes

- The frontend company portrait surface is still focused on read-only viewing and hard delete; creation, section writeback, event append, and tracking updates still mostly go through the agent or tool path
- If you already ran `brew uninstall honeclaw` and later decide to remove data too, you will need to delete `~/.honeclaw` manually
