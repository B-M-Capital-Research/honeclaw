## Goal

Replace one-off ticker exceptions with one shared canonicalization layer and an ordinary Agent-owned research loop that recognizes security-code inputs across supported markets, preserves the user's explicit symbol through provider lookup, distinguishes real symbols from weak acronym collisions, and discloses provider-backed identity or same-symbol quote gaps narrowly without a service-owned whole-answer refusal.

## Scope

- Audit recent production failures and the full ticker path: lexical candidate extraction, confidence/context rules, provider normalization, exact matching, semantic-empty fallback, quote verification, and response-contract routing.
- Cover ordinary US tickers, short symbols that overlap common acronyms, share-class punctuation, exchange-qualified digit-leading symbols, provider suffix aliases, index prefixes, crypto pairs, invalid symbols, and mixed explicit/named comparisons.
- Move market-symbol syntax and provider canonicalization into reusable deterministic helpers rather than scattered residual-language exceptions.
- Preserve false-positive safety: weak finance/technology acronyms must not silently become companies; ambiguous code shapes require stronger ticker context or clarification.
- Add unit, AgentSession, CI-safe contract, live provider, and production Web/SSE regression matrices.
- Keep bounded exact-quote batches, concurrent semantic searches, reusable probe evidence, and concurrent profile enrichment for typed scheduled/heartbeat preparation; interactive turns use the main Agent's current-query `search → exact quote/profile → problem evidence` loop.
- Keep exact ticker identity above weak semantic substring matches: single-stock ETF/ETN/leveraged/yield product names that embed CRWV/RKLB/AAPL-like underlying codes cannot create a false company-name conflict, while Ford/FORD and Apple name fallback remain supported only through word-bounded strong name relations (so Apple does not match Appleseed).
- Treat interactive lexical scans only as candidate seeds. Every nonempty interactive wording—including portfolio/watchlist wording—continues through the same main Agent loop, which reads the complete query, invokes `portfolio(view)` there when membership matters, searches all named securities, then exact-verifies quote/profile evidence; no fixed phrase grammar or closed-ticker shortcut may decide the interactive entity set before the Agent runs.
- Let the Interactive Agent's actual tool choices load evidence into the same context and its complete-query reasoning determine scope, depth, priorities, and answer shape. Web may own only the typed Session-time/neutral-evidence-policy first line; every entity, quote, relationship, candidate, valuation, and conclusion remains Agent-authored from current evidence. Agent-discovered post-run reconstruction is telemetry only: it may not normalize the body, launch a whole-run/seed/repair retry, rewrite content, reject success, or turn genuine provider no-coverage/ambiguity into a generic entity failure. Typed scheduled/heartbeat contracts retain their separate strict server-fact and validation path. Production Interactive finance has no structured finish handoff, source-locator correction, separate tool-free terminal synthesizer, terminal audit, second generation, or post-answer semantic gate.
- Reduce Interactive first-visible-text latency without reopening model-authored speculative tool prose. Every nonempty Interactive request enters open Agent discovery, but finance activates only after a registered, budget-accepted DataFetch call has parseable arguments, a supported `data_type`, all required security targets, and a valid identity-search shape; unsupported/missing-target/malformed/rejected DataFetch and ordinary Web/file/skill turns do not activate finance or deferred ACK. For named companies/securities, the initial Agent round rereads the complete query and returns a separate search call per covered entity with a stable case-sensitive `entity_route` and call-scoped `identity_match`; lowercase/mixed-case tickers in a securities context still use normalized exact-symbol lookup. An explicit route with missing/invalid match mode and any seventh new route—even in the first batch—are rejected before observer/registry/provider-network access and create no ledger state. Active finance rounds call only real business tools under `Required` while the structural route/evidence floor is incomplete. Once the floor is met, those same registered business tools remain available under `Auto`, allowing the Agent either to gather more current evidence or return one natural `Stop + Done` DirectFinal. Production does not expose `finish_research`. Valid aliases with a uniquely matching symbol merge into the accepted route. The hard bounds are six routes, three finance tool batches, 24 total calls, 20 DataFetch calls, and six Web calls; a T0 ACK starts batch/call accounting immediately, including Web-only batches before DataFetch activation. At exhaustion the same Agent's next iteration receives `tools=[]` and naturally answers from existing evidence; this is not the retired handoff/terminal generator.
- For a self-contained strict Interactive main-Agent finance request whose current input has a deterministic explicit security seed, remove synchronous work that does not need to be on the first-attempt critical path: derive active context plus at most four prior user-only references from one durable Session snapshot, restore currently active invoked-skill prompts separately from metadata, skip compact-summary loading and pre-run compaction, and force compaction only on actual context overflow. Referential follow-ups keep full history. Ask for independent DataFetch search and ticker-independent Web/news/filing/industry candidate discovery in one first-round model tool-call batch; the executor remains serial. Wait for route-bound search results before quote/profile/financial/ticker-news calls and never guess a ticker.
- Permit narrowly safe early Web visibility through one typed service-owned line containing only the Session answer time and neutral evidence policy. A deterministic explicit-security fast path may ask the unique downstream sink to ACK it before the first model call; a name-only finance turn waits for the safe DataFetch activation boundary above and also requires every call in that batch to be registered, structurally valid, and known read-only. Ordinary non-finance never ACKs it, rejected/closed/cancelled delivery records no phantom bytes, and the Agent final must reproduce the line byte-for-byte. After ACK, a later batch may enter tool frame/observer/registry/network execution only if every tool exists in the active registry, every argument payload parses and passes structural validation, and every call is classified known read-only; otherwise the whole batch fails before those boundaries. Success emits the final suffix once; later failure appends the fixed explicit research-failure suffix, persists exactly the visible partial bytes, and closes without reset/error flash.
- Bound the complete strict FunctionCalling turn with the converged runner timeouts: one request-level absolute overall deadline created once at `Agent::run` entry plus a fresh configured step deadline for each model, tool, and observer await. Initial discovery, active business model calls, tool execution, and progress observers consume the same remaining overall budget; no phase gets a fresh overall timeout. A failed or timed-out tool execution must append a failed `ToolCallMade` trace—including whether a deadline fired—before the run stops. The Agent and Session share one `hone-core` tool-effect classifier, so an uncertain persistent failure stops inside the current loop and cannot be replayed before the outer boundary sees the trace.
- Keep answer ownership with the Interactive Agent after generation. `response_finalizer.rs` may remove leaked system/internal protocol material, redact unsafe local paths, and stabilize local media references, but Interactive success must not enter legacy market-copy normalization, planning-sentence vetoes, tool-outcome reconstruction, or typed investment completeness refusal. Publish and persist the exact same finalized body. Relationship questions must obtain exact quote/profile evidence for every named entity plus current Web/news/filing evidence before making relationship claims.
- Keep Web model input bounded according to the runner actually selected for the actor. Prior scheduler/heartbeat/job groups and prior failed terminal groups may remain in durable history and UI pagination, but must be removed from the current Interactive runner context so automation and stale failures cannot displace the user's latest query.
- Push the fix, wait for required CI, rebuild runtime binaries, drain/restart through the supervisor, and verify storage/channel/API health.

## Risks / Open Questions

- Some real tickers are ordinary English or technical acronyms; provider existence alone is insufficient when the user's context clearly refers to the concept rather than the security.
- Exchange suffixes such as user-facing `.SH` and provider-facing `.SS` require explicit canonicalization without accepting a different listing or share class.
- Unsupported instruments must remain honest failures; architecture work must not degrade into fuzzy first-result selection, web-memory identity, or wrong-symbol quote acceptance.
- Multi-security completeness must never silently drop a named peer within the six-route research ceiling, but the enforcement point is the main Agent prompt/tool loop plus behavioral regression—not a post-hoc publication veto. Requests naming more than six instruments cannot receive a mechanical all-anchor guarantee without restoring a service semantic parser; the Agent must prioritize user anchors and explicitly disclose bounded omissions. Dynamic observation should record which named instruments received exact current-run provider coverage while preserving the successful body. Auxiliary products/benchmarks likewise cannot become answer-authorizing scope without future structured provenance.
- Prompt guidance must remain rich enough to produce the established investment style, but runtime truth enforcement and answer composition are separate concerns. Do not reintroduce an interactive phrase dictionary or fixed formatter to compensate for a weak model draft; improve Agent context/tool guidance and test the resulting loop instead.
- Free-form interactive financial/news claims cannot be reliably approved or rejected by looking for ticker headings, numbered sections, or matching numbers anywhere in a clause. Keep those parsers on typed scheduled/heartbeat output only; require structured claim provenance before adding strict per-claim enforcement to the interactive path.
- Numeric market/asset words must remain bound to their own source span; whole-query hints, value literals, and pre-resolution deduplication are prohibited.
- Scheduler and heartbeat task prose still needs a typed subject boundary: current live failures include `800G`, `NAND`, truncated `AST`, and `SEC` being sent to quote resolution. Keep this separate from the now-closed CRWV/CWY relation bug and do not mark the umbrella plan done until the existing P2 passes a live scheduler window.
- Long-running Web sessions must remain useful without allowing old automation/failure pairs to dominate the current query. Runtime-only pruning and bounded restore require regressions that prove durable history, public pagination, and scheduled push records are unchanged.
