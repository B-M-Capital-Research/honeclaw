## Goal

Replace one-off ticker exceptions with one shared canonicalization layer and an ordinary Agent-owned research loop that recognizes security-code inputs across supported markets, preserves the user's explicit symbol through provider lookup, distinguishes real symbols from weak acronym collisions, and discloses provider-backed identity or same-symbol quote gaps narrowly without a service-owned whole-answer refusal.

## Scope

- Audit recent production failures and the full ticker path: lexical candidate extraction, confidence/context rules, provider normalization, exact matching, semantic-empty fallback, quote verification, and response-contract routing.
- Cover ordinary US tickers, short symbols that overlap common acronyms, share-class punctuation, exchange-qualified digit-leading symbols, provider suffix aliases, index prefixes, crypto pairs, invalid symbols, and mixed explicit/named comparisons.
- Move market-symbol syntax and provider canonicalization into reusable deterministic helpers rather than scattered residual-language exceptions.
- Preserve false-positive safety: weak finance/technology acronyms must not silently become companies; ambiguous code shapes require stronger ticker context or clarification.
- Add unit, AgentSession, CI-safe contract, live provider, and production Web/SSE regression matrices.
- Keep bounded exact-quote batches, concurrent semantic searches, reusable probe evidence, and concurrent profile enrichment for typed scheduled/heartbeat preparation; interactive turns use the main Agent's current-query `search → exact quote/profile → problem evidence` loop.
- Keep exact ticker identity above weak semantic substring matches: single-stock ETF/ETN/leveraged/yield product names that embed CRWV/RKLB/AAPL-like underlying codes cannot create a false company-name conflict, while Ford/FORD and Apple name fallback remain supported only through word-bounded strong name relations (so Apple does not match Appleseed).
- Treat interactive lexical scans only as candidate seeds. Every nonempty interactive wording—including portfolio/watchlist wording—continues through the same main Agent loop, which reads the complete query, invokes `portfolio(view)` there when membership matters, searches all named securities, then exact-verifies quote/profile evidence; no fixed phrase grammar or closed-ticker shortcut may decide the interactive entity set before the Agent runs.
- Let the Interactive Agent's actual tool choices load evidence into the same context and its complete-query reasoning determine scope, depth, priorities, answer shape, and the first visible time line. Agent-discovered post-run reconstruction is telemetry only: it may not prefix or normalize the body, launch a whole-run/seed/repair retry, rewrite content, reject success, or turn genuine provider no-coverage/ambiguity into a generic entity failure. Typed scheduled/heartbeat contracts retain their separate strict server-fact and validation path. Production Interactive finance has no structured finish handoff, source-locator correction, tool-free terminal, terminal audit, second generation, or post-answer semantic gate.
- Reduce Interactive first-visible-text latency without reopening speculative tool-round output. Every nonempty Interactive request enters open Agent discovery, but finance activates only after a real DataFetch attempt, so ordinary Web/file/skill turns remain ordinary. For named companies/securities, the initial Agent round rereads the complete query and returns one separate search call per named entity with a stable case-sensitive `entity_route` and call-scoped `identity_match`; lowercase/mixed-case tickers in a securities context still use normalized exact-symbol lookup. Active finance rounds call only real business tools under `Required` while the structural route/evidence floor is incomplete. Once the floor is met, those same registered business tools remain available under `Auto`, allowing the Agent either to gather more current evidence or return one natural `Stop + Done` DirectFinal. Production does not expose `finish_research`. The route ledger guides tool choice only and never authorizes or vetoes publication. Tool-capable rounds remain buffered; only the completed natural answer is emitted and persisted once, preventing preamble/reset/flicker without paying for handoff, correction, or a second synthesis.
- Bound the complete strict FunctionCalling turn with the converged runner timeouts: one request-level absolute overall deadline created once at `Agent::run` entry plus a fresh configured step deadline for each model, tool, and observer await. Initial discovery, active business model calls, tool execution, and progress observers consume the same remaining overall budget; no phase gets a fresh overall timeout. A failed or timed-out tool execution must append a failed `ToolCallMade` trace—including whether a deadline fired—before the run stops. The Agent and Session share one `hone-core` tool-effect classifier, so an uncertain persistent failure stops inside the current loop and cannot be replayed before the outer boundary sees the trace.
- Keep answer ownership with the Interactive Agent after generation. `response_finalizer.rs` may remove leaked system/internal protocol material, redact unsafe local paths, and stabilize local media references, but Interactive success must not enter legacy market-copy normalization, planning-sentence vetoes, tool-outcome reconstruction, or typed investment completeness refusal. Publish and persist the exact same finalized body. Relationship questions must obtain exact quote/profile evidence for every named entity plus current Web/news/filing evidence before making relationship claims.
- Keep Web model input bounded according to the runner actually selected for the actor. Prior scheduler/heartbeat/job groups and prior failed terminal groups may remain in durable history and UI pagination, but must be removed from the current Interactive runner context so automation and stale failures cannot displace the user's latest query.
- Push the fix, wait for required CI, rebuild runtime binaries, drain/restart through the supervisor, and verify storage/channel/API health.

- Exchange suffixes such as user-facing `.SH` and provider-facing `.SS` require explicit canonicalization without accepting a different listing or share class.
