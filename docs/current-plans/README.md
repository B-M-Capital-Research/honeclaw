# Current Plans

最后更新：2026-03-18

## 目的

- 为每个需要持续跟踪的活跃任务提供独立计划页
- 支持并行线程同时推进，避免争用 `docs/current-plan.md`

## 文件命名

- 推荐：`<topic>.md`
- 示例：
  - `context-governance.md`
  - `actor-regression-script.md`
  - `web-portfolio-refactor.md`

## 单任务计划页最小结构

```md
# <任务名>

最后更新：YYYY-MM-DD
状态：进行中 | 已完成 | 阻塞

## 目标

- ...

## 涉及文件

- ...

## Todo

- ...
- 验证：...
- 文档同步：...

## 当前进展

- ...

## 阻塞

- 无 / ...

## 风险

- ...
```

## 维护规则

- 只有中等及以上、跨回合 / 跨模块，或需要并行协作 / 交接 / 阻塞管理的任务才进入本目录
- 单次 commit / sync / rebase、轻量脚本或配置修补、无行为变化的小补丁、纯文案 / 格式修正等小任务，默认不在本目录建页
- 同一主题优先复用原文件，不要重复新建
- 任务完成后，从索引页移除；若需要交接，再更新 handoff
- 计划页记录任务过程，handoff 只记录交接所需结论
- 已完成计划默认不长期留在 `docs/current-plans/`；结论已进入 handoff 后，应删除原计划页
- 同一主题在短时间内多次小修时，优先合并到既有 handoff，而不是继续新增碎片交接
