## Context

The product requirement is an ordinary Agent workflow: read the complete user query, identify every named company or security, call DataFetch/Web or other relevant business tools, keep those results in the same model context, and then answer the user naturally. Interactive finance has no configured rule that forbids a useful answer merely because a service-side completeness protocol was not satisfied.

Several successive repairs accidentally moved away from that requirement. `finish_research` became a service-injected control tool; a structured handoff then tried to bind claims to opaque source ids; malformed locators triggered a hidden correction; and a separate tool-free terminal generated the visible answer. Although these phases lived inside one `Agent::run`, they still divided one user turn into a research role, a completion protocol, a correction phase, and a second synthesis. That design produced the same class of failures the product had already rejected: internal protocol errors flashed briefly, valid evidence was followed by fixed “cannot safely publish” copy, and the second synthesis invented facts that the research Agent had not selected.

The decisive production trace came from exact build `d4d44735` for `crwv和英伟达有什么关系`. Ten DataFetch/Web calls succeeded, so FMP, DataFetch, and Tavily were not the outage. The request nevertheless used seven model calls and took 92.435 seconds. Business tools consumed only about 8.9 seconds; after evidence was loaded, the finish-locator round took about 13.993 seconds, locator correction about 19.925 seconds, and the tool-free terminal about 19.363 seconds. The terminal then treated `market_date_new_york` as a New York Stock Exchange close even though the entity evidence said NASDAQ, and expanded an empty inference set into unsupported “core GPU supplier”, “major customer”, “high dependence”, “locked in”, and “multiple binding” claims. The failure was the completion architecture, not provider availability or stale deployment.

Entity and evidence structure still matters before synthesis. `data_fetch(search)` establishes identity candidates only; it does not prove a quote, exchange, relationship, customer concentration, contract, GPU allocation, or valuation input. The same Agent therefore still needs explicit route-aware search and appropriate current-turn follow-up evidence. What changes is ownership: that structural state guides the next model/tool choice and remains observable in telemetry, but it does not create a second completion protocol or authorize the service to replace the Agent's answer.

## Decision

- Production Interactive finance uses one standard function-calling Agent loop, one `AgentContext`, and one model role. After the full user request is read, the Agent may call registered actor-bound business tools or, when it has enough current-turn evidence, return one natural nonempty `Stop + Done` DirectFinal. The same Agent that chose the tools writes the final answer.
- Do not expose `finish_research` in the production Interactive tool schema. Do not run a completion handoff, source-locator correction, tool-free terminal synthesis, terminal semantic audit, post-answer review, second generation, fixed completeness refusal, or market-copy rewrite. The legacy implementation is compiled only for tests; production builds have no flag or public builder that can enable it.
- Enable this research mode through an explicit `AgentRunnerRequest` field, independently of Web-only streaming policy. Every Interactive `function_calling` finance turn selected through main-Agent entity discovery uses the same research loop on Web, Feishu, Discord, and Telegram; only Web receives the early committed-delta policy.
- Build the finance model request from the composed current-turn input plus a narrowly bounded follow-up window: at most four prior user utterances and 4,000 characters, labelled as pronoun/reference context only. Never replay historical assistant claims, tool protocol/results, reasoning, prices, financials, or conclusions. Only when the current strict Interactive input is self-contained and contains a deterministic explicit security seed does the first attempt derive active context and this reference window from one durable Session snapshot while omitting synchronous pre-run compaction and compact-summary loading. Referential follow-ups keep the full restore path. Previously invoked skill prompts are restored separately from Session metadata, subject to current activation, and do not consume the four-user-reference budget. Context overflow still forces compaction and retries with the resulting summary; manual compaction and noninteractive behavior are unchanged.
- Keep the request-local entity/evidence ledger and its route semantics. Every named security receives its own Agent-declared stable, case-sensitive `entity_route`; each search declares `identity_match=exact_symbol|name_or_alias`; lowercase or mixed-case symbols in a securities context still use normalized exact-symbol lookup. Search results are identity-only, and later quote/profile/asset-route evidence must stay bound to the same route and selected symbol. The service does not split natural language on commas, `and`, `和`, `&`, `/`, case, length, or ticker shape.
- Use the structural floor only to guide the ordinary loop. Before the floor, production requests business tools with `Required`; once the floor is satisfied, it uses the real business tools under `Auto`, allowing the Agent either to gather more evidence or return its natural answer. The floor never validates, rejects, retries, suppresses, or replaces a complete DirectFinal. Genuine provider gaps are described narrowly from attempted evidence; they do not become a whole-answer ban.
- Keep every tool-enabled round speculative and buffered until the entity/evidence floor allows a natural final. Tool-round prose, reasoning, resets, and attempt-local errors do not become user-visible answer bytes because the same round may still branch to another tool. On Web, once an eligible final begins with one complete canonical `数据时间：北京时间 …；行情口径：…` line, that line may commit early without its trimmable terminator. Security-clean complete body lines from the same final may then extend the visible prefix; each line's terminator and the current incomplete line remain deferred until a later complete safe line proves those bytes stable. Suspicious lines fall back to final security cleanup. Each delta counts as committed only after the unique downstream publication sink accepts it, and the runner appends exactly those accepted bytes to `committed_visible_prefix`. Cancellation, rejected or closed receivers, ambiguous sinks, and backpressure cannot create phantom persisted bytes. After any visible prefix commits, a later tool call is a protocol error and no persistent tool may execute.
- Ask the model to batch independent evidence work in the earliest possible tool-call round. Named-entity DataFetch searches and ticker-independent Web/news/filing/industry candidate discovery may be selected together; the current executor still awaits those calls serially. Quote/profile/financials and other symbol-dependent calls wait for the corresponding search result and may never guess a ticker from a name, model memory, or the first semantic result.
- Keep all current-turn business tool results in the same Agent context. Web snippets remain bounded snippet evidence rather than page bodies or real-world rank. Relationship, ownership, customer, contract, GPU, valuation, and negative-existence claims must be based on current evidence selected in this loop. If evidence does not establish relationship strength, use a neutral bounded description; do not upgrade it to “core”, “largest/major”, “high dependence”, “locked”, “exclusive”, “priority”, “multiple binding”, or equivalent language.
- The first line for a security, market, or sector answer still uses the current Session Beijing query time and an evidence-based quote basis. Prefer `hone_quote_time.beijing` for quote-source time. `market_date_new_york` and a generic `new_york` date/time establish only a date/timezone context; they do not prove “New York Stock Exchange”, “NYSE”, “close”, or a regular-session close. State an exchange only from structured `exchange` / `exchangeShortName` evidence.
- Empty output, provider/transport error, deadline, malformed lifecycle, or iteration exhaustion remains a genuine failed run. Such failures may not synthesize answer authority or fixed investment-completeness copy. Tool failures stay in the current trace so the Agent can disclose a bounded gap when the provider lifecycle still allows a natural answer.
- Caller-supplied typed scheduled/heartbeat contracts retain their separate deterministic preparation, validation, fallback, and fail-closed behavior. This decision changes open-ended Interactive finance only.

## Consequences

- The Agent sees the complete question and the evidence it requested without a lossy protocol handoff. A tool name/opaque-id mismatch can no longer turn healthy sources into an empty terminal context.
- Removing finish, correction, and terminal rounds eliminates three latency-heavy model phases from the normal path. On Web, the complete canonical header and then security-clean stable body lines can become visible before the same natural answer finishes, while speculative tool rounds remain hidden. Self-contained explicit-security requests also avoid a synchronous compaction/summary read unless context overflow actually requires it.
- The service continues to own structural safety: actor-bound tools, route/evidence telemetry, tool-choice guidance, deadlines, tool-effect replay safety, provider finish/DONE validation, output sanitization, one public terminal, and exact visible/persisted bytes. It does not own open-ended financial prose approval.
- No durable-data migration is required. The mode switch, ledger, and prompts are request-local.
- A safe rollback may keep the whole natural body deferred, but it must not reactivate `finish_research`, locator correction, tool-free terminal generation, semantic review, fixed refusal, or browser request replay.

## Verification / Adoption

- Focused Agent tests must prove that production finance exposes only real registered business tools, uses `Required` before the structural floor and `Auto` afterward, accepts one natural DirectFinal, and makes no finish/correction/terminal/audit model call.
- Entity regressions must cover lowercase and mixed-case exact tickers, CRWV over embedded-name CWY, independent CRWV/NVIDIA routes, Ford/FORD, Apple/Appleseed, legal names/share classes, crypto, broad-market routes, no-coverage attempts, and same-route quote/profile evidence without any service-side query splitter.
- Content regressions must use deliberately different Session, quote, and New York market-date values. They must prove that only the Session value supplies `数据时间`, `hone_quote_time.beijing` supplies quote-source time, a New York date never becomes “纽交所收盘价”, exchange comes only from exchange fields, and unsupported relationship-strength language is absent.
- AgentSession/channel regressions must prove that Web, Feishu, Discord, and Telegram enable the same finance-loop mode independently of stream policy; tool rounds emit no speculative answer; one completed body is emitted once; the persisted assistant body is byte-identical; no fixed completeness refusal or post-answer rewrite runs; and the unique terminal closes the stream without reset or second output. Provider requests may preserve the bounded prior-user wording needed for a follow-up, but must exclude historical assistant/tool claims such as an old `15 USD` quote. A lowercase exact-symbol query must resolve its canonical uppercase provider symbol, and a model-invented retired tool name must be returned to the Agent as an ordinary unknown tool without emitting a public tool-start/tool-failure status.
- Production adoption requires an exact-commit build, zero-active-chat graceful restart, PG/object-storage/API/channel/process health checks, and fresh direct canaries for `crwv和英伟达有什么关系`, `crwv和nbis的估值怎么看`, a single-ticker/no-coverage case, and a non-finance query. Record model-call count, tool duration, first-visible time, final time, one-output/history equality, and zero remaining active chats. Target the normal relationship path at no more than five model calls and under 60 seconds first-visible time while treating correctness and one-output behavior as hard requirements.
- The first-visible latency canary must separately record first provider delta, first committed runner prefix, first successfully queued Web `assistant_delta`, later answer-delta progression, and `run_finished`. An observer callback, rejected SSE send, or persistence record that did not complete successfully is not evidence of delivered text.
- The umbrella ticker plan remains `in_progress` and is not archived because the separately tracked scheduler task-prose/entity P2 (`800G` / `NAND` / `AST` / `SEC`) is still open.

## Risks

- A provider can still answer before gathering semantically sufficient evidence after the floor reaches `Auto`. Address that through same-Agent prompt/tool guidance and behavioral canaries; do not restore a service-side answer veto or second synthesis.
- Tool calls selected in one model batch still execute serially. Incident telemetry attributed only about 8.45 seconds to ten tools, so this is not a deployment blocker for the current repair; if the exact canary still misses the under-60-second first-visible target and tool time becomes dominant, add separately reviewed bounded concurrency for a narrow read-only discovery set.
- Test-only legacy finish/terminal code can drift back into production if compile gates or runner wiring change. The finance CI contract must verify the legacy schema, locator, terminal methods, and canonical-prefix stream state are individually `cfg(test)`, while keeping the production mode, cross-channel request flag, and absence of the retired schema from runner wiring explicit.
- Streaming is limited to a validated complete canonical header and byte-stable, security-clean lines from the continuation of that same eligible final. Never broaden it to trailing whitespace, Markdown preambles, incomplete headers, prompt markers, tool-capable drafts, or an unseen prefix recorded before emitter success.
- The scheduler task-prose/entity P2 has a different typed-origin boundary and remains outside this Interactive decision.
