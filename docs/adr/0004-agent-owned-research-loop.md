## Context

The isolated research-control design fixed one provider behavior but divided one reasoning process into two model roles. Business rounds could load evidence, while a separate completion saw only `continue_research` and `finish_research` and was expected to decide whether that evidence was sufficient. If that completion failed, timed out, returned content, or otherwise violated its narrow protocol, ADR 0003 allowed the runtime to interpret the failure as an implicit/degraded finish and enter terminal synthesis.

The first production canary after the MiniMax clean-EOF transport repair proved that boundary unsafe. The exact query `crwv和英伟达有什么关系` completed as one successful stream with no fixed refusal, reset, second answer, or missing-DONE error, so the provider adapter and public stream lifecycle were working. However, the Agent made only `data_fetch(search)` calls for CRWV and NVIDIA. The isolated control completion then ended through an error-labelled implicit finish, and terminal synthesis asserted company-relationship facts that were not present in the current-turn tool evidence. The canary therefore failed answer quality even though transport and DataFetch identity lookup succeeded.

The first production canary after deploying the initial ADR 0004 implementation (`0cd03abb`) proved that one-loop ownership fixed transport and publication but left a last-mile asymmetry. It produced one answer and one successful terminal in about 42.4 seconds with no reset, error flash, refusal, second generation, or visible/persisted mismatch. Its five Agent calls successfully searched CRWV/NVIDIA, ran one relationship Web search, and fetched batched CRWV/NVDA quotes. FMP and Tavily were healthy. Yet DirectFinal used the quote provider time (`2026-07-18 04:00`) as the answer data time instead of the current Session time and expanded secondary snippets into unsupported customer rank and supply-priority claims. Explicit terminal synthesis had a strong final user contract, while DirectFinal had only an earlier system reminder. A first follow-up attempted to make quote/profile coverage a runtime publication prerequisite; independent review showed that this would recreate the exact generic-error/refusal class the design is meant to remove, while also deadlocking crypto and malformed/budget-error paths. The accepted repair therefore keeps entity/quote/asset-route sequencing in the Agent prompt, retains the ledger for structural guidance and telemetry, gives both completion paths one exact-prefix/source-bounded contract, and never uses that ledger to veto a complete natural answer.

`data_fetch(search)` establishes candidate identity; it does not establish a commercial relationship, ownership, customer concentration, contract, GPU deployment, valuation input, or the absence of any such relationship. A service-side semantic checker cannot safely repair this after generation: it would recreate the brittle refusal/rewrite path that previously replaced valid Agent answers with “cannot safely publish” copy. The research Agent that reads the user query and tool results must retain both evidence-selection and finish authority inside one continuous loop.

## Decision

- Keep one `Agent::run`, one `AgentContext`, and one model role for active Interactive research. After finance activation, each active completion sees the actor-bound business tools. There is no separate `continue_research` completion and no service-owned implicit finish decision.
- Carry `AgentRunnerRequest.timeout` and the configured step timeout into the strict FunctionCalling runner. Convert the request/fallback overall value once at `Agent::run` entry into a single absolute deadline shared by initial model discovery, active model rounds, tool execution, tool observers, terminal synthesis, and committed-prefix terminal recovery; no phase may reset or extend it. Apply a fresh configured step deadline to every individual await. Before returning from a tool failure or deadline, retain a failed `ToolCallMade` with `status=failed`, `isError=true`, and an explicit timeout flag. Use the canonical `hone-core` tool-effect classifier in both the Agent and Session so an uncertain persistent failure stops inside the current loop and cannot be replayed before the outer boundary receives its trace.
- Track a request-local structural evidence ledger inside the Agent loop for guidance and telemetry, not publication authority:
  - `data_fetch(search)` is `identity_only`; it may identify or refine a security but cannot by itself make terminal synthesis eligible;
  - after identity search, record exact-symbol quote plus profile/asset-route attempts, including a crypto-specific `search → crypto_quote` route and a separate broad-market path; unsearched/pre-search symbol calls do not satisfy that diagnostic floor;
  - the Agent prompt—not a server-side answer gate—requires all named securities to be resolved and researched. Genuine no-coverage, malformed calls, budget exhaustion, crypto routing, and unusual wording must still be answerable with bounded disclosure instead of a generic runtime failure.
- After the first accepted active business-tool follow-up, expose `finish_research` alongside the same business tools in `Auto` mode so the Agent may keep researching, return a natural final, or explicitly enter the streamed terminal. Exactly one sole, explicit `finish_research` call whose arguments parse as an empty JSON object is the only transition into the empty-tools terminal completion. A mixed finish plus business calls executes only the business calls; duplicate, malformed, or nonempty-argument finish calls are ignored as control signals and never enter the registry, execution observer, user progress, persistence, or business-tool budget.
- Any complete, nonempty natural-language body that ends with provider `Stop` plus `Done` in an active business completion is the same Agent's `DirectFinal`, even if the advisory ledger reports incomplete coverage. Publish it unchanged through the deferred Interactive boundary without terminal synthesis, retry, rejection, replacement, or a second model generation. Empty output, provider error, timeout, protocol failure, or iteration exhaustion remain genuine failed runs and never synthesize a terminal answer.
- Keep tool-capable rounds speculative and hidden from the public answer. The terminal completion remains tool-free and receives this run's user/tool transcript plus a bounded, explicitly non-evidence copy of prior user requests needed to understand follow-up pronouns or requested analysis methods. Prior assistant/tool facts and prose attached to tool-call messages remain excluded. Provider reasoning signatures needed for MiniMax/Mimo tool-result replay remain on the wire in active and terminal follow-ups but are explicitly non-evidence and are not persisted in the final turn. The canonical first-line commit, exact suffix publication, one buffered terminal-only committed-prefix recovery, honest partial failure, and provider finish/DONE rules from ADR 0003 remain unchanged.
- Give DirectFinal and explicit terminal synthesis the same last-mile evidence contract as the final ephemeral user message. Extract and repeat the exact first-line prefix already generated for this runtime turn rather than asking the model to choose a timestamp. The Session query time comes from one structured clock value shared with `Session 上下文`, the answer suffix, and context-overflow recovery; quote provider time belongs only in `行情口径` and cannot replace it. Prior assistant/tool facts cannot reach the last mile, while bounded prior user requests may remain solely for intent continuity.
- Keep evidence selection open-ended. The complete user query—not a phrase grammar, ticker exception table, or server-side intent classifier—determines which business tools the Agent calls. Relationship questions require exact quote/profile evidence for the named entities plus current Web/news/filing evidence for the claimed relationship; valuation questions require the inputs needed for the stated valuation frame. The structural ledger does not claim that any one non-search call is semantically sufficient.
- Require evidence-limited language in the Agent prompt. “This run did not find evidence” is not proof that a relationship, holding, customer, contract, or event does not exist. Negative factual assertions such as “no equity relationship”, “not a customer”, or “no contract exists” require explicit current evidence; otherwise the Agent may only describe what the current search did or did not establish.
- Describe actual Web-search capability in the tool schema: it is a low-bandwidth basic search that returns at most three titles, URLs, and snippets, not page bodies. A snippet or relationship label may be repeated only within its literal support. It may not be expanded into unstated rights or duties, customer/supplier rank, importance, exclusivity, guarantees, priority, concentration, or roles; the Agent should continue toward company announcements, regulatory filings, or other primary sources when those details matter.
- Keep Interactive post-run reconstruction observational. The service may record entity/evidence coverage and protocol transitions, but it may not prepend, normalize, semantically approve, retry, repair, reject, replace, or append a fixed completeness refusal to the successful Agent answer. Completed Interactive bodies use a dedicated security-only finalizer: it may remove system/internal protocol material, redact unsafe local paths, and stabilize local media references, but it may not rewrite market copy, veto planning-like prose, reconstruct an answer from tool outcomes, or run the scheduled/heartbeat completeness guard. The exact finalized body emitted to the user is also the body persisted in session history. Caller-supplied typed scheduled/heartbeat contracts retain their separate deterministic validation and fail-closed behavior.

## Consequences

- The model that chooses the evidence also decides when it is ready to synthesize, with all business-tool results still in the same context. There is no second control role that can lose the user question, mistake identity lookup for research completion, or add an extra latency-only completion.
- The prompt and advisory ledger make missing search/quote/profile/crypto routing observable without turning an imperfect tool trace into a whole-answer ban.
- A provider that returns a complete natural answer does not pay for an unnecessary terminal round and cannot be replaced by service-authored failure copy. Its answer still receives the exact same time/source/relationship/valuation contract as the explicit terminal path.
- Genuine provider gaps still permit a useful answer after a real evidence attempt. The Agent narrows claims and labels the gap; the runtime does not turn missing fields into a whole-answer ban.
- The service remains responsible for structural safety—tool registration, current-run evidence isolation, stream lifecycle, terminal uniqueness, security-only Interactive cleanup, exact visible/persisted bytes, and typed noninteractive contracts—but not for open-ended semantic publication review.
- Removing the isolated control request also removes one model round from the critical path. Actual first-visible latency still depends on business-tool and terminal generation time and must be measured in production rather than inferred from the architecture.
- One absolute request deadline bounds total FunctionCalling latency instead of multiplying an overall timeout by phases, and the configured step deadline bounds any single stalled model/tool/observer await. A timed-out or failed persistent tool leaves an auditable uncertain trace and fails immediately inside the same loop; failed observer progress is emitted as failure rather than completion.
- No durable-data migration is required. The evidence ledger, completion-signal availability, stable turn clock, and terminal transition are request-local.

## Verification / Adoption

- Focused Agent tests must prove that the advisory ledger distinguishes unsearched/pre-search securities, broad-market calls, equity quote/profile coverage, crypto `search → crypto_quote`, and Web-only evidence; only a sole explicit finish with an empty JSON object enters terminal synthesis, while every complete natural DirectFinal is preserved without retry or veto.
- Regressions must cover mixed/duplicate/malformed finish calls; complete `Stop + Done` DirectFinal with no terminal or second generation even before the advisory floor; empty/error/timeout behavior without implicit terminal authorization; genuine provider no-coverage; relationship evidence containing the exact weak `$6.3B unused capacity` and MFN examples; stable time across context-overflow recovery; and preservation of prior user intent without stale assistant/tool facts.
- Timeout regressions must prove that a pending initial model stream stops at both the configured step boundary and one request-level overall deadline; hanging tool observers are bounded; and a hanging or failed persistent tool produces exactly one failed/timeout `ToolCallMade`, stops the Agent without another model or terminal call, and remains visible to outer retry safety logic. The finance CI contract must lock timeout wiring, the one-overall-deadline helper, observer bounds, shared side-effect classification, failed progress, and failed-trace shape.
- AgentSession regressions must prove that successful Interactive output bypasses legacy market-copy normalization, planning-sentence vetoes, tool-outcome reconstruction, and investment completeness refusal; after security cleanup and media stabilization, the one visible body and persisted assistant body must remain byte-identical.
- Existing provider/session tests must continue to prove requested/effective tool-choice telemetry, typed finish plus internal DONE, MiniMax clean-EOF normalization at the adapter boundary, current-turn-only terminal evidence, canonical-prefix commit/recovery, one terminal, and exact visible/persisted equality.
- Both completion paths must prove the same literal Session-time prefix and evidence contract, with deliberately different Session (`2026-07-19 09:31`) and quote (`2026-07-18 04:00`) timestamps. Relationship regressions must prove that weak transaction/relationship snippets are not expanded into rank, guarantees, priority, concentration, or unstated roles, while no service-side semantic post-processor is restored.
- The finance CI contract must reject restoration of `continue_research`, isolated control completions, control-error-to-terminal routing, malformed finish authorization, any evidence-ledger DirectFinal veto, service-authored Interactive refusal/rewrite/reconstruction, or removal of quote/profile/Web guidance from the Agent contract.
- Adoption requires the full repository gates, an exact-commit build, zero-active-chat graceful restart, PG/S3/API/process health checks, and fresh production replays for `crwv和英伟达有什么关系`, `crwv和英伟达什么关系，估值怎么看`, and a non-finance query. Acceptance requires relationship-specific evidence, no unsupported negative or concentration/contract/GPU claims, one answer and one public terminal event, no error/reset/refusal, exact visible/persisted text, and first-visible timing recorded separately from completion.

## Risks

- Quote/profile coverage is intentionally an advisory structural signal. The Agent can still choose weak evidence for a complex question; prompt/tool descriptions, behavioral regressions, and production canaries must verify question-specific evidence without expanding the runtime into a semantic veto.
- A provider may reject or ignore `Required`. Existing narrow `Required → Auto` compatibility telemetry remains necessary. Before the first accepted follow-up, `Required` guides tool acquisition; afterward the same Agent uses `Auto` so it may continue tools or finish naturally. Any complete fallback body is preserved; the advisory ledger never turns it into a generic error.
- Tool categorization must stay structural and generic. Adding company-specific symbols, question phrases, or hard-coded relationship routes would recreate the architecture this decision removes.
- Iteration limits remain necessary. Reaching one is a normal failed run, not permission to fabricate an answer, publish a fixed “research incomplete” response, or enter terminal synthesis without explicit finish.
