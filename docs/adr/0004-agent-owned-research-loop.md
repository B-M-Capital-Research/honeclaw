## Context

The product requirement is an ordinary Agent workflow: read the complete user query, identify every named company or security, call DataFetch/Web or other relevant business tools, keep those results in the same model context, and then answer the user naturally. Interactive finance has no configured rule that forbids a useful answer merely because a service-side completeness protocol was not satisfied.

Several successive repairs accidentally moved away from that requirement. `finish_research` became a service-injected control tool; a structured handoff then tried to bind claims to opaque source ids; malformed locators triggered a hidden correction; and a separate tool-free terminal generated the visible answer. Although these phases lived inside one `Agent::run`, they still divided one user turn into a research role, a completion protocol, a correction phase, and a second synthesis. That design produced the same class of failures the product had already rejected: internal protocol errors flashed briefly, valid evidence was followed by fixed “cannot safely publish” copy, and the second synthesis invented facts that the research Agent had not selected.

The decisive production trace came from exact build `d4d44735` for `crwv和英伟达有什么关系`. Ten DataFetch/Web calls succeeded, so FMP, DataFetch, and Tavily were not the outage. The request nevertheless used seven model calls and took 92.435 seconds. Business tools consumed only about 8.9 seconds; after evidence was loaded, the finish-locator round took about 13.993 seconds, locator correction about 19.925 seconds, and the tool-free terminal about 19.363 seconds. The terminal then treated `market_date_new_york` as a New York Stock Exchange close even though the entity evidence said NASDAQ, and expanded an empty inference set into unsupported “core GPU supplier”, “major customer”, “high dependence”, “locked in”, and “multiple binding” claims. The failure was the completion architecture, not provider availability or stale deployment.

Entity and evidence structure still matters before synthesis. `data_fetch(search)` establishes identity candidates only; it does not prove a quote, exchange, relationship, customer concentration, contract, GPU allocation, or valuation input. The same Agent therefore still needs explicit route-aware search and appropriate current-turn follow-up evidence. What changes is ownership: that structural state guides the next model/tool choice and remains observable in telemetry, but it does not create a second completion protocol or authorize the service to replace the Agent's answer.

## Decision

- Production Interactive finance uses one standard function-calling Agent loop, one `AgentContext`, and one model role. After the full user request is read, the Agent may call registered actor-bound business tools or, when it has enough current-turn evidence, return one natural nonempty `Stop + Done` DirectFinal. The same Agent that chose the tools writes the final answer.
- Do not expose `finish_research` in the production Interactive tool schema. Do not run a completion handoff, source-locator correction, tool-free terminal synthesis, terminal semantic audit, post-answer review, second generation, fixed completeness refusal, or market-copy rewrite. The legacy implementation is compiled only for tests; production builds have no flag or public builder that can enable it.
- Enable this research mode through an explicit `AgentRunnerRequest` field, independently of Web-only streaming policy. Every Interactive `function_calling` finance turn selected through main-Agent entity discovery uses the same research loop on Web, Feishu, Discord, and Telegram; only Web receives the early committed-delta policy.
- Build the finance model request from the composed current-turn input plus a narrowly bounded follow-up window: at most four prior user utterances and 4,000 characters, labelled as pronoun/reference context only. Never replay historical assistant claims, tool protocol/results, reasoning, prices, financials, or conclusions. Only when the current strict Interactive input is self-contained and contains a deterministic explicit security seed does the first attempt derive active context and this reference window from one durable Session snapshot while omitting synchronous pre-run compaction and compact-summary loading. Referential follow-ups keep the full restore path. Previously invoked skill prompts are restored separately from Session metadata, subject to current activation, and do not consume the four-user-reference budget. Context overflow still forces compaction and retries with the resulting summary; manual compaction and noninteractive behavior are unchanged.
- Keep the request-local entity/evidence ledger and its route semantics. Each Agent-discovered security should receive its own stable, case-sensitive `entity_route`; each search declares `identity_match=exact_symbol|name_or_alias`; lowercase or mixed-case symbols in a securities context still use normalized exact-symbol lookup. Search results are identity-only, and later quote/profile/asset-route evidence must stay bound to the same route and selected symbol. Only a syntactically valid identity search creates a route. If a search carries an explicit `entity_route`, that same call must carry one valid call-scoped `identity_match`; a missing/invalid value is rejected before observer notification, registry execution, or provider/network access and creates no ledger state. The six-route ceiling applies while admitting the first batch as well as later batches, so a seventh new route is rejected before execution without poisoning existing routes. Malformed/out-of-order calls likewise cannot poison the ledger, and an unknown alias with a symbol uniquely matching one accepted route merges into it. Prioritize user-named anchors and normally use at most four slots for screening candidates. This admission-order ceiling deliberately avoids a service natural-language entity parser, so a request naming more than six securities receives explicit bounded-gap disclosure rather than an unbounded crawl.
- Use the structural floor only to guide the ordinary loop. Before the floor, production requests business tools with `Required`; once the floor is satisfied, it uses the real business tools under `Auto`, allowing the Agent either to gather more evidence or return its natural answer. Finance mode independently enforces at most three tool-call batches, 24 total calls, 20 DataFetch calls, and six Web calls. An ACKed service prefix starts that bounded-finance accounting immediately: even a Web-only batch before DataFetch activation consumes one of the three batches plus the applicable total/Web budgets. When a ceiling is reached, the next iteration in the same `Agent::run` receives `tools=[]` and produces one natural evidence-bounded final from its existing context. This does not create a second role, handoff, terminal synthesizer, or semantic publication gate.
- Keep model-authored tool-round prose, reasoning, resets, and attempt-local errors speculative and buffered. On Web finance, configure exactly one typed service-owned first line using the Session answer time and neutral evidence-policy basis. A deterministic explicit-security seed may commit it before the first model call; otherwise it may commit only after a budget-accepted batch contains at least one registered DataFetch call that has a supported `data_type`, every required security target, and a valid identity-search shape where applicable, and every call in the batch is registered, known read-only, and structurally valid. Unsupported DataFetch types, missing/invalid targets, malformed arguments, and rejected identity searches neither activate finance nor make deferred ACK eligible. The final Agent body must reproduce the line byte-for-byte. If hidden-reasoning removal leaves only Unicode whitespace before that exact line, the Session may discard only those leading whitespace bytes; a non-whitespace preamble or actual prefix mismatch still fails closed. Each prefix counts as committed only after the unique downstream publication sink accepts it. Cancellation, rejected or closed receivers, ambiguous sinks, and backpressure cannot create phantom persisted bytes. After commitment, a later batch may continue only when every tool name is present in the active registry, its arguments parse and pass the same structural checks, and the effect classifier marks the call known read-only; otherwise the whole batch fails before an assistant tool frame, observer event, registry execution, or network access.
- Ask the model to batch independent evidence work in the earliest possible tool-call round. Named-entity DataFetch searches and ticker-independent Web/news/filing/industry candidate discovery may be selected together; the current executor still awaits those calls serially. Quote/profile/financials and other symbol-dependent calls wait for the corresponding search result and may never guess a ticker from a name, model memory, or the first semantic result. The hard budgets are authoritative even when prompt guidance is ignored.
- Keep all current-turn business tool results in the same Agent context. Web snippets remain bounded snippet evidence rather than page bodies or real-world rank. Relationship, ownership, customer, contract, GPU, valuation, and negative-existence claims must be based on current evidence selected in this loop. If evidence does not establish relationship strength, use a neutral bounded description; do not upgrade it to “core”, “largest/major”, “high dependence”, “locked”, “exclusive”, “priority”, “multiple binding”, or equivalent language.
- The typed first line for a Web security, market, or sector answer uses the current Session Beijing query time and states only that the body will disclose concrete quote timing and evidence gaps. Prefer `hone_quote_time.beijing` for quote-source time in the Agent body. `market_date_new_york` and a generic `new_york` date/time establish only a date/timezone context; they do not prove “New York Stock Exchange”, “NYSE”, “close”, or a regular-session close. State an exchange only from structured `exchange` / `exchangeShortName` evidence.
- Empty output, provider/transport error, deadline, malformed lifecycle, forced-final tool emission, or iteration exhaustion remains a genuine failed run. Such failures may not synthesize answer authority or investment-completeness copy. If the typed prefix was actually ACKed, failure appends and persists the fixed explicit disclosure `本轮研究未能完成，暂未形成可供参考的标的结论。` and closes as one partial terminal without an error-card flash. Tool failures stay in the current trace so the Agent can disclose a bounded gap when the provider lifecycle still allows a natural answer.
- Caller-supplied typed scheduled/heartbeat contracts retain their separate deterministic preparation, validation, fallback, and fail-closed behavior. This decision changes open-ended Interactive finance only.

## Consequences

- The Agent sees the complete question and the evidence it requested without a lossy protocol handoff. A tool name/opaque-id mismatch can no longer turn healthy sources into an empty terminal context.
- Removing finish, correction, and terminal rounds eliminates three latency-heavy model phases from the normal path. The later refinement also prevents model-created candidate/route fan-out from reopening unbounded rounds. On Web, an ACKed typed first line can become visible before model latency while the same Agent performs only bounded read-only research; the final tail is published once. Self-contained explicit-security requests also avoid a synchronous compaction/summary read unless context overflow actually requires it.
- The service continues to own structural safety: actor-bound tools, route/evidence telemetry, tool-choice guidance, deadlines, tool-effect replay safety, provider finish/DONE validation, output sanitization, one public terminal, and exact visible/persisted bytes. It does not own open-ended financial prose approval.
- No durable-data migration is required. The mode switch, ledger, and prompts are request-local.
- A safe rollback may keep the whole natural body deferred, but it must not reactivate `finish_research`, locator correction, tool-free terminal generation, semantic review, fixed refusal, or browser request replay.

## Verification / Adoption

- Focused Agent tests must prove that production finance exposes only real registered business tools, uses `Required` before the structural floor and `Auto` afterward, accepts one natural DirectFinal, makes no finish/correction/terminal/audit model call, counts Web-only batches after a T0 ACK, and switches the same Agent to `tools=[]` after at most three finance tool batches or an intrinsic call ceiling.
- Entity regressions must cover lowercase and mixed-case exact tickers, CRWV over embedded-name CWY, independent CRWV/NVIDIA routes, Ford/FORD, Apple/Appleseed, legal names/share classes, crypto, broad-market routes, no-coverage attempts, and same-route quote/profile evidence without any service-side query splitter.
- Content regressions must use deliberately different Session, quote, and New York market-date values. They must prove that only the Session value supplies `数据时间`, `hone_quote_time.beijing` supplies quote-source time, a New York date never becomes “纽交所收盘价”, exchange comes only from exchange fields, and unsupported relationship-strength language is absent.
- AgentSession/channel regressions must prove that Web, Feishu, Discord, and Telegram enable the same finance-loop mode independently of Web prefix policy; ordinary non-finance, unsupported/missing-target DataFetch, malformed identity declarations, and budget-rejected DataFetch calls never ACK; exact-seed and deferred finance ACK only at their respective safe boundaries; explicit-route searches with missing/invalid call-scoped `identity_match` and the seventh first-batch route are rejected before registry/network access without ledger pollution; post-ACK unknown aliases, unregistered tools, invalid arguments, and non-read-only calls fail before frame/observer/registry execution; one completed body is emitted once; prefix plus final tail or failure suffix is byte-identical to persistence; no fixed completeness refusal or post-answer rewrite runs; and the unique terminal closes the stream without reset or second output. Provider requests may preserve the bounded prior-user wording needed for a follow-up, but must exclude historical assistant/tool claims such as an old `15 USD` quote. A lowercase exact-symbol query must resolve its canonical uppercase provider symbol, and a model-invented retired tool name must be returned to the Agent as an ordinary unknown tool without emitting a public tool-start/tool-failure status.
- Production adoption requires an exact-commit build, zero-active-chat graceful restart, PG/object-storage/API/channel/process health checks, and a fresh direct replay of `大A有没有类似CRWV、Nebius这样的数据中心的标的` plus proportional relationship/non-finance checks. Record model-call count, tool count/duration, admitted routes, first-visible time, final time, one-output/history equality, and zero remaining active chats. The exact incident query must stay within four model calls, 24 tools, and six routes, with first visible text under 60 seconds and a target below five seconds for its deterministic-seed path; correctness and one-output behavior remain hard requirements.
- The first-visible latency canary must separately record first provider delta, first committed runner prefix, first successfully queued Web `assistant_delta`, later answer-delta progression, and `run_finished`. An observer callback, rejected SSE send, or persistence record that did not complete successfully is not evidence of delivered text.
- The umbrella ticker plan remains `in_progress` and is not archived because the separately tracked scheduler task-prose/entity P2 (`800G` / `NAND` / `AST` / `SEC`) is still open.

## Risks

- A provider can still answer before gathering semantically sufficient evidence after the floor reaches `Auto`, or reach a hard budget with incomplete coverage. Address that through same-Agent prompt/tool guidance, explicit gaps, and behavioral canaries; do not restore a service-side answer veto or second synthesis.
- Tool calls selected in one model batch still execute serially. The failed `b06de76a` canary spent `45.271s` across 105 calls, but the dominant defect was the unbounded count rather than individual tool latency. The intrinsic caps are the current repair; add separately reviewed bounded concurrency only if a capped canary shows tool time is again dominant.
- Test-only legacy finish/terminal code can drift back into production if compile gates or runner wiring change. The finance CI contract must verify the legacy schema, locator, terminal methods, and canonical-prefix stream state are individually `cfg(test)`, while keeping the production mode, cross-channel request flag, and absence of the retired schema from runner wiring explicit.
- Early visibility is limited to the exact typed service prefix or a validated canonical header from the eligible final. Never broaden it to market conclusions, trailing whitespace, Markdown preambles, incomplete headers, prompt markers, model-authored tool drafts, or an unseen prefix recorded before emitter success.
- The scheduler task-prose/entity P2 has a different typed-origin boundary and remains outside this Interactive decision.
