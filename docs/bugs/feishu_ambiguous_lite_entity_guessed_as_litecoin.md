# Bug: Feishu 直聊把歧义股票简称 `lite` 直接猜成 Litecoin，未先澄清实体

- **发现时间**: 2026-04-17 08:02 CST
- **Bug Type**: Business Error
- **严重等级**: P3
- **状态**: New
- **证据来源**:
  - `data/sessions.sqlite3` -> `sessions` / `session_messages`
  - 真实会话：`session_id=Actor_feishu__direct__ou_5fe09f5f16b20c06ee5962d1b6ca7a4cda`
  - `2026-04-17T07:48:17.867414+08:00` 用户发送：`分析目前lite价值`
  - `2026-04-17T07:49:21.569078+08:00` assistant 直接返回 `Litecoin（LTC）当前报价56.36美元...`
  - `2026-04-17T07:49:48.127622+08:00` 用户纠正：`不是，是LITE LUMENTUM`
  - `2026-04-17T07:51:00.689060+08:00` assistant 才改答 `Lumentum（LITE）`

## 端到端链路

1. 用户在 Feishu 直聊里先询问阿里巴巴估值，随后紧接着发送 `分析目前lite价值`。
2. 该输入同时可能指向加密资产 `Litecoin/LTC`，也可能指向美股光模块公司 `Lumentum/LITE`，属于明显歧义实体。
3. 系统没有先做实体澄清，而是直接把 `lite` 解释成 `Litecoin` 并输出完整分析。
4. 用户被迫追加一轮澄清 `不是，是LITE LUMENTUM`，系统第二次才输出目标股票分析。

## 期望效果

- 当用户输入 `lite` 这类高歧义简称时，系统应先做实体校验或一句话澄清，而不是直接猜测并展开长篇分析。
- 在同一会话已经连续讨论股票/公司估值的前提下，若仍无法唯一确定实体，也应优先提示候选项而不是跳到加密资产。

## 当前实现效果

- 系统在没有澄清的情况下直接输出了 `Litecoin（LTC）` 的完整分析。
- 用户必须显式纠正为 `LITE LUMENTUM` 后，系统才回到正确实体并重新生成答案。
- 这说明当前链路在歧义 ticker / 简称识别上仍存在“直接猜实体”的问题，没有执行“模糊实体先校验”的约束。

## 用户影响

- 这是质量类缺陷，不影响消息投递、系统稳定性或数据安全，因此定级为 `P3`。
- 但它会导致用户首轮答案答非所问，浪费一个来回，且在投资分析场景下会明显降低可信度。
- 如果用户没有继续纠正，错误实体会直接把后续分析方向带偏。

## 根因判断

- 实体解析链路对高歧义简称缺少澄清门槛，模型在可勉强映射时倾向直接猜测。
- 当前会话上下文虽然连续讨论股票估值，但系统没有把“近期对话域仍是股票”作为约束优先级，也没有在 `LITE/LTC` 这种容易混淆的场景下触发确认。
- 该问题更像回答策略与实体解析约束缺失，而不是底层数据或渠道故障。

## 下一步建议

- 在 ticker / 简称解析前增加歧义词校验：若同一输入可映射到多个高概率实体，先返回一句澄清。
- 对股票分析场景补会话域偏置：上一轮仍在讨论公司/股票时，优先给出股票候选而不是直接切到加密资产。
- 后续巡检继续关注是否还有类似 `lite`、`rmbus`、简称/外号类输入被直接猜错的会话样本，再判断是否需要升级为通用实体解析缺陷。
