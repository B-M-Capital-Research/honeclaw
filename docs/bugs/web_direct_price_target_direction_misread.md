## 端到端链路

1. Web direct 用户询问 CRWV 短期到 `150` 美元的概率。
2. assistant 读取或保留了当前价、52 周高点等上下文。
3. 答案生成时错误重写用户意图，把“到 150”解释为“继续腰斩到 36”。
4. 用户必须二次澄清后，assistant 才按正确的上行目标回答。

## 期望效果

- 当用户在个股上下文中问“到 150 / 到 $150 / 回 150”且当前价明显低于该目标时，应默认理解为上涨到目标价。
- 如果模型认为目标方向存在歧义，应先用最自然金融语义回答，再附带一句澄清，而不是把主答案建立在反向假设上。
- 对当前价、52 周高点和目标价之间的关系，应做基本 sanity check，避免把显然的上行目标解释成下行幅度。

## 当前实现效果

- assistant 已知道 CRWV 当前约 `$73`、52 周高点约 `$153`，仍把“到150”解释成“继续跌到 `$36`”。
- 第一轮答复没有完成用户问题，且把风险判断方向完全反过来。
- 第二轮在用户澄清后能够给出可用回答，说明投递、会话收口和基础行情链路没有整体故障。

## 用户影响

- 用户需要额外追问才能拿到真正答案，金融判断方向被首答带偏。
- 该问题不影响 Web direct 主功能链路：消息正常落库、assistant 正常收口，后续同会话可纠正；未见错投、数据破坏、系统级未回复、内部路径或 raw tool 外泄。
- 因此这是质量性缺陷；由于不影响功能链路，定级为 `P3`。

## 根因判断

- 初步判断是金融问答的数值目标语义解析缺少方向 sanity check：模型把“150”错误关联到“52 周高点跌幅 / 腰斩”叙事，而不是用当前价与目标价的大小关系校验用户意图。
- 该问题与实体识别、工具链可用性或出站投递无直接关系，更接近 answer contract 对价格目标方向的防错不足。

## 下一步建议

- 为直接金融问答增加价格目标方向约束：当目标价高于当前价时，默认按上行目标；当目标价低于当前价时，默认按下行风险。
- 增加回归样本：`crwv短期到150的概率大不大` 在当前价约 `$73` 时应回答到 `$150` 的上行概率，而不是反向腰斩。
- 如需澄清，应采用“我先按你说的上涨到 $150 回答；若你其实指跌到某价再告诉我”的顺序。
