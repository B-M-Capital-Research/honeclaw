## Summary

Published HONE v0.14.1 with a viewport-safe PC finance-calendar dialog and a fixed persistent WebKit data store for the standalone macOS user app. The GitHub Release contains a verified Universal DMG plus refreshed iOS Simulator/Xcode and CLI assets.

## What Changed

- Moved the finance-calendar modal out of the composer stacking context and into a root Portal; shorter desktop/App windows now scroll inside the modal rather than clipping its lower half.
- Added stable WebKit data-store UUID `484f4e45-5c14-4a71-9b83-636f6d686f6e` for macOS 14+; macOS 12/13 retain WebKit's default persistent fallback.
- Made the iOS release contract read the workspace version instead of hard-coding the obsolete `0.13.0` marketing version.
- Aligned every release version location at `0.14.1` and published bilingual release notes.

## Verification

- Web: typecheck, 229 tests, production build, and `1000x620` browser verification passed. The modal remained within `16..604px` of a `620px` viewport and its send button became fully visible after internal scrolling.
- macOS source: 4 user-app tests and app-specific check passed.
- Local App/DMG: version `0.14.1`, bundle `com.hone.chat`, Universal `arm64/x86_64`, strict ad-hoc signature, valid DMG checksum, and minimal four-file bundle contract passed.
- Persistence smoke: two launches loaded `https://hone-claw.com/chat` and reused exactly one named WebKit store with a dedicated Cookies directory.
- Remote release: workflow `29189572109` succeeded across Apple clients, Linux/macOS CLI assets, checksums, and Homebrew publication.
- Downloaded `HONE-macOS-v0.14.1-universal.dmg` matched `HONE-Apple-SHASUMS256.txt` (`ed7b43861cd8a16a744554133473e6af78ae4d674b15face8cad4b35f7082920`), passed `hdiutil verify`, and contained a Universal, strictly valid `0.14.1` App.

## Risks / Follow-ups

- Existing macOS 14+ users may need one login when first moving from the old unnamed default store to the fixed store. Do not change `HONE_WEBKIT_DATA_STORE_ID` or `com.hone.chat` in routine releases.
- The DMG is ad-hoc signed and not notarized; another Mac may still show a Gatekeeper warning.

## Next Entry Point

Use the GitHub `v0.14.1` release for downloads and `docs/runbooks/public-user-macos-app.md` for future builds. Any login persistence regression should first verify the named data-store UUID and the server's 30-day session cookie before changing the shell architecture.
