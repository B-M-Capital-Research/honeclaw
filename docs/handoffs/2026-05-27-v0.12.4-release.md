## Summary

Prepared the `v0.12.4` formal release from `v0.12.3..HEAD`. The release packages the first Cloud PG / OSS runtime config slice, scheduler commodity guard false-positive fix, Feishu/external error diagnostics, guarded live smoke wrappers, public Web copy alignment, and the existing post-`v0.12.3` documentation updates.

The release task did not open a new active plan because it was a single-session publish execution. The active Cloud PG / OSS migration remains `in_progress`; this release only ships its first runtime config / OSS public-upload slice.

## What Changed

- Bumped the workspace and desktop app version to `0.12.4`.
- Added bilingual user-facing release notes at `docs/releases/v0.12.4.md`.
- Refreshed `resources/architecture.svg` and the companion `resources/architecture.html` for the new release card, recent milestones, current workspace member count, and cloud runtime / local fallback storage wording.
- Hardened the `deep_research` network-failure unit test so it creates a deterministic local connection failure instead of depending on a fixed machine port.
- Hardened the Tauri sidecar wrapper regression so fake-bun runs use a temporary HOME and cannot be shadowed by a real `$HOME/.bun/bin/bun`.

## Risks / Follow-ups

- Cloud PG / OSS Runtime Migration is not complete. Sessions, quota, audit, portfolio, cron, notification prefs, generated artifacts, logs, and some runtime state still use local fallback storage.
- `cloud.strict_no_local_storage=true` should remain off until PG-backed repositories and OSS-backed artifact ownership are implemented for the remaining local dependencies.
- The new manual live smoke wrappers under `tests/regression/manual/` still require real external credentials and network access; they are intentionally not CI gates.
- The GitHub release workflow only starts after the annotated `v0.12.4` tag is pushed. Final remote workflow success must be checked in GitHub Actions after tag push.
