## Risks / Follow-ups

- Remote desktop backend health is not green yet. `https://hone-claw.com/api/meta` does not return compatible `desktop-v1` JSON from this machine; with the system proxy made explicit, the route returns the public web HTML shell, which the desktop log records as JSON decode failures.
- The public site deployment docs describe `hone-claw.com` as Cloudflare Pages plus `/api/public/*` Worker routing. That public surface is not the same as the desktop admin API surface.
- To make remote desktop mode healthy, expose a desktop admin backend endpoint whose `/api/meta` returns `api_version=desktop-v1`, or update local `.env` and `backend.json` to the correct admin backend URL.
- If cloud admin auth is enabled, fill `HONE_DESKTOP_REMOTE_BEARER_TOKEN` locally before relaunch; do not commit the token.

## Next Entry Point

Start with `data/runtime/logs/desktop.log` and look for either:

- `remote backend connected: https://hone-claw.com`, which means the current URL became compatible, or
- persistent `remote backend probe failed url=https://hone-claw.com`, which means the cloud admin URL/routing still needs to be fixed.
