## Summary

Published HONE v0.14.0 with refreshed macOS and iOS remote user clients. The release includes a Universal macOS DMG, an iOS Simulator App, the complete Xcode project, bilingual notes, and updated architecture documentation.

## What Changed

- Aligned all release version locations at 0.14.0.
- Added an explicit Tauri ad-hoc signing identity so the macOS app bundle, not only its linker output, passes strict code-sign verification with hardened runtime.
- Fixed Apple checksum generation to emit portable basenames and self-verify before upload; the corrected checksum asset was also uploaded to the current release.

## Verification

- Release workflow `29181306840`: success across Apple clients, Linux/macOS CLI artifacts, checksums, and Homebrew publication.
- macOS release DMG: `0.14.0`, `com.hone.chat`, Universal `arm64/x86_64`, strict bundle signature verification passed, no ACP/MCP/config/user-data payload.
- iOS Simulator release: `0.14.0`, `com.hone.chat.ios`, Simulator `arm64/x86_64`; Xcode archive points to release commit `60ef12c8`.
- Apple SHA-256 verification passed for all three downloaded assets.

## Risks / Follow-ups

- The repository has no Apple Developer signing/notarization credentials. The DMG is ad-hoc signed and the iOS package is Simulator/Xcode only, not an IPA or TestFlight build.
- GitHub emitted a non-blocking Node 20 deprecation warning for `mozilla-actions/sccache-action@v0.0.9`; update that action separately before GitHub removes compatibility.
