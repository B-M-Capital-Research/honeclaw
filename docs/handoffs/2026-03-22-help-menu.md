# 帮助文档菜单与 Markdown 渲染 (Handoff 2026-03-22)

## 变更说明
在 `packages/app/src/pages/help.tsx` 以及 Tauri 后端中新增了桌面端的帮助文档功能。
主要满足了从项目根目录 `help/` 文件夹下自动枚举 Markdown 文档，并在帮助页面提供选择及渲染预览的需求。

### 新增/修改模块
- 新建了 `help/README.md` 与 `help/启动文档.md` 以提供应用指引。
- 在 `bins/hone-desktop/src/main.rs` 新增了 `get_help_docs` 与 `read_help_doc` Tauri CLI 端点，用以进行文件 I/O。
- 在 `packages/app/src/lib/backend.ts` 暴露了相关的读取 API。
- 使用 `SolidJS`，结合 `marked`、`dompurify` 与 `highlight.js` 重新实现了 `packages/app/src/pages/help.tsx`。

## 验证
此次改动通过以下流程验证无误：
1. `cargo check -p hone-desktop`
2. `bun run typecheck` (在 `packages/app` 目录下执行)

## 后续建议
后续文档的更新可直接提交到 `help/` 目录下，并会在重启帮助菜单后自动进行更新。若涉及更为复杂的文档组织结构（如建立子目录），需要在 `get_help_docs` 指令内迭代支持对子目录的树状抓取，目前仅对一级子目录 `.md` 进行单层平铺检索。
