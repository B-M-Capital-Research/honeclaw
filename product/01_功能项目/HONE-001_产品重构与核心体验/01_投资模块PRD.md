# HONE-001 投资模块 PRD

状态：`draft`
模块负责人：待定
依赖：账号、行情/财务数据、公司主数据、Agent、跟踪
更新时间：2026-07-12

## 1. 需求背景

用户目前需要在行情软件查看数据、在其他地方维护持仓、在内容平台阅读老王观点，再把信息复制给 AI。数据与判断彼此割裂，无法围绕一家公司形成连续研究。

投资模块要成为登录后的默认首页：让用户快速知道自己的组合与关注公司当前状态，并从数据自然进入洞察、Agent 和跟踪。

## 2. 解决的问题

- 在一个入口查看持仓、自选和关键市场状态。
- 查看公司行情、K 线、财务和估值，但不发生证券交易。
- 将公司数据与老王洞察、用户投资主线、公司画像和跟踪任务连接。
- 让 Agent 在用户授权下理解当前公司和组合上下文。

## 3. 用户与范围

### 核心用户

- 已有持仓或自选公司的长期科技投资者。
- 需要持续理解公司变化而不是高频下单的用户。

### 本期范围

- P0：投资首页、持仓、自选、公司搜索与详情、基础行情/K线/财务估值、跨模块入口。
- P1：市场总览、组合分析增强、导入导出、更多证券类型体验。
- P2：高级归因、情景分析、券商只读同步。

### 不做

- 开户、资金、买卖、订单和清算。
- 以分钟级盘口和交易工具服务日内交易。
- 券商级税费、分红再投资和全部公司行动的精确会计核算。

## 4. 信息架构

```text
投资
├── 首页
├── 持仓
├── 自选
├── 市场
└── 公司详情
    ├── 概览
    ├── 行情/K线
    ├── 财务/估值
    ├── 洞察
    ├── 投资主线/画像
    └── 跟踪
```

移动端使用页内切换或二级页面；Web/macOS 可使用列表+公司详情多栏布局。

#### 首页排序逻辑

1. 用户持仓相关内容优先于自选。
2. 高严重度事件优先于普通涨跌。
3. 同级内容按事件时间倒序；未来事件按距现在最近排序。
4. 首页只展示摘要，完整内容进入公司、洞察或跟踪详情。

#### 收益计算口径

- 单标的市值 = 有效数量 × 最新可用价格。
- 未实现盈亏 = 市值 − 持仓成本；空头或特殊证券按其方向计算。
- 今日盈亏优先使用前收价；前收缺失时不展示伪造结果。
- 多币种组合按用户基础币种换算，并展示所用汇率及更新时间。
- 第一阶段由用户维护平均成本，不自动复原每笔交易和税费。

## 6. 关键流程

### 6.1 添加持仓

```text
搜索并确认证券
→ 填写数量/成本/币种
→ 系统校验
→ 用户确认
→ 保存持仓
→ 刷新组合统计
→ 可选建立主线或跟踪
```

校验失败时保留输入；重复证券默认更新现有持仓，不创建第二条，除非未来启用多账户分组。

### 6.2 从公司进入研究

```text
打开公司详情
→ 查看行情和财务
→ 查看关联洞察/主线
→ 询问 Agent
→ 用户确认沉淀主线或任务
```

### 6.3 删除持仓

```text
点击删除
→ 展示受影响数据说明
→ 选择删除后保留自选/完全移除
→ 二次确认
→ 更新组合统计
```

历史会话、画像、洞察和已完成任务不随持仓删除。

## 7. 边界与异常

| 场景 | 处理要求 |
|---|---|
| 行情源不可用 | 展示最近成功值、更新时间和过期标记；收益同步标记不可确认 |
| 公司停牌/退市 | 明确状态，保留历史与用户记录，不从组合静默删除 |
| 拆股/合并/代码变更 | 标记待调整；自动调整必须保留来源和变更记录 |
| 多币种 | 使用用户基础币种汇总，并允许查看原币数据 |
| 成本或数量为空 | 不计算对应盈亏；引导补全，不用 0 代替未知 |
| 负数量/空头 | 数据层允许兼容；界面明确方向，避免与普通持仓混算 |
| 同 ticker 多交易所 | 必须由公司/证券唯一 ID 区分 |
| 用户无会员权限 | 基础投资数据按权限展示；受限洞察只展示摘要 |
| 多端同时编辑 | 冲突时提示重新确认，不静默覆盖成本和数量 |
| 首次用户 | 先引导添加自选或持仓，允许跳过并浏览市场 |

## 8. 验收标准

- [ ] 用户能从零创建自选和持仓，并在首页看到正确聚合。
- [ ] 持仓收益在单币种、缺价格和多币种场景下符合定义。
- [ ] 公司搜索能区分同名、不同交易所和不同证券类型。
- [ ] 公司详情至少串联行情、财务、洞察、Agent 和跟踪。
- [ ] 行情过期、停牌、数据缺失和接口失败均有明确状态。
- [ ] 删除持仓不会误删画像、历史研究和任务结果。
- [ ] 公开分享不包含用户私有投资数据。
- [ ] 现有持仓和期权数据迁移后不丢失。

## 9. 研发拆分建议

5. 洞察、Agent、跟踪跨模块接入。
6. 异常状态、埋点和跨端验收。
