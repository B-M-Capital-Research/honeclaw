# HONE-001 Agent 模块 PRD

状态：`draft`
模块负责人：待定
依赖：LLM/Skills/Tools、会话、用户投资上下文、文件、投资、洞察、跟踪
更新时间：2026-07-12

## 1. 需求背景

当前 HONE 的主要用户能力集中在聊天，但未来 Agent 不能只是一个孤立对话框。它需要理解用户所在的公司、洞察、持仓或事件上下文，并把研究结论转成投资主线、公司画像和持续任务。

Agent 是五入口中间最突出的入口，也是贯穿其他模块的智能能力层。

## 2. 解决的问题

- 用户不再反复解释自己的持仓、关注公司和当前页面背景。
- 将行情、洞察、PDF、图片和用户历史放进同一研究过程。
- 让分析结果可以被保存、引用和继续跟踪，而不是停留在一次聊天中。
- 对有副作用的操作建立明确确认、权限和结果反馈。

## 3. 用户与范围

### 用户

- 需要公司研究、材料解读、持仓分析和复盘的个人投资者。
- 需要创建提醒、监控和定时研究的用户。

### 本期范围

- P0：会话、流式回答、附件、上下文、工具结果、操作确认、跨模块入口、结果沉淀。
- P1：会话搜索/整理、结构化研究模板、长任务进度、多结果对比。
- P2：多 Agent 协作、用户自定义 Agent、复杂工作流编排。

### 不做

- 证券交易和自动下单。
- 在未确认时修改持仓、删除数据或创建长期任务。
- 向普通用户暴露宿主机文件系统、管理员工具或内部运行细节。

## 4. 信息架构

```text
Agent
├── 新对话/最近对话
├── 对话区
│   ├── 上下文
│   ├── 消息与工具过程
│   ├── 结果与操作卡
│   └── 输入/附件
├── 历史会话
└── 已沉淀结果（通过公司/跟踪回看）
```

移动端以单会话为主；Web/macOS 可展示会话列表、对话和上下文侧栏。

## 6. 关键流程

### 6.1 带公司上下文研究

```text
公司详情点击“询问 Agent”
→ 打开/新建会话并显示公司上下文
→ 用户提问
→ Agent 获取授权数据
→ 返回带来源与时间的回答
→ 用户选择保存主线/结果或创建跟踪
```

### 6.2 创建跟踪任务

```text
用户描述跟踪目标
→ Agent 澄清对象、条件和时间
→ 展示任务预览
→ 用户确认
→ 创建任务
→ 返回任务卡
→ 在“跟踪”中可见
```

### 6.3 修改投资主线

```text
Agent 给出新判断
→ 建议更新主线
→ 展示原内容与拟修改内容
→ 用户确认
→ 保存并记录更新时间/来源
```

## 7. 安全与权限

- 普通用户只能调用面向用户开放的 Skills/Tools；管理员能力继续隔离。
- 深度研究如仍为管理员能力，普通用户界面不得承诺可用。
- 非管理员执行必须在 actor 范围内，不能访问其他用户目录、会话和持仓。
- 外部模型只接收完成当前任务所需的最小上下文。
- 对话、附件和精确持仓不进入通用分析埋点。
- 涉及投资结论时区分事实、推断和待验证内容，不伪造最新数据。

## 8. 边界与异常

| 场景 | 处理要求 |
|---|---|
| 用户快速重复发送 | 防重复提交；上一轮进行中时明确排队、阻止或允许并行策略 |
| 页面关闭/网络中断 | 服务端可继续的任务保留状态；重开后恢复，不能重复执行副作用 |
| 流式连接断开 | 尝试恢复；无法恢复时保留已生成内容并提供重试 |
| 工具成功但回答失败 | 展示操作真实结果，不能因回答失败回滚已成功副作用 |
| 回答成功但保存失败 | 回答保留，保存操作可单独重试 |
| 上下文被删除/权限到期 | 标记不可用并要求用户移除或重新授权 |
| 行情过期 | 在回答和数据卡中明确时间与过期状态 |
| 附件含恶意指令 | 作为材料内容处理，不提升权限或改变系统规则 |
| 额度耗尽 | 展示剩余额度/恢复时间（若可得），不吞掉用户输入 |
| 会话过长 | 进行可审计摘要；关键公司、任务和用户确认不能因压缩丢失 |
| 模型切换 | 保持会话可读；说明能力差异，不静默改变已确认任务 |

## 9. 验收标准

- [ ] 用户能从五个入口进入 Agent，并看到正确上下文。
- [ ] 文本、图片和文件输入均具备完整上传、失败和移除状态。
- [ ] 流式回答可停止、恢复失败并标明工具与数据状态。
- [ ] 所有副作用操作先预览确认，重复确认不重复执行。
- [ ] 回答可沉淀为主线、公司研究结果或跟踪任务。
- [ ] 私有持仓、附件和受限洞察不会越权或进入公开分享。
- [ ] 网络中断、模型失败、额度不足和上下文失效均可恢复或明确结束。
- [ ] 现有会话、上传、Skills 和定时任务数据可兼容迁移。

## 10. 研发拆分建议

1. 会话与流式状态重构。
2. 上下文协议和跨模块入口。
3. 附件与多模态结果。
4. 副作用确认与幂等执行。
5. 研究结果/主线沉淀。
6. 跟踪任务闭环、安全和故障恢复。
