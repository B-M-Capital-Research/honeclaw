## 2. 产品定位

HONE 是一个面向长期科技投资者的个人投研工作台，把投资数据、专业洞察、AI Agent 与持续跟踪连接起来，帮助用户长期理解公司、维护投资主线，并及时关注真正影响判断的变化。

HONE 不做证券交易，也不以通用财经新闻聚合为核心。它要帮助用户回答：

> 发生了什么、应该如何理解、与我的投资有什么关系、接下来应该关注什么？

## 3. 目标用户

### 核心用户

- 长期关注科技、AI、半导体、软件、云计算等公司的个人投资者。
- 已有持仓或自选，需要持续跟踪公司，而不是只在交易时看行情。
- 认可老王研究方法，希望持续阅读财报解读、公司判断和短观点。
- 希望 AI 记住自己的持仓、关注公司、投资主线和研究历史。
- 需要 AI 辅助研究和监控，但不希望 AI 替自己做交易决策。

巴芒现有会员、老王内容用户以及未来 SnowDrift 海外用户，是最自然的首批用户。

### 扩展用户

- 将行情、文章、笔记、提醒和 AI 对话分散在多个工具中的个人投资者。
- 暂时没有持仓，但希望通过专业洞察逐步建立公司认知的内容用户。

### 当前不优先服务

- 以高频交易、盘口和下单速度为核心的交易者。
- 希望获得明确买卖指令或收益承诺的用户。
- 只需要通用新闻聚合、与个人研究无关的资讯消费者。

## 4. 用户核心问题

| 问题 | 当前表现 |
|---|---|
| 信息分散 | 行情、持仓、老王观点、资料、提醒和 AI 对话分布在不同工具中 |
| 信息缺少判断 | 普通产品告诉用户发生了什么，却很少解释是否影响长期逻辑 |
| AI 缺少连续性 | 用户每次都要重述持仓和背景，研究结果也难以沉淀 |
| 研究缺少时间管理 | 财报、宏观事件、公司活动和定时任务没有统一跟踪中心 |
| 历史难以复用 | 老王判断、公司画像、投资主线和 Agent 结果没有围绕公司连接起来 |

## 5. 一级产品结构

```text
投资　　洞察　　Agent　　跟踪　　我的
```

- 投资作为默认首页。
- Agent 位于移动端中间，是最突出、最主要的操作入口。
- 五个入口在业务职责上保持一致，但导航形态按端区分：移动端使用五入口底栏；桌面端先展示投资、洞察、跟踪工作台，再展示常驻 AI 研究能力，并将“我的”与快捷设置固定在左下角账户入口；右上角只保留全局搜索。

| 入口 | 主要回答的问题 | 产品角色 |
|---|---|---|
| 投资 | 我的组合和市场现在是什么状态？ | 投资状态与公司研究入口 |
| 洞察 | 老王如何理解这些公司和事件？ | 专业内容与历史判断沉淀 |
| Agent | 帮我理解、研究或完成一件事 | 智能研究与任务发动机 |
| 跟踪 | 接下来有什么值得持续关注？ | 时间、事件、任务和主动服务中心 |
| 我的 | 我的账户、会员和偏好是什么？ | 个人身份与产品配置 |

## 6. 主要功能

### 6.1 投资

**解决的问题**

用户需要像行情软件一样查看公司和组合数据，但不需要交易；数据还需要与洞察、投资主线和研究历史连接。

**面向用户**

拥有持仓或自选、需要长期研究公司的个人投资者。

**核心功能**

- 投资首页：组合市值、收益、仓位、自选变化和今日重点。
- 持仓与自选：数量、成本、市值、收益、仓位和备注。
- 市场：指数、行业、涨跌、财报日历和宏观事件。
- 公司详情：行情、K 线、财务、估值、持仓、投资主线、公司画像和相关洞察。
- 针对公司、指标或事件建立跟踪。

**边界**：不提供开户、入金、下单和清算，不把短期涨跌直接转成交易指令。

### 6.2 洞察

**解决的问题**

老王的财报解读、公司判断和短观点分散，难以按公司和主题回看；普通资讯又缺少稳定的研究视角。

**面向用户**

巴芒会员、老王内容用户，以及希望获得科技公司专业解读的投资者。

**核心功能**

- 短观点：类似微博或推文，可包含图片和图表。
- 深度文章：财报解读、公司研究、行业分析和长期框架。
- 按公司、ticker、行业、主题和内容类型筛选。
- 历史回看、收藏、已读状态和内容关联。
- 从洞察进入公司详情，或基于当前内容询问 Agent。

**边界**：不做大而全的新闻平台，不把未经确认的 AI 内容伪装成老王观点。

### 6.3 Agent

**解决的问题**

通用 AI 不理解用户的投资背景，研究结果也停留在聊天里，无法形成画像、主线、任务和提醒。

**面向用户**

需要 AI 辅助公司研究、材料理解、持仓分析和持续跟踪的用户。

**核心功能**

- 开放式投资研究对话。
- 公司、行业、财报、市场和持仓分析。
- 图片、PDF 和研究材料理解。
- 使用持仓、公司画像和历史洞察作为上下文。
- 更新投资主线，创建监控、提醒和定时研究任务。
- 将研究结果沉淀到公司、持仓或跟踪记录。

**产品形态**：导航名称使用 `Agent`；它既是中间一级入口，也嵌入投资、洞察和跟踪页面，并自动继承当前上下文。

**边界**：不替用户交易，不承诺收益；修改持仓、创建任务等操作需要用户确认。

### 6.4 跟踪

**解决的问题**

用户不知道今天和未来有哪些事件与自己的持仓、自选相关；提醒和任务结果也缺少统一的回看入口。

**面向用户**

有持仓、自选和长期研究对象，需要持续关注变化的用户。

**核心功能**

- 今日：当天需要关注的公司事件、财报、宏观数据、任务和提醒。
- 日历：用周/月视图组织财报、分红、公司活动和研究计划。
- 任务与监控：查看运行中、即将执行、已完成、失败和暂停的任务。
- 历史结果：保存事件摘要、持仓复盘和 Agent 定时报告。
- 优先展示与用户持仓、自选和投资主线相关的内容。

**边界**：`跟踪` 是一级入口，`日历` 是核心视图；不做通用个人日程管理。

### 6.5 我的

**解决的问题**

用户需要统一管理账户、会员、投资偏好、通知和多端设置，同时避免个人中心成为功能杂物间。

**面向用户**

所有已登录用户。

**核心功能**

- 个人资料、登录和设备管理。
- 会员身份与权益。
- 投资偏好和关注方向。
- 通知、摘要和勿扰设置。
- 语言、主题、字号、数据、隐私和安全。
- 帮助、反馈和关于 HONE。

**边界**：Agent 不放入“我的”，任务结果进入“跟踪”而不是藏在设置中。

## 7. 模块如何连接

公司是投资、洞察、Agent 和跟踪共同的连接对象；用户的持仓、自选、投资主线和偏好构成个人投资上下文。

```text
查看持仓或自选
→ 发现行情、财报或公司事件
→ 进入公司详情
→ 阅读老王洞察
→ 询问 Agent
→ 更新投资主线或建立跟踪
→ 后续变化再次提醒用户
```

Agent 创建的持续任务进入“跟踪”；跟踪产生的结果可以继续交给 Agent 分析。研究结果需要沉淀回公司和用户投资上下文，而不是只停留在聊天记录中。

## 8. 多端方向

### 移动端

底部五入口保持：

```text
投资　　洞察　　Agent　　跟踪　　我的
```

Agent 位于中间并突出。左侧偏查看与理解，右侧偏持续管理。

### Web 与 macOS

- 保持相同的信息架构和命名。
- 使用适合大屏的侧边导航和多栏布局。
- Agent 不再以单独导航卡片进入二级研究页；品牌下方先放投资、洞察、跟踪工作台，随后承载新研究、研究记录搜索、最近对话和全部记录。
- 头像、个人中心和快捷设置固定在左下角账户栏；桌面顶栏右侧只保留全局搜索。
- 不提供“返回官网”入口；应用与官网不在当前核心任务链路中。

## 9. 本轮评审事项

- [ ] 核心用户是否准确，是否需要收窄或扩展。
- [ ] 产品定位是否准确表达 HONE 与行情软件、内容产品和通用 AI 的区别。
- [ ] `投资—洞察—Agent—跟踪—我的` 是否可以成为稳定一级架构。
- [ ] 投资是否确定为默认首页，Agent 是否确定为中间突出入口。
- [ ] 跟踪是否能覆盖日历、任务、提醒和历史结果。
- [ ] 是否存在缺失的核心问题，或当前被放得过重的功能。
