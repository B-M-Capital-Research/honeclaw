# HONE-001 洞察模块 PRD

状态：`draft`

更新时间：2026-07-12

## 1. 需求背景

老王的财报解读、公司判断、群内观点和短讨论分散在不同渠道，用户很难按公司和时间回看，也无法从内容自然进入公司数据、个人研究和 Agent 追问。

洞察模块不是通用财经资讯流，而是老王及授权研究者的专业判断库。它同时服务“及时阅读”和“长期检索”。

## 2. 解决的问题

- 用统一帖子结构承载观点、图片和可选深度文章，不要求用户先判断内容类型。
- 让每条内容与公司、行业、主题和时间建立结构化关系。
- 支持历史回看、权限控制、收藏和继续研究。
- 明确区分作者原始观点、编辑加工和 AI 辅助内容。

## 3. 用户与范围

### 用户

- 巴芒会员和老王内容用户。
- 希望学习科技公司研究和长期投资方法的用户。
- 内容编辑与管理员是供给侧角色。

### 本期范围

- P0：统一洞察流、图文帖子、可选深度文章附件、详情、公司/主题关联、权限、基础发布后台。
- P1：收藏/已读、全文搜索、定时发布、多语言、专题合集。
- P2：个性化推荐、协作编辑、复杂内容分发自动化。

### 不做

- 抓取全市场新闻形成大而全资讯平台。
- 本期不开放普通用户发帖和评论；信息架构需要为后续内容社区保留作者、讨论和引用关系。
- 未经老王确认，把 AI 生成判断作为老王署名内容发布。

## 4. 内容模型

### 4.1 统一帖子模型

洞察流不再把“短观点”和“深度文章”作为用户可见分类。每一条内容都是帖子，使用同一套作者区、正文区、媒体区、关联标签和操作区。

| 字段 | 必填 | 说明 |
|---|---|---|
| 正文 | 是 | 一次表达一个主要判断，可在信息流中直接阅读 |
| 作者、发布时间、权限 | 是 | 当前仅老王和授权编辑可发布 |
| 图片 | 否 | 支持单图/多图、替代文本和失败状态 |
| 深度文章附件 | 否 | 帖子可附带文章标题、摘要、阅读时间和详情入口 |
| 公司/主题关联 | 否 | 用于回看、搜索、Agent 上下文和跟踪 |
| 讨论关系 | 预留 | 本期只展示未来社区能力说明，不开放普通用户回复 |

### 4.2 内容状态

```text
草稿 → 待审核 → 已发布 → 已撤回/已归档
```

- 草稿和待审核仅内部可见。
- 已撤回内容对普通用户不返回正文，可保留“内容已撤回”占位。
- 已归档内容仍可通过历史和搜索访问，但默认不进入最新流。

### 4.3 作者与 AI 标识

- `作者` 表示观点责任主体。
- `编辑` 表示整理和发布人员，不改变作者归属。
- AI 仅参与摘要、格式整理或翻译时，需保留人工审核记录。
- AI 独立生成的分析必须显示 AI 标识，不得署名老王。

## 6. 关键流程

### 6.1 阅读并继续研究

```text
打开洞察流
→ 筛选/选择内容
→ 阅读短观点或文章
→ 进入关联公司 或 询问 Agent
→ 继续查看数据、建立自选/跟踪
```

### 6.2 发布内容

```text
创建草稿
→ 选择类型并编辑
→ 关联公司/主题
→ 设置作者与权限
→ 预览
→ 审核/发布
→ 进入信息流并通知有权限用户
```

发布请求重复提交不得产生两份内容；媒体上传失败时不得丢失正文草稿。

## 7. 权限逻辑

| 权限 | 可见范围 |
|---|---|
| 公开 | 未登录可查看全文 |
| 登录可见 | 任意有效登录用户可查看全文 |
| 会员可见 | 有效会员可查看全文；其他用户只看授权摘要 |
| 内部 | 仅授权后台角色可查看 |

- 用户会员到期后，历史收藏仍可见标题和摘要，但正文按当前权限重新鉴权。
- 从 Agent、搜索、公司页和分享链接访问时执行同一权限规则。
- 服务端返回数据时完成正文裁剪，避免受限内容进入客户端缓存。

## 8. 边界与异常

| 场景 | 处理要求 |
|---|---|
| 内容已撤回 | 展示撤回占位和返回入口，不展示缓存正文 |
| 作者/关联公司改名 | 历史内容保留原发布时间与稳定链接，显示最新规范名称 |
| 图片失败 | 正文仍可阅读，展示图片失败状态和替代文本 |
| 外链失效 | 不影响正文；标记链接可能不可用 |
| 用户阅读时权限到期 | 下一次受保护请求重新鉴权，不突然暴露剩余正文 |
| 内容编辑 | 用户分享链接不变；显示已编辑，保留内部版本记录 |
| 同一内容多语言 | 切换到存在且有权限的版本；没有版本时不自动机器翻译伪装原文 |
| 空信息流 | 说明当前筛选无内容并提供清除筛选 |
| 搜索无结果 | 提供公司或主题候选，不虚构内容 |
| Agent 引用受限洞察 | 仅在用户有权限时读取正文，并在回答中标明引用来源 |

## 9. 验收标准

- [ ] 图文帖子和带文章附件的帖子可创建、审核、发布、编辑和撤回。
- [ ] 洞察流不显示内容类型分类，能按公司和主题检索，分页不重复。
- [ ] 四种权限在列表、详情、搜索、分享和 Agent 中一致生效。
- [ ] 每条内容能进入相关公司，并能发起带上下文 Agent 对话。
- [ ] 老王原文、编辑信息和 AI 生成内容有清晰区分。
- [ ] 图片失败、内容撤回、权限变化和空列表均有明确状态。
- [ ] 受限正文不会通过接口、预览、缓存或搜索摘要泄漏。

## 10. 研发拆分建议

1. 内容模型、状态与权限。
2. 内容发布后台与媒体处理。
3. 洞察流、短观点和文章详情。
4. 公司/主题关联与搜索。
5. 收藏/已读、多语言等 P1 能力。
6. Agent 上下文、通知和跨端验收。
