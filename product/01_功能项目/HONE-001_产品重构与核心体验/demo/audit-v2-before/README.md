# HONE Live Demo V2 迭代前审计

视口：`1440 × 1024`
日期：`2026-07-12`

## 1. 投资页

证据：`01-invest.png`

- 健康度：基础结构可用，但右栏明显短于主内容，中下部缺少持续研究信号。
- 主要问题：自定义视图在当前阶段不是必需；右栏只有单条洞察和两个事件，不能支撑“今天要看什么”。
- 迭代方向：增加组合风险、自选快照和本周研究进度，与主表格形成更平衡的纵向节奏。

## 2. 洞察页

证据：`02-insights.png`

- 健康度：长文突出，但整页过于“内容官网”，不像高频洞察流。
- 主要问题：短观点仍被包装成需要进入详情的文章；长短内容的消费动作没有区分。
- 迭代方向：改造为类似社交时间线的混排流。短观点就地完整阅读，长文以帖子中的附加文章卡进入详情。

## 3. Agent 页

证据：`03-agent.png`

- 健康度：三栏工作台架构正确，但默认态空间过多，功能预期过弱。
- 主要问题：左侧导航中 Agent 在未选中时也有额外边框；主区只有三条问题，没有研究模式、线索、最近任务和工具入口。
- 迭代方向：导航状态与其他入口一致；增加研究模式、今日线索、快速工具和最近会话。

## 4. 跟踪页

证据：`04-tracking.png`

- 健康度：事件列表可扫读，但缺少时间管理的整体感。
- 主要问题：默认视图过稀；日历是小型选日器，无法看到一个月的事件分布；新建跟踪只有三个文本框，没有触发、输出和确认逻辑。
- 迭代方向：日历改为整月大视图；默认页增加任务运行态和本周时间轴；创建流程改为三步。

## 5. 我的

证据：`05-me.png`

- 健康度：信息架构基本清楚，但只是设置列表的表层演示。
- 主要问题：数据隐私仅弹出 toast，无内容层；登录头像在左侧栏和顶栏重复。
- 迭代方向：只保留顶栏头像，使其承担账户二级菜单；隐私、设备、偏好等进入可操作的详细弹层。

## 可见性与无障碍风险

- 多处纯图标按钮已有语义化控件，但新增菜单和弹窗后仍需验证可达名称、焦点和对比度。
- 截图只能检查可见层级与布局，不能证明完整的键盘可达性或读屏体验。
