# V4 产品链路审查与实现记录

## 审查范围

- 洞察：统一消息流、图文内容、文章附件、发布与未来社区扩展。
- 跟踪：时间线详情、结构化结果、准备状态、任务管理、日历和历史结果。
- 我的：账户、研究偏好、设备会话、头像快捷设置和订阅支付。

## 用户目标

用户能在不猜测按钮结果的情况下，完成阅读/发布洞察、理解跟踪事件和结果、管理任务与设备、配置高频偏好，并走完整会员升级流程。

## 主要结论

- 原洞察分类把同一作者的内容切成不同产品形态，不利于形成连续内容流；已改成统一帖子 + 可选图片/文章附件。
- 跟踪首页结构可用，但详情、结果、管理任务等核心 CTA 缺失；现已全部补齐可评审状态。
- 我的首页信息完整，但核心设置和商业化入口不够可执行；现已补齐研究偏好、设备安全、快捷设置和订阅支付。
