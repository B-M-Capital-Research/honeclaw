## 视觉判断

### 全局比较

- 保留了参考图的左侧导航、顶部工具区、组合总览、业绩图表、持仓明细、右侧研究栏和 Agent 入口。
- 实现版使用更大的字号与间距，在 `1440 × 1024` 下优先可读性，因此首屏表格展示行数少于视觉源。这是有意识的密度取舍。
- 洞察页吸收编辑化阅读方向；Agent 页吸收上下文、来源和左右研究面板，与投资主屏保持同一视觉系统。

### 聚焦比较

- 组合指标层级清楚，总资产、今日变化、年初至今和风险指标可快速扫读。
- 图表两条线使用线宽和灰度建立主次，不依赖颜色。
- Agent 在桌面导航中与其他入口使用同一状态规则；移动端仍使用居中悬浮黑色主入口。
- 移动组合指标经过迭代后使用两列网格，避免单列导致首屏过长。

6. 第六轮：移除自定义视图，补齐投资右栏的风险、自选和研究进度。
7. 第七轮：把洞察改为社交化混排流，重构 Agent 默认工作区。
8. 第八轮：实现跟踪大日历、三步创建向导，补齐隐私和设备交互。

10. 第十轮：比较资产优先、持仓优先、研究优先三种移动方案，选择持仓优先作为主骨架。
11. 第十一轮：把研究优先方案中的 Agent 信号压缩为单条模块，避免首页演变为内容流或聊天页。
12. 第十二轮：补齐只读连接、对账单、截图、手动录入四级数据入口及三步确认链路。
13. 第十三轮：将公司折线行情升级为 K 线、成交量、成本线和买入点，分别重构桌面和原生移动信息层级。

4. `[P2]` 常驻侧栏需要可用而非静态陈列。已验证输入 `TSMC` 后只保留匹配记录；点击“TSMC 财报问题清单”进入 `conversation=tsmc-earnings` 并展示研究回答；点击“新研究”进入 `new=1` 并恢复默认研究态。
5. 移动端不属于改造范围。`390px` CSS 视口仍显示投资、洞察、Agent、跟踪、我的五入口，桌面研究侧栏不出现，页面水平溢出为 `0`。

### 必检表面

- 间距与布局：桌面侧栏调整为 `248px`，顶栏与主内容同步偏移；Agent 页移除内部 `240px` 轨道后，研究主区获得更多有效宽度。

- 图片与图标：未新增图片资产；加号、搜索、历史与工作台图标均使用 Phosphor，没有占位或自绘图形。
- 文案与内容：使用“新研究”“搜索研究记录”“全部研究记录”等直接动作型文案；删除“返回官网”和重复 Agent 卡片说明。
