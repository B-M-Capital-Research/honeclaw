# HONE-001 投资持仓数据与原生移动体验设计

状态：`direction-review`
负责人：一鸣
更新日期：2026-07-12

## 1. 本轮要解决的问题

这一轮不是继续增加指标，而是要把投资模块从「已经有数据的展示页」升级为「可以长期维护的个人投资账本」。

需要完整回答：

1. 用户怎样把已有持仓带进 HONE？
2. 持仓数据怎样持续同步，同步失败怎样恢复？
3. 数量、成本、交易、分红、外汇与公司行动怎样影响收益？
4. 组合、单股与个人盈亏分别应该用什么图表？
5. 桌面研究桌和原生移动 App 应该如何使用不同的信息密度？

证据目录：`demo/audit-invest-v3-before/`

|---|---|---|---|
| 1. 桌面投资首页 | `01-desktop-invest.png` | 中 | 适合已有数据的用户，但没有持仓导入、同步状态、数据账户与对账入口 |
| 2. 桌面公司详情 | `02-desktop-company.png` | 中下 | 价格用折线表达，缺少 K 线、成交量、成本线、历史交易点与更完整的公司数据 |
| 3. 移动投资首页 | `03-mobile-invest.png` | 下 | 直接把桌面卡片纵向堆叠，首屏只能看到总资产和图表开头，核心持仓不在首屏 |
| 4. 移动公司详情 | `04-mobile-company.png` | 中下 | 操作和 Tab 占用较多垂直空间，行情图占据首屏，持仓盈亏、今日事件和研究信号要继续长距离下滑 |

### 核心判断

- 当前页面假设「用户的持仓数据天然存在」，但实际产品最难的是数据进入、连续同步和对账。
- 移动端不应是响应式网页缩小版，而应使用「首屏状态 + 高密度持仓列表 + 层级化二级页」。
- 公司页的行情、用户持仓和研究上下文应该互相关联：K 线上可以看到用户成本线和历史操作点，但 HONE 不提供交易按钮。

## 3. 富途产品逻辑中值得借鉴的部分

### 已确认事实

- 富途官方帮助中心显示，移动端个股日 K 图支持长按唤起十字线，再进入当日分时数据；桌面端使用双击进入当日分时。[富途官方帮助](https://support.futunn.com/topic43?lang=zh-cn)
- 富途的 K 线支持按历史成交在图上展示买卖点，不同周期可展示成交打点或价格打点。[富途官方帮助](https://support.futunn.com/topic829?from_platform=1&lang=zh-cn)
- 2026-07-09 的富途桌面版记录提到图表指标栏快速切换；2026-06-25 记录提到 K 线周期标签可自定义显隐。[富途官方下载页](https://www.futunn.com/download/windows?lang=zh-CN)

### 对 HONE 的合理借鉴

- 把「价格 + 周期 + 图表 + 指标」放在同一个交互区，不把周期切换放到遥远的设置页。
- 移动端使用长按十字线、点击查看当日详情；桌面端保留鼠标悬停、双击和更多指标。
- 将用户的实际买入、卖出和平均成本叠加到 K 线，用于复盘，而不是诱导用户下单。
- 借鉴移动端的窄屏密度与手势，但不照搬交易、盘口、订单和红绿刺激机制。

## 4. 持仓数据进入 HONE 的优先级

### 推荐的四级输入梯度

| 等级 | 方式 | 用户成本 | 数据连续性 | 建议优先级 |
|---|---|---:|---:|---|
| A | 只读连接券商/投资账户 | 低 | 高 | P1，优先验证覆盖与合规 |
| B | 导入券商对账单 PDF / CSV / XLSX | 中低 | 中 | P0，适合首发 |
| C | 截图或持仓图片识别 | 低 | 低 | P1，用于快速初始导入 |
| D | 手动快速添加 | 高 | 低 | P0，必须作为通用兜底 |

### A. 只读账户连接

用户流程：

```text
选择券商
→ 展示只读范围与隐私说明
→ 在第三方安全连接器中授权
→ 选择需同步的账户
→ 首次同步持仓/余额/历史交易
→ 预览差异
→ 用户确认并建立持续同步
```

设计原则：

- 连接类型默认只读，界面明确说明 HONE 无法下单、转账或查看完整凭证。
- 同步必须展示「最后成功时间 / 数据延迟 / 下次同步 / 是否需要重新授权」。
- 第三方连接会断开或需要重新授权，不能把旧的缓存数据伪装成实时数据。

技术可行性例子：

- Plaid Investments 可返回用户授权的持仓、成本基础与投资交易，但官方文档的默认覆盖主要是美国和加拿大机构，且投资数据通常按日更新。[Plaid Investments](https://plaid.com/docs/investments/)
- SnapTrade 提供券商连接器、只读连接、持仓和交易同步状态，也明确包含重新授权和断开连接的异常场景。[SnapTrade Connections](https://docs.snaptrade.com/docs/connections) [SnapTrade Holdings](https://docs.snaptrade.com/reference/Account%20Information/AccountInformation_getUserHoldings)

> 待验证：中国内地、香港、美国和其他地区券商的实际覆盖、价格、用户授权方式与合规边界，不能在未完成商务/法务调研前承诺「一键连接所有券商」。

### B. 对账单导入

导入不应是「上传后直接覆盖」，而应该是：

```text
上传文件
→ 识别券商/账户/币种/日期
→ 映射交易与持仓字段
→ 展示“新增 / 更新 / 重复 / 无法识别”预览
→ 用户处理冲突
→ 确认写入
→ 生成导入报告与可回滚记录
```

必须处理：重复交易、小数股、多币种、费用、分红、转入转出、拆合股、ticker 更名、无成本数据和未知证券。

### C. 截图识别

- 用户只需上传持仓列表截图，系统识别 ticker、数量、市值和成本。
- 截图只适合建立「当前持仓快照」，不能还原完整交易历史、已实现盈亏和现金流。
- 所有识别结果必须先进入预览和确认，不直接写入。

### D. 手动快速录入

手动路径分两种：

1. **快照模式**：证券 + 数量 + 平均成本，30 秒内完成，先得到当前未实现盈亏。
2. **交易模式**：逐笔录入买入/卖出/分红/费用，用于恢复更准确的已实现盈亏和历史收益。

## 5. 数据模型不能只有「当前数量 + 平均成本」

建议至少分开：

| 对象 | 关键字段 | 用途 |
|---|---|---|
| 数据源 | 券商/导入/截图/手动、最后同步、延迟、状态 | 数据新鲜度和错误恢复 |
| 账户 | 券商、账户类型、币种、地区、是否纳入组合 | 多账户汇总 |
| 证券 | 唯一 ID、ticker、交易所、币种、证券类型 | 避免同名和换码错误 |
| 交易 | 买/卖/分红/费用/转入转出、时间、数量、价格、币种、来源 | 恢复成本和收益历史 |
| 持仓快照 | 时间、数量、成本基础、市值、价格 | 当前组合与对账 |
| 现金流 | 入金、出金、内部转账、外汇换汇 | 组合收益与资金加减的区分 |
| 公司行动 | 分红、拆合股、并购、换码、分拆 | 避免持仓和成本失真 |

## 6. 收益口径与图表

### 图表分工

| 场景 | 推荐图表 | 不应混淆的内容 |
|---|---|---|
| 单只证券行情 | K 线 + 成交量 + 可选 MA/BOLL/MACD | 价格不等于用户收益 |
| 用户单股持仓 | 成本线 + 买卖点 + 未实现盈亏曲线 | 不把公司涨跌幅当作用户的实际收益率 |
| 组合收益 | 时间加权收益率折线 + 基准对比 | 入金/出金不应被计入投资收益 |
| 资产规模 | 总资产折线/面积图 | 需要标注现金流时点 |
| 风险 | 回撤曲线、波动率、集中度 | 不与价格 K 线共用纵轴 |

### 口径建议

- 未实现盈亏 = 当前市值 − 当前持仓成本基础。
- 已实现盈亏必须基于卖出与成本层，不能只依赖当前平均成本。
- 组合表现默认使用时间加权收益率（TWR）排除外部资金流影响；当用户想看「我实际投入的资金赚了多少」时，再补充资金加权收益率（MWR / XIRR）。
- 成本法、税费和券商对账口径因市场不同而变化。HONE 的第一阶段定位是「个人研究与绩效估算」，不应宣称可用于税务或法定对账。

## 7. 桌面与移动不同的信息密度

### 桌面研究桌

- 可同时展示组合摘要、收益曲线、持仓表格和右侧事件/风险。
- 持仓表格可展示数量、成本、市值、今日盈亏、累计盈亏、仓位、数据源和同步时间。
- 公司页使用行情图 + 用户持仓 + 公司数据 + 研究的多栏或分区布局。
- 支持鼠标悬停、键盘快捷键、多指标与不同时间范围。

### 原生移动 App

- 首屏只保留「总资产/今日盈亏」、「同步状态」、「持仓列表」与「今日最重要的 1–2 个信号」。
- 组合收益图放进「表现」二级页，不在首页使用 300px 高的图表占据首屏。
- 使用「资产 / 持仓 / 表现」页内切换，但不重复底部五个一级入口。
- 持仓行用两行完成表达：公司/仓位/市值在左，今日和累计盈亏在右；点击进入公司页。
- 公司页的第一屏优先价格、今日变化、持仓盈亏和紧凑 K 线；详细财务、估值、洞察和主线使用横向切换进入。
- 长按 K 线显示十字线与当日 OHLCV，点击买卖点查看导入的历史操作；图表可全屏。

## 8. 移动投资首页的推荐层级

```text
顶部：投资 + 搜索 + 账户/同步状态

页内切换：资产 | 持仓 | 表现

资产：
- 总资产 / 今日盈亏 / 累计盈亏
- 账户同步状态
- 前 3 大持仓
- 今日 1–2 个关键信号

持仓：
- 高密度持仓列表
- 搜索、排序、账户筛选
- 添加/导入/同步

表现：
- 组合收益率 vs 基准
- 资产规模
- 已实现/未实现盈亏
- 回撤和贡献归因
```

## 9. 边界与异常

| 场景 | 产品处理 |
|---|---|
| 初始同步需要较长时间 | 先展示已连接账户和同步进度，允许离开；完成后通知 |
| 连接过期 | 展示最后成功数据和「需重新授权」，停止展示伪实时收益 |
| 导入与同步数据重复 | 按账户 + 证券 + 日期 + 数量 + 金额去重，冲突交给用户确认 |
| 成本未知 | 展示市值和今日变化，不显示累计盈亏，引导补录成本 |
| 仅有持仓快照 | 标注「收益历史从导入日开始」，不伪造历史曲线 |
| 公司行动 | 展示调整来源与记录，自动调整后仍可回看 |
| 多账户持有同一证券 | 默认汇总，允许展开查看各账户成本和数量 |
| 数据源删除 | 提示删除后是保留最后快照，还是一并删除相关私有数据 |

## 10. 本轮视觉设计会验证什么

接下来先用三套移动投资首页方向验证：

1. 资产优先：强调总资产、同步状态和组合摘要。
2. 持仓优先：把高密度持仓列表提到首屏，更接近行情 App 直觉。
3. 研究优先：在资产和持仓之间嵌入最重要的一个研究信号，体现 HONE 与券商工具的差异。

三套都必须：

- 保留 HONE 黑白灰语言与五入口底栏。
- 明确只读数据同步，不出现买入/卖出按钮。
- 在 `390 × 844` 内显示持仓、同步状态与主要操作，不用大图表占据首屏。
- 展示 2026-07-12 的合理模拟数据和数据时间。

## 11. 待验证

- 首发地区和首批用户主要使用哪些券商。
- 第三方券商连接服务的地区覆盖、缓存时效、合规、定价和删除机制。
- 已实现盈亏和成本法是否在首发阶段必须精确到券商对账口径。
- 是否允许用户将截图、PDF 或 CSV 上传到云端处理，还是需要本地优先的解析策略。
- 移动原生端最终使用 iOS 原生组件还是与 Web 共享前端架构，这将影响长按、全屏图表和手势性能。

## 12. 方向确认与 Live Demo 落地

一鸣于 2026-07-12 确认采用以下融合方向：

- 以方案 2“持仓优先”为主骨架，维持专业投资助手需要的高信息密度。
- 吸收方案 3 的 Agent 研究信号，但首页只保留一条最重要信号，不把投资页变成聊天页或内容流。
- 移动端默认进入持仓，资产与组合收益放入同级页内切换；桌面端保留收益、持仓、事件、数据源和风险的工作台结构。
- 公司详情使用 K 线、成交量、平均成本线和历史买入点，用户持仓数据与公司研究数据明确分层。

已在 `demo/` 中实现并通过视觉与交互校验，具体记录见 `demo/design-qa.md`。
