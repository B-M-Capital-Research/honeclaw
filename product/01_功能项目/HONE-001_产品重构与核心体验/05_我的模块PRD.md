状态：`draft`
模块负责人：待定

更新时间：2026-07-12

## 1. 需求背景

“我的”需要统一承载账户、会员、投资偏好、通知、显示和隐私设置，但不能继续成为业务功能的杂物间。Agent、任务结果和公司研究都应回到各自一级模块。

当前产品已有邀请制手机号登录、账户信息、主题/字号和部分通知偏好；会员仍是占位。新模块需要把这些能力整理成明确、可扩展的账户中心。

## 2. 解决的问题

- 让用户知道自己以什么账号登录、拥有什么权益、在哪些设备上使用。
- 统一配置基础币种、时区、语言、投资偏好和通知规则。
- 让用户清晰管理隐私、安全和数据，而不影响核心研究入口。
- 为后续会员体系建立统一权益判断，但本轮不实现支付。

## 3. 用户与范围

### 用户

- 所有登录用户。
- 会员用户需要查看权益与有效期。

### 本期范围

- P0：账户资料、登录状态、会员状态与权益展示、基础偏好、通知入口、显示设置、帮助与协议。
- P1：设备/会话管理、数据导出、账号注销流程、反馈工单。
- P2：支付、订阅、自动续费、家庭/团队账户。

### 不做

- 把 Agent、任务历史、收藏和公司研究主功能塞进“我的”。
- 本轮不接入真实收款、退款和自动续费；Live Demo 需要完整演示选档、订单、支付和成功状态。
- 展示管理员内部信息和系统配置。

## 6. 关键流程

### 6.1 查看会员权益

```text
进入我的
→ 查看会员状态
→ 打开权益详情
→ 查看各能力可用范围/额度
→ 从权益项进入对应模块
```

如果支付尚未上线，不展示无法完成的购买 CTA；可展示“联系团队/等待开放”等真实状态。

### 6.2 修改时区

```text
选择新时区
→ 预览当前时间和可能受影响的任务
→ 用户确认
→ 保存
→ 跟踪日历和下次执行时间刷新
```

### 6.3 退出登录

```text
点击退出
→ 如有未保存输入则提示
→ 清除会话令牌和本地敏感数据
→ 返回登录页
```

## 7. 边界与异常

| 场景 | 处理要求 |
|---|---|
| 会员接口失败 | 不擅自降级或放行；展示状态暂不可用并允许重试 |
| 会员到期 | 已有数据保留；受限内容重新鉴权，任务按权益规则暂停/降级 |
| 账号被停用 | 终止新请求、清理会话并说明联系渠道 |
| 手机号变更 | 本期不直接编辑；提供受控支持流程 |
| 多端修改偏好 | 服务端版本最终一致；冲突时提示刷新，不静默覆盖关键项 |
| 系统语言无对应内容 | 界面切换，原内容按已有语言展示并明确语言 |
| 通知总开关关闭 | 任务继续按定义运行，结果保留站内历史但不主动打扰 |
| 导出数据量大 | 异步生成并在完成后通知；下载链接限时且需鉴权 |
| 注销申请中 | 明确等待期和可撤销节点；禁止新建长期任务 |
| 联系方式未配置 | 隐藏入口或显示可用替代方式，不使用 `#` 空链接 |

## 8. 验收标准

- [ ] 用户能看懂当前账号、会员状态、有效期和权益。
- [ ] 权益在洞察、Agent 和跟踪模块中由同一服务端结果控制。
- [ ] 用户可设置基础币种、时区、主题、字号和语言。
- [ ] 通知总开关、勿扰和事件偏好与跟踪模块一致。
- [ ] 退出后本地会话和敏感草稿被清除。
- [ ] 会员到期、账号停用、接口失败和多端冲突均有明确处理。
- [ ] 不存在空社区链接、虚假购买按钮或无法兑现的系统推送承诺。
- [ ] 隐私、日志和埋点不泄漏手机号、正文、附件和精确持仓。

## 9. 研发拆分建议

1. 我的首页与账户信息。
2. 会员状态/权益服务与各模块鉴权。
3. 投资、通知和显示偏好。
4. 登录/会话与设备安全增强。
5. 数据导出、注销和反馈等 P1 能力。
6. 跨端同步、异常状态和隐私验收。

## 10. Live Demo 链路确认（2026-07-12）

- 头像菜单前置勿扰模式、深色模式和界面语言示意，用户不必为高频开关进入设置页。
- 研究偏好支持投资周期、关注主题、特别关注问题、证据优先和反方观点配置。
- 设备与登录展示当前设备、其他设备、最近安全活动、单设备退出和退出其他设备。
- 会员链路演示基础版、月付专业版、年付专业版，包含方案选择、剩余价值抵扣、协议确认、支付方式、支付表单和成功权益刷新。
- 支付、语言切换和跨端同步均为 Demo 状态；不产生真实扣款，也不实现完整多语言文案。
