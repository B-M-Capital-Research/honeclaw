# HONE 产品工作区

这里用于产品设计、快速推演和研发交接，不放生产代码。

## 目录

- `00_产品基线/`：相对稳定的产品定位、用户、产品地图和设计原则。
- `01_功能项目/`：每个功能一个目录，集中保存轻量 PRD、Demo 和需求卡片。
- `02_需求池/`：尚未进入正式设计的想法和问题。
- `03_模板/`：轻量 PRD、需求卡片和 Demo 说明模板。
- `99_已归档/`：停止推进或已经完成、无需继续维护的产品项目。

## 功能项目结构

```text
HONE-001_功能名称/
├── PRD.md
├── demo/
│   └── README.md
├── cards/
│   └── HONE-001-C01_需求名称.md
└── assets/                 # 有素材时再创建
```

PRD、Demo 和需求卡片放在同一个功能目录，避免上下文分散。目录 ID 使用 `HONE-001` 递增，需求卡片在功能 ID 后增加 `C01`。

## 工作流程

```text
需求池 → 轻量 PRD → Demo 推演 → 需求卡片 → 开发实现 → 验收 → 归档
```

建议状态：`draft`、`review`、`approved`、`in_development`、`validating`、`shipped`、`archived`。

## 与工程文档的边界

- `product/` 定义用户问题、产品方案、目标体验和验收标准。
- `docs/current-plans/` 只记录需要持续跟踪的工程实施计划。
- 代码和测试是实际产品行为的最终真相源。
- 需求进入开发后，在需求卡片中链接对应 Issue、工程计划、PR 或 commit，不复制多套说明。

## Demo 原则

- Demo 用于验证体验，可以是单页 HTML、小型前端原型或静态页面。
- 优先复用 HONE 现有品牌资产、设计变量、组件模式和真实文案。
- Demo 与生产代码隔离；生产代码不得反向依赖 `product/`。
- 默认使用模拟数据，不接生产数据库，不保存密钥。
