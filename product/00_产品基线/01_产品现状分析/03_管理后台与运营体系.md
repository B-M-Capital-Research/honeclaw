# 管理后台与运营体系

## 后台定位

HONE Console 不是面向普通用户的附属设置页，而是完整的运营与运行控制面。它把聊天前台中隐形的用户数据、任务、能力和故障变成可观察、可操作对象。

## 功能地图

| 模块 | 管理对象 | 当前能力 |
|---|---|---|
| Dashboard | 整体服务 | 运行概览和入口 |
| Sessions | 会话 | 按用户/渠道查看历史和上下文 |
| Users | actor 用户 | 汇总组合、公司画像、投资主线、会话和研究任务 |
| Skills | 投研能力 | 查看 Skill 详情、启停、重置 |
| Tasks | 定时任务 | 创建、编辑、启停、删除和执行历史 |
| Notifications | 主动消息 | 按时间和状态查看投递记录、统计分布 |
| Schedule | 调度全景 | 任务时间表、即时事件、摘要槽位和勿扰时间的交互关系 |
| Research | 深度研究 | 启动任务、跟踪状态、生成/读取 PDF |
| LLM Audit | 模型调用 | 检查提示词、调用结果和执行审计 |
| Logs | 系统日志 | 查询与流式查看运行日志 |
| Task Health | 后台任务 | 查看任务健康状态和失败信息 |
| Settings | 系统配置 | 渠道、模型、通知偏好和其他运行配置 |

后台导航会依据服务端 capability 隐藏不可用模块，所以不同部署方式下看到的控制台并不完全一致。

## 用户管理模型

后台的“用户”实质上是 actor，而不只是传统账号。actor 由渠道、用户 ID、可选群聊 scope 共同确定。管理员可以：

- 创建、启用/停用受邀 Web 用户。
- 重置登录相关信息和 API Key。
- 查看某个 actor 的持仓/关注、公司画像、投资主线、会话和研究。
- 导出、导入、处理冲突或删除公司画像。

这种模型适合多渠道隔离，但也意味着同一个自然人在 Web、飞书和 Discord 中默认可能被识别为多个 actor；当前基线没有发现面向用户的跨渠道统一身份绑定界面。

## 投资组合与公司画像的后台能力

管理员可维护股票和期权，记录数量、均价、策略周期、方向和备注；“仅跟踪”标的作为 watchlist 存在，也可进入事件订阅池。

公司画像是按 actor 分隔的长期研究对象。每份画像包括公司、ticker、别名、行业模板、状态、跟踪频率、重点指标、主档 Markdown 和追加事件历史，并支持 ZIP 导入导出。这说明画像不是一段聊天摘要，而是长期维护的数据资产。

## 研究与运维边界

- 深度研究 Skill 明确标注 `admin_only: true`，预计耗时 1–2 小时，并在前 15 分钟按分钟报告进度。
- 普通股票研究属于用户可调用能力，深度研究属于管理员能力，两者不是同一权限层。
- 日志、LLM 审计、任务健康和配置应用表明当前后台同时服务运营与工程维护。

本轮没有管理员账号，以上后台能力为代码级确认，未进行当前生产环境可用性验证。
