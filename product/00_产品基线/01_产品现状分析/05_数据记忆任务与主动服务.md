# 数据、记忆、任务与主动服务

## 身份与数据隔离

HONE 的基本数据主键是 `ActorIdentity`：

```text
channel + channel_scope（可选） + user_id
```

- 私聊 session 按渠道和用户隔离。
- 群聊 session 按群 scope 共享上下文，但执行期权限仍属于发起 actor。
- 组合、画像、额度、沙箱、通知偏好等按 actor 隔离。

这一模型支撑 Web、飞书、Discord、Telegram、iMessage 等多入口，也形成当前跨渠道身份天然分开的边界。

## 长期记忆结构

| 数据 | 作用 | 当前存储形态 |
|---|---|---|
| 会话 | 恢复对话上下文和历史 | 文件/SQLite；Cloud PG |
| 对话额度 | 控制每天成功请求与在途请求 | SQLite/Cloud PG |
| 投资组合 | 持仓、期权、watchlist、成本、周期和备注 | actor-scoped storage |
| 公司画像 | 公司长期事实、观点、事件和跟踪字段 | Markdown + append-only events；Cloud 对应远端存储 |
| 投资主线 | 全局风格和逐 ticker thesis | 通知偏好/蒸馏数据 |
| 定时任务 | 任务定义、状态、执行历史和站内结果 | JSON + SQLite；Cloud PG |
| 登录与 API Key | 受邀用户、手机号、会话、条款版本、Key 元数据 | SQLite/Cloud PG |
| 文件与上传 | 用户材料、生成图片和研究产物 | 本地 actor 沙箱；Cloud OSS |

Cloud 模式强调 PG/OSS 持久化，不应依赖容器本地磁盘保留核心数据。

## 两套“主动服务”机制

### A. 用户定时任务 Scheduler

定时任务由用户用自然语言创建，再落成结构化 schedule 和 task prompt。当前支持：

- 每日、每周、一次性、工作日、美股交易日、节假日。
- Heartbeat 类型监控。
- 北京时间解释；一次性任务可指定绝对日期。
- 每个 actor 最多同时启用 12 个任务。
- 默认遵守勿扰时间，个别任务可显式绕过。
- 保存执行状态、发送状态、结果预览和错误。

### B. 市场事件 Event Engine

Event Engine 不是让 LLM 持续轮询市场。多数 poller 先用确定性代码拉取、清洗和去重事件，再按订阅、严重度、持仓和偏好路由；LLM 只在新闻重要性判断、摘要或个性化等环节参与。

当前事件类型包括：

- 财报预告、财报发布、电话会纪要。
- 重要新闻。
- 盘中/收盘/盘前盘后价格异动、52 周新高低。
- 分红、拆股、SEC 文件。
- 分析师评级、宏观事件。
- RSS/Telegram 等社交来源内容。

价格微观事件有较短保鲜期；财报、SEC、分红等事实事件可跨夜保留。事件经过去重后，高严重度可即时投递，中低严重度进入摘要；用户可按严重度、类型、来源、持仓、时区、价格阈值和勿扰时段调整。

## 投资主线蒸馏

系统会从用户的公司画像中蒸馏整体投资风格、每个 ticker 的投资主线、最近蒸馏时间，以及因缺画像或失败而跳过的 ticker。

这些字段再参与摘要个性化：印证主线的内容优先，证伪信息保留并标注，与主线关系较弱的噪音可被降低优先级。它构成了“长期记忆影响主动信息流”的闭环。

## “推送”的当前真实含义

当前 Web 用户端的主动消息是：服务端持久化一条 `WebPushMessage`，页面打开时通过 SSE 实时接收，也可在站内推送中心补看。代码中没有发现浏览器 Notification API、Service Worker PushManager 或 Apple 原生 Push 集成。

因此当前可确认的是“站内主动消息 + 在线实时到达”，不能等同于浏览器关闭或 App 退出后仍能出现的系统级通知。飞书、Discord、Telegram 等渠道则可由对应 sink 主动发送，但取决于渠道部署和凭证。

## 数据来源与依赖

- 金融行情和多数结构化市场事件主要来自 Financial Modeling Prep。
- 新闻精读可走网页直抓，再回退 Jina Reader 或 FMP 摘要。
- RSS 和 Telegram 等非 FMP 来源可独立运行。
- 财经日历在 FMP 不可用时仍可展示内置宏观事件，但持仓财报数据会受限。

以上是代码支持的数据链路，不代表每个外部 Key 和来源都已在当前生产部署启用。
