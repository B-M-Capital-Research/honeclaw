# 产品成熟度与现状边界

## 当前成熟度画像

| 维度 | 当前判断 | 依据 |
|---|---|---|
| 核心投研内核 | 较完整 | Persona、Skills、Tools、Runner、长期画像和主动事件已形成体系 |
| Web 用户体验 | 主链路已成形 | 邀请登录、流式 Chat、历史、附件、分享、站内推送和投资上下文均有实现 |
| 原生用户端 | 轻壳阶段 | macOS/iOS 复用线上 Web；没有本地离线 AI 和原生推送 |
| 管理与运维 | 较深入 | 用户、任务、研究、审计、日志、健康和配置具备独立控制面 |
| 多渠道能力 | 工程入口广 | 多个渠道 binary 和统一 actor 模型已存在；真实可用性依赖部署 |
| 主动投研 | 机制较丰富 | Scheduler + Event Engine + 偏好 + 摘要 + 多 sink |
| 商业化 | 尚未形成产品闭环 | 会员仅页面占位；无支付、套餐、订阅和 entitlement |
| 老王/巴芒知识服务 | 尚未接入当前公开仓库 | 未发现 Patreon、群聊素材、会员内容库或老王观点检索链路 |

## 当前可以确认的产品闭环

```text
受邀登录
  → 在 Chat 提出投资问题或上传材料
  → Skill 选择工作流
  → Tool 获取市场数据/用户上下文
  → Agent 在投资纪律和权限边界内生成结果
  → 会话、组合、画像和任务按 actor 持久化
  → Scheduler / Event Engine 在后续产生站内或渠道消息
```

其中“登录后实际生产效果”“外部渠道投递”“长期研究质量”仍需要有效账号、外部 Key 和运行数据才能验证。

## 重要现状差异

这些是基线中的事实差异，不自动构成改版需求：

1. **版本口径不同**：线上路线图仍显示 v0.12.4，仓库 release 已到 v0.14.0。
2. **品牌迁移不完全**：前台使用 HONE，域名、仓库和工程命名仍大量保留 Honeclaw。
3. **两种桌面端容易混淆**：公开 macOS 用户端是远程 Web 壳；另有包含本地 runtime 的自托管桌面端。
4. **iOS 发布边界**：当前 GitHub 发布物用于 Simulator/Xcode 验证，不是可直接安装的真机 IPA 或 App Store 版本。
5. **推送语义有限**：Web 是站内消息和在线 SSE，不是系统级 Web/Apple Push。
6. **会员区域未落地**：页面有会员预告，代码没有商业化权限闭环；社区按钮仍是空链接。
7. **结构化能力入口偏隐性**：普通用户主要从 Chat 触发能力；`/portfolio` 已实现但未进入公共顶栏。
8. **跨渠道身份分散**：actor 默认按渠道与 scope 隔离，未发现普通用户自助合并身份的产品入口。
9. **公开能力与私域知识之间仍有断层**：当前擅长市场数据、用户画像和任务，但未发现老王内容、付费社群材料和会员版权边界的实际接入。
10. **公开页面不能代表全部现状**：路线图和 README 有能力声明；登录后、后台和渠道能力本轮多为代码确认，未全部在线走通。

## 代码层可见的稳定性状态

仓库近期工程工作集中在移动端 Chat 打磨、流式输出、历史分页、Apple 用户端发布、Cloud PG/OSS、事件引擎、飞书诊断和运行时安全。与此同时，`docs/bugs/` 仍记录 Heartbeat 误判、触发时间错位和异常行情锚等运行问题。

因此当前产品并非概念 Demo，但也不能只根据功能数量判断为完全成熟：核心架构和功能面已经很广，真实运行质量仍处于持续修正阶段。

- 当前生产环境每日额度的实际配置值。
- 管理后台各 capability 在生产环境的开启情况。
- FMP、Web Search、Jina、图像生成与深度研究的生产凭证和成功率。
- 飞书、Discord、Telegram、iMessage 当前实际在线状态。
- Event Engine 的真实订阅覆盖、投递成功率与摘要质量。
- macOS DMG 在真实机器上的安装体验；iOS 真机体验。
- 任何付费会员、Patreon 或老王知识库接入。

## 与后续产品工作的隔离规则

- 本目录固定在上述 commit 的“现状证据”。未来代码变化不在这里悄然覆盖，应另建新快照或明确更新基线。
- 新方向进入 `product/02_需求池/`。
- 已决定推进的功能进入 `product/01_功能项目/HONE-xxx_功能名称/`，再形成轻量 PRD、Demo 和需求卡片。
- 后续可以引用本分析作为“为什么改”的证据，但不要把本分析本身改造成需求文档。
