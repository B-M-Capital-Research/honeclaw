# Profile Framework

## 目标

`profile.md` 是“当前仍然成立的最佳观点”，不是历史流水账。

写画像时，优先保证：

- 这是当前 view，不是把所有旧结论都塞进去
- 重点是 thesis、为什么成立、关键观察变量、证伪条件
- 用自然语言、表格、项目符号都可以，但不要强行填 schema

## 推荐 section

### 投资主张

- 一句话总结这家公司当前为什么值得长期跟踪
- 可补一句：当前最核心的赔率/质量/结构性变化

### Thesis

这一节是画像中心。

至少回答四件事：

- 当前多空要点是什么
- 为什么现在这么判断
- 最关键的 3-5 个观察变量是什么
- 什么事实会证伪或改写 thesis

如果合适，可单独写出：

- `bull case`
- `base case`
- `bear case`
- `disconfirming evidence`

### 商业模式

- 怎么赚钱
- 收入结构
- 成本结构
- 单位经济
- 周期性
- 一次性 / 经常性收入占比

### 行业与竞争格局

- 市场空间
- 行业集中度
- 竞争格局
- 替代品
- 上下游议价权
- 进入壁垒

### 护城河

- 品牌
- 网络效应
- 切换成本
- 规模优势 / 成本优势
- 渠道控制
- 牌照 / 监管壁垒
- 数据资产 / 生态位

明确写出 `moat_trend`：

- strengthening
- stable
- weakening

### 管理层与治理

- 创始人 / CEO / CFO / 核心高管
- 稳定性与激励机制
- 资本配置能力
- 对外沟通可信度
- 是否长期主义

### 财务质量

- 收入增长质量
- 毛利率 / 营业利润率
- ROIC
- FCF 转化
- 负债结构
- 再投资效率
- 稀释
- 并购质量

### 资本配置

- 分红
- 回购
- 再投资
- 并购
- 研发
- 去杠杆

判断标准不是动作本身，而是是否提升长期每股价值。

### 关键经营指标

这一节必须行业化。

- 工业 / 制造 / 国防：订单、积压订单、book-to-bill、产能利用率、交付节奏
- SaaS：ARR、RPO、NRR、客户留存、seat expansion、deferred revenue
- 消费：同店、客单价、复购率、渠道库存、品牌溢价
- 金融：净息差、拨备、不良、资本充足率、负债成本
- 半导体 / 硬科技：产能、良率、产品 mix、ASP、设计 win、库存周期

### 估值框架

- 哪种方法最适合看这家公司：DCF、SOTP、EV/EBITDA、P/E、P/B、FCF yield、NAV 等
- 核心假设是什么
- 哪些变量最敏感
- 什么时候估值区间会影响 thesis / 仓位讨论

### 风险台账

不要只写“有竞争风险”。

优先写会真正破坏 thesis 的风险：

- 监管
- 技术替代
- 客户集中
- 地缘政治
- 原材料 / 库存
- 融资
- 管理层失误
- 财务失真
- 商业模式破坏

建议单列 `disconfirming evidence`。

### 关键跟踪清单

- 季度必看什么
- 财报后必查什么
- 哪类事件会触发立即重看 thesis
- 哪些指标进入拐点时必须回写画像

### 未决问题

- 现在还不知道，但答案会显著改变判断的问题
- 不确定的地方不要伪装成确定结论

## 写作纪律

- 事实和解释尽量分开
- 写判断时，尽量附上“为什么”
- 写乐观结论时，也补上证伪条件
- 除非真的重要，不要把短期价格噪音写进画像
