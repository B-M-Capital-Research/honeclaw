# Profile Framework

## 目标

`profile.md` 是“当前仍然成立的最佳观点”，不是历史流水账。

写画像时，优先保证：

- 这是当前 view，不是把所有旧结论都塞进去
- 重点是 投资主线、为什么成立、关键观察变量、证伪条件
- 当长期判断已经变化时，直接改正文，不要在原句后面不断打补丁
- 用自然语言、表格、项目符号都可以，但不要强行填 schema

## 必须的文件头

`profile.md` 第一行必须是 YAML frontmatter，至少声明 `ticker`：

```markdown
---
ticker: GOOGL
---

# Alphabet (GOOGL)
```

- 双 ticker 用斜杠分隔，如 `ticker: GOOGL / GOOG`
- 目录 `<profile_id>` 用 lowercase ticker（双 ticker 写 `googl-goog`），不要用公司名
- frontmatter 是下游 投资主线 蒸馏 / global digest 识别 ticker 的**唯一可靠路径**；缺了这一行，整份画像会被悄悄跳过

可选的额外 frontmatter 字段：`name`、`exchange`、`updated`，按需写。

## 推荐 section

### 投资主张

- 一句话总结这家公司当前为什么值得长期跟踪
- 可补一句：当前最核心的赔率/质量/结构性变化

### 投资主线

这一节是画像中心。

至少回答四件事：

- 当前多空要点是什么
- 为什么现在这么判断
- 最关键的 3-5 个观察变量是什么
- 什么事实会证伪或改写 投资主线

这一节也应尽量吸收已经稳定下来的用户视角：

- 用户自己明确表达过的核心看法
- 你与用户此前已经达成一致的判断逻辑
- 不应被遗忘的前提、偏好或约束

如果合适，可单独写出：

- `bull case`
- `base case`
- `bear case`
- `disconfirming evidence`

### 用户视角与偏好

如果用户长期会基于自己的偏好做研究取舍，建议单列这一节。

- 用户偏好的公司特征、赛道位置、风险收益结构
- 用户明确反感或回避的点，例如高杠杆、强周期、重融资依赖、治理差
- 用户偏好的估值口味，例如更看重自由现金流、PEG、订单确定性还是资产重估
- 只记录相对稳定、会反复影响判断的偏好；一次性情绪不要写成长期画像

### 商业模式

- 怎么赚钱
- 收入结构
- 成本结构
- 单位经济
- 周期性
- 一次性 / 经常性收入占比

### 行业与竞争格局

- 市场空间
- 行业集中度
- 竞争格局
- 替代品
- 上下游议价权
- 进入壁垒

### 护城河

- 品牌
- 网络效应
- 切换成本
- 规模优势 / 成本优势
- 渠道控制
- 牌照 / 监管壁垒
- 数据资产 / 生态位

明确写出 `moat_trend`：

- strengthening
- stable
- weakening

### 管理层与治理

- 创始人 / CEO / CFO / 核心高管
- 稳定性与激励机制
- 资本配置能力
- 对外沟通可信度
- 是否长期主义

### 财务质量

- 收入增长质量
- 毛利率 / 营业利润率
- ROIC
- FCF 转化
- 负债结构
- 再投资效率
- 稀释
- 并购质量

### 资本配置

- 分红
- 回购
- 再投资
- 并购
- 研发
- 去杠杆

判断标准不是动作本身，而是是否提升长期每股价值。

### 关键经营指标

这一节必须行业化。

- 工业 / 制造 / 国防：订单、积压订单、book-to-bill、产能利用率、交付节奏
- SaaS：ARR、RPO、NRR、客户留存、seat expansion、deferred revenue
- 消费：同店、客单价、复购率、渠道库存、品牌溢价
- 金融：净息差、拨备、不良、资本充足率、负债成本
- 半导体 / 硬科技：产能、良率、产品 mix、ASP、设计 win、库存周期

### 估值框架

- 哪种方法最适合看这家公司：DCF、SOTP、EV/EBITDA、P/E、P/B、FCF yield、NAV 等
- 核心假设是什么
- 哪些变量最敏感
- 当前估值结论、估值区间或估值锚点是什么
- 用户对“便宜 / 合理 / 昂贵”的主观阈值如果比较稳定，也应顺手记下
- 什么时候估值区间会影响 投资主线 / 仓位讨论

### 风险台账

不要只写“有竞争风险”。

优先写会真正破坏 投资主线 的风险：

- 监管
- 技术替代
- 客户集中
- 地缘政治
- 原材料 / 库存
- 融资
- 管理层失误
- 财务失真
- 商业模式破坏

建议单列 `disconfirming evidence`。

### 关键跟踪清单

- 季度必看什么
- 财报后必查什么
- 哪类事件会触发立即重看 投资主线
- 哪些指标进入拐点时必须回写画像

### 未决问题

- 现在还不知道，但答案会显著改变判断的问题
- 不确定的地方不要伪装成确定结论

## 写作纪律

- 事实和解释尽量分开
- 写判断时，尽量附上“为什么”
- 写乐观结论时，也补上证伪条件
- 只要本轮研究形成了稳定的用户偏好、共识逻辑或估值结论，就不要只留在对话里，应写回画像对应 section
- 旧判断如果已经失效，就直接删改主画像对应表述，不要保留成“先这样、后那样”的流水账正文
- 除非真的重要，不要把短期价格噪音写进画像
